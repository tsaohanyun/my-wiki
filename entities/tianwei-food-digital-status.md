# 天味食品数字化现状

## 概述

天味食品是一家调味品制造企业，其数字化现状反映出中国传统食品制造业的典型痛点：黑箱交流、数据孤岛、流程缺失、依赖经验。

## 车间痛点 ^[raw/articles/tianwei-food-pain-points.md]

### 信息沟通

- **黑箱交流严重**：大多流程在微信、电话、口头上进行，需多次确认甚至靠经验判断
- **生产数据延后**：数据上传和跟踪有很大延后性，不实时录入系统
- **大量自建表格和纸质单据**：各部门对接数据需再汇总后建自建表格

### 生产执行

- **配料靠经验**：配料投放及设备使用靠现场计算和经验
- **物料放错**：配料过程中存在物料放错的问题
- **异常检测缺失**：设备/阀门无法正常运行、管道堵塞无法检测和预防
- **反应过程不可见**：釜内过程反应（电机电流、重量等）不能用数据清晰反映
- **温度控制难**：物料加入速度、反应程度和搅拌速率导致温度难以控制
- **物料分批投入管控弱**：非一次性投入，每次添加数量和准确性缺乏管控

### 设备与系统

- **DCS/中控/设备问题多**：涉及技改较多，设备检测数据不准，中控控制不准
- **缺少报工环节**：通过目测、经验方式判断
- **缺少看板**：车间缺少安全标语和生产数据展示
- **缺少系统维护专员**：车间主任管理范围杂，无时间维护系统

## 仓库痛点 ^[raw/articles/tianwei-food-pain-points.md]

| 问题 | 描述 |
|------|------|
| 标识位置不合理 | 编码贴桶身，需多次移动才能核对 |
| 库房分散杂乱 | 位置分散，摆放物料多而散乱 |
| 余位不可视 | 需频繁到库房查看 |
| 订单微信沟通 | 零散低效 |
| 堆放混乱 | 老员工也需每日盘点才能定位物料 |
| 无库位管理 | 自行绘制表格，难以按时间轴追溯 |
| 联系方式单一 | 电话沟通，发货数量/型号需二次确认 |
| ERP/OA时效差 | 流程后续补充填写 |
| 包装标识常变 | 客户要求变化，对接困难，缺 SRM |
| 库存不可视 | 需与车间反复确认 |
| 领料流程混乱 | 先签单后领料/先领料后补单，账务不符 |

## 计划部痛点 ^[raw/articles/tianwei-food-pain-points.md]

- 原料出入库不及时
- 月订货计划无法在 ERP 生成生产订单
- 大量线下沟通及记录

## 数字化需求总结

| 领域 | 核心需求 |
|------|----------|
| 生产执行 | MES 系统，实时数据采集，配料管控，异常预警 |
| 仓储管理 | WMS 系统，库位管理，库存可视化，条码/RFID |
| 计划排程 | ERP 深化应用，计划自动转订单 |
| 数据展示 | 车间看板，实时数据大屏 |
| 协同沟通 | 替代微信/电话的系统化流程，SRM 供应商管理 |
