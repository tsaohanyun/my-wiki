## 一、概述：什么是 SIE-IIDP？

SIE-IIDP 是一个**企业级工业互联网数据平台**，为制造业提供 MES（制造执行系统）功能。其前端设计风格是一种典型的**中国企业级后台管理系统风格**，以强品牌色、高数据密度、功能主义为核心特征。

技术栈基于：
- **Vue.js 2.x + Element UI** 底层框架
- **SMI Design** 自研设计系统/组件库
- **Tech Framework** 自研微前端架构
- 微前端内部应用体系（base-inner、workFlow-inner、user-inner、permission-inner）

## 三、排版系统

### 3.1 根字号缩放策略

核心特征：**html 根元素 font-size: 100px**

这是一个极具特色的做法：
- `1rem = 100px`
- 所有尺寸以 rem 为单位，形成直观的映射关系
- `0.14rem = 14px`、`0.32rem = 32px`、`0.10rem = 10px`

### 3.2 字号层级

| 用途 | 大小 |
|------|------|
| 表格内容 | 0.14rem (14px) |
| 按钮文字 | 0.14rem (14px) |
| 表单标签 | 0.14rem (14px) |
| 对话框标题 | 12-14px |
| 小号文字 | 12px |

### 3.3 字体族

- 字体栈：`-apple-system, BlinkMacSystemFont, 'Microsoft YaHei', 'PingFang SC', sans-serif`
- 显式中日文字体优先（Microsoft YaHei / PingFang SC），无自定义Web字体
- 全部元素 `font: inherit; font-size: 100%`（reset）
- **行高：`line-height: 1.5`**（紧凑但不极端，区别于极端的 `line-height: 1`）

> **注意**：这与一般认知中"企业后台行高极紧"有所不同，SIE-IIDP 在表格和表单等密集区域仍保持适中的阅读行距。

```
┌──────────────────────────────────────────────────┐
│  顶部 Header (#3D4547 深灰, 60px)                │
│  [MOM Logo] [☰]       [🏭 租户选择] [中文▼] [🔔³] [👤用户名▼]│
├────────┬─────────────────────────────────────────┤
│        │  面包屑导航 (红色文字 #AC2A48)            │
│        │  首页 > 生产管理 > 工单管理                │
│ 侧边栏  ├─────────────────────────────────────────┤
│ (红色)  │  标签页栏 (红色下划线激活, 蓝色悬停)       │
│        ├─────────────────────────────────────────┤
│ #AC2A48│  主内容区                                  │
│        │  ┌─ 搜索/筛选工具栏                       │
│ 白色SVG │  ├─ 工具栏 (按钮组) + 分页控件 (右侧)    │
│  图标   │  ├─ 数据表格 (灰底表头 #f2f3f5 + 斑马纹) │
│        │  ├─ (可选) 主子表详情区域                  │
│ 220px  │  └─ 表单 / 对话框 (Element UI)            │
│        │                                            │
└────────┴──────────────────────────────────────────┘
```

### 布局特点：
1. **双层布局架构**：顶部深色 Header（#3D4547, 60px）+ 红色侧边栏（#AC2A48, 220px），内容区 `margin-left: 220px; margin-top: 60px`
2. **顶部固定 Header**：60px 高度，`position: fixed`，z-index 200，包含系统Logo、收起按钮、租户选择器、语言切换、消息中心、用户头像/下拉菜单
3. **面包屑导航**：红色文字指示当前位置，`>` 分隔符
4. **标签页切换**：红色底部下划线标识激活标签，蓝色悬停
5. **主内容区**：表格/表单为主，可选主子表上下布局（主表行点击联动子表）
6. **分页控件位于工具栏右侧**：每页条数选择器 + 总数 + 页码下拉在工具栏右端，不使用底部独立分页栏

## 五、组件设计语言

### 5.1 侧边栏 (Sidebar)
- **背景色**：`#AC2A48`（#AC2A48）
- **激活项背景**：`#BF0016`（深红）
- **侧边栏Logo区**：内置系统Logo/标题，白色文字，`font-size: 0.18rem; font-weight: 700`，带底部半透明分割线
- **图标**：20×20 SVG，白色填充，圆形徽章风格
- **图标激活色**：`#0878CB`（蓝色）
- **非激活透明度**：1（完全可见，不降低透明度），即 `--sidebar-noactive-opacity: 1`
- **菜单文字**：白色，`font-size: 0.14rem`
- **位置**：`position: fixed; left: 0; top: 60px; height: calc(100vh - 60px)`
- **z-index**：100（低于 Header 的 200）

### 5.2 数据表格 (Table)
- **表头背景**：`#f2f3f5`（浅灰）
- **斑马纹**：偶数行 `background: #f5f5f5`（原始 SPA 通过 vxe-table `.row--stripe` 实现）
- **已查看行**：`#EDF1F7`（浅蓝灰）
- **选中行（复选框选中）**：`#53aefc` 边框 + 半透明背景，使用 `.row-selected` 类
- **行点击高亮（主子表联动）**：使用独立的 `.row-active` 类，`background: rgba(172,42,72,0.10)`，与复选框选中互不干扰
- **复选框列**：表格默认第一列为复选框，支持全选 + 行选，选中 ≥1 行后启用工具栏批量操作按钮
- **切换视图按钮**：`#677177`
- **搜索按钮**：`var(--button-primary-bg, #677177)`
- **重置按钮**：白色背景 + `#677177` 边框
- **多标签页表格**：每个 Tab 的表格有独立的复选框作用域，使用 `getActiveTabContent()` 限定选择操作范围
- 基于 Element UI `el-table` + 自定义 `vxe-table`

### 5.3 标签页 (Tabs)
- **激活态**：底部 `#AC2A48` 下划线 + `#AC2A48` 文字色
- **悬停态**：`rgba(70, 144, 199, 1)` 文字色
- 基于 Element UI `el-tabs`

### 5.4 对话框 (Dialog)
- **标题栏背景**：`#AC2A48`
- **标题栏文字**：白色
- **确认按钮**：`#AC2A48`（悬停：`#f22731`）
- **圆角**：5px（较大，是少数圆角明显的组件）
- 基于 Element UI `el-dialog`

### 5.5 按钮 (Button)
- **主按钮**：`--button-primary-bg`（默认 `#AD053D` 或 `#677177`）
- **圆角**：**2px**（极小，近乎直角！这是关键特征）
- **内边距**：`0 16px`
- **高度**：`32px`
- **文字按钮**：干净简洁
- **线框按钮**：白色背景 + 边框
- **危险按钮**：`background: var(--brand-red)`
- **工具栏按钮组**：紧凑排列
- **按钮排列顺序规则**（从左到右）：新增 → 编辑 → 删除 → 其他业务操作。每页条数选择器用 `margin-left: auto` 推到工具栏最右侧
- 基于 Element UI `el-button` + 自定义 `smi-btn`

### 5.6 表单 (Form)
- 基于 Element UI `el-form` / `el-form-item`
- 表单项间距：`margin-bottom: 0.12rem`
- 支持多种数据源配置表单（SQL、API、WebSocket、Kafka、RabbitMQ、Excel、OPC UA、视频流）
- 紧凑布局，标签左对齐
- 输入框高度：32px，`border: 1px solid var(--border-color)`，`border-radius: var(--radius)`（2px）
- 输入框聚焦：`border-color: #0878CB` + 蓝色阴影

### 5.7 顶部 Header

SIE-IIDP 的顶部 Header 是一个重要的布局组件，**必须包含在所有页面中**：

| 属性 | 规格 |
|------|------|
| 背景色 | `#3D4547`（深灰） |
| 高度 | 60px |
| 定位 | `position: fixed; top: 0; left: 0; right: 0` |
| z-index | 200（高于侧边栏的 100） |
| 内边距 | `padding: 0 20px` |

**左侧区域**：系统 Logo（白色，`font-size: 0.18rem; font-weight: 700`）+ 菜单收起按钮

**右侧区域**：租户选择器 → 语言切换 → 消息中心（红色角标 `#AC2A48`）→ 用户头像/姓名下拉菜单

**用户下拉菜单**：
- 👤 个人中心
- 🔑 修改密码
- 分割线
- 🚪 退出登录

### 5.8 状态标签 (Status Badges)

SIE-IIDP 的状态标签**不使用底色色块**，仅用纯文字 + 颜色区分状态：

| 状态 | 色值 | 含义 |
|------|------|------|
| 成功 | `#2e7d32` | 绿色 — 正常/成功/生产中 |
| 警告 | `#e65100` | 橙色 — 警告/暂停/即将完成 |
| 危险 | `#c62828` | 红色 — 危险/取消/异常 |
| 信息 | `#1565c0` | 蓝色 — 信息/待下达/待处理 |
| 默认 | `#616161` | 灰色 — 默认/已完成 |

> **禁止**给状态标签加 `background`、`padding`、`border-radius` 等色块样式，只用文字颜色区分。

### 5.9 主子表上下布局（核心模式）

当页面包含"1个主列表 + N个关联详情"时（如订单列表 + 生产进度 + 异常记录），SIE-IIDP 采用**上方主表 + 下方子表标签页**的上下布局，而非平级标签页切换。

**交互规则**：
1. **主表**（上方）：核心列表数据（搜索栏 + 工具栏 + 表格 + 分页），是页面的主体
2. **子表**（下方）：用标签页组织多个关联信息，标签页样式同 5.3 标签页组件
3. **行点击联动**：点击主表某行 → `.row-active` 高亮 → 子表自动加载对应数据
4. **区分两种选中状态**：
   - `row-active`（行点击联动子表）→ `background: rgba(172,42,72,0.10)`，红色调
   - `row-selected`（复选框批量操作）→ `#53aefc` 半透明背景，蓝色调
   - 两种样式独立共存，互不干扰
5. **子表标签页**：每个标签页内容区只含一种元素类型（一个表格 / 一组字段），不混合多种元素
6. **子表标签页不加数字角标**：标签页名称后不显示数量badge
7. **子表不重复显示当前选择的信息**：主表已通过高亮标识，子表无需额外提示

## 八、总体设计哲学

### 8.1 设计理念关键词
- **企业优先**：一切为数据展示和操作效率服务
- **品牌驱动**：#AC2A48贯穿始终，强识别性，但仅限侧边栏/对话框/标签激活态等特定位置
- **功能主义**：极低圆角（2px）、最少阴影、无渐变装饰
- **高密度**：表格多列、表单多字段、信息紧凑，面向工厂操作员场景优化
- **结构化**：Header → 侧边栏 → 面包屑 → 标签页 → 内容区，清晰的层级导航
- **简洁优先**：一个按钮只出现一次，不搞重复入口；批量操作按钮始终可见（未选中时禁用灰显）
- **无动画**：企业后台不需要花哨动效，所有交互即时应答

### 8.2 色彩使用规则（必须遵守）

| 颜色 | 使用范围 | 禁止使用 |
|------|---------|---------|
| `#AC2A48` | 侧边栏背景、面包屑文字、对话框标题栏、标签页激活态、确认按钮、红点标记 | 内容区背景、大段文字、表格 |
| `#3D4547` | 顶部 Header 背景 | 仅此处使用 |
| `#677177` | 主操作按钮（查询、搜索、提交） | 不能用于品牌色场景 |
| `#0878CB` | 侧边栏图标激活态、输入框聚焦边框、标签页悬停文字 | 不用于大面积背景 |
| `#53aefc` | 表格选中行边框（复选框选中） | 仅表格场景 |

### 8.3 与其他设计风格对比

| 维度 | SIE-IIDP | Apple Design | Ant Design | Material Design |
|------|----------|-------------|-----------|-----------------|
| 信息密度 | 极高 | 低 | 中高 | 中 |
| 双层布局 | Header + 侧边栏双层（红+灰） | 纯侧边栏或纯顶栏 | 顶栏 + 侧边栏 | 顶栏 + 侧边栏 |
| 圆角 | 2px (极小) | 大圆角 | 2-6px | 4-8px |
| 阴影 | 极少 | 极轻 | 中等 | 多层次 |
| 主色调 | 红色 (#AC2A48) | 中性灰白 | 蓝色 | 蓝色 |
| 动效 | 极少 | 无 | 丰富 | 丰富 |
| 留白 | 紧凑 | 大量 | 适中 | 适中 |
| 目标用户 | 工厂操作员 | 消费者 | 企业用户 | 混合 |
| 设计哲学 | 功能主义 | 极简优雅 | 企业效率 | 材料隐喻 |

- 内部应用（inner-app）体系实现模块化加载
- 每个业务模块独立打包部署

- CSS变量实现主题定制

- 内容区上方面包屑：首页 > 生产管理 > 工单管理（红色文字）
- 标签页：工单列表 | 工艺路线 | ...
- 工具栏：搜索框 + 查询/重置按钮（灰色），右侧为分页控件（每页条数 + 总数 + 页码下拉）
- 数据表格：灰色表头 #f2f3f5，偶数行斑马纹 #f5f5f5，默认第一列复选框

## 十一、常见设计陷阱 (Pitfalls)

经过深入分析和实际应用，以下是在 SIE-IIDP 风格设计中容易犯的错误：

| # | 陷阱 | 正确做法 |
|---|------|---------|
| 1 | 使用大圆角（>2px） | 所有按钮/输入框/表单圆角严格 2px，对话框允许 5px |
| 2 | 将 #AC2A48 用于内容区背景或大面积文字 | 仅限侧边栏、面包屑、对话框标题、标签激活态、确认按钮 |
| 3 | 使用阴影装饰（box-shadow） | 企业后台用边框（border）区分区域，几乎不用阴影 |
| 4 | 使用渐变装饰 | 所有组件纯色填充，无渐变 |
| 5 | 忘记设置 html font-size: 100px | 所有 rem 值依赖此基准，不设置则尺寸完全错乱 |
| 6 | 侧边栏图标使用其他颜色 | 图标 SVG 必须 fill="white"，圆形徽章风格 |
| 7 | 表格表头使用非 #f2f3f5 的灰色 | 表头背景固定为 #f2f3f5 |
| 8 | 使用 emoji 代替 SVG 图标 | 所有导航图标使用 20×20 SVG，不用 emoji/字体图标 |
| 9 | 使用颜色昵称（"通威红""中国红"等） | 所有颜色直接使用 HEX 值（#AC2A48、#3D4547 等） |
| 10 | 省略顶部 Header | SIE-IIDP 是 Header + 侧边栏双层布局，不是纯侧边栏 |
| 11 | 使用底部独立分页栏 | 分页控件在工具栏右侧（每页条数 + 总数 + 页码下拉） |
| 12 | 给状态标签加底色色块 | 状态只用纯文字 + 颜色区分 |
| 13 | 用平级标签页替代主子表布局 | 1主列表 + N详情时用上方主表 + 下方子表标签页 |
| 14 | 子表标签页中混合多种元素 | 每个子表标签页只包含一种元素（表格/字段/图片） |
| 15 | 子表标签页中放数量badge | 标签页名称后不加数字角标 |
| 16 | 子表重复显示"当前选择：XXX"信息 | 主表已通过行高亮标识，子表不重复提示 |
| 17 | 复选框跨Tab混淆 | 多Tab表格时复选框选择必须限定在当前活动Tab作用域内 |
| 18 | 主表行点击联动干扰复选框选中 | 行点击联动（row-active）与复选框选中（row-selected）是独立交互 |
| 19 | 使用动画效果 | 企业后台不需要动效，所有交互即时应答 |

## 十二、总结

SIE-IIDP 设计风格是一种**典型的中国大型制造企业后台管理系统风格**。其核心特征可概括为：

1. **#AC2A48** (红) 作为品牌主色，深度嵌入每个UI组件，但使用范围严格限定
2. **#3D4547 深灰顶栏 + #AC2A48 红色侧边栏** 的双层布局架构，Header 固定 60px，侧边栏 220px
3. **100px根字号** rem缩放体系，形成独特的尺寸规范
4. **2px极小圆角** 的按钮和输入框，体现功能主义美学
5. **白色圆形徽章SVG图标**，在红色侧边栏上形成强烈对比，激活态可变为蓝色
6. **高密度数据展示**，带斑马纹表格、复选框列、批量操作，面向工厂操作场景优化
7. **主子表上下布局模式**，主表行点击联动子表标签页，区分两种选中状态
8. **状态标签纯文字颜色区分**（无底色色块），保持信息密度
9. **工具栏紧凑排列**：按钮按"新增→编辑→删除→其他"顺序，分页控件在右侧
10. **无动画、无渐变、极少阴影**的高度功能主义设计
11. **Element UI + SMI Design** 双层组件体系
12. **微前端模块化**架构（Tech Framework，inner-app 体系），支持大规模企业应用

这是一种**高度实用主义**的设计风格，美学让位于功能效率，品牌色贯穿始终形成统一识别，是工业互联网领域的一个典型参考案例。
