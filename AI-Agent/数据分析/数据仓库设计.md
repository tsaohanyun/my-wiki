# 数据仓库设计

## 概述

数据仓库（Data Warehouse）是面向主题的、集成的、相对稳定的、反映历史变化的数据集合，用于支持管理决策。本文档涵盖维度建模、星型模型、雪花模型和ETL流程设计。

---

## 维度建模

维度建模是数据仓库设计的核心方法论，由Ralph Kimball提出，强调从业务过程出发构建数据模型。

### 核心概念

| 概念 | 说明 | 示例 |
|------|------|------|
| 事实表 | 存储业务度量（可加性数值） | 订单金额、销售数量 |
| 维度表 | 存储描述性属性 | 时间维度、商品维度、用户维度 |
| 粒度 | 事实表中每行数据的业务含义 | 每笔订单、每天汇总 |
| 退化维度 | 直接存在事实表中的维度 | 订单号、交易流水号 |

### 建模步骤

1. **选择业务过程**：确定需要分析的核心业务（如订单、访问、广告点击）
2. **声明粒度**：确定事实表的最细粒度（事务、周期快照、累积快照）
3. **确认维度**：确定描述事实的上下文（Who/What/When/Where/How）
4. **确认事实**：确定需要度量的数值型指标

## 星型模型

星型模型是最常用的数据仓库模型，以一个事实表为中心，周围连接多个维度表。

### 电商订单星型模型

```sql
-- ========================
-- 时间维度表
-- ========================
CREATE TABLE dim_date (
    date_key        INT PRIMARY KEY COMMENT '日期键 YYYYMMDD',
    full_date       DATE NOT NULL COMMENT '完整日期',
    year            SMALLINT NOT NULL COMMENT '年份',
    quarter         TINYINT NOT NULL COMMENT '季度(1-4)',
    month           TINYINT NOT NULL COMMENT '月份(1-12)',
    month_name      VARCHAR(10) NOT NULL COMMENT '月份名称',
    week_of_year    TINYINT NOT NULL COMMENT '年内周数',
    day_of_week     TINYINT NOT NULL COMMENT '周几(1=周一)',
    day_name        VARCHAR(10) NOT NULL COMMENT '星期名称',
    is_weekend      BOOLEAN NOT NULL COMMENT '是否周末',
    is_holiday      BOOLEAN DEFAULT FALSE COMMENT '是否节假日',
    holiday_name    VARCHAR(50) COMMENT '节假日名称',
    fiscal_year     SMALLINT COMMENT '财务年',
    fiscal_quarter  TINYINT COMMENT '财务季度'
) COMMENT '时间维度表';

-- ========================
-- 用户维度表（SCD Type 2）
-- ========================
CREATE TABLE dim_user (
    user_key        BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '用户代理键',
    user_id         BIGINT NOT NULL COMMENT '用户自然键',
    username        VARCHAR(50) NOT NULL COMMENT '用户名',
    email           VARCHAR(100) COMMENT '邮箱',
    phone           VARCHAR(20) COMMENT '手机号',
    gender          VARCHAR(10) COMMENT '性别',
    age_group       VARCHAR(20) COMMENT '年龄段',
    city            VARCHAR(50) COMMENT '城市',
    province        VARCHAR(50) COMMENT '省份',
    vip_level       VARCHAR(20) COMMENT 'VIP等级',
    register_date   DATE COMMENT '注册日期',
    channel         VARCHAR(50) COMMENT '注册渠道',
    -- SCD Type 2 字段
    effective_date  DATE NOT NULL COMMENT '生效日期',
    expiry_date     DATE DEFAULT '9999-12-31' COMMENT '失效日期',
    is_current      BOOLEAN DEFAULT TRUE COMMENT '是否当前有效'
) COMMENT '用户维度表(SCD2)';

-- ========================
-- 商品维度表
-- ========================
CREATE TABLE dim_product (
    product_key     BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '商品代理键',
    product_id      BIGINT NOT NULL COMMENT '商品自然键',
    product_name    VARCHAR(200) NOT NULL COMMENT '商品名称',
    sku_code        VARCHAR(50) COMMENT 'SKU编码',
    category_l1     VARCHAR(50) NOT NULL COMMENT '一级分类',
    category_l2     VARCHAR(50) COMMENT '二级分类',
    category_l3     VARCHAR(50) COMMENT '三级分类',
    brand           VARCHAR(100) COMMENT '品牌',
    supplier        VARCHAR(100) COMMENT '供应商',
    unit_price      DECIMAL(12,2) COMMENT '标准单价',
    cost_price      DECIMAL(12,2) COMMENT '成本价',
    weight          DECIMAL(8,2) COMMENT '重量(kg)',
    is_active       BOOLEAN DEFAULT TRUE COMMENT '是否在售'
) COMMENT '商品维度表';

-- ========================
-- 订单事实表
-- ========================
CREATE TABLE fact_order (
    order_key       BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '订单代理键',
    order_id        BIGINT NOT NULL COMMENT '订单编号(退化维度)',
    date_key        INT NOT NULL COMMENT '下单日期键',
    user_key        BIGINT NOT NULL COMMENT '用户代理键',
    product_key     BIGINT NOT NULL COMMENT '商品代理键',
    -- 度量值
    quantity        INT NOT NULL COMMENT '购买数量',
    unit_price      DECIMAL(12,2) NOT NULL COMMENT '成交单价',
    order_amount    DECIMAL(12,2) NOT NULL COMMENT '订单金额',
    discount_amount DECIMAL(12,2) DEFAULT 0 COMMENT '优惠金额',
    pay_amount      DECIMAL(12,2) NOT NULL COMMENT '实付金额',
    cost_amount     DECIMAL(12,2) COMMENT '成本金额',
    profit_amount   DECIMAL(12,2) COMMENT '利润',
    -- 外键约束
    FOREIGN KEY (date_key) REFERENCES dim_date(date_key),
    FOREIGN KEY (user_key) REFERENCES dim_user(user_key),
    FOREIGN KEY (product_key) REFERENCES dim_product(product_key)
) COMMENT '订单事实表';

-- 创建索引
CREATE INDEX idx_fact_order_date ON fact_order(date_key);
CREATE INDEX idx_fact_order_user ON fact_order(user_key);
CREATE INDEX idx_fact_order_product ON fact_order(product_key);
```

### 典型查询示例

```sql
-- 查询2024年各品类月度销售趋势
SELECT 
    d.year,
    d.month,
    d.month_name,
    p.category_l1,
    COUNT(DISTINCT f.order_id) AS order_count,
    SUM(f.quantity) AS total_quantity,
    SUM(f.pay_amount) AS total_revenue,
    SUM(f.profit_amount) AS total_profit,
    ROUND(SUM(f.profit_amount) / NULLIF(SUM(f.pay_amount), 0) * 100, 2) AS profit_rate
FROM fact_order f
JOIN dim_date d ON f.date_key = d.date_key
JOIN dim_product p ON f.product_key = p.product_key
WHERE d.year = 2024
GROUP BY d.year, d.month, d.month_name, p.category_l1
ORDER BY d.month, total_revenue DESC;

-- 用户RFM分析
WITH rfm AS (
    SELECT 
        u.user_id,
        u.username,
        DATEDIFF(CURDATE(), MAX(d.full_date)) AS recency,
        COUNT(DISTINCT f.order_id) AS frequency,
        SUM(f.pay_amount) AS monetary
    FROM fact_order f
    JOIN dim_user u ON f.user_key = u.user_key AND u.is_current = TRUE
    JOIN dim_date d ON f.date_key = d.date_key
    WHERE d.full_date >= DATE_SUB(CURDATE(), INTERVAL 365 DAY)
    GROUP BY u.user_id, u.username
)
SELECT 
    *,
    NTILE(5) OVER (ORDER BY recency DESC) AS r_score,
    NTILE(5) OVER (ORDER BY frequency ASC) AS f_score,
    NTILE(5) OVER (ORDER BY monetary ASC) AS m_score
FROM rfm
ORDER BY monetary DESC;
```

## 雪花模型

雪花模型是星型模型的扩展，将维度表进一步规范化，减少数据冗余但增加查询复杂度。

```sql
-- 品类维度表（规范化）
CREATE TABLE dim_category (
    category_key    INT AUTO_INCREMENT PRIMARY KEY,
    category_id     INT NOT NULL,
    category_name   VARCHAR(50) NOT NULL,
    parent_key      INT COMMENT '父品类键',
    category_level  TINYINT NOT NULL COMMENT '层级',
    FOREIGN KEY (parent_key) REFERENCES dim_category(category_key)
) COMMENT '品类维度表(雪花模型)';

-- 品牌维度表
CREATE TABLE dim_brand (
    brand_key       INT AUTO_INCREMENT PRIMARY KEY,
    brand_id        INT NOT NULL,
    brand_name      VARCHAR(100) NOT NULL,
    brand_country   VARCHAR(50) COMMENT '品牌国家',
    brand_group     VARCHAR(100) COMMENT '品牌集团'
) COMMENT '品牌维度表';

-- 雪花模型的商品维度表（引用品类和品牌）
CREATE TABLE dim_product_snowflake (
    product_key     BIGINT AUTO_INCREMENT PRIMARY KEY,
    product_id      BIGINT NOT NULL,
    product_name    VARCHAR(200) NOT NULL,
    category_key    INT NOT NULL,
    brand_key       INT NOT NULL,
    unit_price      DECIMAL(12,2),
    FOREIGN KEY (category_key) REFERENCES dim_category(category_key),
    FOREIGN KEY (brand_key) REFERENCES dim_brand(brand_key)
) COMMENT '商品维度表(雪花模型)';
```

### 星型 vs 雪花模型对比

| 特性 | 星型模型 | 雪花模型 |
|------|----------|----------|
| 查询性能 | 更快（少JOIN） | 较慢（多JOIN） |
| 数据冗余 | 有冗余 | 冗余少 |
| 维护成本 | 低 | 高 |
| ETL复杂度 | 简单 | 复杂 |
| 适用场景 | OLAP、BI报表 | OLTP+OLAP混合 |
| 推荐度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

## ETL流程

ETL（Extract-Transform-Load）是数据仓库的核心数据管道。

### ETL 架构示例

```python
"""
数据仓库ETL Pipeline
使用Python + SQLAlchemy实现
"""
import pandas as pd
from datetime import datetime, timedelta
from sqlalchemy import create_engine, text
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class ETLPipeline:
    """ETL数据管道"""
    
    def __init__(self, source_conn, dw_conn):
        self.source_engine = create_engine(source_conn)
        self.dw_engine = create_engine(dw_conn)
        self.batch_date = datetime.now().strftime('%Y-%m-%d')
    
    def extract_orders(self, execution_date: str) -> pd.DataFrame:
        """Extract: 从源系统提取订单数据"""
        query = f"""
            SELECT 
                o.order_id,
                o.order_date,
                o.user_id,
                o.status,
                o.total_amount,
                o.discount_amount,
                oi.product_id,
                oi.quantity,
                oi.unit_price,
                oi.item_amount
            FROM orders o
            JOIN order_items oi ON o.order_id = oi.order_id
            WHERE o.order_date = '{execution_date}'
              AND o.status IN ('completed', 'shipped')
        """
        df = pd.read_sql(query, self.source_engine)
        logger.info(f"Extracted {len(df)} order records for {execution_date}")
        return df
    
    def transform_orders(self, df: pd.DataFrame) -> pd.DataFrame:
        """Transform: 数据清洗、转换、派生"""
        # 1. 数据清洗
        df = df.dropna(subset=['order_id', 'user_id', 'product_id'])
        df = df[df['quantity'] > 0]
        df = df[df['unit_price'] > 0]
        
        # 2. 类型转换
        df['order_date'] = pd.to_datetime(df['order_date'])
        df['date_key'] = df['order_date'].dt.strftime('%Y%m%d').astype(int)
        
        # 3. 派生字段
        df['profit_amount'] = df['item_amount'] - df['item_amount'] * 0.6  # 假设成本60%
        df['pay_amount'] = df['item_amount'] - df['discount_amount']
        
        # 4. 查找代理键
        df = self._lookup_surrogate_keys(df)
        
        logger.info(f"Transformed {len(df)} records")
        return df
    
    def _lookup_surrogate_keys(self, df: pd.DataFrame) -> pd.DataFrame:
        """查找维度表的代理键"""
        # 查找用户代理键
        user_query = """
            SELECT user_key, user_id 
            FROM dim_user 
            WHERE is_current = TRUE
        """
        user_dim = pd.read_sql(user_query, self.dw_engine)
        df = df.merge(user_dim, on='user_id', how='left')
        
        # 查找商品代理键
        product_query = "SELECT product_key, product_id FROM dim_product"
        product_dim = pd.read_sql(product_query, self.dw_engine)
        df = df.merge(product_dim, on='product_id', how='left')
        
        return df
    
    def load_fact_orders(self, df: pd.DataFrame):
        """Load: 加载到事实表"""
        target_cols = [
            'order_id', 'date_key', 'user_key', 'product_key',
            'quantity', 'unit_price', 'item_amount', 'discount_amount',
            'pay_amount', 'profit_amount'
        ]
        
        # 重命名列以匹配目标表
        df_load = df[target_cols].rename(columns={'item_amount': 'order_amount'})
        
        # Upsert 写入
        df_load.to_sql(
            'fact_order', self.dw_engine,
            if_exists='append', index=False, method='multi', chunksize=1000
        )
        logger.info(f"Loaded {len(df_load)} records into fact_order")
    
    def run(self, execution_date: str):
        """执行完整ETL流程"""
        logger.info(f"=== ETL Pipeline Started for {execution_date} ===")
        start_time = datetime.now()
        
        try:
            # Extract
            raw_df = self.extract_orders(execution_date)
            
            # Transform
            transformed_df = self.transform_orders(raw_df)
            
            # Load
            self.load_fact_orders(transformed_df)
            
            elapsed = (datetime.now() - start_time).total_seconds()
            logger.info(f"=== ETL Pipeline Completed in {elapsed:.1f}s ===")
            
        except Exception as e:
            logger.error(f"ETL Pipeline Failed: {e}")
            raise

# 使用示例
# pipeline = ETLPipeline(
#     source_conn='mysql+pymysql://user:pass@source-db/ecommerce',
#     dw_conn='mysql+pymysql://user:pass@dw-db/data_warehouse'
# )
# pipeline.run('2024-01-15')
```

### ETL 最佳实践 - 幂等设计

```python
class IdempotentETL:
    """幂等ETL：可安全重复执行"""
    
    def load_with_partition_replace(self, df, table_name, partition_col, partition_value):
        """分区替换策略：先删后插"""
        with self.dw_engine.begin() as conn:
            # 先删除该分区数据
            conn.execute(text(
                f"DELETE FROM {table_name} WHERE {partition_col} = :pv"
            ), {'pv': partition_value})
            
            # 再插入新数据
            df.to_sql(table_name, conn, if_exists='append', index=False)
        
        logger.info(f"Replaced partition {partition_value} in {table_name}")
```

## 数据仓库分层架构

| 层次 | 名称 | 说明 | 典型技术 |
|------|------|------|----------|
| ODS | 操作数据层 | 原始数据副本，保持源系统结构 | MySQL Binlog、CDC |
| DWD | 明细数据层 | 清洗、标准化后的明细数据 | Spark、Flink |
| DWS | 汇总数据层 | 按主题聚合的宽表 | Spark SQL |
| ADS | 应用数据层 | 面向具体应用的结果表 | BI工具、API |
| DIM | 维度层 | 统一的维度表 | 全量/增量刷新 |

```sql
-- ODS层：原始数据镜像
CREATE TABLE ods_order (
```

## 最佳实践

1. **选择合适的粒度**：粒度是数据仓库设计最重要的决策，一旦确定难以更改
2. **代理键优先**：维度表使用自增代理键而非业务自然键
3. **SCD策略**：根据业务需求选择SCD Type 1（覆盖）、Type 2（全历史）或Type 3（保留前值）
4. **一致性维度**：跨业务过程共享一致的维度表（总线架构）
5. **事实表设计**：优先选择事务粒度，避免过度汇总
6. **ETL幂等**：确保ETL任务可安全重复执行
7. **数据质量**：在ETL各环节设置数据质量检查点
8. **增量加载**：优先使用增量加载而非全量刷新
9. **文档化**：维护数据字典和血缘关系图
