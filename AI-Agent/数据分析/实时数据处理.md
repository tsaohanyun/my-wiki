# 实时数据处理

## 概述

实时数据处理是指对持续到达的数据流进行即时处理和分析的技术。相比批处理，流处理能提供更低延迟的数据洞察，适用于实时监控、实时推荐、实时风控等场景。本文档涵盖流处理基础概念、Apache Flink、Spark Streaming和Kafka Streams。

---

## 流处理基础概念

### 核心术语

| 术语 | 说明 |
|------|------|
| Event Time | 事件实际发生的时间 |
| Processing Time | 事件被处理的时间 |
| Ingestion Time | 事件进入流处理系统的时间 |
| Watermark | 衡量事件时间进展的机制，用于处理乱序 |
| Window | 对无界数据的有界切分（滚动、滑动、会话） |
| State | 流处理中需要维护的中间状态 |
| Checkpoint | 状态的一致性快照，用于故障恢复 |

### 窗口类型

```
┌─────────────────────────────────────────────┐
│  滚动窗口 (Tumbling Window)                  │
│  [====][====][====][====]                    │
│  固定大小，不重叠                              │
├─────────────────────────────────────────────┤
│  滑动窗口 (Sliding Window)                   │
│  [======]                                    │
│    [======]                                  │
│      [======]                                │
│  固定大小，可重叠                              │
├─────────────────────────────────────────────┤
│  会话窗口 (Session Window)                   │
│  [==]   [====]  [===]                        │
│  基于活跃间隔动态划分                          │
└─────────────────────────────────────────────┘
```

## Apache Flink

Apache Flink 是分布式流处理引擎，提供精确一次（Exactly-Once）语义、事件时间处理和丰富的状态管理。

### Flink SQL 实时指标计算

```sql
-- 创建Kafka数据源表
CREATE TABLE user_events (
    user_id     BIGINT,
    event_type  STRING,
    page_url    STRING,
    event_time  TIMESTAMP(3),
    -- 定义事件时间与水印
    WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND
) WITH (
    'connector' = 'kafka',
    'topic' = 'user-events',
    'properties.bootstrap.servers' = 'kafka-broker:9092',
    'properties.group.id' = 'flink-analytics',
    'format' = 'json',
    'scan.startup.mode' = 'latest-offset'
);

-- 创建输出表（写入Kafka）
CREATE TABLE page_view_stats (
    window_start    TIMESTAMP(3),
    window_end      TIMESTAMP(3),
    page_url        STRING,
    view_count      BIGINT,
    unique_users    BIGINT,
    avg_session_dur DOUBLE
) WITH (
    'connector' = 'kafka',
    'topic' = 'page-view-stats',
    'properties.bootstrap.servers' = 'kafka-broker:9092',
    'format' = 'json'
);

-- 每分钟页面浏览统计
INSERT INTO page_view_stats
SELECT
    window_start,
    window_end,
    page_url,
    COUNT(*) AS view_count,
    COUNT(DISTINCT user_id) AS unique_users,
    AVG(session_duration) AS avg_session_dur
FROM TABLE(
    TUMBLE(TABLE user_events, DESCRIPTOR(event_time), INTERVAL '1' MINUTE)
)
WHERE event_type = 'page_view'
GROUP BY window_start, window_end, page_url;
```

### Flink DataStream API - 实时风控

```java
import org.apache.flink.streaming.api.datastream.DataStream;
import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;
import org.apache.flink.streaming.api.windowing.assigners.TumblingEventTimeWindows;
import org.apache.flink.streaming.api.windowing.time.Time;

public class RealTimeRiskDetection {
    
    public static void main(String[] args) throws Exception {
        StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();
        env.setParallelism(4);
        // 启用检查点，每60秒一次
        env.enableCheckpointing(60000);
        
        // 从Kafka读取交易数据
        DataStream<Transaction> transactions = env
            .addSource(new FlinkKafkaConsumer<>(
                "transactions",
                new TransactionDeserializer(),
                kafkaProperties
            ))
            .assignTimestampsAndWatermarks(
                WatermarkStrategy
                    .<Transaction>forBoundedOutOfOrderness(Duration.ofSeconds(5))
                    .withTimestampAssigner((event, timestamp) -> event.getTimestamp())
            );
        
        // 规则1: 同一用户5分钟内超过3次大额交易
        DataStream<Alert> highFreqAlerts = transactions
            .filter(t -> t.getAmount() > 10000)
            .keyBy(Transaction::getUserId)
            .window(TumblingEventTimeWindows.of(Time.minutes(5)))
            .process(new ProcessWindowFunction<Transaction, Alert, Long, TimeWindow>() {
                @Override
                public void process(Long userId, Context ctx,
                                    Iterable<Transaction> elements, Collector<Alert> out) {
                    List<Transaction> txns = new ArrayList<>();
                    elements.forEach(txns::add);
                    if (txns.size() >= 3) {
                        double totalAmount = txns.stream()
                            .mapToDouble(Transaction::getAmount).sum();
                        out.collect(new Alert(
                            userId,
                            "HIGH_FREQ_LARGE_TXN",
                            "用户" + userId + "在5分钟内有" + txns.size() +
                            "笔大额交易，总计" + totalAmount + "元",
                            AlertLevel.HIGH
                        ));
                    }
                }
            });
        
        // 规则2: 异常金额检测（Z-score方法）
        DataStream<Alert> amountAnomalyAlerts = transactions
            .keyBy(Transaction::getUserId)
            .process(new AmountAnomalyDetector(3.0)); // Z-score > 3 视为异常
        
        // 合并告警写入下游
        highFreqAlerts.union(amountAnomalyAlerts)
            .addSink(new AlertSink());
        
        env.execute("Real-time Risk Detection");
    }
}
```

```python
from pyflink.table import EnvironmentSettings, TableEnvironment
from pyflink.table.expressions import col, lit

env_settings = EnvironmentSettings.in_streaming_mode()
t_env = TableEnvironment.create(env_settings)

# 设置checkpoint
t_env.get_config().set("execution.checkpointing.interval", "60s")

# 定义源表
t_env.execute_sql("""
    CREATE TABLE orders_source (
        order_id     BIGINT,
        user_id      BIGINT,
        product_id   BIGINT,
        quantity     INT,
        amount       DECIMAL(12,2),
        order_time   TIMESTAMP(3),
        WATERMARK FOR order_time AS order_time - INTERVAL '10' SECOND
    ) WITH (
        'connector' = 'kafka',
        'topic' = 'orders',
        'properties.bootstrap.servers' = 'localhost:9092',
        'format' = 'json',
        'scan.startup.mode' = 'latest-offset'
    )
""")

# 定义输出表
t_env.execute_sql("""
    CREATE TABLE orders_sink (
        window_start  TIMESTAMP(3),
        product_id    BIGINT,
        order_count   BIGINT,
        total_amount  DECIMAL(14,2),
        avg_amount    DECIMAL(12,2)
    ) WITH (
        'connector' = 'jdbc',
        'url' = 'jdbc:mysql://localhost:3306/analytics',
        'table-name' = 'product_order_stats',
        'username' = 'root',
        'password' = 'password'
    )
""")

# 实时聚合查询
t_env.execute_sql("""
    INSERT INTO orders_sink
    SELECT
        window_start,
        product_id,
        COUNT(*) AS order_count,
        SUM(amount) AS total_amount,
        AVG(amount) AS avg_amount
    FROM TABLE(
        TUMBLE(TABLE orders_source, DESCRIPTOR(order_time), INTERVAL '5' MINUTE)
    )
    GROUP BY window_start, product_id
""")
```

## Spark Streaming

Spark Streaming 使用微批处理模型，适合对延迟要求不那么严格（秒级）的场景。

### 结构化流处理 - 实时仪表板

```python
from pyspark.sql import SparkSession
from pyspark.sql.functions import (
    col, window, count, avg, sum as spark_sum,
    from_json, current_timestamp, expr
)
from pyspark.sql.types import (
    StructType, StructField, StringType,
    LongType, DoubleType, TimestampType
)

# 创建Spark Session
spark = SparkSession.builder \
    .appName("RealTimeDashboard") \
    .config("spark.sql.shuffle.partitions", "8") \
    .config("spark.streaming.backpressure.enabled", "true") \
    .getOrCreate()

# 定义数据Schema
event_schema = StructType([
    StructField("user_id", LongType(), True),
    StructField("event_type", StringType(), True),
    StructField("page", StringType(), True),
    StructField("duration_sec", DoubleType(), True),
    StructField("timestamp", TimestampType(), True)
])

# 从Kafka读取数据流
raw_stream = spark.readStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "localhost:9092") \
    .option("subscribe", "user-events") \
    .option("startingOffsets", "latest") \
    .load()

# 解析JSON
events = raw_stream \
    .selectExpr("CAST(value AS STRING)") \
    .select(from_json(col("value"), event_schema).alias("data")) \
    .select("data.*") \
    .withWatermark("timestamp", "30 seconds")

# 实时统计：每5分钟的页面浏览量和独立用户数
page_stats = events \
    .filter(col("event_type") == "page_view") \
    .groupBy(
        window(col("timestamp"), "5 minutes", "1 minute"),
        col("page")
    ) \
    .agg(
        count("*").alias("view_count"),
        count("user_id").alias("unique_users"),
        avg("duration_sec").alias("avg_duration")
    )

# 输出到控制台（调试用）
console_query = page_stats.writeStream \
    .outputMode("update") \
    .format("console") \
    .option("truncate", "false") \
    .trigger(processingTime="30 seconds") \
    .start()

# 输出到Redis（实时仪表板）
def write_to_redis(batch_df, batch_id):
    """将结果写入Redis"""
    import redis
    r = redis.Redis(host='localhost', port=6379, db=0)
    for row in batch_df.collect():
        key = f"page_stats:{row.page}:{row.window.start}"
        r.hset(key, mapping={
            "view_count": str(row.view_count),
            "unique_users": str(row.unique_users),
            "avg_duration": str(round(row.avg_duration, 2))
        })
        r.expire(key, 3600)  # 1小时过期

redis_query = page_stats.writeStream \
    .outputMode("update") \
    .foreachBatch(write_to_redis) \
    .trigger(processingTime="30 seconds") \
    .start()

spark.streams.awaitAnyTermination()
```

### Spark Streaming 状态管理

```python
from pyspark.sql.functions import collect_set, size

# 有状态处理：用户会话分析
session_events = events \
    .withWatermark("timestamp", "5 minutes") \
    .groupBy(
        col("user_id"),
        window(col("timestamp"), "30 minutes", "5 minutes")
    ) \
    .agg(
        count("*").alias("event_count"),
        collect_set("page").alias("visited_pages"),
        spark_sum("duration_sec").alias("total_duration")
    ) \
    .withColumn("page_count", size(col("visited_pages"))) \
    .filter(col("event_count") > 10)  # 过滤高活跃用户

# 输出到Kafka
session_events.selectExpr(
    "CAST(user_id AS STRING) AS key",
    "to_json(struct(*)) AS value"
).writeStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "localhost:9092") \
    .option("topic", "user-sessions") \
    .outputMode("update") \
    .option("checkpointLocation", "/tmp/checkpoint/sessions") \
    .start()
```

## Kafka Streams

Kafka Streams 是一个轻量级的客户端库，适合嵌入到Java应用中进行流处理。

```java
import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.*;

import java.time.Duration;
import java.util.Properties;

public class RealTimeMetrics {
    
    public static void main(String[] args) {
        Properties config = new Properties();
        config.put(StreamsConfig.APPLICATION_ID_CONFIG, "real-time-metrics");
        config.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        config.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, Serdes.String().getClass());
        config.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, Serdes.String().getClass());
        // 精确一次语义
        config.put(StreamsConfig.PROCESSING_GUARANTEE_CONFIG, "exactly_once_v2");
        
        StreamsBuilder builder = new StreamsBuilder();
        
        // 从输入topic读取事件
        KStream<String, String> events = builder.stream("user-events");
        
        // 解析、过滤、聚合
        KTable<Windowed<String>, Long> pageViewCounts = events
            .mapValues(value -> parseEvent(value))
            .filter((key, event) -> event.getType().equals("page_view"))
            .groupBy((key, event) -> event.getPage(), 
                     Serialized.with(Serdes.String(), eventSerde))
            .windowedBy(TimeWindows.ofSizeWithNoGrace(Duration.ofMinutes(5)))
            .count();
        
        // 输出到结果topic
        pageViewCounts.toStream()
            .map((windowedKey, count) -> {
                String key = windowedKey.key() + "@" + windowedKey.window().start();
                String value = String.format(
                    "{\"page\":\"%s\",\"window_start\":%d,\"count\":%d}",
                    windowedKey.key(), windowedKey.window().start(), count
                );
                return KeyValue.pair(key, value);
            })
            .to("page-view-counts", Produced.with(Serdes.String(), Serdes.String()));
        
        // 启动应用
        KafkaStreams streams = new KafkaStreams(builder.build(), config);
        
        // 优雅关闭
        Runtime.getRuntime().addShutdownHook(new Thread(streams::close));
        
        streams.start();
    }
}
```

### Kafka Streams 交互式查询

```java
// 支持从外部查询流处理的实时状态
public class InteractiveQueryService {
    
    private final KafkaStreams streams;
    
    public Long getPageViewCount(String page) {
        // 从本地状态存储查询
        ReadOnlyWindowStore<String, Long> store = streams.store(
            StoreQueryParameters.fromNameAndType(
                "page-view-counts-store", 
                QueryableStoreTypes.windowStore()
            )
        );
        
        // 查询最近5分钟的数据
        Instant now = Instant.now();
        Instant fiveMinAgo = now.minus(Duration.ofMinutes(5));
        
        try (WindowStoreIterator<Long> iterator = 
                store.fetch(page, fiveMinAgo, now)) {
            long total = 0;
            while (iterator.hasNext()) {
                total += iterator.next().value;
            }
            return total;
        }
    }
}
```

## 最佳实践

1. **时间语义明确**：始终使用事件时间而非处理时间，确保结果可重放一致
2. **合理设置水印**：水印过小导致数据丢失，过大导致延迟增大
3. **状态后端选择**：大状态使用RocksDB，小状态使用内存
4. **检查点配置**：根据延迟容忍度和恢复时间设置合理的checkpoint间隔
5. **背压处理**：启用背压机制，防止下游处理不过来导致OOM
6. **序列化优化**：使用Kryo或Avro替代Java默认序列化，提升性能
7. **监控告警**：监控延迟、吞吐量、checkpoint失败率等关键指标
8. **幂等输出**：下游存储使用幂等写入，保证exactly-once语义
9. **死信队列**：异常数据写入死信队列，避免阻塞主流
10. **资源隔离**：不同优先级的作业使用独立的资源槽位
