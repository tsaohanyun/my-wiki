# 前端面试题集

## 概述

本文档整理前端开发面试中的高频题目，涵盖 JavaScript 核心、CSS 布局、主流框架（React/Vue）、性能优化和手写代码题。

## 一、JavaScript 核心

### 1.1 数据类型与类型判断

**基本类型：** `undefined`, `null`, `boolean`, `number`, `string`, `symbol`, `bigint`
**引用类型：** `Object`（含 Array, Function, Date, RegExp 等）

**类型判断方法对比：**

| 方法 | 能力 | 注意事项 |
|-----|------|---------|
| `typeof` | 判断基本类型 | `typeof null === 'object'`（历史Bug） |
| `instanceof` | 判断引用类型 | 不能跨 iframe/窗口 |
| `Object.prototype.toString` | 精确判断 | 最佳实践 |
| `Array.isArray()` | 判断数组 | 推荐 |

```javascript
// 最佳类型判断
function getType(obj) {
  return Object.prototype.toString.call(obj).slice(8, -1).toLowerCase();
}
// getType([]) → 'array'
// getType(null) → 'null'
```

### 1.2 闭包

**定义：** 函数有权访问其词法作用域中的变量，即使函数在其作用域外执行。

**经典面试题：**

```javascript
// 题目1: var 声明的 for 循环
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 输出: 3, 3, 3

// 解决方案1: let
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 输出: 0, 1, 2

// 解决方案2: IIFE
for (var i = 0; i < 3; i++) {
  (function(j) {
    setTimeout(() => console.log(j), 0);
  })(i);
}
// 输出: 0, 1, 2
```

### 1.3 原型与原型链

```javascript
// 原型链核心关系
// 实例.__proto__ === 构造函数.prototype
// 构造函数.prototype.__proto__ === Object.prototype
// Object.prototype.__proto__ === null

function Person(name) { this.name = name; }
Person.prototype.sayHi = function() { console.log(`Hi, ${this.name}`); };

const p = new Person('Alice');
p.sayHi(); // Hi, Alice
console.log(p.__proto__ === Person.prototype); // true
console.log(Person.prototype.__proto__ === Object.prototype); // true
```

**关键概念：**
- `prototype` 是函数特有的属性
- `__proto__` 是所有对象都有的属性
- 原型链的尽头是 `null`

### 1.4 this 指向

| 调用方式 | this 指向 |
|---------|----------|
| 普通函数调用 | `window` / `undefined`（严格模式） |
| 对象方法调用 | 调用该方法的对象 |
| `new` 调用 | 新创建的实例对象 |
| `call/apply/bind` | 指定的第一个参数 |
| 箭头函数 | 定义时所在作用域的 `this` |

### 1.5 事件循环

```javascript
console.log('1'); // 同步

setTimeout(() => console.log('2'), 0); // 宏任务

Promise.resolve().then(() => console.log('3')); // 微任务

console.log('4'); // 同步

// 执行顺序: 1 → 4 → 3 → 2
```

**宏任务：** setTimeout, setInterval, I/O, UI Rendering
**微任务：** Promise.then, MutationObserver, queueMicrotask

**执行顺序：** 同步代码 → 微任务队列清空 → 一个宏任务 → 微任务队列清空 → 循环

### 1.6 Promise

```javascript
// Promise 状态: pending → fulfilled / rejected

// 题目: Promise 链式调用
Promise.resolve(1)
  .then(val => { console.log(val); return val + 1; }) // 1
  .then(val => { console.log(val); return val + 1; }) // 2
  .catch(err => console.log(err))
  .finally(() => console.log('done')); // done
```

**面试常考：**
- Promise.all / Promise.race / Promise.allSettled / Promise.any 的区别
- async/await 原理（Generator + 自动执行器）
- 并发控制（实现 limit 并发的请求池）

### 1.7 ES6+ 重点

| 特性 | 说明 |
|-----|------|
| `let/const` | 块级作用域、暂时性死区 |
| 箭头函数 | 没有自己的 `this`、`arguments` |
| 解构赋值 | 数组/对象解构、默认值 |
| 模板字符串 | 多行字符串、插值 |
| 扩展运算符 `...` | 数组/对象展开 |
| `Map/Set` | 新数据结构 |
| Proxy/Reflect | 元编程能力 |
| Class 语法 | 原型链的语法糖 |
| 模块化 `import/export` | ES Module |
| 可选链 `?.` / 空值合并 `??` | 安全访问 |

## 二、CSS

### 2.1 盒模型

- **标准盒模型：** `width = content`，`box-sizing: content-box`
- **IE盒模型：** `width = content + padding + border`，`box-sizing: border-box`

### 2.2 居中方案

```css
/* Flex 居中（最常用） */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Grid 居中 */
.parent {
  display: grid;
  place-items: center;
}

/* 绝对定位 + transform */
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* 绝对定位 + margin auto */
.child {
  position: absolute;
  top: 0; right: 0; bottom: 0; left: 0;
  margin: auto;
  width: 100px;
  height: 100px;
}
```

### 2.3 Flexbox 关键属性

| 属性 | 说明 |
|-----|------|
| `flex-direction` | 主轴方向 row/column |
| `justify-content` | 主轴对齐 |
| `align-items` | 交叉轴对齐 |
| `flex-wrap` | 是否换行 |
| `flex: 1` | `flex-grow: 1; flex-shrink: 1; flex-basis: 0%` |

### 2.4 BFC（块级格式化上下文）

**触发条件：**
- `overflow: hidden/auto/scroll`
- `float` 不为 `none`
- `position: absolute/fixed`
- `display: flow-root / flex / grid / inline-block`

**作用：**
- 清除浮动
- 防止 margin 重叠
- 阻止元素被浮动元素覆盖

### 2.5 响应式设计

```css

  .container { flex-direction: column; }
}

/* rem / vw / vh / clamp */

.responsive-text {
  font-size: clamp(1rem, 2vw, 1.5rem);
}
```

```
!important > 内联样式(1000) > ID(100) > 类/伪类/属性(10) > 标签/伪元素(1) > 通配符(0)
```

## 三、框架

### 3.1 React

**核心概念：**
- Virtual DOM 与 Diff 算法
- 生命周期（mount → update → unmount）
- Hooks: `useState`, `useEffect`, `useMemo`, `useCallback`, `useRef`, `useContext`
- 状态管理: Context, Redux, Zustand, Jotai
- 性能优化: `React.memo`, `useMemo`, `useCallback`, 虚拟列表

**高频面试题：**

1. **React 的 Diff 算法原理？**
   - 同层比较（Tree Diff）
   - 列表使用 key（Component Diff）
   - 同类型组件继续比较，不同类型直接替换（Element Diff）

2. **useEffect vs useLayoutEffect？**
   - `useEffect` 异步执行，不阻塞浏览器渲染
   - `useLayoutEffect` 同步执行，在 DOM 变更后、浏览器绘制前

3. **React Fiber 是什么？**
   - 可中断、可恢复的协调过程
   - 将渲染工作拆分为多个小单元（fiber）
   - 支持优先级调度和并发模式

4. **Redux 数据流？**
   ```
   Action → Dispatcher → Reducer → Store → View
   ```

### 3.2 Vue

**核心概念：**
- 响应式原理（Vue 2: Object.defineProperty; Vue 3: Proxy）
- 组件通信（props/emit, provide/inject, Vuex/Pinia, EventBus）
- 生命周期钩子
- 计算属性 vs 侦听器
- 虚拟 DOM 与 Diff

**高频面试题：**

1. **Vue 2 vs Vue 3 响应式原理？**
   - Vue 2: `Object.defineProperty`（无法监听数组索引变化和新增属性）
   - Vue 3: `Proxy`（全面代理，性能更好）

2. **computed 和 watch 的区别？**
   - `computed`: 有缓存，依赖不变时不会重新计算
   - `watch`: 无缓存，监听值变化时执行回调

3. **Vue 的 nextTick 原理？**
   - 利用微任务（Promise.then）在下次 DOM 更新后执行回调

4. **v-for 中为什么要加 key？**
   - 帮助 Diff 算法识别节点，减少不必要的 DOM 操作

| 预加载 | `<link rel="preload">` |
| 缓存策略 | 强缓存、协商缓存 |

### 4.2 渲染性能

| 优化手段 | 说明 |
|---------|------|
| 减少 DOM 操作 | 使用 DocumentFragment |
| 防抖/节流 | 限制事件触发频率 |

| CSS 硬件加速 | transform, opacity |
| requestAnimationFrame | 动画优化 |
| Web Worker | CPU 密集任务 |

### 4.3 防抖与节流

```javascript
// 防抖（Debounce）：多次触发只执行最后一次
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流（Throttle）：固定时间间隔执行一次
function throttle(fn, interval) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}
```

### 4.4 核心 Web 指标

| 指标 | 含义 | 目标 |
|-----|------|------|
| LCP (Largest Contentful Paint) | 最大内容绘制 | < 2.5s |
| FID (First Input Delay) | 首次输入延迟 | < 100ms |
| CLS (Cumulative Layout Shift) | 累积布局偏移 | < 0.1 |
| INP (Interaction to Next Paint) | 交互到下次渲染 | < 200ms |

## 五、手写代码题

### 5.1 实现 new

```javascript
function myNew(constructor, ...args) {
  const obj = Object.create(constructor.prototype);
  const result = constructor.apply(obj, args);
  return result instanceof Object ? result : obj;
}
```

### 5.2 实现 call/apply/bind

```javascript
// call
Function.prototype.myCall = function(context = window, ...args) {
  const fn = Symbol('fn');
  context[fn] = this;
  const result = context[fn](...args);
  delete context[fn];
  return result;
};

// bind
Function.prototype.myBind = function(context = window, ...args) {
  const self = this;
  return function(...innerArgs) {
    return self.apply(context, [...args, ...innerArgs]);
  };
};
```

### 5.3 实现 Promise

```javascript
class MyPromise {
  constructor(executor) {
    this.state = 'pending';
    this.value = undefined;
    this.reason = undefined;
    this.onFulfilledCallbacks = [];
    this.onRejectedCallbacks = [];

    const resolve = (value) => {
      if (this.state === 'pending') {
        this.state = 'fulfilled';
        this.value = value;
        this.onFulfilledCallbacks.forEach(fn => fn());
      }
    };

    const reject = (reason) => {
      if (this.state === 'pending') {
        this.state = 'rejected';
        this.reason = reason;
        this.onRejectedCallbacks.forEach(fn => fn());
      }
    };

    try {
      executor(resolve, reject);
    } catch (err) {
      reject(err);
    }
  }

  then(onFulfilled, onRejected) {
    onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : v => v;
    onRejected = typeof onRejected === 'function' ? onRejected : e => { throw e; };

    const promise2 = new MyPromise((resolve, reject) => {
      const fulfilledTask = () => {
        queueMicrotask(() => {
          try {
            const x = onFulfilled(this.value);
            this.resolvePromise(promise2, x, resolve, reject);
          } catch (err) {
            reject(err);
          }
        });
      };

      const rejectedTask = () => {
        queueMicrotask(() => {
          try {
            const x = onRejected(this.reason);
            this.resolvePromise(promise2, x, resolve, reject);
          } catch (err) {
            reject(err);
          }
        });
      };

      if (this.state === 'fulfilled') fulfilledTask();
      else if (this.state === 'rejected') rejectedTask();
      else {
        this.onFulfilledCallbacks.push(fulfilledTask);
        this.onRejectedCallbacks.push(rejectedTask);
      }
    });

    return promise2;
  }

  resolvePromise(promise2, x, resolve, reject) {
    if (promise2 === x) return reject(new TypeError('Chaining cycle'));
    if (x instanceof MyPromise) {
      x.then(resolve, reject);
    } else if (x !== null && typeof x === 'object' || typeof x === 'function') {
      try {
        const then = x.then;
        if (typeof then === 'function') {
          then.call(x, y => this.resolvePromise(promise2, y, resolve, reject), reject);
        } else {
          resolve(x);
        }
      } catch (err) {
        reject(err);
      }
    } else {
      resolve(x);
    }
  }
}
```

### 5.4 深拷贝

```javascript
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);
  if (map.has(obj)) return map.get(obj); // 处理循环引用

  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone);

  for (const key of Reflect.ownKeys(obj)) {
    clone[key] = deepClone(obj[key], map);
  }
  return clone;
}
```

### 5.5 发布订阅模式

```javascript
class EventEmitter {
  constructor() {
    this.events = {};
  }

  on(event, callback) {
    (this.events[event] || (this.events[event] = [])).push(callback);
    return this;
  }

  off(event, callback) {
    if (!this.events[event]) return this;
    this.events[event] = this.events[event].filter(fn => fn !== callback);
    return this;
  }

  once(event, callback) {
    const wrapper = (...args) => {
      callback(...args);
      this.off(event, wrapper);
    };
    this.on(event, wrapper);
    return this;
  }

  emit(event, ...args) {
    (this.events[event] || []).forEach(fn => fn(...args));
    return this;
  }
}
```

### 5.6 实现 instanceOf

```javascript
function myInstanceOf(obj, constructor) {
  let proto = Object.getPrototypeOf(obj);
  while (proto) {
    if (proto === constructor.prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
  return false;
}
```

### 5.7 数组扁平化

```javascript
// 递归
function flatten(arr, depth = Infinity) {
  return depth > 0
    ? arr.reduce((acc, val) =>
        acc.concat(Array.isArray(val) ? flatten(val, depth - 1) : val), [])
    : arr.slice();
}

// 迭代（栈）
function flattenIterative(arr) {
  const stack = [...arr];
  const result = [];
  while (stack.length) {
    const item = stack.pop();
    if (Array.isArray(item)) {
      stack.push(...item);
    } else {
      result.unshift(item);
    }
  }
  return result;
}
```

### 5.8 并发请求控制

```javascript
async function concurrentRequest(urls, limit = 3) {
  const results = [];
  const executing = new Set();

  for (const url of urls) {
    const promise = fetch(url).then(res => {
      results.push(res);
      executing.delete(promise);
    });
    executing.add(promise);

    if (executing.size >= limit) {
      await Promise.race(executing);
    }
  }

  await Promise.all(executing);
  return results;
}
```

## 六、网络与浏览器

### 6.1 HTTP 缓存

| 类型 | 头部 | 说明 |
|-----|------|------|
| 强缓存 | `Cache-Control: max-age` / `Expires` | 不发请求，直接用缓存 |
| 协商缓存 | `ETag` + `If-None-Match` | 发请求验证，304 则用缓存 |
| 协商缓存 | `Last-Modified` + `If-Modified-Since` | 同上（精度秒级） |

### 6.2 浏览器渲染流程

```
HTML → DOM Tree
CSS → CSSOM
DOM + CSSOM → Render Tree → Layout → Paint → Composite
```

### 6.3 重排与重绘

- **重排（Reflow）：** 改变几何属性（width, height, margin 等），触发布局重新计算
- **重绘（Repaint）：** 改变外观属性（color, background 等），不改变布局
- **优化：** 使用 `transform` 代替 `top/left`，使用 `visibility: hidden` 代替 `display: none`
