# 数据结构与算法面试

## 概述

数据结构与算法是技术面试中的核心考察内容，几乎所有的中大型公司都会在面试中涉及。本文档涵盖常见算法题型、解题模板、复杂度分析以及高效的刷题策略。

## 一、核心数据结构

### 1.1 数组与字符串

| 数据结构 | 典型应用 | 常见复杂度 |
|---------|---------|-----------|
| 数组 | 随机访问、双指针 | 访问 O(1), 插入/删除 O(n) |
| 动态数组 | 可变长度存储 | 均摊 O(1) 追加 |
| 字符串 | 模式匹配、回文 | 取决于操作 |

**高频题目：**
- 两数之和（HashMap）
- 最长无重复字符子串（滑动窗口）
- 三数之和（排序 + 双指针）
- 反转字符串 / 翻转单词
- 字符串相乘、大数加法

### 1.2 链表

**高频题目：**
- 反转链表（迭代 / 递归）
- 合并两个有序链表
- 检测环 / 找环入口（快慢指针）
- 链表中倒数第K个节点
- 删除链表倒数第N个节点
- 两两交换节点 / K个一组反转
- 回文链表

```python
# 反转链表模板
def reverseList(head):
    prev, cur = None, head
    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    return prev
```

- 用栈实现队列

### 1.4 树

**高频题目：**
- 二叉树的前/中/后/层序遍历
- 二叉树的最大深度 / 最小深度
- 对称二叉树
- 翻转二叉树
- 最近公共祖先（LCA）
- 二叉搜索树验证 / 搜索 / 插入 / 删除

- 从前序和中序重建二叉树
- 二叉树展开为链表

```python
# BFS层序遍历模板
from collections import deque
def levelOrder(root):
    if not root: return []
    result = []
    queue = deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):
            node = queue.popleft()
            level.append(node.val)
            if node.left: queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result
```

## 二、核心算法范式

### 2.1 双指针

```python
# 快慢指针模板（原地修改）
def twoPointer(nums):
    slow = 0
    for fast in range(len(nums)):
        if condition(nums[fast]):
            nums[slow] = nums[fast]
            slow += 1
    return slow
```

**适用场景：** 原地数组修改、有序数组操作、链表操作

### 2.2 滑动窗口

```python
# 滑动窗口模板
def slidingWindow(s):
    window = {}
    left = 0
    result = 0
    for right in range(len(s)):
        c = s[right]
        window[c] = window.get(c, 0) + 1
        while needShrink(window):
            d = s[left]
            window[d] -= 1
            left += 1
        result = max(result, right - left + 1)
    return result
```

**适用场景：** 子串/子数组问题，连续区间问题

### 2.3 二分查找

```python
# 二分查找模板（左闭右闭）
def binarySearch(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

**适用场景：** 有序数组查找、旋转数组、查找边界、数学问题中的二分

### 2.4 回溯算法

```python
# 回溯模板
def backtrack(path, choices):
    if isSolution(path):
        result.append(path[:])
        return
    for choice in choices:
        if isValid(choice):
            path.append(choice)
            backtrack(path, newChoices)
            path.pop()  # 撤销选择
```

**适用场景：** 全排列、子集、组合、N皇后、解数独、分割问题

### 2.5 动态规划

```python
# 动态规划模板
def dp():
    # 1. 定义状态
    dp = [0] * (n + 1)
    # 2. 初始化
    dp[0] = base_case
    # 3. 状态转移
    for i in range(1, n + 1):
        dp[i] = transition(dp[i-1], ...)
    # 4. 返回结果
    return dp[n]
```

**分类：**
- **一维DP：** 爬楼梯、打家劫舍、单词拆分
- **二维DP：** 最长公共子序列、编辑距离、不同路径
- **背包问题：** 0-1背包、完全背包、分割等和子集
- **区间DP：** 戳气球、矩阵链乘法
- **状态压缩DP：** 旅行商问题、小集合子集枚举
- **树形DP：** 打家劫舍III、二叉树最大路径和

### 2.6 贪心算法

**适用场景：** 区间调度、跳跃游戏、分发糖果、股票买卖

### 2.7 排序算法

| 排序 | 时间(平均) | 时间(最坏) | 空间 | 稳定 |
|-----|----------|----------|------|------|
| 快速排序 | O(nlogn) | O(n²) | O(logn) | ❌ |
| 归并排序 | O(nlogn) | O(nlogn) | O(n) | ✅ |
| 堆排序 | O(nlogn) | O(nlogn) | O(1) | ❌ |
| 桶排序 | O(n+k) | O(n²) | O(n+k) | ✅ |

### 2.8 并查集

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # 路径压缩
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py: return False
        if self.rank[px] < self.rank[py]:
            px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]:
            self.rank[px] += 1
        return True
```

**适用场景：** 连通分量、朋友圈、最小生成树（Kruskal）

## 三、复杂度分析速查

### 时间复杂度

| 复杂度 | 含义 | 典型算法 |
|-------|------|---------|
| O(1) | 常数 | 哈希表查找 |
| O(logn) | 对数 | 二分查找 |
| O(n) | 线性 | 遍历数组 |
| O(nlogn) | 线性对数 | 排序、归并 |
| O(n²) | 平方 | 嵌套循环 |
| O(n³) | 立方 | 矩阵乘法 |
| O(2ⁿ) | 指数 | 暴力递归 |
| O(n!) | 阶乘 | 全排列 |

### 空间复杂度注意事项
- 递归的调用栈深度
- DP中是否可以状态压缩（滚动数组）
- 是否可以使用原地算法

## 四、刷题策略

### 4.1 分阶段计划

**第一阶段：基础（1-2个月）**
- 按数据结构分类刷题
- 每类完成 10-15 道 Easy/Medium
- 重点理解模板和套路

**第二阶段：进阶（1-2个月）**
- 按算法范式分类刷题
- Medium 为主，少量 Hard
- 总结每类题型的解题模板

**第三阶段：冲刺（2-4周）**
- 高频题 / 面经题集中突破
- 限时模拟（Medium 20分钟，Hard 40分钟）
- 公司专项（如 Google 偏 DP/Graph，Facebook 偏数组/树）

### 4.2 推荐刷题顺序

1. **Hot 100** → 100题经典精选
2. **剑指 Offer** → 国内面试高频
3. **算法小抄 / 程序员面试金典** → 分类专题
4. **公司标签高频题** → 针对性准备

### 4.3 刷题技巧

- ✅ **先思考再编码：** 至少花 5 分钟理解题意和思考方案
- ✅ **多解法对比：** 暴力 → 优化 → 最优，说出复杂度分析
- ✅ **画图辅助：** 链表、树、DP 的状态转移表先画出来
- ✅ **总结规律：** 同类题放在一起，提炼模板
- ✅ **间隔重复：** 隔 3 天 / 7 天 / 15 天重做错题
- ❌ **不要死磕：** 超过 30 分钟没有思路就看题解
- ❌ **不要只看不练：** 每道题必须亲手编码

### 4.4 面试中的算法沟通技巧

1. **澄清问题：** 边界条件、输入规模、数据类型
2. **举例验证：** 先用一个例子 walkthrough
3. **先说思路：** 从暴力解开始，逐步优化
4. **编码时口述：** 一边写一边解释
5. **测试验证：** 写完后用例子跑一遍
6. **复杂度分析：** 主动说出时间和空间复杂度
