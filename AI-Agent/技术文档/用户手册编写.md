## 1. 概述

### 1.1 用户文档分类

| 类型 | 目标读者 | 核心目标 | 示例 |
|------|---------|---------|------|
| 用户指南 | 最终用户 | 理解产品功能，学会使用 | 「如何创建项目」 |
| 操作手册 | 运维/管理员 | 完成特定操作任务 | 「数据库备份操作步骤」 |
| FAQ | 所有用户 | 快速解决常见问题 | 「忘记密码怎么办？」 |
| 快速入门 | 新用户 | 5 分钟内上手 | 「3 步快速开始」 |
| 故障排除 | 遇到问题的用户 | 自助解决问题 | 「常见错误码说明」 |

### 1.2 用户文档 vs 开发者文档

| 维度 | 用户文档 | 开发者文档 |
|------|---------|-----------|
| 读者 | 非技术人员 | 技术人员 |
| 语言 | 通俗易懂 | 专业术语 |
| 重点 | 操作步骤 | 技术原理 |
| 示例 | 界面截图 | 代码示例 |
| 格式 | 图文并茂 | 文字为主 |

### 3.2 标准操作程序（SOP）模板

```markdown
## SOP-001: 数据库备份操作

### 操作信息

| 项目 | 内容 |
|------|------|
| 操作名称 | PostgreSQL 数据库全量备份 |
| 执行频率 | 每日凌晨 2:00（自动）/ 按需（手动） |
| 预计耗时 | 15-30 分钟（视数据量） |
| 风险等级 | 🟢 低 |
| 所需权限 | DBA |

### 前置条件

- [ ] 确认备份存储空间充足（至少 50GB 可用）
- [ ] 确认没有正在运行的大批量写入操作
- [ ] 通知相关团队备份窗口

### 操作步骤

#### 1. 检查磁盘空间

\```bash
# 检查备份目录可用空间
df -h /backup

# 确认至少有 50GB 可用空间
\```

#### 2. 执行备份

\```bash
# 全量备份（使用 pg_dump）
pg_dump -h localhost -U postgres -F c -f /backup/db_$(date +%Y%m%d).dump mydb

# 验证备份文件
ls -lh /backup/db_*.dump
\```

#### 3. 验证备份完整性

\```bash
# 恢复到临时数据库验证
createdb test_restore
pg_restore -d test_restore /backup/db_20260628.dump

# 检查表数量
psql -d test_restore -c "SELECT count(*) FROM information_schema.tables WHERE table_schema = 'public';"

# 清理临时数据库
dropdb test_restore
\```

#### 4. 记录备份日志

在 `/var/log/backup.log` 中记录：

\```
[2026-06-28 02:15:00] BACKUP_SUCCESS db_20260628.dump 2.3GB
\```

### 验证标准

- ✅ 备份文件已生成且大小 > 0
- ✅ 恢复测试通过
- ✅ 日志已记录

### 异常处理

| 异常情况 | 处理方式 |
|----------|---------|
| 磁盘空间不足 | 清理旧备份或扩容 |
| 备份中断 | 检查数据库状态，重新执行 |
| 恢复验证失败 | 检查日志，联系 DBA |

### 回滚

如备份导致数据库异常：

\```bash
# 终止 pg_dump 进程
pkill pg_dump

# 检查数据库状态
psql -c "SELECT pg_is_in_recovery();"
\```
```

### 3.3 应急操作模板

```markdown
## SOP-EMERGENCY-001: 服务宕机应急处理

### 响应级别：🔴 紧急（P0）

### 快速诊断流程

```
收到告警
  │
  ├── 1. 确认告警（1分钟内）
  │    └── 检查是否为误报
  │
  ├── 2. 初步评估（2分钟内）
  │    ├── 影响范围：哪些服务/用户受影响？
  │    ├── 严重程度：P0/P1/P2？
  │    └── 通知相关方
  │
  ├── 3. 应急处理（5分钟内）
  │    ├── 尝试重启服务
  │    ├── 切换到备机
  │    └── 降级非核心功能
  │
  └── 4. 恢复确认
       └── 验证服务恢复正常
```

### 联系人

| 角色 | 姓名 | 电话 | 备用联系人 |
|------|------|------|-----------|
| 运维负责人 | 张三 | 138-xxxx-xxxx | 李四 |
| DBA | 王五 | 139-xxxx-xxxx | 赵六 |
| 架构师 | 钱七 | 137-xxxx-xxxx | 孙八 |

### 常见故障处理

#### 服务无响应

1. 检查进程状态：`systemctl status myapp`
2. 检查端口：`netstat -tlnp | grep 8080`
3. 检查日志：`journalctl -u myapp --since "10 min ago"`
4. 尝试重启：`systemctl restart myapp`
5. 如未恢复，切换备机

#### 数据库连接耗尽

1. 查看连接数：`SELECT count(*) FROM pg_stat_activity;`
2. 终止空闲连接：`SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE state = 'idle' AND query_start < now() - interval '30 min';`
3. 检查连接池配置
```

## 💳 账户与计费

### Q: 如何修改密码？

1. 登录后点击右上角头像 →「账户设置」
2. 选择「安全」标签
3. 点击「修改密码」
4. 输入当前密码和新密码
5. 点击「保存」

> 💡 密码要求：至少 8 位，包含大写字母、小写字母和数字。

---

### Q: 忘记密码怎么办？

1. 访问登录页面
2. 点击「忘记密码？」
3. 输入注册邮箱
4. 查收重置邮件（可能在垃圾邮件文件夹中）
5. 点击邮件中的链接，设置新密码

> ⚠️ 重置链接有效期为 30 分钟。

---

### Q: 如何升级/降级套餐？

1. 进入「账户设置 → 套餐管理」
2. 选择目标套餐
3. 确认变更详情
4. **升级**：立即生效，按比例补差价
5. **降级**：下个计费周期生效

### Q: 数据存储在哪里？安全吗？

- 数据存储在 **AWS 云服务器**（区域：us-east-1）
- 传输使用 **TLS 1.3** 加密
- 存储使用 **AES-256** 加密
- 每日自动备份，保留 **30 天**
- 通过 **SOC 2 Type II** 认证

---

3. 选择格式：JSON / CSV / Excel
4. 点击「开始导出」
5. 导出完成后会收到邮件通知（通常 5-15 分钟）

## 🐛 技术问题

### Q: 页面加载很慢怎么办？

**请依次尝试以下方法：**

1. **清除浏览器缓存**：Ctrl+Shift+Delete（Windows）/ Cmd+Shift+Delete（Mac）
2. **更换浏览器**：推荐使用 Chrome 或 Edge 最新版
3. **检查网络**：确保网络连接稳定
4. **关闭浏览器扩展**：部分扩展可能影响页面加载
5. **联系客服**：如以上方法无效，请提供以下信息联系客服：
   - 浏览器类型和版本
   - 操作系统
   - 问题截图
   - 发生时间

---

### Q: 出现 "Error 500" 怎么办？

这是服务器内部错误，通常不是您的问题。

1. 请等待 1-2 分钟后刷新页面
2. 如果持续出现，请 [提交工单](https://support.example.com)
3. 提供截图和发生时间，我们会尽快修复

## 5. 写作原则与技巧

### 5.1 黄金原则：以用户为中心

```
写作前问自己：
1. 读者是谁？（新手/专家/管理员）
2. 读者想做什么？（完成任务/解决问题/了解功能）
3. 读者在什么场景？（紧急/悠闲/学习）
4. 读者有什么前置知识？
```

### 5.2 语言风格指南

| 场景 | ✅ 推荐 | ❌ 避免 |
|------|---------|---------|
| 称呼用户 | 「你」 | 「您」（过于正式）/「用户」（疏远） |
| 语气 | 友好、直接 | 说教、命令 |
| 句长 | 短句（<20字） | 长句嵌套 |
| 术语 | 首次出现时解释 | 直接使用专业术语 |
| 否定 | 正面表述 | 「不得」「禁止」 |
| 被动 | 主动语态 | 被动语态 |

**示例：**

❌ 「系统将会被用户进行配置以确保其正常运行。」
✅ 「你需要配置系统才能正常使用。」

❌ 「如遇操作不当导致数据丢失，本平台概不负责。」
✅ 「请按照步骤操作。如果遇到问题，可以随时联系我们。」

### 5.3 步骤编写规范

**✅ 好的步骤：**
```
1. 点击左上角的「文件」菜单
2. 选择「导入」
3. 在弹出的窗口中选择文件
4. 点击「打开」
5. 等待导入完成（通常 < 10 秒）
```

**❌ 差的步骤：**
```
首先你需要找到文件菜单然后点击它之后选择导入功能
然后在窗口中找到你要导入的文件点击确认就可以了
```

### 5.4 截图与标注规范

- **截图尺寸**：宽度统一（推荐 800px 或 1200px）
- **标注方式**：用红色方框/箭头标注关键位置
- **编号标注**：步骤多时用数字标注
- **裁剪**：只截取相关区域，不要全屏截图
- **更新**：界面变更时及时更新截图

## 8. 文档维护与更新

### 8.1 文档生命周期

```
创建 → 评审 → 发布 → 维护 → 归档/更新
```

### 8.2 定期审查计划

| 频率 | 审查内容 | 负责人 |
|------|---------|--------|
| 每周 | 新增 FAQ 条目 | 客服团队 |
| 每月 | 检查截图是否过时 | 文档团队 |
| 每季度 | 全面审查所有文档 | 产品 + 文档 |
| 每次发版 | 更新对应功能文档 | 产品经理 |

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| 文档满意度 | > 4.0/5 | 文章底部评分 |

| 自助解决率 | > 60% | 工单来源分析 |
| 文档覆盖率 | > 90% | 功能/文档比 |
