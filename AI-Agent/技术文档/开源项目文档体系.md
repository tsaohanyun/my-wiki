## 1. 开源项目文档全景图

### 1.1 文档体系结构

```
project/
├── README.md              # 项目入口，第一印象
├── CONTRIBUTING.md        # 贡献指南
├── CHANGELOG.md           # 版本变更记录
├── LICENSE                # 开源许可证
├── CODE_OF_CONDUCT.md     # 行为准则
├── SECURITY.md            # 安全策略
├── SUPPORT.md             # 支持与求助
├── docs/                  # 详细文档
│   ├── getting-started.md # 快速开始
│   ├── architecture.md    # 架构设计
│   ├── api/               # API 文档
│   └── faq.md             # 常见问题
├── .github/
│   ├── ISSUES_TEMPLATE/   # Issue 模板
│   ├── PULL_REQUEST_TEMPLATE.md
│   └── workflows/         # CI/CD 文档
└── examples/              # 示例代码
```

### 1.2 文档优先级

| 优先级 | 文档 | 必要性 |
|--------|------|--------|
| P0 | README.md | 必须 |
| P0 | LICENSE | 必须 |
| P1 | CONTRIBUTING.md | 强烈推荐 |
| P1 | CHANGELOG.md | 强烈推荐 |
| P2 | CODE_OF_CONDUCT.md | 推荐 |
| P2 | SECURITY.md | 推荐 |
| P3 | docs/ 目录 | 按需 |

## 2. README.md

### 2.1 README 核心原则

> **用户打开你的项目页面，前 5 秒看到的内容决定了他是否继续阅读。**

### 2.2 README 完整模板

```markdown
# Project Name

> 一句话描述：这个项目是什么，解决什么问题。

[![Build Status](https://img.shields.io/badge/build-passing-brightgreen)]
[![Coverage](https://img.shields.io/badge/coverage-95%25-brightgreen)]
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)]
[![npm version](https://img.shields.io/npm/v/project-name)]
[![Stars](https://img.shields.io/github/stars/user/repo)]

## ✨ 特性

- 🚀 **高性能** - 比 XX 快 10 倍
- 📦 **零依赖** - 无第三方依赖
- 🔒 **类型安全** - 完整 TypeScript 支持
- 🌍 **国际化** - 支持 20+ 语言

## 📦 安装

\```bash
npm install project-name
# 或
yarn add project-name
# 或
pnpm add project-name
\```

## 🚀 快速开始

\```javascript
import { createApp } from 'project-name';

const app = createApp({
  name: 'My App',
  port: 3000,
});

app.start();
\```

|------|------|

## 3. CONTRIBUTING.md

### 3.1 完整模板

```markdown
# 贡献指南

首先，感谢你考虑为本项目做贡献！🎉

## 📋 目录

- [行为准则](#行为准则)
- [环境准备](#环境准备)
- [开发流程](#开发流程)
- [代码规范](#代码规范)
- [提交规范](#提交规范)
- [Pull Request 流程](#pull-request-流程)
- [Issue 报告](#issue-报告)
- [项目结构](#项目结构)

## 行为准则

参与本项目的所有贡献者都需要遵守 [行为准则](CODE_OF_CONDUCT.md)。

## 环境准备

### 前置要求

- Node.js >= 18
- Git
- Docker（可选，用于集成测试）

### 克隆与安装

\```bash
git clone https://github.com/user/repo.git
cd repo
npm install
\```

### 项目结构

\```
src/
├── core/        # 核心逻辑
├── utils/       # 工具函数
├── types/       # 类型定义
└── index.ts     # 入口文件
tests/           # 测试文件
docs/            # 文档
examples/        # 示例
\```

## 开发流程

1. **Fork** 本仓库
2. **克隆** 你的 Fork 到本地
3. 创建 **功能分支**：`git checkout -b feat/your-feature`
4. 编写代码并 **添加测试**
5. 运行测试：`npm test`
6. 运行 lint：`npm run lint`
7. **提交** 代码
8. 创建 **Pull Request**

### 分支命名规范

| 类型 | 前缀 | 示例 |
|------|------|------|
| 新功能 | `feat/` | `feat/add-dark-mode` |
| Bug 修复 | `fix/` | `fix/memory-leak` |
| 文档 | `docs/` | `docs/update-readme` |
| 重构 | `refactor/` | `refactor/auth-module` |
| 测试 | `test/` | `test/e2e-suite` |
| 性能 | `perf/` | `perf/optimize-query` |

## 代码规范

- 使用 **ESLint + Prettier** 格式化代码
- 函数必须有 **JSDoc 注释**
- 新增公共 API 必须有 **TypeScript 类型**
- 复杂逻辑必须有 **行内注释**

## 提交规范

本项目使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

\```
<type>(<scope>): <subject>

<body>

<footer>
\```

| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档变更 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具变更 |

**示例：**
\```
feat(auth): 添加 OAuth2 登录支持

- 支持 Google、GitHub 登录
- 添加 token 自动刷新机制
- 更新相关文档

Closes #123
\```

## Pull Request 流程

1. 确保 PR 对应的分支已经 **rebase** 到最新的 main
2. PR 标题遵循 Conventional Commits 规范
3. 填写 PR 模板中的所有必填项
4. 确保 CI 检查全部通过
5. 至少获得 **1 位** maintainer 的 approve
6. 不要自己 merge（除非你是 maintainer）

### PR 检查清单

- [ ] 代码通过了 lint 检查
- [ ] 添加了必要的测试
- [ ] 所有测试通过
- [ ] 更新了相关文档
- [ ] Commit message 遵循规范
- [ ] PR 描述清晰完整

## Issue 报告

### Bug 报告模板

\```markdown
**Bug 描述**
简要描述遇到的问题。

**复现步骤**
1. 进入 '...'
2. 点击 '...'
3. 看到 '...'

**预期行为**
描述你期望发生的事情。

**实际行为**
描述实际发生了什么。

**环境信息**
- OS: [e.g. macOS 14.0]
- Node.js: [e.g. 18.17.0]
- 项目版本: [e.g. 1.2.3]

如有，请附上截图或错误日志。
\```

描述你考虑过的其他方案。

**额外信息**
其他相关信息。
\```

## 4. CHANGELOG.md

### 4.1 Keep a Changelog 规范

```markdown
# Changelog

本项目的重要变更记录。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，
版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [Unreleased]

### Added
- 支持自定义中间件顺序

### Changed
- 优化了日志输出的格式

### Fixed
- 修复了在高并发下连接池泄漏的问题 (#142)

## [1.2.0] - 2026-05-15

### Added
- 🎉 新增 OAuth2 认证模块 (#120)
- 新增 `createMiddleware()` API (#125)
- 支持自定义错误处理器 (#128)

### Changed
- **BREAKING**: `config.port` 改为 `config.server.port`
- 升级依赖 `express` 到 4.19.x
- 改进 TypeScript 类型推导

### Deprecated
- `oldMethod()` 已废弃，请使用 `newMethod()` 代替

### Removed
- 移除了对 Node.js 14 的支持

### Fixed
- 修复 Windows 路径兼容性问题 (#130)
- 修复 `response.time` 在代理后不正确的问题 (#135)

### Security
- 升级 `lodash` 修复原型污染漏洞 (CVE-2024-xxxx)

## [1.1.0] - 2026-03-01

### Added
- 初始发布

### 变更类型说明

| 类型 | 说明 |
|------|------|
| `Added` | 新增功能 |
| `Changed` | 已有功能的变更 |
| `Deprecated` | 即将废弃的功能 |
| `Removed` | 已移除的功能 |
| `Fixed` | Bug 修复 |
| `Security` | 安全相关修复 |
```

### 4.2 自动生成 CHANGELOG

| 工具 | 说明 |
|------|------|
| [standard-version](https://github.com/conventional-changelog/standard-version) | 自动版本管理 + CHANGELOG |
| [release-please](https://github.com/googleapis/release-please) | GitHub Action 自动发布 |
| [changesets](https://github.com/changesets/changesets) | Monorepo 友好 |
| [semantic-release](https://github.com/semantic-release/semantic-release) | 全自动化发布 |

## 5. LICENSE

### 5.1 许可证选择指南

```
希望别人怎样使用你的代码？
│
├── 允许闭源使用（宽松型）
│   ├── MIT ─────────── 最流行，最简单
│   ├── Apache 2.0 ──── 包含专利授权
│   └── BSD 3-Clause ── 类似 MIT
│
├── 要求衍生作品也开源（Copyleft）
│   ├── GPL v3 ──────── 强 Copyleft
│   ├── LGPL v3 ─────── 弱 Copyleft（适合库）
│   └── AGPL v3 ─────── 网络服务也必须开源
│
└── 不确定
    └── 建议选 MIT（最宽松，最简单）
```

### 5.2 许可证兼容性矩阵

|  | MIT | Apache 2.0 | GPL v3 | AGPL v3 |
|--|-----|-----------|--------|---------|
| MIT | ✅ | ✅ | ✅ | ✅ |
| Apache 2.0 | ✅ | ✅ | ✅ | ✅ |
| GPL v3 | ✅ | ✅ | ✅ | ✅ |
| AGPL v3 | ✅ | ✅ | ✅ | ✅ |

> 使用 [choosealicense.com](https://choosealicense.com/) 获取完整许可证文本。

## 8. 文档自动化

### 8.1 自动化文档检查

```yaml
# .github/workflows/docs.yml
name: Documentation Check
on: [pull_request]
jobs:
  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check markdown links
        uses: gaurav-nelson/github-action-markdown-link-check@v1
      - name: Lint markdown
        uses: DavidAnson/markdownlint-cli2-action@v15
      - name: Check spelling
        uses: streetsidesoftware/cspell-action@v5
```
