# 响应式编程

> 响应式编程（Reactive Programming）是一种面向**数据流**和**变化传播**的编程范式。通过声明式的数据流管道，自动响应异步事件和数据变化，实现非阻塞、高吞吐量的应用。

---

## 核心概念

### 响应式宣言四大原则

```
┌─────────────────────────────────────────────┐
│              响应式系统特征                     │
├──────────────┬──────────────────────────────┤
│  响应的       │  系统及时响应（保持 SLA）       │
│  (Responsive)│  用户交互体验好                 │
├──────────────┼──────────────────────────────┤
│  弹性的       │  面对故障保持响应               │
│  (Resilient) │  故障隔离、自动恢复             │
├──────────────┼──────────────────────────────┤
│  弹性伸缩的   │  负载变化时自动伸缩             │
│  (Elastic)   │  无中心瓶颈、可分片             │
├──────────────┼──────────────────────────────┤
│  消息驱动的   │  异步消息传递                   │
│  (Message    │  松耦合、位置透明               │
│   Driven)    │                                │
└──────────────┴──────────────────────────────┘
```

### 数据流核心模型

```
Source ──map──► filter ──flatMap──► reduce ──► Sink
  │                                              │
  ▼                                              ▼
产生数据                                      消费数据
```

## 一、Reactive Streams 规范

Reactive Streams 是一个标准化规范（JDK 9 中被纳入 `java.util.concurrent.Flow`），定义了异步流处理的非阻塞背压标准。

### 四个核心接口

```java
// 1. Publisher：数据生产者
@FunctionalInterface
public interface Publisher<T> {
    void subscribe(Subscriber<? super T> subscriber);
}

// 2. Subscriber：数据消费者
public interface Subscriber<T> {
    void onSubscribe(Subscription subscription); // 订阅时调用
    void onNext(T item);                          // 收到数据
    void onError(Throwable throwable);            // 发生错误
    void onComplete();                            // 数据结束
}

// 3. Subscription：订阅链接
public interface Subscription {
    void request(long n);   // 请求 n 条数据（背压机制）
    void cancel();          // 取消订阅
}

// 4. Processor：既是生产者又是消费者
public interface Processor<T, R> extends Subscriber<T>, Publisher<R> {
}
```

### JDK 9 Flow API 使用

```java
import java.util.concurrent.Flow;
import java.util.concurrent.SubmissionPublisher;

public class ReactiveDemo {
    public static void main(String[] args) throws InterruptedException {
        // 创建发布者
        SubmissionPublisher<String> publisher = new SubmissionPublisher<>();

        // 创建订阅者
        Flow.Subscriber<String> subscriber = new Flow.Subscriber<>() {
            private Flow.Subscription subscription;

            @Override
            public void onSubscribe(Flow.Subscription subscription) {
                this.subscription = subscription;
                subscription.request(1); // 请求第一条数据
            }

            @Override
            public void onNext(String item) {
                System.out.println("Received: " + item);
                subscription.request(1); // 请求下一条
            }

            @Override
            public void onError(Throwable throwable) {
                System.err.println("Error: " + throwable.getMessage());
            }

            @Override
            public void onComplete() {
                System.out.println("Done!");
            }
        };

        // 建立订阅关系
        publisher.subscribe(subscriber);

        // 发布数据
        publisher.submit("Hello");
        publisher.submit("Reactive");
        publisher.submit("World");

        // 关闭发布者
        publisher.close();

        // 等待异步处理完成
        Thread.sleep(1000);
    }
}
```

## 二、RxJava

RxJava 是 ReactiveX 在 JVM 上的实现，是 Java 生态中最成熟的响应式库。

### 基本操作

```java
import io.reactivex.rxjava3.core.*;
import io.reactivex.rxjava3.schedulers.Schedulers;

public class RxJavaDemo {

    public void basicExample() {
        // 创建 Observable
        Observable<String> observable = Observable.create(emitter -> {
            emitter.onNext("Hello");
            emitter.onNext("Reactive");
            emitter.onNext("World");
            emitter.onComplete();
        });

        // 订阅
        observable
            .map(String::toUpperCase)
            .filter(s -> s.length() > 4)
            .subscribe(
                item -> System.out.println("Received: " + item),
                error -> System.err.println("Error: " + error),
                () -> System.out.println("Completed!")
            );
    }

    public void dataPipeline() {
        // 模拟数据流水线
        Observable.range(1, 100)
            // 在 IO 线程处理
            .observeOn(Schedulers.io())
            .map(this::fetchUserData)
            // 过滤有效数据
            .filter(user -> user.isActive())
            // 转换为 DTO
            .map(this::toDto)
            // 批量处理：每 10 个收集一次
            .buffer(10)
            // 在计算线程处理批量
            .observeOn(Schedulers.computation())
            .flatMap(batch -> saveBatch(batch).toObservable())
            // 回到主线程
            .observeOn(Schedulers.single())
            .subscribe(
                result -> System.out.println("Saved batch: " + result),
                Throwable::printStackTrace,
                () -> System.out.println("All data processed!")
            );
    }

    // Observable vs Single vs Flowable
    public void observableTypes() {
        // Observable: 0..N 个事件，无背压
        Observable<Integer> obs = Observable.just(1, 2, 3);

        // Single: 恰好 1 个事件或错误
        Single<String> single = Single.fromCallable(() -> fetchData());

        // Maybe: 0 或 1 个事件
        Maybe<String> maybe = Maybe.fromCallable(() -> findUser().orElse(null));

        // Completable: 只有完成/错误，无数据
        Completable completable = Completable.fromAction(this::performAction);

        // Flowable: 0..N 个事件，支持背压
        Flowable<Integer> flowable = Flowable.range(1, 1_000_000);
    }

    private User fetchUserData(int id) { /* ... */ return new User(); }
    private UserDto toDto(User user) { /* ... */ return new UserDto(); }
    private Single<Integer> saveBatch(List<UserDto> batch) { /* ... */ return Single.just(batch.size()); }
    private String fetchData() { return "data"; }
    private java.util.Optional<String> findUser() { return java.util.Optional.of("user"); }
    private void performAction() { }
}
```

### Hot vs Cold Observable

```java
// Cold Observable：每个订阅者独立收到完整数据
Observable<Integer> cold = Observable.range(1, 5);
cold.subscribe(v -> System.out.println("A: " + v)); // A: 1..5
cold.subscribe(v -> System.out.println("B: " + v)); // B: 1..5（独立流）

// Hot Observable：所有订阅者共享同一个流
ConnectableObservable<Integer> hot = Observable.range(1, 5).publish();
hot.subscribe(v -> System.out.println("A: " + v));
hot.connect(); // 开始发射
hot.subscribe(v -> System.out.println("B: " + v)); // B 可能只收到部分

// 使用 Subject 创建 Hot Observable
PublishSubject<String> subject = PublishSubject.create();
subject.subscribe(v -> System.out.println("Subscriber 1: " + v));
subject.onNext("First");  // 只有 Subscriber 1 收到
subject.subscribe(v -> System.out.println("Subscriber 2: " + v));
subject.onNext("Second"); // 两个 Subscriber 都收到
subject.onComplete();
```

## 三、Project Reactor

Project Reactor 是 Spring 生态的响应式库，是 Spring WebFlux 的核心。

### Mono 和 Flux

```java
import reactor.core.publisher.*;
import reactor.core.scheduler.Schedulers;
import java.time.Duration;

public class ReactorDemo {

    // Mono: 0 或 1 个元素
    public Mono<String> getUserById(String id) {
        return Mono.fromCallable(() -> database.findById(id))
            .onErrorResume(e -> Mono.empty());
    }

    // Flux: 0..N 个元素
    public Flux<Order> getOrdersForUser(String userId) {
        return Flux.fromStream(() -> database.findByUserId(userId).stream());
    }

    // 链式操作
    public void pipelineExample() {
        Flux.range(1, 10)
            .log("before-filter")
            .filter(n -> n % 2 == 0)
            .map(n -> n * n)
            .log("after-map")
            .subscribe(
                result -> System.out.println("Result: " + result),
                error -> error.printStackTrace(),
                () -> System.out.println("Completed")
            );
    }

    // 异步与线程调度
    public void asyncExample() {
        Flux<String> flux = Flux.interval(Duration.ofMillis(500))  // 每 500ms 发射
            .take(5)                                                // 取前 5 个
            .map(i -> "Event " + i)
            .subscribeOn(Schedulers.boundedElastic())               // 在弹性线程池上订阅
            .publishOn(Schedulers.parallel());                       // 在并行线程上处理

        flux.subscribe(s -> System.out.println(
            Thread.currentThread().getName() + ": " + s));
    }

    // 错误处理
    public void errorHandling() {
        Flux<Integer> flux = Flux.range(1, 10)
            .map(i -> {
                if (i == 5) throw new RuntimeException("Boom at 5!");
                return i * 10;
            })
            // 策略1：返回默认值
            // .onErrorReturn(-1)

            // 策略2：返回备用 Flux
            // .onErrorResume(e -> Flux.just(50, 60, 70))

            // 策略3：重试
            .retryWhen(reactor.util.retry.Retry
                .backoff(3, Duration.ofSeconds(1))
                .maxBackoff(Duration.ofSeconds(10)));

        flux.subscribe(
            v -> System.out.println("Got: " + v),
            e -> System.err.println("Final error: " + e)
        );
    }

    // 组合多个流
    public void combiningFlows() {
        Mono<String> userMono = getUserById("123");
        Mono<List<Order>> ordersMono = getOrdersForUser("123").collectList();

        // zip：等待所有完成
        Mono.zip(userMono, ordersMono)
            .map(tuple -> {
                String user = tuple.getT1();
                List<Order> orders = tuple.getT2();
                return new UserProfile(user, orders);
            })
            .subscribe(profile -> System.out.println(profile));

        // merge：合并交错
        Flux.merge(
            Flux.interval(Duration.ofMillis(100)).take(3).map(i -> "A" + i),
            Flux.interval(Duration.ofMillis(150)).take(3).map(i -> "B" + i)
        ).subscribe(System.out::println);
    }
}
```

### Spring WebFlux 响应式 Controller

```java
import org.springframework.web.bind.annotation.*;
import org.springframework.web.reactive.function.client.WebClient;
import reactor.core.publisher.*;

@RestController
@RequestMapping("/api/users")
public class UserController {

    private final UserRepository repository;
    private final WebClient webClient;

    @GetMapping("/{id}")
    public Mono<User> getUser(@PathVariable String id) {
        return repository.findById(id)
            .switchIfEmpty(Mono.error(new UserNotFoundException(id)));
    }

    @GetMapping
    public Flux<User> getAllUsers() {
        return repository.findAll();
    }

    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public Mono<User> createUser(@RequestBody CreateUserRequest request) {
        return Mono.just(request)
            .map(req -> new User(req.name(), req.email()))
            .flatMap(repository::save);
    }

    // SSE（Server-Sent Events）流
    @GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> streamUsers() {
        return Flux.interval(Duration.ofSeconds(1))
            .map(seq -> ServerSentEvent.<String>builder()
                .id(String.valueOf(seq))
                .event("user-update")
                .data("Update #" + seq)
                .build());
    }

    // 响应式调用外部 API
    public Mono<WeatherInfo> getWeather(String city) {
        return webClient.get()
            .uri("/weather/{city}", city)
            .retrieve()
            .bodyToMono(WeatherInfo.class)
            .timeout(Duration.ofSeconds(5))
            .onErrorResume(e -> Mono.just(WeatherInfo.unknown()));
    }
}
```

## 四、背压（Backpressure）

背压是消费者向生产者传达其处理能力的机制，防止生产速度超过消费速度导致 OOM。

```
生产者 (快) ══╗                         ╔══ 消费者 (慢)
              ║  1000 events/sec        ║   100 events/sec
              ╠═════════════════════════╣
              ║     背压信号 (n=100)     ║
              ╚═════════════════════════╝
                    ↑ request(n)

策略：
├── BUFFER：缓存所有未处理数据（可能 OOM）
├── DROP：丢弃新数据
├── LATEST：只保留最新的
├── ERROR：抛出异常
└── 自定义：MISSING（不处理背压）
```

### RxJava 背压示例

```java
import io.reactivex.rxjava3.core.*;
import io.reactivex.rxjava3.schedulers.Schedulers;

public class BackpressureDemo {

    public void overflowExample() {
        // 快速生产 + 慢速消费 = 背压问题
        Flowable.range(1, 1_000_000)
            // 策略1：缓冲（默认 128）
            // .onBackpressureBuffer(1000, () -> {}, BackpressureOverflowStrategy.ERROR)

            // 策略2：丢弃无法处理的数据
            // .onBackpressureDrop(item -> System.out.println("Dropped: " + item))

            // 策略3：只保留最新值
            .onBackpressureLatest()

            .observeOn(Schedulers.io(), false, 128) // 缓冲区大小 128
            .map(this::slowProcess)
            .subscribe(
                item -> System.out.println("Processed: " + item),
                Throwable::printStackTrace,
                () -> System.out.println("Done")
            );
    }

    // 自定义 Subscriber 控制拉取速率
    public void customBackpressure() {
        Flowable.create(emitter -> {
            for (int i = 0; i < Integer.MAX_VALUE; i++) {
                // 检查下游是否取消
                if (emitter.isCancelled()) break;
                // 在发之前检查请求
                while (!emitter.isCancelled()) {
                    if (emitter.requested() > 0) {
                        emitter.onNext(i);
                        break;
                    }
                    // 等待请求
                    Thread.sleep(1);
                }
            }
            emitter.onComplete();
        }, BackpressureStrategy.BUFFER);
    }

    private int slowProcess(int item) throws InterruptedException {
        Thread.sleep(100); // 模拟慢速处理
        return item * 2;
    }
}
```

### Reactor 背压

```java
import reactor.core.publisher.*;
import reactor.core.scheduler.Schedulers;

public class ReactorBackpressure {

    public void rateLimiting() {
        // 使用 limitRate 控制预取速率
        Flux.range(1, Integer.MAX_VALUE)
            .publishOn(Schedulers.parallel(), 64) // 预取 64
            .limitRate(32)                        // 降低预取到 32
            .map(this::process)
            .subscribe(System.out::println);
    }

    public void onBackpressureStrategy() {
        Flux<Integer> fastProducer = Flux.create(sink -> {
            for (int i = 0; i < 10000; i++) {
                sink.next(i);
            }
            sink.complete();
        }, FluxSink.OverflowStrategy.BUFFER); // BUFFER / DROP / LATEST / ERROR

        fastProducer
            .onBackpressureBuffer(1000)
            .onBackpressureDrop()
            .onBackpressureLatest()
            .subscribe(System.out::println);
    }

    private int process(int i) { return i; }
}
```

## 最佳实践

### ✅ 推荐做法

1. **用 Flowable 处理大数据流**：超过 1000 个元素或无限流时使用 Flowable 而非 Observable
2. **始终处理错误**：每个 subscribe 都应该有 error handler
3. **合理设置线程**：IO 操作用 `boundedElastic`，CPU 密集用 `parallel`
4. **及时取消订阅**：防止内存泄漏，特别是在 Android/前端场景
5. **利用背压**：不要无脑 buffer，分析场景选择合适的溢出策略

### ❌ 避免

1. **阻塞调用**：响应式流中绝对不要调用阻塞 I/O
2. **混用命令式和响应式**：不要在响应式链中修改外部状态
3. **深层嵌套的 flatMap**：超过 3 层考虑拆分
4. **忽略背压**：生产者快于消费者时必须有背压策略
