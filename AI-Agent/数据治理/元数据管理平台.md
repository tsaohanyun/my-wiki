# 元数据管理平台

> 元数据管理是数据治理的基础设施，通过构建统一的数据目录、血缘关系和数据地图，让数据"可发现、可理解、可信任"。

## 概述

元数据是"描述数据的数据"，分为三类：

- **技术元数据**：表结构、字段类型、索引、分区等
- **业务元数据**：业务定义、数据标准、指标口径等
- **操作元数据**：ETL 日志、数据量、更新频率、访问记录等

元数据管理平台的核心价值：**让正确的用户在正确的时间找到正确的数据**。

## 数据目录

### 数据目录架构

```
┌─────────────────────────────────────────────┐
│              数据目录门户                     │
│   搜索 │ 浏览 │ 推荐 │ 标签 │ 评价          │
├─────────────────────────────────────────────┤
│              目录服务层                       │
│   资产注册 │ 分类分级 │ 标签管理 │ 评分       │
├─────────────────────────────────────────────┤
│              元数据存储层                     │
│   图数据库 │ 关系数据库 │ 搜索引擎           │
├─────────────────────────────────────────────┤
│              元数据采集层                     │
│   自动采集 │ 人工录入 │ API对接              │
└─────────────────────────────────────────────┘
```

### 数据资产登记

| 属性类别 | 包含内容 |
|----------|----------|
| **基础信息** | 名称、描述、负责人、所属部门 |
| **技术信息** | 数据库、Schema、表名、字段列表 |
| **业务信息** | 业务域、业务定义、数据标准 |
| **质量信息** | 质量评分、质量规则、问题记录 |
| **使用信息** | 访问量、热门查询、使用趋势 |

### 分类分级体系

```
数据分类：
├── 按业务域：客户数据、财务数据、运营数据、产品数据
├── 按数据类型：结构化、半结构化、非结构化
└── 按敏感级别：公开、内部、机密、绝密

数据分级：
├── L1 - 公开数据：可对外发布
├── L2 - 内部数据：企业内部共享
├── L3 - 敏感数据：授权访问
├── L4 - 机密数据：严格管控
└── L5 - 绝密数据：最高安全等级
```

## 数据血缘

### 血缘类型

| 类型 | 说明 | 应用场景 |
|------|------|----------|
| **表级血缘** | 表与表之间的依赖关系 | 影响分析、变更评估 |
| **字段级血缘** | 字段级别的数据流转 | 数据溯源、问题定位 |
| **SQL 血缘** | 从 SQL 语句解析血缘 | 自动化采集 |
| **任务血缘** | ETL/调度任务的依赖关系 | 任务编排、故障定位 |

### 血缘解析技术

1. **SQL 解析**：通过 AST（抽象语法树）解析 SQL 语句中的表和字段引用
2. **日志分析**：从 ETL 工具日志中提取数据流转信息
3. **API Hook**：拦截数据处理框架的 API 调用
4. **正则匹配**：对非标准 SQL 或脚本进行模式匹配

### 血缘应用场景

- **影响分析**：上游表结构变更时，评估下游影响范围
- **数据溯源**：指标数据异常时，追溯到数据源头
- **合规审计**：追踪敏感数据的流转路径
- **变更管理**：数据迁移或重构前的影响评估

## 数据地图

### 数据地图功能

```
数据地图
├── 全局视图
│   ├── 数据资产分布（按域、按部门）
│   ├── 数据流转全景图
│   └── 数据热度分析
├── 关系视图
│   ├── 实体关系图
│   ├── 血缘关系图
│   └── 影响分析图
├── 搜索发现
│   ├── 全文搜索
│   ├── 标签筛选
│   └── 智能推荐
└── 分析视图
    ├── 数据使用趋势
    ├── 数据质量趋势
    └── 数据增长趋势
```

### 数据地图可视化

- **知识图谱**：基于图数据库展示实体关系
- **拓扑图**：展示系统间数据流转拓扑
- **热力图**：展示数据资产的使用热度
- **时间线**：展示数据资产的变更历史

## 元数据采集

### 采集方式

| 方式 | 适用场景 | 优缺点 |
|------|----------|--------|
| **自动采集** | 数据库、ETL 工具、BI 平台 | 覆盖面广，需适配器 |
| **API 对接** | 云服务、SaaS 平台 | 实时性好，依赖 API 能力 |
| **爬虫采集** | 文档、Wiki、代码仓库 | 覆盖非结构化，需定制 |
| **人工录入** | 业务定义、数据标准 | 准确性高，效率低 |

### 采集架构

```
数据源                      采集引擎                    元数据存储
┌──────┐    JDBC/ODBC    ┌──────────┐    写入     ┌──────────┐
│MySQL │ ──────────────→ │          │ ─────────→ │ 图数据库  │
└──────┘                 │          │            │(Neo4j)   │
┌──────┐    Hive API     │  采集    │            ├──────────┤
│Hive  │ ──────────────→ │  调度器  │ ─────────→ │ 搜索引擎  │
└──────┘                 │          │            │(ES)      │
┌──────┐    REST API     │  (Airflow│            ├──────────┤
│BI工具│ ──────────────→ │  )       │ ─────────→ │ 关系数据库│
└──────┘                 │          │            │(MySQL)   │
┌──────┐    Git API      │          │            └──────────┘
│代码库│ ──────────────→ │          │
└──────┘                 └──────────┘
```

### 采集策略

- **全量采集**：首次采集或定期全量刷新
- **增量采集**：基于 binlog 或时间戳的增量更新
- **事件驱动**：DDL 变更时触发采集
- **定时轮询**：按计划定期检查更新

## 实施方法论

### 阶段一：元数据盘点（2-4周）

1. 梳理数据源清单和系统架构
2. 确定元数据采集范围和优先级
3. 定义元数据模型和属性标准
4. 评估技术选型

### 阶段二：平台搭建（6-8周）

1. 部署元数据管理平台
2. 开发数据源适配器
3. 配置自动采集任务
4. 构建搜索和浏览功能

### 阶段三：血缘建设（4-6周）

1. 实现 SQL 解析引擎
2. 构建表级和字段级血缘
3. 开发影响分析功能
4. 可视化血缘关系图

### 阶段四：运营推广（持续）

1. 推广数据目录使用
2. 完善业务元数据（人工补充业务定义）
3. 建立元数据质量监控
4. 持续优化搜索和推荐
