# 大模型训练工程

> 训练百亿至万亿参数的大语言模型（LLM）需要系统化的工程能力：**分布式并行策略**、**显存优化**、**混合精度训练**、**梯度累积** 等。本文涵盖从单卡优化到千卡集群的全栈训练工程实践。

---

## 1. 训练并行策略全景

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **数据并行 (DP)** | 每卡持有完整模型副本，切分数据 | 模型可放入单卡 |
| **张量并行 (TP)** | 将单个层的权重矩阵切分到多卡 | 超大单层（如 Attention/FFN） |
| **流水线并行 (PP)** | 将模型按层切分到不同卡 | 深层模型 |
| **ZeRO** | 优化器状态/梯度/参数分片 | 通用，配合 DP |
| **专家并行 (EP)** | 不同专家分布到不同卡 | MoE 模型 |

```
3D 并行 = TP × PP × DP（实际生产中最常用）

GPU 0  GPU 1  |  GPU 2  GPU 3
[TP Group 0 ] | [TP Group 1 ]     ← 张量并行
    ↓              ↓
[ Pipeline Stage 0 → Stage 1 ]    ← 流水线并行
    ↓              ↓
[ Data Parallel Replica 0, 1 ]    ← 数据并行
```

## 2. ZeRO 优化

ZeRO（Zero Redundancy Optimizer）是 DeepSpeed 的核心创新，通过分片消除数据并行中的内存冗余。

| Stage | 分片内容 | 显存节省 | 通信开销 |
|-------|----------|----------|----------|
| ZeRO-1 | 优化器状态 | ~4× | 与 DP 相同 |
| ZeRO-2 | 优化器状态 + 梯度 | ~8× | 略增 |
| ZeRO-3 | 优化器状态 + 梯度 + 参数 | ~N× | 显著增加 |

## 3. Megatron-LM 张量并行

Megatron-LM 由 NVIDIA 开发，实现了高效的**张量并行（TP）**和**流水线并行（PP）**。

### 3.1 张量并行原理

对于线性层 $Y = XW$，Megatron 将权重 $W$ 按列或行切分：

```python
# 列并行：W 按列切分为 [W_1, W_2, ..., W_n]
#   每卡计算 Y_i = X @ W_i，最终 all-gather 合并
# 行并行：W 按行切分
#   每卡计算 Y_i = X_i @ W_i，最终 all-reduce 求和

import torch.nn as nn

class ColumnParallelLinear(nn.Module):
    """Megatron 列并行线性层"""
    def __init__(self, in_features, out_features, world_size, rank):
        super().__init__()
        assert out_features % world_size == 0
        self.out_features_per_partition = out_features // world_size
        self.weight = nn.Parameter(torch.empty(
            self.out_features_per_partition, in_features
        ))
        self.bias = nn.Parameter(torch.empty(self.out_features_per_partition))
        self.rank = rank
        nn.init.kaiming_uniform_(self.weight, a=math.sqrt(5))

    def forward(self, x):
        output = F.linear(x, self.weight, self.bias)
        return output  # 各卡独立输出，后续 all-gather


class RowParallelLinear(nn.Module):
    """Megatron 行并行线性层"""
    def __init__(self, in_features, out_features, world_size, rank):
        super().__init__()
        assert in_features % world_size == 0
        self.in_features_per_partition = in_features // world_size
        self.weight = nn.Parameter(torch.empty(
            out_features, self.in_features_per_partition
        ))
        self.bias = nn.Parameter(torch.empty(out_features))
        self.rank = rank

    def forward(self, x):
        # x 已按特征维度切分
        local_output = F.linear(x, self.weight)
        # all-reduce 合并各卡结果
        output = torch.distributed.all_reduce(local_output, op=torch.distributed.ReduceOp.SUM)
        return local_output + self.bias
```

## 4. 混合精度训练

### 4.1 BF16 vs FP16

| 格式 | 位宽 | 指数位 | 尾数位 | 动态范围 | 精度 |
|------|------|--------|--------|----------|------|
| FP32 | 32 | 8 | 23 | ~10^38 | 高 |
| FP16 | 16 | 5 | 10 | ~10^5 | 中 |
| BF16 | 16 | 8 | 7 | ~10^38 | 低 |

> **推荐**：Ampere 架构（A100）及以上优先使用 **BF16**，数值范围与 FP32 相同，无需 loss scaling。

### 4.2 自动混合精度（AMP）

```python
from torch.cuda.amp import autocast, GradScaler

model = MyModel().cuda()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)

scaler = GradScaler()  # FP16 需要，BF16 不需要

for batch in dataloader:
    inputs, labels = batch
    inputs, labels = inputs.cuda(), labels.cuda()

    optimizer.zero_grad()

    # 混合精度前向传播
    with autocast(dtype=torch.bfloat16):  # 或 torch.float16
        outputs = model(inputs)
        loss = loss_fn(outputs, labels)

    # 反向传播
    if scaler.is_enabled():
        scaler.scale(loss).backward()
        scaler.unscale_(optimizer)
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        scaler.step(optimizer)
        scaler.update()
    else:
        loss.backward()
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        optimizer.step()
```

## 5. 梯度累积

在显存有限的情况下模拟大 batch size：

```python
GRADIENT_ACCUMULATION_STEPS = 8
MICRO_BATCH_SIZE = 4
# 有效 batch size = MICRO_BATCH_SIZE × GRADIENT_ACCUMULATION_STEPS × world_size

optimizer.zero_grad()

for step, batch in enumerate(dataloader):
    inputs, labels = batch

    with autocast(dtype=torch.bfloat16):
        outputs = model(inputs)
        loss = loss_fn(outputs, labels)
        # 关键：除以累积步数
        loss = loss / GRADIENT_ACCUMULATION_STEPS

    # 使用 model.backward() 而非 loss.backward()（DeepSpeed 场景）
    if use_deepspeed:
        model.backward(loss)
    else:
        loss.backward()

    # 每累积 N 步才更新
    if (step + 1) % GRADIENT_ACCUMULATION_STEPS == 0:
        if use_deepspeed:
            model.step()
        else:
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            optimizer.step()
            optimizer.zero_grad()
```

## 6. 激活检查点

用计算换显存——只保存部分中间激活值，其余在反向传播时重新计算：

```python
from torch.utils.checkpoint import checkpoint

class TransformerBlock(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.attn = MultiHeadAttention(config)
        self.ffn = FeedForward(config)
        self.use_checkpoint = config.use_activation_checkpointing

    def forward(self, x):
        if self.use_checkpoint and self.training:
            # 检查点：前向不保存中间激活
            x = checkpoint(self._forward, x, use_reentrant=False)
        else:
            x = self._forward(x)
        return x

    def _forward(self, x):
        x = x + self.attn(x)
        x = x + self.ffn(x)
        return x

# 全局启用
model.gradient_checkpointing_enable()
```

## 7. FSDP（PyTorch 原生分布式）

FSDP（Fully Sharded Data Parallel）是 PyTorch 官方对标 ZeRO-3 的方案：

```python
from torch.distributed.fsdp import (
    FullyShardedDataParallel as FSDP,
    ShardingStrategy,
    MixedPrecision,
    CPUOffload,
)
from torch.distributed.fsdp.wrap import (
    transformer_auto_wrap_policy,
    size_based_auto_wrap_policy,
)
import functools

# 混合精度策略
bf16_policy = MixedPrecision(
    param_dtype=torch.bfloat16,
    reduce_dtype=torch.bfloat16,
    buffer_dtype=torch.bfloat16,
)

# 自动包装策略
auto_wrap_policy = functools.partial(
    transformer_auto_wrap_policy,
    transformer_layer_cls={TransformerBlock},
)

# 初始化 FSDP
model = FSDP(
    model,
    sharding_strategy=ShardingStrategy.FULL_SHARD,  # 等价 ZeRO-3
    mixed_precision=bf16_policy,
    auto_wrap_policy=auto_wrap_policy,
    cpu_offload=CPUOffload(offload_params=False),
    device_id=torch.cuda.current_device(),
    forward_prefetch=True,
    backward_prefetch="backward_pre",
    activation_checkpointing=True,
)

# 使用标准训练循环
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)
for batch in dataloader:
    loss = model(batch)
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()

# 保存模型
from torch.distributed.fsdp import FullStateDictConfig, StateDictType
full_state_config = FullStateDictConfig(offload_to_cpu=True, rank0_only=True)
with FSDP.state_dict_type(model, StateDictType.FULL_STATE_DICT, full_state_config):
    state_dict = model.state_dict()
    if torch.distributed.get_rank() == 0:
        torch.save(state_dict, "model.pt")
```

### 8.2 常见训练问题排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| OOM（显存溢出） | batch/micro_batch 太大 | 减小 micro_batch_size，增大 gradient_accumulation |
| Loss = NaN | 梯度爆炸/数值溢出 | 启用 gradient clipping，切换 BF16 |
| 训练缓慢 | ZeRO-3 通信开销 | 降低 ZeRO stage 或使用 ZeRO-2 |
| GPU 利用率低 | 数据加载瓶颈 | 增加 DataLoader workers，使用预取 |
| Loss 不下降 | 学习率不当 | 调整 warmup，检查 lr scheduler |

## 9. 最佳实践

1. **从预训练 checkpoint 开始**：SFT/微调优先使用预训练权重，不要从头训练。
2. **合理设置并行度**：TP × PP = 节点内 GPU 数，DP = 节点数。TP 一般取 2/4/8。
3. **BF16 优先**：A100/H100 上使用 BF16 训练，避免 FP16 的 loss scaling 复杂性。
4. **梯度裁剪必开**：`max_norm=1.0` 是几乎所有 LLM 训练的标配。
5. **学习率调度**：使用 Cosine Decay + Linear Warmup，warmup 步数约 2% 总步数。
6. **弹性容错**：使用 DeepSpeed 的 checkpoint 机制支持断点续训。
7. **通信优化**：开启 `overlap_comm`、调整 `reduce_bucket_size` 以重叠计算与通信。
