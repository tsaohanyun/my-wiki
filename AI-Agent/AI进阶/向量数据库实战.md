# 向量数据库实战

> 向量数据库是 RAG（检索增强生成）和语义搜索的核心基础设施。它们专为存储和检索高维向量（Embedding）而设计，支持**近似最近邻搜索（ANN）**，在毫秒级完成百万至十亿级向量的相似度查询。

---

## 1. 核心概念

| 概念 | 说明 |
|------|------|
| **Embedding** | 将文本/图像/音频映射为高维向量（如 768 维、1536 维） |
| **ANN 搜索** | 近似最近邻搜索，牺牲少量精度换取巨大速度提升 |
| **余弦相似度** | $\cos(A, B) = \frac{A \cdot B}{\|A\| \|B\|}$ |
| **欧氏距离** | $d(A, B) = \sqrt{\sum_i (a_i - b_i)^2}$ |
| **Metadata Filter** | 结合标量过滤的向量搜索（hybrid search） |

### 1.1 主流向量数据库对比

| 数据库 | 类型 | 部署方式 | 索引算法 | 特点 |
|--------|------|----------|----------|------|
| **Pinecone** | 云原生 SaaS | 托管 | HNSW + PQ | 全托管、零运维 |
| **Milvus** | 分布式开源 | 自托管/K8s | HNSW, IVF, DiskANN | 高性能、可扩展 |
| **Weaviate** | 开源 | Docker/K8s | HNSW | 内置模块化向量化 |
| **Chroma** | 轻量级开源 | 嵌入式/Server | HNSW | 简单易用、本地优先 |
| **Qdrant** | 开源 | Docker | HNSW | Rust 实现、高性能 |
| **pgvector** | PostgreSQL 扩展 | 自托管 | IVF, HNSW | 与 SQL 深度集成 |

## 2. 索引算法详解

### 2.1 HNSW（分层导航小世界图）

**HNSW（Hierarchical Navigable Small World）** 是当前最广泛使用的 ANN 索引算法。

```
原理：
  - 构建多层图结构，上层稀疏、下层密集
  - 搜索从最上层开始粗略定位，逐层下降精细搜索
  - 类似跳表的思想，时间复杂度 O(log N)

关键参数：
  M:        每个节点的最大连接数（通常 16-64）
  ef_construction: 构建时的搜索宽度（通常 200-500）
  ef_search:       查询时的搜索宽度（通常 50-200，越大越精确越慢）
```

```python
# HNSW 参数调优示例（以 hnswlib 为例）
import hnswlib
import numpy as np

# 生成示例数据
dim = 768
num_elements = 1_000_000
data = np.random.randn(num_elements, dim).astype(np.float32)

# 创建索引
index = hnswlib.Index(space='cosine', dim=dim)

# 初始化索引
index.init_index(max_elements=num_elements, ef_construction=400, M=32)

# 添加数据（支持增量添加）
index.add_items(data, ids=np.arange(num_elements))

# 设置查询参数
index.set_ef(100)  # 查询时 ef，影响召回率和速度

# 查询
query = np.random.randn(1, dim).astype(np.float32)
labels, distances = index.knn_query(query, k=10)

print(f"Top-10 results: {labels[0]}")
print(f"Distances: {distances[0]}")

# 保存与加载
index.save_index("hnsw_index.bin")
index.load_index("hnsw_index.bin", max_elements=num_elements)
```

### 2.2 IVF（倒排文件索引）

```
原理：
  1. K-Means 聚类将向量空间划分为 nlist 个簇
  2. 查询时只搜索最近的 nprobe 个簇
  3. 可与 PQ（乘积量化）结合压缩存储

参数：
  nlist:  聚类中心数量（通常 sqrt(N) 到 4*sqrt(N)）
  nprobe: 查询时搜索的簇数（通常 nlist 的 1%-10%）
```

```python
# IVF + PQ 使用 FAISS
import faiss
import numpy as np

dim = 768
num_elements = 1_000_000
data = np.random.randn(num_elements, dim).astype(np.float32)

# 量化器
quantizer = faiss.IndexFlatIP(dim)

# IVFPQ 索引
# nlist: 聚类中心数, m: 子向量数, nbits: 每个子向量编码位数
nlist = 4096
m = 64          # 必须是 dim 的因子
nbits = 8

index = faiss.IndexIVFPQ(quantizer, dim, nlist, m, nbits)
index.metric_type = faiss.METRIC_INNER_PRODUCT

# 训练（IVF 需要先训练聚类中心）
train_data = data[:min(50000, num_elements)]  # 使用子集训练
index.train(train_data)

# 添加数据
index.add(data)

# 设置 nprobe
index.nprobe = 64  # 搜索 64 个最近的簇

# 查询
query = np.random.randn(1, dim).astype(np.float32)
distances, labels = index.search(query, 10)
```

### 2.3 索引算法对比

| 指标 | Flat | HNSW | IVF | IVF+PQ | DiskANN |
|------|------|------|-----|--------|---------|
| 构建速度 | 快 | 中 | 中 | 中 | 慢 |
| 查询速度 | 慢 | 很快 | 快 | 很快 | 快 |
| 召回率 | 100% | 高(99%+) | 中高 | 中 | 高 |
| 内存占用 | 高 | 高 | 中 | 低 | 极低 |
| 增量更新 | 支持 | 支持 | 支持 | 受限 | 受限 |
| 适用规模 | <10万 | <1000万 | <1亿 | <10亿 | >10亿 |

## 3. Pinecone 实战

```python
from pinecone import Pinecone, ServerlessSpec

# 初始化
pc = Pinecone(api_key="YOUR_API_KEY")

# 创建索引
pc.create_index(
    name="documents",
    dimension=1536,
    metric="cosine",
    spec=ServerlessSpec(
        cloud="aws",
        region="us-east-1",
    ),
)

# 获取索引
index = pc.Index("documents")

# 插入向量
import openai

openai.api_key = "YOUR_API_KEY"

texts = [
    "Transformer is a neural network architecture for NLP.",
    "Vector databases enable semantic search at scale.",
    "DeepSpeed enables training of trillion-parameter models.",
]

# 生成 Embedding
embeddings = []
for text in texts:
    response = openai.embeddings.create(
        input=text,
        model="text-embedding-3-small",
    )
    embeddings.append(response.data[0].embedding)

# 批量插入
index.upsert(
    vectors=[
        {
            "id": f"doc_{i}",
            "values": embeddings[i],
            "metadata": {
                "text": texts[i],
                "source": "wiki",
                "category": "AI/ML",
            },
        }
        for i in range(len(texts))
    ]
)

# 语义搜索（带 metadata 过滤）
query_response = openai.embeddings.create(
    input="How to train large language models?",
    model="text-embedding-3-small",
)
query_embedding = query_response.data[0].embedding

results = index.query(
    vector=query_embedding,
    top_k=5,
    include_metadata=True,
    filter={
        "category": {"$eq": "AI/ML"},
    },
)

for match in results["matches"]:
    print(f"Score: {match['score']:.4f} | {match['metadata']['text']}")
```

## 4. Milvus 实战

```python
from pymilvus import MilvusClient, DataType
import numpy as np

# 连接（Milvus 2.4+ 使用 MilvusClient API）
client = MilvusClient(uri="http://localhost:19530")

# 创建 Collection Schema
schema = MilvusClient.create_schema(auto_id=True, enable_dynamic_field=True)
schema.add_field("id", DataType.INT64, is_primary=True)
schema.add_field("embedding", DataType.FLOAT_VECTOR, dim=768)
schema.add_field("text", DataType.VARCHAR, max_length=2048)
schema.add_field("category", DataType.VARCHAR, max_length=64)

# 索引配置
index_params = client.prepare_index_params()
index_params.add_index(
    field_name="embedding",
    index_type="HNSW",
    metric_type="COSINE",
    params={"M": 32, "efConstruction": 400},
)

# 创建 Collection
client.create_collection(
    collection_name="knowledge_base",
    schema=schema,
    index_params=index_params,
)

# 插入数据
data = [
    {
        "embedding": np.random.randn(768).tolist(),
        "text": "Transformer architecture explanation",
        "category": "DeepLearning",
    }
    for _ in range(1000)
]

client.insert(collection_name="knowledge_base", data=data)

# 向量搜索（带标量过滤）
query_vector = np.random.randn(768).tolist()

results = client.search(
    collection_name="knowledge_base",
    data=[query_vector],
    limit=10,
    search_params={"params": {"ef": 100}},
    filter='category == "DeepLearning"',
    output_fields=["text", "category"],
)

for hits in results:
    for hit in hits:
        print(f"Distance: {hit['distance']:.4f} | {hit['entity']['text']}")
```

## 5. Weaviate 实战

```python
import weaviate
from weaviate.embedded import EmbeddedOptions

# 连接 Weaviate（支持内置 Embedding 模块）
client = weaviate.connect_to_local(
    host="localhost",
    port=8080,
    headers={"X-OpenAI-Api-Key": "YOUR_OPENAI_API_KEY"},
)

# 创建 Schema（Collection）
from weaviate.classes.config import Configure, Property, DataType

if client.collections.exists("Document"):
    client.collections.delete("Document")

documents = client.collections.create(
    name="Document",
    vectorizer_config=Configure.Vectorizer.text2vec_openai(
        model="text-embedding-3-small"
    ),
    properties=[
        Property(name="content", data_type=DataType.TEXT),
        Property(name="category", data_type=DataType.TEXT),
    ],
    # 混合搜索配置
    vector_index_config=Configure.VectorIndex.hnsw(
        ef_construction=128,
        ef=64,
        cleanup_interval_seconds=300,
    ),
)

# 插入数据（自动生成 Embedding）
with documents.batch.dynamic() as batch:
    batch.add_object(
        properties={
            "content": "Transformer uses self-attention mechanism.",
            "category": "NLP",
        },
    )
    batch.add_object(
        properties={
            "content": "HNSW is an efficient ANN search algorithm.",
            "category": "VectorDB",
        },
    )

# 语义搜索
from weaviate.classes.query import MetadataQuery

results = documents.query.near_text(
    query="attention mechanism",
    limit=5,
    return_metadata=MetadataQuery(distance=True),
    filters=weaviate.classes.query.Filter.by_property("category").equal("NLP"),
)

for obj in results.objects:
    print(f"Distance: {obj.metadata.distance:.4f}")
    print(f"Content: {obj.properties['content']}\n")

# 混合搜索（向量 + BM25 关键词）
results = documents.query.hybrid(
    query="attention mechanism transformer",
    query_properties=["content"],
    alpha=0.5,  # 0=纯关键词, 1=纯向量
    limit=5,
)

client.close()
```

## 6. Chroma 实战

```python
import chromadb

# 本地持久化客户端
client = chromadb.PersistentClient(path="./chroma_db")

# 创建 Collection
collection = client.create_collection(
    name="knowledge_base",
    metadata={"hnsw:space": "cosine"},  # 可选: cosine, l2, ip
)

# 插入数据（Chroma 默认使用自带的 Embedding 模型）
collection.add(
    ids=["doc1", "doc2", "doc3"],
    documents=[
        "LangChain is a framework for developing LLM applications.",
        "Vector databases store high-dimensional embeddings.",
        "RLHF aligns language models with human preferences.",
    ],
    metadatas=[
        {"source": "wiki", "topic": "LLM"},
        {"source": "wiki", "topic": "VectorDB"},
        {"source": "wiki", "topic": "Alignment"},
    ],
)

# 使用自定义 Embedding 模型
from chromadb.utils.embedding_functions import OpenAIEmbeddingFunction

openai_ef = OpenAIEmbeddingFunction(
    api_key="YOUR_API_KEY",
    model_name="text-embedding-3-small",
)

collection = client.create_collection(
    name="knowledge_base_openai",
    embedding_function=openai_ef,
)

# 查询
results = collection.query(
    query_texts=["How to build LLM applications?"],
    n_results=3,
    where={"topic": "LLM"},  # Metadata 过滤
    include=["documents", "metadatas", "distances"],
)

for i, doc in enumerate(results["documents"][0]):
    dist = results["distances"][0][i]
    meta = results["metadatas"][0][i]
    print(f"Distance: {dist:.4f} | Topic: {meta['topic']} | {doc[:80]}...")
```

## 7. RAG 系统集成

```python
"""
完整的 RAG Pipeline：文档分块 → Embedding → 向量存储 → 检索 → 生成
"""
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_community.vectorstores import Chroma
from langchain.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# 1. 文档分块
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_size_overlap=50,
    separators=["\n\n", "\n", ". ", " "],
)

chunks = text_splitter.split_text(long_document_text)

# 2. 生成 Embedding 并存储到 Chroma
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = Chroma.from_texts(
    texts=chunks,
    embedding=embeddings,
    collection_name="rag_docs",
    persist_directory="./chroma_db",
)

# 3. 创建检索器
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 5},
)

# 4. RAG Chain
template = """Based on the following context, answer the question.

Context:
{context}

Question: {question}

Answer:"""

prompt = ChatPromptTemplate.from_template(template)
llm = ChatOpenAI(model="gpt-4", temperature=0)

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# 5. 查询
answer = rag_chain.invoke("What is the Transformer architecture?")
print(answer)
```

## 8. 性能调优最佳实践

1. **选择合适的 Embedding 模型**：参考 [MTEB Leaderboard](https://huggingface.co/spaces/mteb/leaderboard) 选择最佳模型。
2. **Chunk 大小调优**：一般 256-512 tokens，保留 10-20% 重叠避免上下文截断。
3. **索引参数调优**：
   - HNSW: `M=16~48`, `ef_construction=200~500`, `ef=50~200`
   - IVF: `nlist=sqrt(N)`, `nprobe=nlist*0.05`
4. **批量操作**：插入和查询都应批量处理，减少网络往返。
5. **Metadata 过滤优化**：高选择性过滤条件优先，减少向量搜索范围。
6. **数据预处理**：标准化文本、去重、去除无效内容，提升检索质量。
7. **监控召回率**：定期评估检索质量，使用 Recall@K 指标。
8. **冷热分层**：高频查询数据使用 HNSW（内存），冷数据使用 DiskANN/PQ（磁盘）。
