# 微前端架构

微前端是一种将前端应用分解为更小、更简单的片段的架构风格，允许多个团队独立开发和部署各自的模块。

## 1. 微前端核心概念

### 1.1 什么是微前端
微前端借鉴了微服务的理念，将大型前端应用拆分为多个小型应用，每个应用可以独立开发、测试和部署。

### 1.2 微前端优势
- **技术栈无关**：不同模块可以使用不同技术栈
- **独立开发部署**：团队可以独立工作，提升效率
- **渐进式升级**：可以逐步升级旧系统
- **故障隔离**：一个模块崩溃不影响其他模块

## 2. Single-SPA

### 2.1 Single-SPA简介
Single-SPA是一个微前端框架，允许在一个页面中使用多个前端框架。

### 2.2 主应用配置

```javascript
// main-app/src/index.js
import { registerApplication, start } from 'single-spa';

// 注册子应用
registerApplication({
  name: '@myorg/react-app',
  app: () => System.import('@myorg/react-app'),
  activeWhen: ['/react'],
  customProps: { authToken: 'abc123' },
});

registerApplication({
  name: '@myorg/vue-app',
  app: () => System.import('@myorg/vue-app'),
  activeWhen: ['/vue'],
});

start();
```

### 2.3 React子应用

```javascript
// react-app/src/index.js
import React from 'react';
import ReactDOM from 'react-dom';
import singleSpaReact from 'single-spa-react';
import App from './App';

const lifecycles = singleSpaReact({
  React,
  ReactDOM,
  rootComponent: App,
  errorBoundary(err, info, props) {
    return <div>Error: {err.message}</div>;
  },
});

export const { bootstrap, mount, unmount } = lifecycles;
```

### 2.4 Vue子应用

```javascript
// vue-app/src/main.js
import { createApp } from 'vue';
import singleSpaVue from 'single-spa-vue';
import App from './App.vue';

const vueLifecycles = singleSpaVue({
  createApp,
  appOptions: {
    render() {
      return h(App, { props: this.props });
    },
  },
});

export const { bootstrap, mount, unmount } = vueLifecycles;
```

## 3. qiankun

### 3.1 qiankun简介
qiankun是蚂蚁金服开源的微前端框架，基于Single-SPA，提供了更完善的解决方案。

### 3.2 主应用配置

```javascript
// main-app/src/index.js
import { registerMicroApps, start, setDefaultMountApp } from 'qiankun';

registerMicroApps([
  {
    name: 'react-app',
    entry: '//localhost:7100',
    container: '#subapp-container',
    activeRule: '/react',
    props: {
      routerBase: '/react',
      authToken: 'token123',
    },
  },
  {
    name: 'vue-app',
    entry: '//localhost:7200',
    container: '#subapp-container',
    activeRule: '/vue',
    props: {
      routerBase: '/vue',
    },
  },
]);

setDefaultMountApp('/react');
start();
```

### 3.3 子应用配置（React）

```javascript
// react-app/src/index.js
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

let root = null;

function render(props) {
  const { container } = props;
  const dom = container
    ? container.querySelector('#root')
    : document.getElementById('root');
  
  root = ReactDOM.createRoot(dom);
  root.render(<App />);
}

if (!window.__POWERED_BY_QIANKUN__) {
  render({});
}

export async function bootstrap() {
  console.log('React app bootstrapped');
}

export async function mount(props) {
  console.log('React app mounted', props);
  render(props);
}

export async function unmount() {
  root.unmount();
  root = null;
}
```

## 4. Module Federation

### 4.1 Module Federation简介
Module Federation是Webpack 5的新特性，允许在运行时动态加载远程模块。

### 4.2 Host应用配置

```javascript
// host/webpack.config.js
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');

module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'host',
      remotes: {
        remoteApp: 'remoteApp@http://localhost:3001/remoteEntry.js',
      },
      shared: {
        react: { singleton: true, requiredVersion: '^18.0.0' },
        'react-dom': { singleton: true, requiredVersion: '^18.0.0' },
      },
    }),
  ],
};
```

### 4.3 Remote应用配置

```javascript
// remote/webpack.config.js
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');

module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'remoteApp',
      filename: 'remoteEntry.js',
      exposes: {
        './Button': './src/components/Button',
        './utils': './src/utils',
      },
      shared: {
        react: { singleton: true, requiredVersion: '^18.0.0' },
        'react-dom': { singleton: true, requiredVersion: '^18.0.0' },
      },
    }),
  ],
};
```

### 4.4 使用远程组件


## 5. 沙箱隔离

### 5.1 JS沙箱

```javascript
// 简单的JS沙箱实现
class JSSandbox {
  constructor() {
    this.proxy = null;
    this.sandboxRunning = false;
  }

  active() {
    const rawWindow = window;
    const fakeWindow = {};
    
    this.proxy = new Proxy(fakeWindow, {
      get(target, key) {
        return target[key] || rawWindow[key];
      },
      set(target, key, value) {
        target[key] = value;
        return true;
      },
    });
    
    this.sandboxRunning = true;
  }

  inactive() {
    this.sandboxRunning = false;
  }

  exec(code) {
    if (!this.sandboxRunning) return;
    
    const fn = new Function('window', 'self', code);
    fn(this.proxy, this.proxy);
  }
}
```

### 5.2 CSS沙箱

```javascript
// CSS隔离方案
class CSSSandbox {
  constructor(appName) {
    this.appName = appName;
    this.styleElement = null;
  }

  // Shadow DOM隔离
  createShadowContainer(container) {
    const shadow = container.attachShadow({ mode: 'closed' });
    return shadow;
  }

  // Scoped CSS
  scopeCSS(css, scope) {
    return css.replace(/([^\r\n,{}]+)(,(?=[^}]*{)|\s*{)/g, (match) => {
      if (match.startsWith('@') || match.startsWith('from') || match.startsWith('to')) {
        return match;
      }
      return `.${scope} ${match}`;
    });
  }

  // 动态样式注入
  injectStyle(css) {
    this.styleElement = document.createElement('style');
    this.styleElement.setAttribute('data-app', this.appName);
    this.styleElement.textContent = this.scopeCSS(css, this.appName);
    document.head.appendChild(this.styleElement);
  }

  removeStyle() {
    if (this.styleElement) {
      document.head.removeChild(this.styleElement);
    }
  }
}
```

### 6.2 自定义事件总线

```javascript
// 简单的事件总线实现
class EventBus {
  constructor() {
    this.events = {};
  }

  on(event, callback) {
    if (!this.events[event]) {
      this.events[event] = [];
    }
    this.events[event].push(callback);
  }

  emit(event, ...args) {
    if (this.events[event]) {
      this.events[event].forEach(callback => callback(...args));
    }
  }

  off(event, callback) {
    if (this.events[event]) {
      this.events[event] = this.events[event].filter(cb => cb !== callback);
    }
  }
}

// 使用
const eventBus = new EventBus();
window.__MICRO_APP_EVENT_BUS__ = eventBus;
```

## 8. 最佳实践

### 8.1 技术栈选择
1. **qiankun**：适合大多数场景，生态成熟
2. **Module Federation**：适合Webpack 5项目，性能好
3. **Single-SPA**：需要高度定制时使用

### 8.2 应用拆分原则
1. **按业务域拆分**：每个子应用对应一个业务模块
2. **按团队拆分**：每个团队负责一个子应用
3. **独立部署**：子应用可以独立构建和部署

### 8.3 性能优化
1. **预加载**：配置prefetch预加载子应用
2. **共享依赖**：使用externals或Module Federation共享公共库
3. **懒加载**：按需加载子应用资源

### 8.4 开发调试
1. **本地联调**：配置代理指向本地开发环境
2. **独立运行**：子应用支持独立运行和调试
3. **统一日志**：建立统一的日志和监控体系

## 9. 常见问题

### Q1: 子应用样式冲突怎么办？
**A**: 
1. 使用CSS Modules或CSS-in-JS
2. 添加命名空间前缀
3. 使用Shadow DOM隔离

### Q2: 如何处理子应用的全局错误？
**A**: 
1. 在子应用中添加错误边界
2. 主应用监听unhandledrejection
3. 统一错误上报和处理

### Q3: 子应用之间如何共享数据？
**A**: 
1. 使用qiankun的globalState
2. 使用URL参数传递
3. 使用localStorage/sessionStorage
