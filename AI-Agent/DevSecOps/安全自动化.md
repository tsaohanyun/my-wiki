## 1. 安全测试分层

```
┌─────────────────────────────────────────────────────────┐
│                    安全面测试分层                          │
├───────────┬───────────┬───────────┬─────────────────────┤
│   IDE     │  Pre-Commit │  CI/CD   │   运行时            │
│  (开发期)  │  (提交前)    │ (构建期)  │   (部署后)          │
├───────────┼───────────┼───────────┼─────────────────────┤
│ IDE 插件   │ Git Hooks  │ SAST     │ DAST                │
│ Semgrep    │ Secret     │ SCA      │ IAST                │
│ Snyk      │ Pre-commit │ License  │ RASP                │
│           │            │ Container│ Fuzz Testing        │
│           │            │ Scan     │ Pen Test            │
└───────────┴───────────┴───────────┴─────────────────────┘
```

| 类型 | 全称 | 检测方式 | 优势 | 局限 |
|------|------|---------|------|------|
| **SAST** | Static Application Security Testing | 不运行代码，静态分析源码 | 早期检测，覆盖率高 | 误报率高 |
| **DAST** | Dynamic Application Security Testing | 运行状态下黑盒测试 | 发现真实可利用漏洞 | 覆盖率有限，发现晚 |
| **IAST** | Interactive Application Security Testing | 运行时插桩，灰盒测试 | 误报率极低 | 需要插桩代理 |

## 2. SAST — 静态应用安全测试

### 2.1 Semgrep — 轻量级多语言 SAST

```bash
# 安装
pip install semgrep

# 扫描项目
semgrep scan --config=auto .

# 使用特定规则集
semgrep scan --config=p/default .
semgrep scan --config=p/owasp-top-ten .
semgrep scan --config=p/security-audit .
semgrep scan --config=p/java .
semgrep scan --config=p/typescript .

# 输出 SARIF 格式（CI 集成）
semgrep scan --config=auto --sarif -o semgrep-results.sarif .

# 输出 JSON
semgrep scan --config=auto --json -o semgrep-results.json .
```

#### 自定义 Semgrep 规则

```yaml
# .semgrep/rules.yml
rules:
  # 检测硬编码密码
  - id: hardcoded-password
    pattern: password = "..."
    message: 检测到硬编码密码
    languages: [python, java, javascript]
    severity: ERROR
    metadata:
      cwe: "CWE-798: Use of Hard-coded Credentials"
      owasp: "A07:2021 – Identification and Authentication Failures"

  # 检测 SQL 注入
  - id: sql-injection-string-format
    patterns:
      - pattern: |
          $CUR.execute("..." + $VAR + "...")
      - pattern-not: |
          $CUR.execute("..." + "?", ...)
    message: 可能存在 SQL 注入
    languages: [python]
    severity: ERROR

  # 检测不安全的反序列化
  - id: unsafe-deserialization
    pattern: pickle.loads($DATA)
    message: 不安全的反序列化，可能导致远程代码执行
    languages: [python]
    severity: ERROR
    metadata:
      cwe: "CWE-502: Deserialization of Untrusted Data"

  # 检测 Node.js 中危险的 require
  - id: dangerous-require
    pattern-either:
      - pattern: eval($REQ.body.$X)
      - pattern: child_process.exec($REQ.body.$X)
    message: 用户输入直接进入危险函数
    languages: [javascript]
    severity: ERROR
```

### 2.3 CodeQL (GitHub) — 语义化代码分析

```yaml
# .github/workflows/codeql.yml
name: CodeQL Analysis

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  schedule:
    - cron: "0 2 * * 1"  # 每周一扫描

jobs:
  analyze:
    runs-on: ubuntu-latest
    permissions:
      actions: read
      contents: read
      security-events: write
    strategy:
      matrix:
        language: [python, javascript, java]
    steps:
      - uses: actions/checkout@v4
      - name: Initialize CodeQL
        uses: github/codeql-action/init@v3
        with:
          languages: ${{ matrix.language }}
          queries: security-extended
      - name: Autobuild
        uses: github/codeql-action/autobuild@v3
      - name: Perform CodeQL Analysis
        uses: github/codeql-action/analyze@v3
        with:
          category: "/language:${{ matrix.language }}"
```

### 3.1 OWASP ZAP — 开源 DAST 工具

```bash
# Docker 运行 ZAP 扫描
docker run -t owasp/zap2docker-stable zap-baseline.py \
  -t https://example.com \
  -g gen.conf \
  -r zap-report.html \
  -J zap-report.json

# 完全扫描（爬虫 + 主动扫描）
docker run -t owasp/zap2docker-stable zap-full-scan.py \
  -t https://example.com \
  -g gen.conf \
  -r zap-report.html \
  -J zap-report.json \
  -n context.context

# API 扫描 (OpenAPI)
docker run -t owasp/zap2docker-stable zap-api-scan.py \
  -t https://example.com/openapi.json \
  -f openapi \
  -r zap-report.html
```

### 3.2 ZAP 自动化配置

```yaml
# zap-automation.yaml
env:
  contexts:
    - name: "production"
      urls:
        - "https://example.com"
      includePaths:
        - "https://example.com/.*"
      excludePaths:
        - "https://example.com/logout.*"
      authentication:
        type: "form-based"
        loginUrl: "https://example.com/login"
        loginRequestData: "username=admin&password=admin"

  - type: openapi
    parameters:
      apiFile: "openapi.yaml"
      targetUrl: "https://example.com"
  - type: spider
    parameters:
      context: "production"
      maxDuration: 60
  - type: activeScan
    parameters:
      context: "production"
      policy: "API-MINIMAL"
  - type: report
    parameters:
      template: "traditional-html"
      reportDir: "/zap/wrk/"
      reportFile: "zap-report.html"
```

### 3.3 CI/CD 集成 ZAP

```yaml
# .gitlab-ci.yml
dast:
  stage: test
  image: owasp/zap2docker-stable
  variables:
    target_website: "https://staging.example.com"
  script:
    - zap-baseline.py -t $target_website -J zap-report.json -r zap-report.html
    # 如果有 HIGH 级别漏洞则失败
    - |
      HIGH_COUNT=$(jq '[.site[].alerts[]? | select(.riskcode == "3")] | length' zap-report.json)
      if [ "$HIGH_COUNT" -gt 0 ]; then
        echo "发现 $HIGH_COUNT 个高危漏洞"
        exit 1
      fi
  artifacts:
    paths:
      - zap-report.html
      - zap-report.json
    when: always
  allow_failure: false
```

## 5. 安全门禁策略

### 5.1 安全门禁决策模型

```
                    CI/CD Pipeline
                         │
                         ▼
              ┌─────────────────────┐
              │   安全扫描执行        │
              │  (SAST/SCA/Secret)  │
              └──────────┬──────────┘
                         │
                         ▼
              ┌─────────────────────┐
              │  漏洞分级 & 计数      │
              │  CRITICAL / HIGH /  │
              │  MEDIUM / LOW       │
              └──────────┬──────────┘
                         │
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
    ┌────────────┐ ┌──────────┐ ┌───────────┐
    │ CRITICAL   │ │ HIGH     │ │ MEDIUM/   │
    │ → BLOCK    │ │ → WARN   │ │ LOW       │
    │ 构建失败    │ │ PR 标记   │ │ → 报告    │
    └────────────┘ └──────────┘ └───────────┘
```

### 5.2 GitHub Actions 安全门禁

```yaml
# .github/workflows/security-gate.yml
name: Security Gate

on: [pull_request]

jobs:
  security-gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # 1. Secret 扫描
      - name: Secret Scan (Gitleaks)
        uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      # 2. SAST 扫描
      - name: Semgrep Scan
        uses: returntocorp/semgrep-action@v1
        with:
          config: >-
            p/default
            p/owasp-top-ten
            p/security-audit

      # 3. SCA 扫描
      - name: Trivy FS Scan
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: fs
          scan-ref: .
          severity: CRITICAL,HIGH
          format: json
          output: trivy-results.json

      # 4. 安全门禁判断
      - name: Evaluate Security Gate
        run: |
          CRITICAL=$(jq '[.Results[].Vulnerabilities[]? | select(.Severity == "CRITICAL")] | length' trivy-results.json)
          HIGH=$(jq '[.Results[].Vulnerabilities[]? | select(.Severity == "HIGH")] | length' trivy-results.json)

          echo "CRITICAL 漏洞数: $CRITICAL"
          echo "HIGH 漏洞数: $HIGH"

          if [ "$CRITICAL" -gt 0 ]; then
            echo "❌ 安全门禁失败：存在 $CRITICAL 个严重漏洞"
            exit 1
          fi

          if [ "$HIGH" -gt 5 ]; then
            echo "❌ 安全门禁失败：高危漏洞超过阈值 ($HIGH > 5)"
            exit 1
          fi

          echo "✅ 安全门禁通过"
```

```yaml
# .github/workflows/auto-merge-security-prs.yml
name: Auto-merge Security PRs

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  auto-merge:
    runs-on: ubuntu-latest
    if: contains(github.event.pull_request.labels.*.name, 'security')
    steps:
      - name: Wait for checks
        uses: lewagon/wait-on-check-action@v1.3.1
        with:
          ref: ${{ github.event.pull_request.head.sha }}
          check-name: "tests"
          repo-token: ${{ secrets.GITHUB_TOKEN }}
          wait-interval: 30

      - name: Auto-merge
        if: success()
        run: |
          gh pr merge ${{ github.event.pull_request.number }} \
            --squash \
            --delete-branch \
            --repo ${{ github.repository }}
        env:
          GH_TOKEN: ${{ secrets.GH_AUTO_MERGE_TOKEN }}
```

## 7. 安全 Bot

### 7.1 GitHub Security Bot

GitHub 内置的 Dependabot 和 CodeQL 自动化 Bot，无需额外配置：

- **Dependabot alerts** — 自动检测依赖漏洞并创建 PR
- **Code scanning alerts** — CodeQL 扫描结果自动出现在 PR 中
- **Secret scanning** — 检测提交中的密钥泄露

### 7.2 自定义安全 Bot (GitHub App)

```python
#!/usr/bin/env python3
"""安全 Bot — 自动响应安全相关 PR"""

import os
import json
import requests
from github import Github

GITHUB_TOKEN = os.environ["GITHUB_TOKEN"]
REPO_NAME = os.environ["GITHUB_REPOSITORY"]
SEVERITY_THRESHOLD = "HIGH"

g = Github(GITHUB_TOKEN)
repo = g.get_repo(REPO_NAME)

def check_pr_security(pr_number: int):
    """检查 PR 的安全扫描结果"""
    pr = repo.get_pull(pr_number)

    # 检查是否有安全标签
    labels = [label.name for label in pr.get_labels()]

    # 获取 CodeQL 扫描告警
    alerts_url = f"https://api.github.com/repos/{REPO_NAME}/code-scanning/alerts"
    headers = {"Authorization": f"token {GITHUB_TOKEN}"}
    response = requests.get(alerts_url, headers=headers)
    alerts = response.json()

    new_alerts = [
        a for a in alerts
        if a.get("state") == "open"
        and a.get("rule", {}).get("security_severity_level") in ["critical", "high"]
    ]

    if new_alerts:
        # 添加评论
        comment = f"""## ⚠️ 安全扫描告警

检测到 **{len(new_alerts)}** 个高危/严重安全告警：

"""
        for alert in new_alerts[:5]:
            rule = alert.get("rule", {})
            comment += f"- **{rule.get('security_severity_level', 'unknown').upper()}**: {rule.get('description', 'N/A')}\n"

        comment += "\n请修复以上安全问题后再合并。"
        pr.create_issue_comment(comment)

        # 请求更改
        pr.create_review(
            event="REQUEST_CHANGES",
            body="安全扫描发现问题，请修复后再合并。"
        )

        # 添加标签
        pr.add_to_labels("security-blocked")
    else:
        # 安全通过
        pr.create_issue_comment("✅ 安全扫描通过，无高危问题。")
        pr.add_to_labels("security-approved")

# Webhook 入口
def handle_webhook(event_data: dict):
    if event_data.get("action") == "opened":
        pr_number = event_data["pull_request"]["number"]
        check_pr_security(pr_number)
```

### 7.3 Slack/Teams 安全通知 Bot

```python
#!/usr/bin/env python3
"""安全告警 Slack 通知 Bot"""

import json
import requests
from datetime import datetime

SLACK_WEBHOOK = os.environ["SLACK_WEBHOOK_URL"]

def send_security_alert(alert_data: dict):
    """发送安全告警到 Slack"""
    severity_colors = {
        "CRITICAL": "#FF0000",
        "HIGH": "#FF8C00",
        "MEDIUM": "#FFD700",
        "LOW": "#00CED1"
    }

    severity = alert_data.get("severity", "LOW")
    color = severity_colors.get(severity, "#808080")

    payload = {
        "channel": "#security-alerts",
        "attachments": [{
            "color": color,
            "title": f"🚨 安全告警 — {alert_data.get('title', 'Unknown')}",
            "fields": [
                {"title": "严重程度", "value": severity, "short": True},
                {"title": "漏洞类型", "value": alert_data.get("type", "N/A"), "short": True},
                {"title": "受影响组件", "value": alert_data.get("component", "N/A"), "short": True},
                {"title": "CVSS 评分", "value": str(alert_data.get("cvss", "N/A")), "short": True},
                {"title": "详情", "value": alert_data.get("description", ""), "short": False},
                {"title": "修复建议", "value": alert_data.get("remediation", ""), "short": False},
            ],
            "footer": f"Security Bot | {datetime.now().isoformat()}",
            "actions": [
                {
                    "type": "button",
                    "text": "查看详情",
                    "url": alert_data.get("url", "#")
                },
                {
                    "type": "button",
                    "text": "创建工单",
                    "url": f"https://jira.example.com/create?issue={json.dumps(alert_data)}"
                }
            ]
        }]
    }

    response = requests.post(SLACK_WEBHOOK, json=payload)
    return response.status_code == 200
```

## 8. 最佳实践

### ✅ 推荐做法

| 实践 | 说明 |
|------|------|
| **分层安全测试** | SAST（编码期）+ SCA（构建期）+ DAST（部署后）多管齐下 |
| **安全门禁渐进式** | 先报告 → 后阻断，让团队有时间适应 |
| **减少误报** | 使用 IAST 替代部分 SAST，或定制规则排除已知误报 |
| **修复而非仅报告** | 使用 Dependabot/Semgrep autofix 自动创建修复 PR |
| **开发者教育** | 在 IDE 中集成安全插件，实时反馈 |
| **安全指标仪表盘** | 跟踪 MTTR（平均修复时间）、漏洞密度、误报率 |
| **定期评估工具覆盖** | 确保新语言/框架有对应的安全扫描 |

### ❌ 避免

| 反模式 | 风险 |
|--------|------|
| 仅依赖一种工具 | 不同工具覆盖不同漏洞类型 |
| 阻断所有级别漏洞 | 导致安全疲劳，开发者开始忽略告警 |
| 安全扫描在部署后 | 发现问题太晚，修复成本高 |
| 不更新规则库 | 新漏洞模式无法被检测 |
| 无漏洞修复 SLA | 漏洞长期存在，成为攻击目标 |
