# 合规自动化

> 合规自动化（Compliance as Code）将安全合规标准（如 CIS Benchmarks、PCI-DSS、SOC 2）转化为**可执行、可验证的自动化检查**，消除手工审计的低效与偏差。

## 1. 合规框架概览

| 框架 | 适用场景 | 关键控制域 |
|------|---------|-----------|
| **CIS Benchmarks** | OS / 云平台 / 应用安全配置 | 系统加固、访问控制、日志审计 |
| **PCI-DSS** | 信用卡支付处理 | 数据加密、网络隔离、漏洞管理 |
| **SOC 2** | 云服务 SaaS | 安全性、可用性、保密性 |
| **NIST 800-53** | 美国联邦系统 | 访问控制、审计、事件响应 |
| **ISO 27001** | 信息安全管理体系 | 风险管理、资产管理 |
| **HIPAA** | 医疗健康数据 | 数据保护、访问审计 |
| **GDPR** | 欧盟个人数据 | 数据隐私、用户权利 |

## 2. CIS Benchmarks

CIS (Center for Internet Security) Benchmarks 是业界公认的**安全配置基线标准**，覆盖 100+ 技术栈。

### 2.1 CIS Benchmark 分级

| 级别 | 说明 | 影响 |
|------|------|------|
| **Level 1** | 基础安全配置，最小影响 | 推荐所有环境 |
| **Level 2** | 深度安全配置，需评估影响 | 推荐高安全需求环境 |

### 2.2 CIS Benchmark 示例（Linux）

| 控制项 | Level | 描述 |
|--------|-------|------|
| 1.1.1.1 | L1 | 确保禁用 cramfs 文件系统 |
| 1.4.1 | L1 | 确保 permissions on bootloader config 已设置 |
| 4.2.1.1 | L1 | 确保 rsyslog 已安装 |
| 5.2.1 | L1 | 确保 SSH Protocol 设置为 2 |
| 6.1.2 | L1 | 确保 permissions on /etc/passwd 已设置 (644) |
| 6.1.10 | L1 | 确保没有 world-writable 文件 |

## 4. OpenSCAP

### 4.1 OpenSCAP 安装

```bash
# RHEL / CentOS / Rocky Linux
yum install -y openscap-scanner scap-security-guide

# Ubuntu / Debian
apt install -y openscap-scanner ssg-base

# 查看可用的 SCAP 安全指南
ls /usr/share/xml/scap/ssg/content/
# 输出示例:
# ssg-ubuntu2204-ds.xml
# ssg-rhel9-ds.xml
# ssg-debian12-ds.xml
```

### 4.2 运行合规扫描

```bash
# 扫描 Ubuntu 22.04 (CIS Benchmark)
oscap xccdf eval \
  --profile xccdf_org.ssgproject.content_profile_cis_server_l1 \
  --results scan-results.xml \
  --report scan-report.html \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml

# 仅扫描特定规则
oscap xccdf eval \
  --profile xccdf_org.ssgproject.content_profile_cis_server_l1 \
  --rule xccdf_org.ssgproject.content_rule_sshd_set_keepalive \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml

# 生成修复脚本
oscap xccdf generate fix \
  --profile xccdf_org.ssgproject.content_profile_cis_server_l1 \
  --output remediation.sh \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml
```

### 4.3 OpenSCAP 验证修复

```bash
# 1. 生成修复脚本
oscap xccdf generate fix \
  --profile xccdf_org.ssgproject.content_profile_cis_server_l1 \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml > remediation.sh

# 2. 审查修复脚本
less remediation.sh

# 3. 执行修复
chmod +x remediation.sh
./remediation.sh

# 4. 重新扫描验证
oscap xccdf eval \
  --profile xccdf_org.ssgproject.content_profile_cis_server_l1 \
  --results scan-after.xml \
  --report scan-after.html \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml

# 5. 对比前后差异
oscap xccdf eval \
  --profile xccdf_org.ssgproject.content_profile_cis_server_l1 \
  --results-arf scan-after-arf.xml \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml
```

### 4.4 CI/CD 集成

```yaml
# .gitlab-ci.yml
cis-compliance:
  stage: test
  image: openscap/openscap:latest
  script:
    # 安装安全指南
    - yum install -y scap-security-guide
    # 运行扫描
    - oscap xccdf eval
        --profile xccdf_org.ssgproject.content_profile_cis_server_l1
        --results results.xml
        --report report.html
        /usr/share/xml/scap/ssg/content/ssg-rhel9-ds.xml
    # 提取通过率
    - |
      PASS=$(xmllint --xpath 'string(//TestResult/score[@system="urn:xccdf:scoring:default"])' results.xml)
      echo "合规得分: $PASS"
      # 要求通过率 > 90%
      COMPLIANCE_THRESHOLD=90
      SCORE_INT=$(echo "$PASS * 100" | bc | cut -d. -f1)
      if [ "$SCORE_INT" -lt "$COMPLIANCE_THRESHOLD" ]; then
        echo "❌ 合规不达标: $SCORE_INT% < $COMPLIANCE_THRESHOLD%"
        exit 1
      fi
      echo "✅ 合规达标: $SCORE_INT%"
  artifacts:
    paths:
      - report.html
      - results.xml
    expire_in: 90 days
```

### 5.4 Kubernetes 加固检查清单

```yaml
# Kyverno 策略 — 实施 CIS Kubernetes Benchmark
---
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: cis-k8s-benchmark
spec:
  validationFailureAction: Enforce
  rules:
    # 1. 禁止特权容器 (CIS 5.2.1)
    - name: disallow-privileged-containers
      match:
        resources:
          kinds: [Pod]
      validate:
        message: "禁止特权容器 (CIS 5.2.1)"
        pattern:
          spec:
            containers:
              - securityContext:
                  privileged: "false"

    # 2. 禁止 hostPID/hostIPC (CIS 5.2.2, 5.2.3)
    - name: disallow-host-pid-ipc
      match:
        resources:
          kinds: [Pod]
      validate:
        message: "禁止 hostPID 和 hostIPC (CIS 5.2.2/5.2.3)"
        pattern:
          spec:
            =(hostPID): "false"
            =(hostIPC): "false"

    # 3. 必须设置 resource limits (CIS 5.2.4)
    - name: require-resource-limits
      match:
        resources:
          kinds: [Pod]
      validate:
        message: "必须设置 resource limits (CIS 5.2.4)"
        pattern:
          spec:
            containers:
              - resources:
                  limits:
                    memory: "?*"
                    cpu: "?*"

    # 4. 只读 root 文件系统 (CIS 5.2.7)
    - name: require-readonly-rootfs
      match:
        resources:
          kinds: [Pod]
      validate:
        message: "root 文件系统必须为只读 (CIS 5.2.7)"
        pattern:
          spec:
            containers:
              - securityContext:
                  readOnlyRootFilesystem: true

    # 5. 必须以非 root 运行 (CIS 5.2.6)
    - name: require-non-root-user
      match:
        resources:
          kinds: [Pod]
      validate:
        message: "必须以非 root 用户运行 (CIS 5.2.6)"
        pattern:
          spec:
            containers:
              - securityContext:
                  runAsNonRoot: true
```

## 6. 合规报告生成

### 6.1 Chef InSpec — Compliance as Code

```ruby
# controls/cis_ubuntu.rb — CIS Ubuntu 合规控制

control "cis-ubuntu-1.1.1.1" do
  impact 1.0
  title "Ensure mounting of cramfs filesystems is disabled"
  desc "The cramfs filesystem type is a compressed read-only Linux filesystem."
  tag cis: ["ubuntu", "1.1.1.1"]
  tag level: 1

  describe kernel_module("cramfs") do
    it { should_not be_loaded }
    it { should be_disabled }
    it { should be_blacklisted }
  end
end

control "cis-ubuntu-5.2.1" do
  impact 1.0
  title "Ensure SSH Protocol is set to 2"
  desc "SSH supports two protocols: SSH v1 and SSH v2"
  tag cis: ["ubuntu", "5.2.1"]
  tag level: 1

  describe sshd_config do
    its("Protocol") { should cmp "2" }
  end
end

control "cis-ubuntu-4.2.1.1" do
  impact 1.0
  title "Ensure rsyslog is installed"
  desc "The rsyslog software is recommended"
  tag cis: ["ubuntu", "4.2.1.1"]
  tag level: 1

  describe package("rsyslog") do
    it { should be_installed }
  end
end
```

```bash
# 运行 InSpec 扫描
inspec exec controls/cis_ubuntu.rb \
  --target ssh://user@server \
  --input-file attributes.yml \
  --reporter cli json:/reports/inspec-report.json html:/reports/inspec-report.html

# 使用 Chef Automate 上传合规报告
inspec compliance upload profiles/cis-ubuntu2204
```

### 6.2 自动化合规仪表盘

```yaml
# compliance-dashboard-values.yaml (Grafana + InSpec)
grafana:
  dashboards:
    - name: "CIS Compliance Dashboard"
      panels:
        - title: "合规通过率趋势"
          type: graph
          datasource: Prometheus
          targets:
            - expr: 'compliance_score_percent'
              legendFormat: "{{hostname}}"
        - title: "未合规控制项 Top 10"
          type: table
          datasource: PostgreSQL
          targets:
            - query: |
                SELECT control_id, title, COUNT(*) as failures
                FROM compliance_results
                WHERE result = 'failed'
                GROUP BY control_id, title
                ORDER BY failures DESC
                LIMIT 10
```

## 7. 审计日志

### 7.1 Linux 审计

```bash
# 安装 auditd
yum install -y audit    # RHEL
apt install -y auditd   # Ubuntu

# 启动审计服务
systemctl enable --now auditd

# 添加审计规则 (CIS 推荐)
# 监控 /etc/passwd 变更
auditctl -w /etc/passwd -p wa -k identity
# 监控 /etc/group 变更
auditctl -w /etc/group -p wa -k identity
# 监控 /etc/sudoers 变更
auditctl -w /etc/sudoers -p wa -k scope
# 监控登录事件
auditctl -w /var/log/lastlog -p wa -k logins
# 监控 SSH 配置变更
auditctl -w /etc/ssh/sshd_config -p wa -k ssh_config
```

```bash
# /etc/audit/rules.d/audit.rules (持久化规则)
## CIS Audit Rules
-D
-b 8192
-f 1

# Identity changes
-w /etc/group -p wa -k identity
-w /etc/passwd -p wa -k identity
-w /etc/gshadow -p wa -k identity
-w /etc/shadow -p wa -k identity
-w /etc/security/opasswd -p wa -k identity

# Network environment
-w /etc/hosts -p wa -k system-locale
-w /etc/sysconfig/network -p wa -k system-locale

# SSH configuration
-w /etc/ssh/sshd_config -p wa -k ssh_config

# Cron configuration
-w /etc/cron.d -p wa -k cron
-w /etc/cron.daily -p wa -k cron
-w /etc/cron.hourly -p wa -k cron
-w /etc/cron.monthly -p wa -k cron
-w /etc/cron.weekly -p wa -k cron

# Kernel modules
-a always,exit -F arch=b64 -S init_module -S delete_module -k modules

# File deletion
-a always,exit -F arch=b64 -S unlink -S unlinkat -S rename -S renameat -k delete
```

### 7.2 Kubernetes 审计日志

```yaml
# /etc/kubernetes/audit-policy.yaml
apiVersion: audit.k8s.io/v1
kind: Policy
# 不记录以下请求
omitStages:
  - RequestReceived
rules:
  # 记录 Secret 操作
  - level: RequestResponse
    resources:
      - group: ""
        resources: ["secrets", "configmaps"]
    verbs: ["create", "update", "patch", "delete"]

  # 记录认证授权变更
  - level: RequestResponse
    resources:
      - group: "rbac.authorization.k8s.io"
        resources: ["roles", "rolebindings", "clusterroles", "clusterrolebindings"]
    verbs: ["create", "update", "patch", "delete"]

  # 记录 Pod/Deployment 变更
  - level: Request
    resources:
      - group: ""
        resources: ["pods"]
      - group: "apps"
        resources: ["deployments", "daemonsets", "statefulsets"]
    verbs: ["create", "update", "patch", "delete"]

  # 记录其他写操作
  - level: Metadata
    verbs: ["create", "update", "patch", "delete"]

  # 不记录只读请求
  - level: None
    verbs: ["get", "list", "watch"]
```

## 8. 最佳实践

### ✅ 推荐做法

| 实践 | 说明 |
|------|------|
| **Compliance as Code** | 将合规规则编码为可执行检查（InSpec/OpenSCAP） |
| **定期扫描 + 事件驱动** | 每周基线扫描 + 变更时即时扫描 |
| **合规基线管理** | 使用 IaC (Terraform/Ansible) 管理基线配置 |
| **审计日志集中存储** | 发送到 SIEM (ELK/Splunk)，不可篡改 |
| **合规报告自动化** | CI/CD 自动生成合规报告并存储 |
| **持续监控而非年度审计** | 实时合规仪表盘替代一次性审计 |
| **最小权限原则** | 基于 RBAC 严格限制访问权限 |

### ❌ 避免

| 反模式 | 风险 |
|--------|------|
| 手工检查合规 | 低效、易遗漏、不可重现 |
| 合规检查只在部署前 | 配置漂移无法及时发现 |
| 审计日志存本地 | 可被篡改，不符合合规要求 |
| 未加密审计日志 | 日志中可能包含敏感信息 |
| 无合规修复 SLA | 发现问题但长期不修复 |
