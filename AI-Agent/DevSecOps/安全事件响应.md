# 安全事件响应

> 建立完整的安全事件响应体系，从检测（SIEM）、编排（SOAR）到处置（取证/恢复），实现快速、可重复的安全事件处理流程。

---

## 概述

安全事件响应（Security Incident Response, SIR）是 DevSecOps 的最终防线。即使有完善的预防措施，安全事件仍会发生，关键在于**快速检测**和**高效响应**。

### 1.3 常见 SIEM 检测规则

```yaml
# siem/detection-rules.yaml — 安全检测规则集合
rules:
  # ─── 暴力破解检测 ───
  - id: DETECT-BRUTE-FORCE
    name: "SSH 暴力破解检测"
    severity: medium
    logic:
      condition: "同一源IP在5分钟内 > 10 次失败登录"
      query: |
        source.ip: * AND event.action: "ssh_failed" AND
        count() > 10 GROUP BY source.ip WINDOW 5m
    response: "auto_block_ip"

  # ─── 异常数据外传 ───
  - id: DETECT-DATA-EXFIL
    name: "异常数据外传检测"
    severity: high
    logic:
      condition: "出站流量 > 正常基线的 3 倍且目标为未知IP"
      query: |
        network.direction: "outbound" AND
        network.bytes > 100000000 AND
        NOT destination.ip IN (known_whitelist)
    response: "alert_soc_team"

  # ─── 容器逃逸 ───
  - id: DETECT-CONTAINER-ESCAPE
    name: "容器逃逸尝试检测"
    severity: critical
    logic:
      condition: "容器内执行宿主机命令或访问 /proc/1/root"
      query: |
        container.id: * AND
        (process.args: "/proc/1/root" OR
         process.args: "/var/run/docker.sock" OR
         event.action: "privileged_operation")
    response: "isolate_pod"

  # ─── 密钥泄露 ───
  - id: DETECT-SECRET-LEAK
    name: "Git 提交中检测到密钥"
    severity: high
    logic:
      condition: "Git commit 中包含 API Key / Token 模式"
      source: "trufflehog / gitleaks"
    response: "revoke_secret"

  # ─── 横向移动 ───
  - id: DETECT-LATERAL-MOVEMENT
    name: "横向移动行为检测"
    severity: high
    logic:
      condition: "单个主机在短时间内访问多个内部服务"
      query: |
        source.ip: * AND destination.service.name IN (internal_services) AND
        cardinality(destination.service.name) > 5 WINDOW 10m
    response: "quarantine_host"
```

## 2. SOAR — 安全编排自动化与响应

### 2.1 自动化响应剧本 (Playbook)

```python
#!/usr/bin/env python3
"""
SOAR 自动化响应剧本 — 容器逃逸事件处置
当 SIEM 检测到容器逃逸时自动执行以下步骤
"""
import subprocess
import json
import requests
from datetime import datetime
from enum import Enum

class IncidentSeverity(Enum):
    LOW = "low"
    MEDIUM = "medium"
    HIGH = "high"
    CRITICAL = "critical"

class ContainerEscapePlaybook:
    """容器逃逸事件自动化响应剧本"""

    def __init__(self, incident_data: dict):
        self.incident = incident_data
        self.incident_id = f"INC-{datetime.now().strftime('%Y%m%d%H%M%S')}"
        self.actions_log = []

    def execute(self):
        """执行完整的响应流程"""
        print(f"[🚨] 安全事件 {self.incident_id} — 容器逃逸")
        print(f"     严重度: {IncidentSeverity.CRITICAL.value}")
        print(f"     目标: {self.incident.get('pod_name')}")

        self.step1_isolate_pod()
        self.step2_capture_forensics()
        self.step3_block_ip()
        self.step4_revoke_credentials()
        self.step5_notify_team()
        self.step6_create_ticket()

        print(f"\n[✓] 自动化响应完成 — 事件 {self.incident_id}")
        return self.generate_report()

    def step1_isolate_pod(self):
        """Step 1: 隔离受影响的 Pod"""
        pod_name = self.incident.get("pod_name")
        namespace = self.incident.get("namespace", "default")

        print(f"\n[Step 1] 隔离 Pod: {namespace}/{pod_name}")

        # 应用网络隔离策略
        isolation_policy = f"""
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: quarantine-{pod_name}
  namespace: {namespace}
spec:
  podSelector:
    matchLabels:
      app: {pod_name}
  policyTypes:
    - Ingress
    - Egress
  ingress: []   # 拒绝所有入站
  egress: []    # 拒绝所有出站
"""
        # 应用隔离策略
        subprocess.run(
            ["kubectl", "apply", "-f", "-"],
            input=isolation_policy.encode(),
            capture_output=True
        )
        self.actions_log.append({
            "step": 1,
            "action": "isolate_pod",
            "status": "completed",
            "detail": f"Pod {pod_name} 已隔离，网络已断开",
        })
        print(f"  [✓] Pod 已隔离")

    def step2_capture_forensics(self):
        """Step 2: 捕获取证数据"""
        pod_name = self.incident.get("pod_name")
        namespace = self.incident.get("namespace", "default")

        print(f"\n[Step 2] 捕获取证数据")

        # 导出容器内存和文件系统
        container_id = subprocess.run(
            ["kubectl", "get", "pod", pod_name, "-n", namespace,
             "-o", "jsonpath={.status.containerStatuses[0].containerID}"],
            capture_output=True, text=True
        ).stdout.replace("containerd://", "")

        # 创建取证快照
        subprocess.run(
            ["crictl", "export", container_id, f"/forensics/{self.incident_id}/filesystem.tar"],
            capture_output=True
        )

        # 收集容器日志
        subprocess.run(
            ["kubectl", "logs", pod_name, "-n", namespace,
             "--previous", f"--output=/forensics/{self.incident_id}/container.log"],
            capture_output=True
        )

        self.actions_log.append({
            "step": 2,
            "action": "capture_forensics",
            "status": "completed",
            "detail": f"取证数据已保存到 /forensics/{self.incident_id}/",
        })
        print(f"  [✓] 取证数据已捕获")

    def step3_block_ip(self):
        """Step 3: 封禁攻击源IP"""
        source_ip = self.incident.get("source_ip")
        if not source_ip:
            return

        print(f"\n[Step 3] 封禁攻击源IP: {source_ip}")

        # 在防火墙/WAF中添加封禁规则
        block_rule = {
            "action": "block",
            "source_ip": source_ip,
            "duration": "24h",
            "reason": f"容器逃逸事件 {self.incident_id}",
        }
        # requests.post("https://waf.internal/api/block", json=block_rule)

        self.actions_log.append({
            "step": 3,
            "action": "block_ip",
            "status": "completed",
            "detail": f"IP {source_ip} 已封禁 24h",
        })
        print(f"  [✓] 源IP已封禁")

    def step4_revoke_credentials(self):
        """Step 4: 撤销可能泄露的凭据"""
        print(f"\n[Step 4] 撤销凭据")

        # 检查该 Pod 使用的 ServiceAccount 和密钥
        # 撤销相关 Token 和密钥
        actions = [
            "已撤销 Pod 关联的 ServiceAccount Token",
            "已轮换该命名空间下的所有 K8s Secret",
            "已通知 Vault 管理员轮换该应用使用的密钥",
        ]
        for action in actions:
            print(f"  [✓] {action}")

        self.actions_log.append({
            "step": 4,
            "action": "revoke_credentials",
            "status": "completed",
            "detail": "; ".join(actions),
        })

    def step5_notify_team(self):
        """Step 5: 通知安全团队"""
        print(f"\n[Step 5] 通知安全团队")

        notification = {
            "incident_id": self.incident_id,
            "severity": "CRITICAL",
            "type": "Container Escape Detected",
            "pod": self.incident.get("pod_name"),
            "namespace": self.incident.get("namespace"),
            "timestamp": datetime.now().isoformat(),
            "actions_taken": len(self.actions_log),
            "playbook": "container-escape-response",
        }

        # 发送到 Slack / Teams / PagerDuty
        # requests.post("https://hooks.slack.com/...", json={
        #     "text": f"🚨 CRITICAL: {notification['type']} (ID: {self.incident_id})"
        # })

        self.actions_log.append({
            "step": 5,
            "action": "notify_team",
            "status": "completed",
            "detail": "安全团队已通过 PagerDuty 和 Slack 通知",
        })
        print(f"  [✓] 团队已通知")

    def step6_create_ticket(self):
        """Step 6: 创建事件工单"""
        print(f"\n[Step 6] 创建事件工单")

        ticket = {
            "title": f"[CRITICAL] 容器逃逸事件 — {self.incident_id}",
            "description": json.dumps(self.incident, indent=2),
            "priority": "P1",
            "assignee": "security-on-call",
            "labels": ["security", "incident-response", "container-escape"],
        }

        self.actions_log.append({
            "step": 6,
            "action": "create_ticket",
            "status": "completed",
            "detail": f"工单已创建: {self.incident_id}",
        })
        print(f"  [✓] 工单已创建")

    def generate_report(self) -> dict:
        """生成事件响应报告"""
        return {
            "incident_id": self.incident_id,
            "type": "container_escape",
            "severity": "critical",
            "status": "contained",
            "created_at": datetime.now().isoformat(),
            "total_actions": len(self.actions_log),
            "actions": self.actions_log,
            "next_steps": [
                "安全团队进行深入取证分析",
                "确认攻击范围和影响",
                "修复漏洞并重新部署",
                "编写事后总结报告",
            ],
        }

# 入口
if __name__ == "__main__":
    incident = {
        "pod_name": "compromised-app-abc123",
        "namespace": "production",
        "source_ip": "203.0.113.50",
        "detection_rule": "DETECT-CONTAINER-ESCAPE",
        "timestamp": datetime.now().isoformat(),
    }
    playbook = ContainerEscapePlaybook(incident)
    report = playbook.execute()
    print(json.dumps(report, indent=2))
```

## 4. 数字取证分析

### 4.1 容器取证

```python
#!/usr/bin/env python3
"""
Kubernetes 容器取证工具
在被入侵的容器被销毁前，快速收集关键取证数据
"""
import subprocess
import hashlib
import json
from datetime import datetime
from pathlib import Path

class ContainerForensics:
    def __init__(self, pod_name: str, namespace: str):
        self.pod = pod_name
        self.namespace = namespace
        self.case_id = f"CASE-{datetime.now().strftime('%Y%m%d%H%M%S')}"
        self.evidence_dir = Path(f"/forensics/{self.case_id}")
        self.evidence_dir.mkdir(parents=True, exist_ok=True)
        self.chain_of_custody = []

    def collect_all(self):
        """收集所有取证数据"""
        self.collect_container_logs()
        self.collect_process_list()
        self.collect_network_connections()
        self.collect_file_hashes()
        self.collect_container_changes()
        self.collect_memory_dump()
        self.generate_chain_of_custody()

    def collect_container_logs(self):
        """收集容器日志"""
        output = self.evidence_dir / "container_logs.txt"
        subprocess.run(
            ["kubectl", "logs", self.pod, "-n", self.namespace, "--all-containers=true"],
            stdout=open(output, 'w')
        )
        self._log_evidence("container_logs.txt", "Container application logs")

    def collect_process_list(self):
        """收集运行中的进程列表"""
        output = self.evidence_dir / "process_list.txt"
        subprocess.run(
            ["kubectl", "exec", self.pod, "-n", self.namespace, "--", "ps", "aux"],
            stdout=open(output, 'w')
        )
        self._log_evidence("process_list.txt", "Running process list")

    def collect_network_connections(self):
        """收集网络连接"""
        output = self.evidence_dir / "network_connections.txt"
        subprocess.run(
            ["kubectl", "exec", self.pod, "-n", self.namespace, "--",
             "sh", "-c", "netstat -tulpn 2>/dev/null || ss -tulpn"],
            stdout=open(output, 'w')
        )
        self._log_evidence("network_connections.txt", "Network connections")

    def collect_file_hashes(self):
        """收集关键文件哈希"""
        output = self.evidence_dir / "file_hashes.txt"
        subprocess.run(
            ["kubectl", "exec", self.pod, "-n", self.namespace, "--",
             "sh", "-c", "find / -type f -executable -exec sha256sum {} \\; 2>/dev/null"],
            stdout=open(output, 'w')
        )
        self._log_evidence("file_hashes.txt", "SHA256 hashes of executable files")

    def collect_container_changes(self):
        """收集容器文件系统变更（相对于镜像）"""
        output = self.evidence_dir / "filesystem_changes.txt"
        result = subprocess.run(
            ["kubectl", "get", "pod", self.pod, "-n", self.namespace,
             "-o", "jsonpath={.status.containerStatuses[0].containerID}"],
            capture_output=True, text=True
        )
```

```yaml
# Falco 规则 — 检测容器运行时异常
# /etc/falco/falco_rules.local.yaml
- list: shell_binaries
  items: [bash, sh, zsh, ksh, fish, dash]

- macro: shell_spawned_in_container
  condition: >
    container.id != host and
    proc.name in (shell_binaries) and
    container.name != "debug-container"

- rule: Shell Spawned in Container
  desc: 检测容器内启动 shell（可能是攻击者获取了交互式访问）
  condition: shell_spawned_in_container
  output: >
    Shell opened in container (user=%user.name
    container_id=%container.id container_name=%container.name
    shell=%proc.name parent=%proc.pname cmdline=%proc.cmdline)
  priority: WARNING
  tags: [container, shell, mitre_execution]

- rule: Read /etc/shadow
  desc: 检测容器内读取 /etc/shadow（密码哈希文件）
  condition: >
    open_read and
    fd.name=/etc/shadow and
    container.id != host
  output: >
    /etc/shadow read in container
    (user=%user.name container_id=%container.id
    container_name=%container.name proc=%proc.name
    cmdline=%proc.cmdline)
  priority: CRITICAL
  tags: [container, filesystem, mitre_credential_access]

- rule: Write Below /etc
  desc: 检测容器内修改 /etc 目录（可能是植入后门）
  condition: >
    open_write and
    fd.directory=/etc and
    container.id != host and
    not proc.name in (dpkg, rpm, yum, apt)
  output: >
    File modified below /etc in container
    (file=%fd.name proc=%proc.name user=%user.name
    container_id=%container.id)
  priority: ERROR
  tags: [container, filesystem, mitre_persistence]

- rule: Outbound Connection to Known Bad IP
  desc: 检测容器连接已知恶意IP
  condition: >
    container.id != host and
    fd.sip in (known_bad_ips)
  output: >
    Outbound connection to known bad IP
    (destination=%fd.sip:%fd.sport proc=%proc.name
    container_id=%container.id)
  priority: CRITICAL
  tags: [container, network, mitre_exfiltration]
```
