# 密钥管理实践

> 在云原生环境中安全地存储、分发和轮换密钥（Secrets），防止密钥泄露到代码仓库、日志或运行时环境中。

---

## 概述

密钥管理是 DevSecOps 的基础环节。一个完整的密钥管理体系涵盖以下维度：

```
┌──────────────────────────────────────────────────────────┐
│                    密钥管理生命周期                        │
├──────────┬──────────┬──────────┬──────────┬──────────────┤
│  创建    │  存储    │  分发    │  使用    │  轮换/销毁    │
│          │          │          │          │              │
│ 密钥生成 │ 加密存储 │ 注入运行 │ 运行时   │ 定期轮换     │
│ 密钥分类 │ 访问控制 │ 时环境   │ 访问审计 │ 自动化更新   │
│ 权限分配 │ 审计日志 │ 自动挂载 │ 最小权限 │ 安全销毁     │
└──────────┴──────────┴──────────┴──────────┴──────────────┘
```

### 常见密钥管理工具对比

| 工具 | 部署方式 | 适用场景 | 动态密钥 | K8s集成 | 开源 |
|------|---------|---------|---------|---------|------|
| **HashiCorp Vault** | 自托管/Cloud | 企业级通用密钥管理 | ✅ | ✅ CSI | ✅ |
| **Sealed Secrets** | K8s原生 | GitOps加密密钥 | ❌ | ✅ Controller | ✅ |
| **SOPS** | CLI | 加密配置文件 | ❌ | GitOps | ✅ |
| **AWS Secrets Manager** | 云服务 | AWS生态 | ✅ | ✅ CSI | ❌ |
| **External Secrets Operator** | K8s Operator | 统一外部密钥源 | 取决于后端 | ✅ | ✅ |

## 1. HashiCorp Vault

### 1.1 Vault 架构与部署

```yaml
# k8s/vault/values.yaml — Helm Chart 配置
server:
  enabled: true
  image:
    repository: hashicorp/vault
    tag: "1.16.1"
  
  # 高可用模式
  ha:
    enabled: true
    replicas: 3
    raft:
      enabled: true
      setNodeId: true
      config: |
        cluster_name = "vault-cluster"
        storage "raft" {
          path = "/vault/data"
          retry_join {
            leader_api_addr = "http://vault-0.vault-internal:8200"
          }
          retry_join {
            leader_api_addr = "http://vault-1.vault-internal:8200"
          }
        }
  
  # 资源限制
  resources:
    requests:
      memory: "512Mi"
      cpu: "250m"
    limits:
      memory: "1Gi"
      cpu: "500m"

# 自动撤销策略
injector:
  enabled: true
  replicas: 2
  resources:
    requests:
      memory: "128Mi"
      cpu: "100m"
```

### 1.2 密钥存储策略配置

```bash
# 启用 KV 密钥引擎（V2 支持版本控制）
vault secrets enable -path=secret kv-v2

# 配置最大版本数（保留最近10个版本）
vault write secret/config max_versions=10

# 存储应用密钥
vault kv put secret/myapp/database \
  username="app_user" \
  password="S3cur3P@ssw0rd!" \
  host="db.internal" \
  port=5432

# 配置访问策略
cat > /tmp/app-policy.hcl << 'EOF'
path "secret/data/myapp/database" {
  capabilities = ["read"]
}

path "secret/metadata/myapp/database" {
  capabilities = ["list", "read"]
}

# 禁止写入/删除（应用只需读取）
path "secret/data/myapp/database" {
  capabilities = ["deny"]
}
EOF

vault policy write myapp-policy /tmp/app-policy.hcl
```

### 1.3 动态数据库密钥

```bash
# 启用数据库密钥引擎
vault secrets enable database

# 配置 PostgreSQL 动态凭据
vault write database/config/my-postgresql \
  plugin_name=postgresql-database-plugin \
  connection_url="postgresql://{{username}}:{{password}}@db.internal:5432/myapp?sslmode=disable" \
  allowed_roles="app-role" \
  username="vault_admin" \
  password="VaultAdminPassword"

# 创建角色 — 每次请求生成新凭据，TTL 1小时
vault write database/roles/app-role \
  db_name=my-postgresql \
  creation_statements="CREATE ROLE \"{{name}}\" WITH LOGIN PASSWORD '{{password}}' VALID UNTIL '{{expiration}}'; \
    GRANT SELECT ON ALL TABLES IN SCHEMA public TO \"{{name}}\";" \
  default_ttl="1h" \
  max_ttl="24h"

# 生成动态凭据
vault read database/creds/app-role
# 输出:
# username    v-root-app-role-abc123
# password    temp-password-xyz789
# lease_id    database/creds/app-role/xxx
# lease_duration  1h
```

### 1.4 Vault Agent Sidecar 注入

```yaml
# K8s Deployment — Vault Agent Sidecar 自动注入
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
  labels:
    app: myapp
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
      annotations:
        vault.hashicorp.com/agent-inject: "true"
        vault.hashicorp.com/role: "myapp-role"
        # 注入数据库密钥到文件
        vault.hashicorp.com/agent-inject-secret-db-config: "secret/data/myapp/database"
        # 使用模板渲染配置文件
        vault.hashicorp.com/agent-inject-template-db-config: |
          {{- with secret "secret/data/myapp/database" -}}
          DATABASE_URL=postgresql://{{ .Data.data.username }}:{{ .Data.data.password }}@{{ .Data.data.host }}:{{ .Data.data.port }}/myapp
          {{- end }}
    spec:
      serviceAccountName: myapp-sa
      containers:
        - name: myapp
          image: myapp:latest
          env:
            - name: DATABASE_URL_FILE
              value: /vault/secrets/db-config
```

### 1.5 Vault 自动轮换

```bash
# 自动轮换策略 — 利用 lease renewal
# Vault Agent 配置
cat > /tmp/vault-agent.hcl << 'EOF'
auto_auth {
  method "kubernetes" {
    mount_path = "auth/kubernetes"
    config = {
      role = "myapp-role"
    }
  }

  sink "file" {
    config = {
      path = "/vault/token"
    }
  }
}

# 模板渲染 + 自动轮换

  static_secret_render_interval = "5m"  # 每5分钟刷新
}

  source      = "/vault/templates/db.tpl"
  destination = "/vault/secrets/db-config"
  command     = ["sh", "-c", "kill -HUP $(cat /app/pid)"]  # 密钥变更后通知应用
}
EOF
```

## 2. Sealed Secrets (Bitnami)

### 2.1 架构

```
开发者                      K8s集群
│                          ┌──────────────────────────────┐
│  kubeseal 加密            │                              │
│  sealed-secret.yaml ────▶│  SealedSecret CRD            │
│  (安全提交到Git)          │      │                       │
│                          │      ▼                       │
│                          │  Sealed Secrets Controller   │
│                          │  (持有私钥解密)               │
│                          │      │                       │
│                          │      ▼                       │
│                          │  Secret (明文，注入Pod)       │
└──────────────────────────┴──────────────────────────────┘
```

### 2.2 安装与使用

```bash
# 安装 Sealed Secrets Controller
helm repo add sealed-secrets https://bitnami-labs.github.io/sealed-secrets
helm install sealed-secrets sealed-secrets/sealed-secrets \
  --namespace kube-system \
  --set fullnameOverride=sealed-secrets-controller \
  --set keyrenewperiod="720h"  # 30天自动轮换密钥

# 安装 kubeseal CLI
wget https://github.com/bitnami-labs/sealed-secrets/releases/download/v0.27.3/kubeseal-0.27.3-linux-amd64.tar.gz
tar xzf kubeseal-*.tar.gz
sudo install -m 755 kubeseal /usr/local/bin/kubeseal

# 创建普通 Secret 并加密
echo -n 'admin' | kubectl create secret generic db-secret \
  --dry-run=client --from-file=password=/dev/stdin -o yaml | \
  kubeseal --format yaml > sealed-secret.yaml

# sealed-secret.yaml 可以安全提交到 Git
cat sealed-secret.yaml
```

```yaml
# 示例: sealed-secret.yaml（可安全提交到Git仓库）
apiVersion: bitnami.com/v1alpha1
kind: SealedSecret
metadata:
  name: db-secret
  namespace: production
spec:
  encryptedData:
    password: AgB3F...加密后的Base64数据...==
  template:
    metadata:
      name: db-secret
      namespace: production
    type: Opaque
```

### 2.4 备份与恢复密钥

```bash
# 备份 Sealed Secrets 私钥（关键！）
kubectl get secret -n kube-system \
  -l sealedsecrets.bitnami.com/sealed-secrets-key=active \
  -o yaml > sealed-secrets-key-backup.yaml

# 在灾难恢复时导入
kubectl apply -f sealed-secrets-key-backup.yaml

# 轮换密钥
kubectl annotate secret -n kube-system \
  sealed-secrets-key-xxxxx \
  sealedsecrets.bitnami.com/sealed-secrets-key-rotation="true"
```

## 3. SOPS (Secrets OPerationS)

### 3.1 配置与使用

```yaml
# .sops.yaml — SOPS 配置文件（放在仓库根目录）
creation_rules:
  # 使用 AWS KMS 加密 production 环境
  - path_regex: secrets/production/.*\.yaml$
    kms: arn:aws:kms:us-east-1:123456789012:key/abcd-1234
    aws_profile: production

  # 使用 GCP KMS 加密 staging 环境
  - path_regex: secrets/staging/.*\.yaml$
    gcp_kms: projects/my-project/locations/global/keyRings/sops/cryptoKeys/sops-key

  # 使用 Age 加密开发环境
  - path_regex: secrets/dev/.*\.yaml$
    age: age1xy...recipient-key...

  # 使用 PGP
  - path_regex: secrets/pgp/.*\.yaml$
    pgp: FBC7B9D2A3E4F5G6H7I8J9K0L1M2N3O4P5Q6R7S8
```

```bash
# 创建并加密密钥文件
cat > secrets/production/app-secrets.yaml << 'EOF'
database:
  host: db.production.internal
  port: 5432
  username: prod_user
  password: SuperSecretPassword123!
api:
  key: sk-prod-api-key-456
EOF

# 加密文件
sops --encrypt --in-place secrets/production/app-secrets.yaml

# 查看加密后的文件（敏感值被加密，结构可见）
cat secrets/production/app-secrets.yaml
# database:
#   host: db.production.internal          ← 非敏感值保持明文
#   port: 5432
#   username: prod_user
#   password: ENC[AES256_GCM,data:abc...,iv:def...,tag:ghi...,type:str]
# api:
#   key: ENC[AES256_GCM,data:xyz...]

# 解密查看（需要 KMS/Age 权限）
sops --decrypt secrets/production/app-secrets.yaml

# 原地编辑（自动解密→编辑→加密）
sops secrets/production/app-secrets.yaml

# 轮换密钥（重新加密）
sops updatekeys secrets/production/app-secrets.yaml
```

## 4. External Secrets Operator (ESO)

### 4.1 架构与配置

```yaml
# 1. 创建 SecretStore — 对接外部密钥源
apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: vault-backend
  namespace: production
spec:
  provider:
    vault:
      server: "https://vault.internal:8200"
      path: "secret"
      version: "v2"
      auth:
        kubernetes:
          mountPath: "kubernetes"
          role: "external-secrets"
          serviceAccountRef:
            name: external-secrets-sa
---
# 2. 创建 ExternalSecret — 定义需要拉取的密钥
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: app-db-secret
  namespace: production
spec:
  refreshInterval: 1h  # 每小时自动刷新
  secretStoreRef:
    name: vault-backend
    kind: SecretStore
  target:
    name: app-db-secret  # 生成的 K8s Secret 名称
    creationPolicy: Owner
    template:
      type: Opaque
      data:
        DATABASE_URL: "postgresql://{{ .username }}:{{ .password }}@{{ .host }}:{{ .port }}/myapp"
  data:
    - secretKey: username
      remoteRef:
        key: myapp/database
        property: username
    - secretKey: password
      remoteRef:
        key: myapp/database
        property: password
    - secretKey: host
      remoteRef:
        key: myapp/database
        property: host
```

## 5. 密钥轮换策略

### 5.1 轮换框架

```python
#!/usr/bin/env python3
"""
密钥自动轮换框架
- 定期检查密钥年龄
- 生成新密钥并更新到 Vault/Secret Manager
- 通知应用刷新
- 保留旧密钥一段时间作为回退
"""
import boto3
import datetime
from dataclasses import dataclass

@dataclass
class SecretMetadata:
    name: str
    created_at: datetime.datetime
    last_rotated: datetime.datetime
    rotation_period_days: int

class SecretRotationManager:
    def __init__(self, vault_client=None, kms_client=None):
        self.vault = vault_client
        self.kms = kms_client or boto3.client('secretsmanager')
    
    def check_rotation_needed(self, secrets: list[SecretMetadata]) -> list[str]:
        """检查哪些密钥需要轮换"""
        now = datetime.datetime.now(datetime.timezone.utc)
        needs_rotation = []
        
        for secret in secrets:
            age = (now - secret.last_rotated).days
            if age >= secret.rotation_period_days:
                needs_rotation.append(secret.name)
                print(f"[!] {secret.name} 已 {age} 天未轮换（上限 {secret.rotation_period_days} 天）")
        
        return needs_rotation
    
    def rotate_secret(self, secret_name: str, rotation_type: str = "password"):
        """执行密钥轮换"""
        print(f"[*] 开始轮换密钥: {secret_name}")
        
        # 1. 生成新密钥
        new_value = self._generate_secret(rotation_type)
        
        # 2. 更新密钥存储（保留旧版本）
        self._update_secret_store(secret_name, new_value)
        
        # 3. 通知应用刷新（通过 webhook / signal）
        self._notify_applications(secret_name)
        
        # 4. 标记旧密钥为待删除（30天后清理）
        self._schedule_old_secret_cleanup(secret_name)
        
        print(f"[✓] 密钥 {secret_name} 轮换完成")
    
    def _generate_secret(self, rotation_type: str) -> str:
        import secrets as pysecrets
        import string
        alphabet = string.ascii_letters + string.digits + "!@#$%^&*"
        return ''.join(pysecrets.choice(alphabet) for _ in range(32))
    
    def _update_secret_store(self, name: str, value: str):
        if self.vault:
            self.vault.secrets.kv.v2.create_or_update_secret(
                path=name, secret={"value": value}
            )
        else:
            self.kms.put_secret_value(
                SecretId=name, SecretString=value
            )
    
    def _notify_applications(self, secret_name: str):
        """通过事件通知应用刷新密钥"""

        pass
    
    def _schedule_old_secret_cleanup(self, secret_name: str):
        """设置旧密钥的清理计划"""

# 定时任务（CronJob / Lambda / Cloud Function）
```

### 5.2 轮换策略矩阵

| 密钥类型 | 轮换周期 | 方式 | 回退窗口 |
|---------|---------|------|---------|
| 数据库密码 | 30-90天 | Vault 动态凭据 | 旧凭据24h过期 |
| API Key | 90天 | 双Key共存 | 旧Key 7天有效 |
| TLS证书 | 90天 | cert-manager自动 | 旧证书立即失效 |
| KMS加密密钥 | 365天 | 密钥版本化 | 旧版本可解密 |
| JWT签名密钥 | 30天 | JWKS轮换 | 旧密钥 Kid 保留7天 |
| Sealed Secrets私钥 | 365天 | Controller自动 | 旧密钥可解密封 |

## 最佳实践

### 🔹 密钥安全原则

1. **永不硬编码**：密钥不进代码仓库，使用 pre-commit hook 检测
2. **最小权限**：应用只读取所需的密钥，路径级控制
3. **加密传输**：密钥传输使用 mTLS，存储使用 envelope encryption
4. **审计日志**：记录所有密钥访问操作，定期审查
5. **自动轮换**：避免人工轮换，减少暴露窗口
6. **备份策略**：加密备份密钥管理系统的主密钥

### 🔹 GitOps 兼容

```yaml
# .github/workflows/secret-scan.yml
name: Secret Scanning (Pre-Commit)
on: [push, pull_request]

jobs:
  trufflehog:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # 需要完整历史进行扫描
      - name: TruffleHog OSS
        uses: trufflesecurity/trufflehog@main
        with:
          path: .
          extra_args: --only-verified  # 仅报告已验证的密钥
```

### 🔹 多环境管理
