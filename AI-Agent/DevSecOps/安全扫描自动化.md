# 安全扫描自动化

> 在 CI/CD 流水线中集成多种安全扫描工具，实现 **Shift-Left Security**，在开发早期发现并修复安全漏洞。

---

## 概述

安全扫描自动化是 DevSecOps 的核心实践之一，通过在软件开发生命周期（SDLC）的各个阶段嵌入自动化安全扫描工具，实现持续安全检测。主要包括五类扫描：

| 扫描类型 | 全称 | 扫描目标 | 典型工具 | 执行阶段 |
|---------|------|---------|---------|---------|
| **SAST** | Static Application Security Testing | 源代码 | SonarQube, Semgrep, CodeQL | 编码 / 构建 |
| **DAST** | Dynamic Application Security Testing | 运行时应用 | OWASP ZAP, Burp Suite | 测试 / 预发布 |
| **SCA** | Software Composition Analysis | 第三方依赖 | Trivy, Snyk, Dependabot | 构建 / CI |
| **容器扫描** | Container Image Scanning | 容器镜像 | Trivy, Grype, Clair | 构建 / 部署前 |
| **IaC扫描** | Infrastructure as Code Scanning | Terraform/K8s/CloudFormation | Checkov, tfsec, KICS | 基础设施变更 |

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  编码    │───▶│  构建    │───▶│  测试    │───▶│  预发布  │───▶│  生产    │
│          │    │          │    │          │    │          │    │          │
│ SAST     │    │ SCA      │    │ DAST     │    │ DAST     │    │ 持续监控  │
│ 密钥检测  │    │ 容器扫描  │    │ 渗透测试  │    │ IaC扫描  │    │          │
│ IaC扫描  │    │          │    │          │    │          │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

## 1. SAST — 静态应用安全测试

### 1.1 Semgrep 配置示例

```yaml
# .semgrep.yml — Semgrep 自定义规则
rules:
  - id: detect-hardcoded-secrets
    patterns:
      - pattern-either:
          - pattern: password = "$PWD"
          - pattern: api_key = "$KEY"
          - pattern: token = "$TOKEN"
    message: "检测到硬编码的密钥/密码，请使用环境变量或密钥管理工具"
    languages: [python, javascript, go, java]
    severity: ERROR
    metadata:
      cwe: "CWE-798: Use of Hard-coded Credentials"
      owasp: "A07:2021 – Identification and Authentication Failures"

  - id: python-sql-injection
    patterns:
      - pattern: |
          cursor.execute(f"SELECT ... {user_input}")
    message: "检测到可能的SQL注入，请使用参数化查询"
    languages: [python]
    severity: ERROR

  - id: insecure-deserialization
    pattern: pickle.loads($DATA)
    message: "pickle.loads 可能导致不安全的反序列化"
    languages: [python]
    severity: WARNING
```

### 1.2 GitHub CodeQL 配置

```yaml
# .github/workflows/codeql-analysis.yml
name: "CodeQL Security Analysis"

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
  schedule:
    - cron: '0 2 * * 1'  # 每周一凌晨2点

jobs:
  analyze:
    runs-on: ubuntu-latest
    permissions:
      actions: read
      contents: read
      security-events: write

    strategy:
      fail-fast: false
      matrix:
        language: [python, javascript, java, go]

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Initialize CodeQL
        uses: github/codeql-action/init@v3
        with:
          languages: ${{ matrix.language }}
          queries: security-extended, security-and-quality

      - name: Autobuild
        uses: github/codeql-action/autobuild@v3

      - name: Perform CodeQL Analysis
        uses: github/codeql-action/analyze@v3
        with:
          category: "/language:${{ matrix.language }}"
```

## 2. DAST — 动态应用安全测试

### 2.1 OWASP ZAP 自动化扫描

```yaml
# .github/workflows/dast-zap.yml
name: DAST - OWASP ZAP Scan

on:
  workflow_dispatch:
  schedule:
    - cron: '0 4 * * *'  # 每天凌晨4点执行

jobs:
  zap-scan:
    runs-on: ubuntu-latest
    steps:
      - name: ZAP Baseline Scan
        uses: zaproxy/action-baseline@v0.13.0
        with:
          target: 'https://staging.example.com'
          cmd_options: '-a -j'  # 启用主动扫描和AJAX Spider
          docker_name: 'ghcr.io/zaproxy/zaproxy:stable'
          issue_title: 'DAST安全扫描报告'
          fail_action: true
          allow_issue_writing: true
```

### 2.2 ZAP 自动化脚本

```python
#!/usr/bin/env python3
"""OWASP ZAP 自动化DAST扫描脚本"""
import time
from zapv2 import ZAPv2

target = "https://staging.example.com"
zap = ZAPv2(apikey="your-api-key", proxies={"http": "http://localhost:8080", "https": "http://localhost:8080"})

# 1. 访问目标
print(f"[*] 访问目标: {target}")
zap.urlopen(target)
time.sleep(2)

# 2. Spider 扫描
print("[*] 启动 Spider 扫描...")
scan_id = zap.spider.scan(target)
while int(zap.spider.status(scan_id)) < 100:
    print(f"  Spider 进度: {zap.spider.status(scan_id)}%")
    time.sleep(2)

# 3. Active Scan（主动扫描）
print("[*] 启动 Active Scan...")
scan_id = zap.ascan.scan(target)
while int(zap.ascan.status(scan_id)) < 100:
    print(f"  Active Scan 进度: {zap.ascan.status(scan_id)}%")
    time.sleep(5)

# 4. 生成报告
print("[*] 生成报告...")
report = zap.core.htmlreport()
with open("zap-report.html", "w") as f:
    f.write(report)

# 5. 检查高危漏洞
alerts = zap.core.alerts(baseurl=target)
high_risk = [a for a in alerts if a.get("riskcode") == "3"]
if high_risk:
    print(f"[!] 发现 {len(high_risk)} 个高危漏洞！")
    exit(1)
else:
    print("[✓] 未发现高危漏洞")
```

## 3. SCA — 软件成分分析

### 3.1 Trivy 依赖扫描

```yaml
# .github/workflows/sca-trivy.yml
name: SCA - Dependency Scan

on:
  push:
    branches: [main, develop]
  pull_request:

jobs:
  trivy-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Trivy FS Scan (依赖)
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: 'fs'
          scan-ref: '.'
          severity: 'CRITICAL,HIGH'
          format: 'sarif'
          output: 'trivy-results.sarif'
          exit-code: '1'
          ignore-unfixed: true

      - name: Upload to GitHub Security
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: trivy-results.sarif
```

## 4. 容器镜像扫描

### 4.1 Trivy 容器扫描

```bash
# 扫描容器镜像
trivy image --severity CRITICAL,HIGH \
  --ignore-unfixed \
  --format json \
  --output trivy-image-report.json \
  my-registry/app:latest

# 与构建流水线集成（构建后立即扫描）
docker build -t app:latest .
trivy image --exit-code 1 --severity CRITICAL app:latest
```

### 4.2 GitLab CI 容器扫描

```yaml
# .gitlab-ci.yml
container_scanning:
  stage: test
  image:
    name: aquasec/trivy:latest
    entrypoint: [""]
  variables:
    TRIVY_IMAGE: "${CI_REGISTRY_IMAGE}:${CI_COMMIT_SHA}"
    TRIVY_SEVERITY: "CRITICAL,HIGH"
  script:
    - trivy --version
    - trivy image --exit-code 1 --severity ${TRIVY_SEVERITY}
        --ignore-unfixed
        --format template
        --template "@/contrib/junit.tpl"
        --output trivy-junit.xml
        ${TRIVY_IMAGE}
  artifacts:
    reports:
      junit: trivy-junit.xml
```

### 4.3 策略门禁 — 严重漏洞阻断部署

```python
#!/usr/bin/env python3
"""扫描结果质量门禁检查"""
import json
import sys

def check_scan_gate(report_file: str, max_critical: int = 0, max_high: int = 5):
    """检查扫描结果是否通过安全门禁"""
    with open(report_file) as f:
        report = json.load(f)

    critical_count = sum(1 for v in report.get("Results", [{}])[0].get("Vulnerabilities", []) if v.get("Severity") == "CRITICAL")
    high_count = sum(1 for v in report.get("Results", [{}])[0].get("Vulnerabilities", []) if v.get("Severity") == "HIGH")

    print(f"[安全门禁] CRITICAL: {critical_count} (上限: {max_critical}), HIGH: {high_count} (上限: {max_high})")

    if critical_count > max_critical:
        print(f"[✗] 门禁未通过: CRITICAL 漏洞 ({critical_count}) 超过上限 ({max_critical})")
        sys.exit(1)

    if high_count > max_high:
        print(f"[✗] 门禁未通过: HIGH 漏洞 ({high_count}) 超过上限 ({max_high})")
        sys.exit(1)

    print("[✓] 安全门禁通过")
    sys.exit(0)

if __name__ == "__main__":
    check_scan_gate("trivy-image-report.json", max_critical=0, max_high=5)
```

## 5. IaC 安全扫描

### 5.1 Checkov — Terraform / K8s / CloudFormation

```yaml
# checkov 配置文件
# .checkov.yml
framework:
  - terraform
  - kubernetes
  - dockerfile
  - github_actions
directory:
  - infra/
  - k8s/
skip-check:
  - CKV_AWS_18  # 确认跳过的规则（需有理由）
compact: true
output:
  - cli
  - sarif
soft-fail: false
```

```bash
# 扫描 Terraform
checkov -d infra/terraform/ --framework terraform --output sarif --output-file checkov-results.sarif

# 扫描 Kubernetes 清单
checkov -d k8s/ --framework kubernetes --check CKV_K8S_*

# 扫描 Dockerfile
checkov -f Dockerfile --framework dockerfile
```

### 5.2 tfsec — Terraform 专项扫描

```hcl
# tfsec 在 Terraform 中的注释控制
# main.tf
resource "aws_s3_bucket" "data" {
  bucket = "my-sensitive-data-bucket"

  # tfsec:ignore:aws-s3-enable-versioning 已知风险，版本控制由其他方式管理
  versioning {
    enabled = false
  }

  # tfsec:ignore:aws-s3-encryption-customer-key 使用默认KMS密钥
  server_side_encryption_configuration {
    rule {
      apply_server_side_encryption_by_default {
        sse_algorithm = "aws:kms"
      }
    }
  }
}
```

### 5.3 综合流水线集成

```yaml
# .github/workflows/security-pipeline.yml
name: DevSecOps Security Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:

jobs:
  # ─── SAST ───
  semgrep:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: returntocorp/semgrep-action@v1
        with:
          config: >-
            p/default
            p/owasp-top-ten
            p/security-audit

  # ─── SCA ───
  dependency-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: aquasecurity/trivy-action@master
        with:
          scan-type: fs
          severity: CRITICAL,HIGH
          exit-code: '1'

  # ─── 容器扫描 ───
  container-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: docker build -t app:scan .
      - uses: aquasecurity/trivy-action@master
        with:
          image-ref: app:scan
          severity: CRITICAL,HIGH
          exit-code: '1'
          ignore-unfixed: true

  # ─── IaC 扫描 ───
  iac-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Checkov
        uses: bridgecrewio/checkov-action@v12
        with:
          directory: infra/
          framework: terraform kubernetes
          output_format: sarif
          output_file_path: checkov.sarif
          soft_fail: false

  # ─── 安全门禁 ───
  security-gate:
    needs: [semgrep, dependency-scan, container-scan, iac-scan]
    runs-on: ubuntu-latest
    if: ${{ !cancelled() }}
    steps:
      - name: Check All Jobs Passed
        run: |
          if [[ "${{ contains(needs.*.result, 'failure') }}" == "true" ]]; then
            echo "[✗] 安全扫描门禁未通过，阻止合并"
            exit 1
          fi
          echo "[✓] 所有安全扫描通过"
```

## 最佳实践

### 🔹 扫描策略

| 实践 | 说明 |
|------|------|
| **Shift-Left** | 尽早在开发阶段嵌入扫描，IDE插件 + Pre-commit Hook |
| **分层扫描** | 不同阶段使用不同工具，互补覆盖 |
| **门禁策略** | 仅阻断 CRITICAL 漏洞，HIGH 漏洞跟踪修复 |
| **增量扫描** | 大型项目仅扫描变更文件，提升效率 |
| **基线管理** | 使用 `.trivyignore` / `baseline` 管理已知可接受风险 |

### 🔹 误报处理

```python
# 使用 suppress 机制管理误报
# semgrep-ignore-patterns.yml
# 这些发现已确认为误报，不会在 CI 中报告
paths:
  - "tests/**/*"        # 测试代码不扫描硬编码密钥
  - "**/fixtures/**"    # 测试fixture跳过
```

### 🔹 性能优化

- **并行执行**：多语言项目使用 matrix 策略并行扫描
- **缓存依赖**：缓存工具二进制和数据库（Trivy DB 等）
- **增量扫描**：基于 git diff 仅扫描变更文件
- **异步报告**：扫描结果异步发送到安全平台，不阻塞流水线

### 🔹 度量与报告

| 指标 | 描述 |
|------|------|
| MTTR (Mean Time to Remediate) | 漏洞从发现到修复的平均时间 |
| 漏洞密度 | 每千行代码的漏洞数量 |
| 扫描覆盖率 | 被扫描代码/资产占总体的百分比 |
| 修复率 | 已修复漏洞占总发现漏洞的百分比 |
