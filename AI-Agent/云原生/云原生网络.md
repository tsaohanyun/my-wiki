# 云原生网络

## 概述

Kubernetes 网络模型要求每个 Pod 拥有唯一 IP，Pod 间可以直接通信。CNI（Container Network Interface）负责实现 Pod 网络，NetworkPolicy 提供网络隔离，Ingress Controller 管理外部访问入口。

## CNI 选择

### CNI 对比

| 特性 | Calico | Cilium | Flannel | Weave Net |
|------|--------|--------|---------|-----------|
| 网络模型 | BGP/VXLAN | eBPF/VXLAN | VXLAN | VXLAN/Sleeve |
| NetworkPolicy | ✅ 完整 | ✅ 增强 | ❌ | ✅ 基础 |
| 加密 | WireGuard | WireGuard/IPsec | ❌ | IPsec |
| 性能 | 高 | 极高 | 中 | 中 |
| 可观测性 | 中 | 高 | 低 | 中 |
| 复杂度 | 中 | 高 | 低 | 低 |
| 适用场景 | 通用生产 | 高性能/安全 | 简单环境 | 小规模 |

## NetworkPolicy

### 默认拒绝策略

```yaml
# 拒绝所有入站流量
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-ingress
  namespace: production
spec:
  podSelector: {}
  policyTypes:
    - Ingress
---
# 拒绝所有出站流量
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-egress
  namespace: production
spec:
  podSelector: {}
  policyTypes:
    - Egress
---
# 允许 DNS 查询
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-dns
  namespace: production
spec:
  podSelector: {}
  policyTypes:
    - Egress
  egress:
    - to:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: kube-system
      ports:
        - protocol: UDP
          port: 53
        - protocol: TCP
          port: 53
```

### 应用级网络策略

### Ingress 资源

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp-ingress
  namespace: production
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    nginx.ingress.kubernetes.io/force-ssl-redirect: "true"
    nginx.ingress.kubernetes.io/proxy-body-size: "50m"
    nginx.ingress.kubernetes.io/proxy-read-timeout: "60"
    nginx.ingress.kubernetes.io/proxy-send-timeout: "60"
    nginx.ingress.kubernetes.io/proxy-connect-timeout: "10"
    nginx.ingress.kubernetes.io/rate-limit: "100"
    nginx.ingress.kubernetes.io/rate-limit-window: "1m"
    nginx.ingress.kubernetes.io/enable-cors: "true"
    nginx.ingress.kubernetes.io/cors-allow-origin: "https://frontend.example.com"
    nginx.ingress.kubernetes.io/configuration-snippet: |
      more_set_headers "X-Frame-Options: DENY";
      more_set_headers "X-Content-Type-Options: nosniff";
    cert-manager.io/cluster-issuer: letsencrypt-prod
spec:
  ingressClassName: nginx
  tls:
    - hosts:
        - myapp.example.com
        - api.myapp.example.com
      secretName: myapp-tls
  rules:
    - host: myapp.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: frontend-svc
                port:
                  number: 80
    - host: api.myapp.example.com
      http:
        paths:
          - path: /v1
            pathType: Prefix
            backend:
              service:
                name: backend-api-v1
                port:
                  number: 8080
          - path: /v2
            pathType: Prefix
            backend:
              service:
                name: backend-api-v2
                port:
                  number: 8080
```

### Gateway API（新一代）

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: production-gateway
  namespace: istio-system
spec:
  gatewayClassName: istio
  listeners:
    - name: https
      protocol: HTTPS
      port: 443
      hostname: "*.example.com"
      tls:
        mode: Terminate
        certificateRefs:
          - name: wildcard-tls
      allowedRoutes:
        namespaces:
          from: Selector
          selector:
            matchLabels:
              gateway-access: "true"
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: myapp-route
  namespace: production
spec:
  parentRefs:
    - name: production-gateway
      namespace: istio-system
  hostnames:
    - myapp.example.com
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /api
      backendRefs:
        - name: backend-api
          port: 8080
      timeouts:
        request: 30s
```

## 最佳实践

1. **零信任网络**：默认拒绝所有流量，按需开放
2. **最小权限**：NetworkPolicy 精确控制 Pod 间通信
3. **命名空间隔离**：不同环境使用独立命名空间，配合 NetworkPolicy 隔离
4. **DNS 策略**：始终允许 DNS 出站流量
5. **加密通信**：启用 WireGuard/IPsec 加密 Pod 间流量
6. **Ingress 高可用**：多副本 + 拓扑分散，避免单点故障
7. **限流保护**：Ingress 层配置限流和熔断
8. **监控告警**：监控网络延迟、丢包率、连接数
9. **CIDR 规划**：合理规划 Pod/Service CIDR，避免地址冲突
10. **IPv6 双栈**：评估并规划 IPv6 支持
