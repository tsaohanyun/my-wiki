# 设计模式进阶

设计模式是解决软件设计中常见问题的可复用方案。本文覆盖常用的创建型、结构型和行为型模式。

## 1. 观察者模式 (Observer)

定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖者都会收到通知。

```typescript
interface Observer<T> {
  update(event: T): void;
}

class EventEmitter<T extends Record<string, any>> {
  private listeners = new Map<keyof T, Set<Observer<any>>>();

  on<K extends keyof T>(event: K, observer: Observer<T[K]>): void {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, new Set());
    }
    this.listeners.get(event)!.add(observer);
  }

  off<K extends keyof T>(event: K, observer: Observer<T[K]>): void {
    this.listeners.get(event)?.delete(observer);
  }

  emit<K extends keyof T>(event: K, data: T[K]): void {
    this.listeners.get(event)?.forEach((obs) => obs.update(data));
  }
}

// 使用
interface AppEvents {
  login: { userId: string };
  logout: {};
  error: { message: string; code: number };
}

const emitter = new EventEmitter<AppEvents>();

const loginLogger: Observer<{ userId: string }> = {
  update(data) {
    console.log(`User logged in: ${data.userId}`);
  },
};

emitter.on("login", loginLogger);
emitter.emit("login", { userId: "user-123" });
```

## 2. 策略模式 (Strategy)

定义一系列算法，把它们封装起来，使它们可以互相替换。

```typescript
interface SortStrategy {
  sort<T>(arr: T[], compare: (a: T, b: T) => number): T[];
}

class BubbleSort implements SortStrategy {
  sort<T>(arr: T[], compare: (a: T, b: T) => number): T[] {
    const result = [...arr];
    for (let i = 0; i < result.length; i++) {
      for (let j = 0; j < result.length - i - 1; j++) {
        if (compare(result[j], result[j + 1]) > 0) {
          [result[j], result[j + 1]] = [result[j + 1], result[j]];
        }
      }
    }
    return result;
  }
}

class QuickSort implements SortStrategy {
  sort<T>(arr: T[], compare: (a: T, b: T) => number): T[] {
    if (arr.length <= 1) return arr;
    const pivot = arr[0];
    const left = arr.slice(1).filter((x) => compare(x, pivot) <= 0);
    const right = arr.slice(1).filter((x) => compare(x, pivot) > 0);
    return [...this.sort(left, compare), pivot, ...this.sort(right, compare)];
  }
}

class Sorter {
  constructor(private strategy: SortStrategy) {}

  setStrategy(strategy: SortStrategy) {
    this.strategy = strategy;
  }

  sort<T>(arr: T[], compare: (a: T, b: T) => number): T[] {
    return this.strategy.sort(arr, compare);
  }
}

const sorter = new Sorter(new QuickSort());
const sorted = sorter.sort([3, 1, 4, 1, 5], (a, b) => a - b);
```

## 3. 工厂模式 (Factory)

### 简单工厂

```typescript
interface Logger {
  log(message: string): void;
  error(message: string): void;
}

class ConsoleLogger implements Logger {
  log(msg: string) { console.log(`[LOG] ${msg}`); }
  error(msg: string) { console.error(`[ERR] ${msg}`); }
}

class FileLogger implements Logger {
  constructor(private filePath: string) {}
  log(msg: string) { /* 写入文件 */ }
  error(msg: string) { /* 写入文件 */ }
}

class LoggerFactory {
  static create(type: "console" | "file", options?: any): Logger {
    switch (type) {
      case "console": return new ConsoleLogger();
      case "file": return new FileLogger(options?.path ?? "app.log");
      default: throw new Error(`Unknown logger type: ${type}`);
    }
  }
}
```

### 抽象工厂

```typescript
interface UIFactory {
  createButton(): Button;
  createInput(): Input;
}

interface Button { render(): string; }
interface Input { render(): string; }

class DarkThemeFactory implements UIFactory {
  createButton(): Button {
    return { render() { return '<button class="dark-btn">Click</button>'; } };
  }
  createInput(): Input {
    return { render() { return '<input class="dark-input" />'; } };
  }
}

class LightThemeFactory implements UIFactory {
  createButton(): Button {
    return { render() { return '<button class="light-btn">Click</button>'; } };
  }
  createInput(): Input {
    return { render() { return '<input class="light-input" />'; } };
  }
}
```

## 4. 单例模式 (Singleton)

确保一个类只有一个实例，并提供全局访问点。

```typescript
class DatabaseConnection {
  private static instance: DatabaseConnection | null = null;
  private connected = false;

  private constructor(private connectionString: string) {}

  static getInstance(connStr?: string): DatabaseConnection {
    if (!DatabaseConnection.instance) {
      if (!connStr) throw new Error("Connection string required for first init");
      DatabaseConnection.instance = new DatabaseConnection(connStr);
    }
    return DatabaseConnection.instance;
  }

  async connect(): Promise<void> {
    if (this.connected) return;
    console.log(`Connecting to ${this.connectionString}...`);
    this.connected = true;
  }

  async query<T>(sql: string): Promise<T[]> {
    if (!this.connected) await this.connect();
    console.log(`Executing: ${sql}`);
    return [] as T[];
  }
}

// 使用
const db = DatabaseConnection.getInstance("postgres://localhost/mydb");
await db.connect();
```

## 5. 装饰器模式 (Decorator)

动态地给对象添加额外职责，比继承更灵活。

```typescript
interface HttpRequest {
  url: string;
  method: string;
  headers: Record<string, string>;
  body?: string;
}

interface HttpHandler {
  handle(request: HttpRequest): Promise<Response>;
}

// 基础处理器
class BasicHttpHandler implements HttpHandler {
  async handle(request: HttpRequest): Promise<Response> {
    return fetch(request.url, {
      method: request.method,
      headers: request.headers,
      body: request.body,
    });
  }
}

// 日志装饰器
class LoggingHandler implements HttpHandler {
  constructor(private inner: HttpHandler) {}

  async handle(request: HttpRequest): Promise<Response> {
    console.log(`→ ${request.method} ${request.url}`);
    const start = Date.now();
    const response = await this.inner.handle(request);
    console.log(`← ${response.status} (${Date.now() - start}ms)`);
    return response;
  }
}

// 认证装饰器
class AuthHandler implements HttpHandler {
  constructor(private inner: HttpHandler, private token: string) {}

  async handle(request: HttpRequest): Promise<Response> {
    request.headers["Authorization"] = `Bearer ${this.token}`;
    return this.inner.handle(request);
  }
}

// 重试装饰器
class RetryHandler implements HttpHandler {
  constructor(private inner: HttpHandler, private maxRetries = 3) {}

  async handle(request: HttpRequest): Promise<Response> {
    for (let i = 0; i <= this.maxRetries; i++) {
      try {
        return await this.inner.handle(request);
      } catch (err) {
        if (i === this.maxRetries) throw err;
        await new Promise((r) => setTimeout(r, 1000 * 2 ** i));
      }
    }
    throw new Error("Unreachable");
  }
}

// 组合使用
const client = new RetryHandler(
  new AuthHandler(
    new LoggingHandler(new BasicHttpHandler()),
    "my-token"
  ),
  3
);
```

## 6. 代理模式 (Proxy)

为另一个对象提供替代以控制对它的访问。

```typescript
interface DataStore {
  get(key: string): Promise<string | null>;
  set(key: string, value: string): Promise<void>;
}

class RealDataStore implements DataStore {
  async get(key: string) { /* 从数据库读取 */ return null; }
  async set(key: string, value: string) { /* 写入数据库 */ }
}

// 缓存代理
class CachingProxy implements DataStore {
  private cache = new Map<string, { value: string; expiry: number }>();

  constructor(
    private inner: DataStore,
    private ttlMs: number = 60_000
  ) {}

  async get(key: string): Promise<string | null> {
    const cached = this.cache.get(key);
    if (cached && cached.expiry > Date.now()) return cached.value;
    const value = await this.inner.get(key);
    if (value !== null) {
      this.cache.set(key, { value, expiry: Date.now() + this.ttlMs });
    }
    return value;
  }

  async set(key: string, value: string): Promise<void> {
    await this.inner.set(key, value);
    this.cache.set(key, { value, expiry: Date.now() + this.ttlMs });
  }
}

// 日志代理
class LoggingProxy implements DataStore {
  constructor(private inner: DataStore) {}

  async get(key: string) {
    console.log(`GET ${key}`);
    return this.inner.get(key);
  }

  async set(key: string, value: string) {
    console.log(`SET ${key} = ${value.substring(0, 50)}`);
    return this.inner.set(key, value);
  }
}
```

## 7. 适配器模式 (Adapter)

将一个类的接口转换为客户期望的另一个接口。

```typescript
// 已有的第三方 SDK
interface LegacyPaymentSDK {
  makePurchase(amountCents: number, currency: string): { success: boolean; txnId: string };
}

// 新系统期望的接口
interface PaymentGateway {
  charge(amount: number, currency: string): Promise<{ ok: boolean; transactionId: string }>;
}

// 适配器
class LegacyPaymentAdapter implements PaymentGateway {
  constructor(private sdk: LegacyPaymentSDK) {}

  async charge(amount: number, currency: string) {
    const result = this.sdk.makePurchase(Math.round(amount * 100), currency);
    return { ok: result.success, transactionId: result.txnId };
  }
}

// 使用
const legacySdk = new SomeLegacySDK();
const gateway: PaymentGateway = new LegacyPaymentAdapter(legacySdk);
const result = await gateway.charge(9.99, "USD");
```

## 最佳实践

| 原则 | 说明 |
|------|------|
| 优先组合优于继承 | 使用装饰器、策略、代理等模式 |
| 面向接口编程 | 依赖抽象而非具体实现 |
| 单一职责 | 每个类只做一件事 |
| 开闭原则 | 对扩展开放，对修改关闭 |
| 避免过度设计 | 简单场景不需要模式 |
