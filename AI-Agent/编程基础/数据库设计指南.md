# 数据库设计指南

## 概述

本指南提供数据库设计的方法论和最佳实践。

## 1. 设计流程

### 需求分析

```
1. 收集业务需求
2. 识别实体和关系
3. 确定属性和约束
4. 定义业务规则
```

### 概念设计

```
1. 绘制ER图
2. 定义实体和关系
3. 确定基数和参与度
4. 验证业务规则
```

### 逻辑设计

```
1. 转换ER图为关系模型
2. 规范化表结构
3. 定义主键和外键
4. 设计索引策略
```

### 物理设计

```
1. 选择存储引擎
2. 设计分区策略
3. 优化存储参数
4. 设计备份策略
```

## 2. ER图绘制

### 实体表示

```
┌─────────────────┐
│      User       │
├─────────────────┤
│ PK  id          │
│     name        │
│     email       │
│     created_at  │
└─────────────────┘
```

### 关系表示

```
一对一关系
User ────── Profile

一对多关系
User ──────< Order

多对多关系
User >─────< Role
```

### 关系基数

```
1:1  一对一
1:N  一对多
M:N  多对多
```

## 3. 规范化

### 第一范式（1NF）

```sql
-- ❌ 违反1NF（字段包含多个值）
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    skills VARCHAR(200)  -- "Java,Python,SQL"
);

-- ✅ 符合1NF
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100)
);

CREATE TABLE user_skills (
    user_id INT,
    skill VARCHAR(50),
    PRIMARY KEY (user_id, skill),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 第二范式（2NF）

```sql
-- ❌ 违反2NF（部分依赖）
CREATE TABLE order_items (
    order_id INT,
    product_id INT,
    product_name VARCHAR(100),  -- 依赖product_id
    quantity INT,
    PRIMARY KEY (order_id, product_id)
);

-- ✅ 符合2NF
CREATE TABLE order_items (
    order_id INT,
    product_id INT,
    quantity INT,
    PRIMARY KEY (order_id, product_id),
    FOREIGN KEY (order_id) REFERENCES orders(id),
    FOREIGN KEY (product_id) REFERENCES products(id)
);

CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100)
);
```

### 第三范式（3NF）

```sql
-- ❌ 违反3NF（传递依赖）
CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    department_id INT,
    department_name VARCHAR(100)  -- 依赖department_id
);

-- ✅ 符合3NF
CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    department_id INT,
    FOREIGN KEY (department_id) REFERENCES departments(id)
);

CREATE TABLE departments (
    id INT PRIMARY KEY,
    name VARCHAR(100)
);
```

## 4. 反规范化

### 适当冗余

```sql
-- 冗余常用字段，减少JOIN
CREATE TABLE orders (
    id INT PRIMARY KEY,
    customer_id INT,
    customer_name VARCHAR(100),  -- 冗余字段
    customer_email VARCHAR(100), -- 冗余字段
    total DECIMAL(10,2),
    created_at TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
);
```

### 汇总表

```sql
-- 创建汇总表，提高统计查询性能
CREATE TABLE daily_sales (
    date DATE PRIMARY KEY,
    total_orders INT,
    total_amount DECIMAL(12,2),
    avg_order_amount DECIMAL(10,2)
);
```

## 5. 主键设计

### 自增主键

```sql
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100)
);
```

### UUID主键

```sql
CREATE TABLE users (
    id CHAR(36) PRIMARY KEY,  -- UUID
    name VARCHAR(100)
);
```

### 业务主键

```sql
CREATE TABLE orders (
    order_no VARCHAR(32) PRIMARY KEY,  -- 业务单号
    customer_id INT,
    total DECIMAL(10,2)
);
```

## 6. 外键设计

### 外键约束

```sql
CREATE TABLE orders (
    id INT PRIMARY KEY,
    customer_id INT NOT NULL,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
        ON DELETE RESTRICT
        ON UPDATE CASCADE
);
```

### 级联操作

```sql
-- RESTRICT：阻止删除/更新
-- CASCADE：级联删除/更新
-- SET NULL：设置为NULL
-- SET DEFAULT：设置为默认值
-- NO ACTION：类似RESTRICT

FOREIGN KEY (customer_id) REFERENCES customers(id)
    ON DELETE CASCADE
    ON UPDATE CASCADE
```

## 7. 索引设计

### 索引类型

```sql
-- 主键索引（自动创建）
PRIMARY KEY (id)

-- 唯一索引
UNIQUE INDEX uk_email (email)

-- 普通索引
INDEX idx_name (name)

-- 组合索引
INDEX idx_status_time (status, created_at)

-- 全文索引
FULLTEXT INDEX ft_content (content)
```

### 索引原则

```sql
-- 1. 选择性高的字段
CREATE INDEX idx_email ON users(email);

-- 2. 查询频繁的字段
CREATE INDEX idx_status ON orders(status);

-- 3. 组合索引遵循最左前缀
CREATE INDEX idx_status_time ON orders(status, created_at);

-- 4. 避免过度索引
-- 单表索引不超过5个
```

## 8. 数据类型选择

### 整数类型

```sql
TINYINT     -- 1字节，-128~127
SMALLINT    -- 2字节，-32768~32767
MEDIUMINT   -- 3字节，-8388608~8388607
INT         -- 4字节，-2147483648~2147483647
BIGINT      -- 8字节，-9223372036854775808~9223372036854775807
```

### 字符串类型

```sql
CHAR(n)     -- 定长字符串
VARCHAR(n)  -- 变长字符串
TEXT        -- 长文本
```

### 日期时间类型

```sql
DATE        -- 日期
TIME        -- 时间
DATETIME    -- 日期时间
TIMESTAMP   -- 时间戳
```

### 数值类型

```sql
DECIMAL(M,D)  -- 精确数值
FLOAT         -- 单精度浮点
DOUBLE        -- 双精度浮点
```

## 9. 表设计模板

### 用户表

```sql
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    status TINYINT NOT NULL DEFAULT 1 COMMENT '1:active, 0:inactive',
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 订单表

```sql
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_no VARCHAR(32) NOT NULL UNIQUE,
    customer_id INT NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    total DECIMAL(10,2) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_customer_id (customer_id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at),
    FOREIGN KEY (customer_id) REFERENCES customers(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

## 10. 设计评审清单

### 结构评审

- [ ] 是否符合三范式（或明确的反范式理由）
- [ ] 主键设计是否合理
- [ ] 外键关系是否正确
- [ ] 字段类型是否合适
- [ ] 是否有审计字段

### 性能评审

- [ ] 索引设计是否合理
- [ ] 是否有冗余字段
- [ ] 大表是否考虑分区
- [ ] 查询SQL是否高效

### 安全评审

- [ ] 敏感数据是否加密
- [ ] 是否有访问控制
- [ ] 是否有审计日志
