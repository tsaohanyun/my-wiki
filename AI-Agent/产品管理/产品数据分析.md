# 产品数据分析

## 概述

产品数据分析是通过数据驱动产品决策的核心能力。本文档涵盖指标体系搭建、漏斗分析、留存分析和 A/B 测试等关键方法。

---

## 一、指标体系

### 1.1 北极星指标（North Star Metric）

北极星指标是衡量产品核心价值的单一关键指标，应满足：

- ✅ 反映用户获得的核心价值
- ✅ 能预测长期业务成功
- ✅ 可衡量、可行动
- ✅ 不易被短期操纵

| 产品类型 | 北极星指标示例 |
|----------|---------------|
| 社交产品 | 日活跃用户数（DAU） |
| 电商产品 | 月 GMV（成交总额） |
| SaaS 产品 | 月活跃付费用户数 |
| 内容产品 | 用户总使用时长 |
| 工具产品 | 周活跃用户完成任务数 |
| 游戏产品 | 日活跃用户游戏时长 |

### 1.3 指标定义模板

| 字段 | 内容 |
|------|------|
| 指标名称 | 日活跃用户数（DAU） |
| 英文名 | Daily Active Users |
| 定义 | 当日至少有1次有效行为的去重用户数 |
| 计算方式 | COUNT(DISTINCT user_id) WHERE event_date = today AND action_type IN ('login', 'browse', 'purchase', ...) |
| 数据来源 | 用户行为日志表 |
| 统计周期 | 自然日（UTC+8） |
| 负责人 | 数据团队 |
| 更新频率 | 每日 |
| 告警阈值 | 日环比下降 > 10% 或 周同比下降 > 15% |
| 关联指标 | MAU、DAU/MAU 比值 |

## 二、漏斗分析

### 2.1 漏斗分析原理

漏斗分析用于追踪用户在完成目标过程中的每一步转化情况，识别流失最严重的环节。

### 2.2 典型漏斗模型

**电商购买漏斗：**
```
访问首页          100,000 (100%)
    │
    ▼ 转化率 60%
浏览商品           60,000 (60%)
    │
    ▼ 转化率 40%
商品详情页         24,000 (24%)
    │
    ▼ 转化率 30%
加入购物车          7,200 (7.2%)
    │
    ▼ 转化率 50%
提交订单            3,600 (3.6%)
    │
    ▼ 转化率 70%
完成支付            2,520 (2.52%)
```

**SaaS 注册漏斗：**
```
访问官网            50,000 (100%)
    │
    ▼ 转化率 25%
点击注册            12,500 (25%)
    │
    ▼ 转化率 60%
填写信息             7,500 (15%)
    │
    ▼ 转化率 80%
邮箱验证             6,000 (12%)
    │
    ▼ 转化率 50%
完成引导             3,000 (6%)
    │
    ▼ 转化率 40%
创建第一个项目       1,200 (2.4%)
```

### 2.3 漏斗分析模板

```sql
-- 电商购买漏斗 SQL 示例
WITH funnel AS (
  SELECT
    COUNT(DISTINCT CASE WHEN event = 'page_view' THEN user_id END) AS step1_visit,
    COUNT(DISTINCT CASE WHEN event = 'product_view' THEN user_id END) AS step2_browse,
    COUNT(DISTINCT CASE WHEN event = 'add_to_cart' THEN user_id END) AS step3_cart,
    COUNT(DISTINCT CASE WHEN event = 'submit_order' THEN user_id END) AS step4_order,
    COUNT(DISTINCT CASE WHEN event = 'payment_success' THEN user_id END) AS step5_pay
  FROM user_events
  WHERE event_date BETWEEN '2025-06-01' AND '2025-06-30'
)
SELECT
  step1_visit AS '访问',
  step2_browse AS '浏览',
  step3_cart AS '加购',
  step4_order AS '下单',
  step5_pay AS '支付',
  ROUND(step2_browse / step1_visit * 100, 2) AS '浏览转化率%',
  ROUND(step3_cart / step2_browse * 100, 2) AS '加购转化率%',
  ROUND(step4_order / step3_cart * 100, 2) AS '下单转化率%',
  ROUND(step5_pay / step4_order * 100, 2) AS '支付转化率%',
  ROUND(step5_pay / step1_visit * 100, 2) AS '整体转化率%'
FROM funnel;
```

### 2.4 漏斗优化策略

| 流失环节 | 可能原因 | 优化策略 |
|----------|----------|----------|
| 访问→浏览 | 页面加载慢、内容不吸引 | 优化首屏加载、个性化推荐 |
| 浏览→加购 | 价格高、信息不足 | 价格优化、丰富详情页 |
| 加购→下单 | 流程复杂、运费高 | 简化流程、满减包邮 |
| 下单→支付 | 支付方式少、信任度低 | 增加支付方式、安全保障展示 |

## 三、留存分析

### 3.1 留存类型

| 留存类型 | 定义 | 适用场景 |
|----------|------|----------|
| 新用户留存 | 注册后第 N 天仍活跃 | 衡量新用户体验 |
| 老用户留存 | 活跃用户在后续周期仍活跃 | 衡量产品粘性 |
| 功能留存 | 使用某功能的用户再次使用 | 评估功能价值 |
| 渠道留存 | 特定渠道获取用户的留存 | 评估渠道质量 |

### 3.2 留存率计算

```
第 N 日留存率 = 第 N 日仍活跃的用户数 / 第 0 日的新增用户数 × 100%

示例：
- 第 0 日新增：1,000 人
- 第 1 日留存：400 人 → 次日留存率 = 40%
- 第 7 日留存：250 人 → 7日留存率 = 25%
- 第 30 日留存：150 人 → 30日留存率 = 15%
```

### 3.3 留存曲线分析

```
留存率
100% ┤ ●
     │  ╲
 80% ┤   ╲
     │    ╲
 60% ┤     ╲
     │      ╲
 40% ┤       ╲────────────────────  ← 理想留存曲线
     │        ╲
 20% ┤         ╲__________________  ← 实际留存曲线
     │
  0% ┤──────────────────────────────
     0    1    7   14   30   60   90 (天)

解读：
- 次日留存（Day 1）：反映新用户首次体验
- 7日留存（Day 7）：反映产品初期粘性
- 30日留存（Day 30）：反映产品长期价值
- 留存曲线趋于平稳的点 = 产品的真实用户群
```

### 3.4 留存分析 SQL 模板

```sql
-- 新用户留存分析
WITH new_users AS (
  SELECT user_id, MIN(event_date) AS register_date
  FROM user_events
  WHERE event = 'register'
  GROUP BY user_id
),
retention AS (
  SELECT
    DATEDIFF(e.event_date, n.register_date) AS day_n,
    COUNT(DISTINCT e.user_id) AS retained_users
  FROM new_users n
  JOIN user_events e ON n.user_id = e.user_id
  WHERE n.register_date BETWEEN '2025-06-01' AND '2025-06-07'
  GROUP BY DATEDIFF(e.event_date, n.register_date)
)
SELECT
  day_n,
  retained_users,
  ROUND(retained_users / (SELECT COUNT(*) FROM new_users 
    WHERE register_date BETWEEN '2025-06-01' AND '2025-06-07') * 100, 2) AS retention_rate
FROM retention
WHERE day_n BETWEEN 0 AND 30
ORDER BY day_n;
```

### 3.5 留存提升策略

| 策略 | 具体措施 | 预期效果 |
|------|----------|----------|
| 优化新用户引导 | 简化注册流程、展示核心价值 | 提升次日留存 5-10% |
| 构建 Aha Moment | 快速让用户感受到产品价值 | 提升 7日留存 10-15% |
| Push/邮件召回 | 对沉默用户进行精准触达 | 提升回访率 3-5% |
| 建立用户习惯 | 签到、积分、成就系统 | 提升 30日留存 8-12% |
| 社交绑定 | 关注、好友、社群 | 提升长期留存 15-20% |

## 四、A/B 测试

### 4.1 A/B 测试流程

```
1. 提出假设
   "将购买按钮从灰色改为橙色，能提升点击率"

2. 设计实验
   ├── 对照组（A）：灰色按钮
   ├── 实验组（B）：橙色按钮
   ├── 流量分配：50% / 50%
   └── 最小样本量：5,000/组

3. 计算样本量
   - 基准转化率：3%
   - 最小可检测效应（MDE）：10% 相对提升
   - 显著性水平：95%（α=0.05）
   - 统计功效：80%（β=0.2）
   → 每组需要约 50,000 样本

4. 执行实验
   ├── 确保随机分配
   ├── 监控护栏指标
   └── 运行足够时长（至少 1-2 个完整周期）

5. 分析结果
   ├── 计算转化率差异
   ├── 统计显著性检验
   ├── 计算置信区间
   └── 分析细分群体

6. 做出决策
   ├── 推全量 / 继续迭代 / 放弃
   └── 记录实验结论
```

### 4.2 A/B 测试指标体系

| 指标类型 | 说明 | 示例 |
|----------|------|------|
| 主要指标 | 实验要提升的核心指标 | 购买转化率 |
| 次要指标 | 辅助验证的指标 | 客单价、浏览时长 |
| 护栏指标 | 确保实验不造成负面影响 | 页面加载时间、错误率 |
| 分段指标 | 不同用户群体的表现 | 新/老用户、不同设备 |

### 4.4 A/B 测试报告模板

```markdown
# A/B 测试报告：购买按钮颜色优化

## 实验概述
- **实验ID**：EXP-2025-042
- **实验周期**：2025-06-01 ~ 2025-06-14
- **实验平台**：iOS + Android
- **负责团队**：产品增长团队

## 假设
将购买按钮从灰色（#999999）改为橙色（#FF6600），

## 实验设计

## 实验结果

|------|--------|--------|------|-----|--------|

| 指标 | 对照组 | 实验组 | 状态 |
|------|--------|--------|------|

| 错误率 | 0.1% | 0.1% | ✅ 正常 |

| 用户群 | 对照组 | 实验组 | 差异 |
|--------|--------|--------|------|
| iOS 用户 | 3.45% | 3.92% | +13.6% |
| Android 用户 | 2.98% | 3.25% | +9.1% |
| 新用户 | 2.10% | 2.65% | +26.2% |
| 老用户 | 3.80% | 3.95% | +3.9% |

## 结论与建议
1. 实验组（橙色按钮）显著提升了购买转化率（+11.5%）
```

### 4.5 常见 A/B 测试陷阱

| 陷阱 | 说明 | 避免方法 |
|------|------|----------|
| 偷看结果 | 提前查看导致误判 | 预设实验时长，到时间再看 |
| 多重比较 | 测试太多变体 | 使用 Bonferroni 校正 |
| 辛普森悖论 | 整体趋势与分组相反 | 分析细分群体 |
| 新奇效应 | 用户因新鲜感而短期改变 | 延长实验时间 |
| 样本污染 | 用户在组间切换 | 使用设备ID而非用户ID |
| 季节性影响 | 特殊时期的数据偏差 | 包含完整周期 |

## 五、常用分析框架

### 5.1 用户分群（RFM 模型）

| 维度 | 定义 | 高分标准 |
|------|------|----------|
| R（Recency） | 最近一次消费时间 | 7天内 |
| F（Frequency） | 消费频率 | 月消费 ≥ 4次 |
| M（Monetary） | 消费金额 | 月消费 ≥ 500元 |

| 用户分群 | R | F | M | 策略 |
|----------|---|---|---|------|
| 高价值用户 | 高 | 高 | 高 | VIP服务，专属权益 |
| 潜力用户 | 高 | 低 | 高 | 提升频次，推送召回 |
| 流失风险用户 | 低 | 高 | 高 | 紧急召回，专属优惠 |
| 新用户 | 高 | 低 | 低 | 引导激活，培养习惯 |

### 5.2 归因分析模型

| 模型 | 说明 | 适用场景 |
|------|------|----------|
| 首次触达 | 100% 归因于第一次接触 | 评估获客渠道 |
| 末次触达 | 100% 归因于最后一次接触 | 评估转化渠道 |
| 线性归因 | 平均分配给所有触点 | 全链路评估 |
| 时间衰减 | 越近的触点权重越高 | 评估短期效果 |
| 数据驱动 | 基于算法自动分配 | 精准归因（需大量数据） |
