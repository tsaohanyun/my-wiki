# 容器安全最佳实践

## 概述

容器安全涵盖整个容器生命周期，从镜像构建、存储、部署到运行时。本文档提供全面的容器安全最佳实践，帮助构建安全的容器化应用环境。

## 安全架构

```
┌─────────────────────────────────────────────────────────┐
│                    供应链安全                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐   │
│  │ 代码扫描 │→│ 镜像构建 │→│ 镜像扫描 │→│ 镜像签名 │   │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    运行时安全                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐   │
│  │ 网络策略 │→│ 准入控制 │→│ 进程监控 │→│ 日志审计 │   │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘   │
└─────────────────────────────────────────────────────────┘
```

## 镜像扫描

### 1. 构建安全镜像

```dockerfile
# Dockerfile.best-practices
# 使用最小基础镜像
FROM alpine:3.18 AS builder

# 安装安全更新
RUN apk update && apk upgrade && \
    apk add --no-cache \
    ca-certificates \
    tzdata && \
    rm -rf /var/cache/apk/*

# 创建非root用户
RUN addgroup -g 1001 -S appgroup && \
    adduser -u 1001 -S appuser -G appgroup

# 多阶段构建
FROM scratch

# 从builder复制必要文件
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/
COPY --from=builder /usr/share/zoneinfo /usr/share/zoneinfo
COPY --from=builder /etc/passwd /etc/passwd

# 复制应用二进制
COPY --chown=appuser:appgroup ./app /app

# 使用非root用户运行
USER appuser

# 设置只读文件系统
VOLUME ["/tmp"]

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD ["/app", "healthcheck"]

ENTRYPOINT ["/app"]
```

### 3. Trivy镜像扫描

```yaml
# GitHub Actions - Trivy扫描
name: Image Security Scan

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Build image
        run: docker build -t myapp:${{ github.sha }} .
      
      - name: Run Trivy vulnerability scanner
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: 'myapp:${{ github.sha }}'
          format: 'sarif'
          output: 'trivy-results.sarif'
          severity: 'CRITICAL,HIGH'
          exit-code: '1'
      
      - name: Upload Trivy scan results to GitHub Security tab
        uses: github/codeql-action/upload-sarif@v2
        if: always()
        with:
          sarif_file: 'trivy-results.sarif'
```

```bash

# 验证签名
cosign verify --key cosign.pub registry.example.com/myapp:v1.0.0
```

### 1. Pod安全标准

```yaml

  name: production

```

### 2. 安全上下文配置

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: secure-app
spec:
  template:
    spec:
      # Pod级别安全上下文
      securityContext:
        runAsNonRoot: true
        runAsUser: 1000
        runAsGroup: 1000
        fsGroup: 1000
        seccompProfile:
          type: RuntimeDefault
      
      # 服务账号配置
      serviceAccountName: app-sa
      automountServiceAccountToken: false
      
      containers:
      - name: app
        image: registry.example.com/app:v1.0.0
        
        # 容器级别安全上下文
        securityContext:
          allowPrivilegeEscalation: false
          readOnlyRootFilesystem: true
          runAsNonRoot: true
          runAsUser: 1000
          capabilities:
            drop:
              - ALL
            add:
              - NET_BIND_SERVICE
          seccompProfile:
            type: RuntimeDefault
        
        # 资源限制
        resources:
          requests:
            memory: "128Mi"
            cpu: "100m"
          limits:
            memory: "256Mi"
            cpu: "500m"
        
        # 只读挂载
        volumeMounts:
        - name: tmp
          mountPath: /tmp
        - name: cache
          mountPath: /app/cache
      
      volumes:
      - name: tmp
        emptyDir:
          medium: Memory
          sizeLimit: 100Mi
      - name: cache
        emptyDir:
          sizeLimit: 1Gi
```

```yaml
    when:
    - key: request.headers[x-api-key]
```

## 准入控制

### 1. OPA/Gatekeeper策略

```yaml
# 禁止特权容器
apiVersion: templates.gatekeeper.sh/v1
kind: ConstraintTemplate
metadata:
  name: k8spspprivilegedcontainer
spec:
  crd:
    spec:
      names:
        kind: K8sPSPPrivilegedContainer
  targets:
    - target: admission.k8s.gatekeeper.sh
      rego: |
        package k8spspprivileged
        
        violation[{"msg": msg, "details": {}}] {
            c := input_containers[_]
            c.securityContext.privileged
            msg := sprintf("Privileged container is not allowed: %v, securityContext: %v", [c.name, c.securityContext])
        }
        
        input_containers[c] {
            c := input.review.object.spec.containers[_]
        }
        
        input_containers[c] {
            c := input.review.object.spec.initContainers[_]
        }

---
apiVersion: constraints.gatekeeper.sh/v1beta1
kind: K8sPSPPrivilegedContainer
metadata:
  name: psp-privileged-container
spec:
  match:
    kinds:
      - apiGroups: [""]
        kinds: ["Pod"]
    namespaces:
      - production
```

### 2. Kyverno策略

```yaml
# 要求资源限制
apiVersion: kyverno.io/v1

  name: require-resource-limits
```

## 安全监控

### 1. Falco运行时监控

```yaml
# Falco规则
- rule: Unauthorized Process in Container
  desc: Detect process not in allowed list
  condition: >
    spawned_process and container and
    not proc.name in (allowed_processes) and
    not proc.pname in (allowed_parent_processes)
  output: >
    Unauthorized process started (user=%user.name command=%proc.cmdline 
    container_id=%container.id container_name=%container.name 
    image=%container.image.repository:%container.image.tag)
  priority: WARNING
  tags: [container, process, mitre_execution]

- rule: Sensitive File Access
  desc: Detect access to sensitive files
  condition: >
    open_read and container and
    (fd.name startswith /etc/shadow or
     fd.name startswith /etc/passwd or
     fd.name startswith /etc/sudoers)
  output: >
    Sensitive file opened for reading (user=%user.name 
    file=%fd.name command=%proc.cmdline 
    container_id=%container.id container_name=%container.name)
  priority: CRITICAL
  tags: [container, filesystem, mitre_credential_access]
```

- [ ] 使用最小基础镜像（Alpine、Distroless）
