# 数据中台架构

> 数据中台是企业级数据能力平台，核心目标是**实现数据资产的统一管理、复用和赋能**。通过数据采集、治理、服务和运营四大体系，将分散的数据沉淀为可复用的数据资产，为前台业务提供高效的数据服务能力。

## 二、OneData 方法论

阿里提出的 **OneData** 是数据中台的核心方法论，包含三大核心理念：

### 2.1 OneModel — 统一数据规范

> 解决"数据标准不统一"问题，定义全公司统一的**数据分层模型**。

#### 数据分层架构

| 层级 | 名称 | 说明 | 示例 |
|------|------|------|------|
| **ODS** | 操作数据层 | 原始数据1:1同步 | `ods_user_log_di` |
| **DWD** | 明细数据层 | 清洗、标准化后的明细 | `dwd_user_event_di` |
| **DWS** | 汇总数据层 | 按主题预聚合 | `dws_user_active_da` |
| **DIM** | 维度层 | 公共维度表 | `dim_user_info` |
| **ADS** | 应用数据层 | 面向应用的结果表 | `ads_user_profile_result` |

#### 命名规范

```
层级_业务域_数据内容_时间粒度

示例：
ods_trade_order_di      → ODS层 | 交易域 | 订单明细 | 日增量
dwd_user_register_di    → DWD层 | 用户域 | 注册明细 | 日增量
dws_user_active_1d_da   → DWS层 | 用户域 | 活跃1天汇总 | 日全量
dim_product_info        → DIM层 | 商品信息维度
ads_sales_dashboard_1d  → ADS层 | 销售看板 | 日
```

#### 时间粒度后缀

| 后缀 | 含义 |
|------|------|
| `_di` | 日增量（Daily Increment） |
| `_df` | 日全量（Daily Full） |
| `_hi` | 小时增量（Hourly Increment） |
| `_da` | 日聚合（Daily Aggregation） |
| `_wa` | 周聚合（Weekly Aggregation） |
| `_ma` | 月聚合（Monthly Aggregation） |

### 2.2 OneID — 统一实体识别

> 解决"同一实体在不同系统中ID不同"的问题。

```
业务系统A: user_id = 1001 (MySQL)
业务系统B: member_id = "M2001" (CRM)
日志系统C: device_id = "abc123" (埋点日志)

         ↓ OneID 统一

统一ID: one_uid = "U0000001"
映射关系: one_uid → [user_id, member_id, device_id]
```

#### ID-Mapping 实现逻辑

```python
# 基于图算法（连通子图）进行 ID 归一化
# 1. 收集所有 ID 对的关联关系
#    (user_id, device_id), (member_id, device_id), ...

# 2. 构建图
import networkx as nx

G = nx.Graph()

# 添加边（来自不同系统的事件）
edges = [
    ("user_1001", "device_abc123"),
    ("member_M2001", "device_abc123"),
    ("user_1001", "phone_138xxxx"),
    ("member_M2001", "email_a@b.com"),
    ("user_2002", "device_def456"),
    ("phone_138xxxx", "member_M2001"),  # 交叉关联
]
G.add_edges_from(edges)

# 3. 查找连通子图
for component in nx.connected_components(G):
    component_list = list(component)
    print(f"同一实体: {component_list}")

# 输出：
# 同一实体: ['user_1001', 'device_abc123', 'member_M2001', 'phone_138xxxx', 'email_a@b.com']
# 同一实体: ['user_2002', 'device_def456']

# 4. 分配统一 OneID
for idx, component in enumerate(nx.connected_components(G)):
    one_id = f"U{idx:07d}"
    for node in component:
        # 写入 ID 映射表
        print(f"{one_id} → {node}")
```

### 2.3 OneService — 统一数据服务

> 将数据封装为标准 API，统一对外提供数据服务能力。

```
数据表/视图 → 自动生成 RESTful API → 前端/应用消费

GET  /api/v1/user/profile?one_uid=U0000001
POST /api/v1/sales/summary  {"start_date": "2026-06-01", "end_date": "2026-06-28"}
```

## 三、数据采集层

### 3.1 采集方式总览

| 数据源 | 采集方式 | 工具 | 延迟 |
|--------|----------|------|------|
| 关系型数据库 | 批量同步 | DataX / Sqoop | 小时级 |
| 关系型数据库 | 增量同步（CDC） | Canal / Debezium / Flink CDC | 秒级 |
| 日志文件 | 实时采集 | Flume / Filebeat / Logstash | 秒级 |
| 消息系统 | 实时管道 | Kafka / Pulsar | 毫秒级 |
| API 接口 | 定时拉取 | 自定义脚本 / Airflow | 小时级 |
| IoT 设备 | 实时上报 | MQTT / EMQX | 毫秒级 |

### 3.2 CDC 同步方案（推荐 Flink CDC）

```java
// Flink CDC: MySQL → 数据湖（Iceberg/Hudi）
import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;
import org.apache.flink.cdc.connectors.mysql.source.MySqlSource;

StreamExecutionEnvironment env =
    StreamExecutionEnvironment.getExecutionEnvironment();
env.enableCheckpointing(60000);  // 60s Checkpoint

// 配置 MySQL CDC Source
MySqlSource<String> mySqlSource = MySqlSource.<String>builder()
    .hostname("mysql-primary")
    .port(3306)
    .databaseList("ecommerce")       // 监听数据库
    .tableList("ecommerce.orders", "ecommerce.users", "ecommerce.products")
    .username("cdc_user")
    .password("cdc_password")
    .serverTimeZone("Asia/Shanghai")
    .deserializer(new JsonDebeziumDeserializationSchema())
    .includeSchemaChanges(true)       // 包含 DDL 变更
    .build();

env.fromSource(mySqlSource,
    WatermarkStrategy.noWatermarks(),
    "MySQL CDC")
   .sinkTo(icebergSink);              // 写入 Iceberg
```

## 四、数据治理层

### 4.1 元数据管理

> 元数据是"关于数据的数据"，分为**技术元数据**和**业务元数据**。

| 类别 | 内容 | 示例 |
|------|------|------|
| 技术元数据 | 表结构、列类型、分区信息 | `dwd_order.di, column: order_id BIGINT` |
| 业务元数据 | 业务含义、数据域、负责人 | 交易域, 负责人: 张三, SLA: T+1 |
| 操作元数据 | 访问记录、产出时间、依赖关系 | 上游表: ods_order, 产出时间: 02:00 |

### 4.2 数据血缘

```sql
-- 构建血缘关系（以列级血缘为例）
```

### 4.3 数据质量监控

```python
# 数据质量规则框架示例
from dataclasses import dataclass
from enum import Enum

class RuleType(Enum):
    COMPLETENESS = "完整性"      # 空值率
    ACCURACY = "准确性"         # 值域校验
    CONSISTENCY = "一致性"      # 跨表一致性
    TIMELINESS = "时效性"       # 数据到达时间
    UNIQUENESS = "唯一性"       # 主键唯一
    VALIDITY = "有效性"         # 格式/枚举校验

@dataclass
class QualityRule:
    rule_id: str
    table_name: str
    column_name: str
    rule_type: RuleType
    rule_sql: str
    threshold: float           # 通过阈值（0~1）
    severity: str              # BLOCK / WARN / INFO

# 规则示例
rules = [
    QualityRule(
        rule_id="QR001",
        table_name="dwd_user_register_di",
        column_name="user_id",
        rule_type=RuleType.UNIQUENESS,
        rule_sql="""
            SELECT COUNT(*) - COUNT(DISTINCT user_id)
            FROM dwd_user_register_di
            WHERE dt = '${dt}'
        """,
        threshold=0.0,
        severity="BLOCK"        # 不通过则阻断下游
    ),
    QualityRule(
        rule_id="QR002",
        table_name="dwd_user_register_di",
        column_name="phone",
        rule_type=RuleType.COMPLETENESS,
        rule_sql="""
            SELECT
                COUNT(CASE WHEN phone IS NULL OR phone = '' THEN 1 END)
                / COUNT(*) AS null_rate
            FROM dwd_user_register_di
            WHERE dt = '${dt}'
        """,
        threshold=0.05,         # 空值率 <= 5%
        severity="WARN"
    ),
    QualityRule(
        rule_id="QR003",
        table_name="dwd_order_detail_di",
        column_name="amount",
        rule_type=RuleType.ACCURACY,
        rule_sql="""
            SELECT
                COUNT(CASE WHEN amount < 0 OR amount > 1000000 THEN 1 END)
                / COUNT(*) AS anomaly_rate
            FROM dwd_order_detail_di
            WHERE dt = '${dt}'
        """,
        threshold=0.01,         # 异常率 <= 1%
        severity="BLOCK"
    ),
    QualityRule(
        rule_type=RuleType.TIMELINESS,
        rule_id="QR004",
        table_name="dws_user_active_da",
        column_name="dt",
        rule_sql="""
            SELECT CASE
                WHEN MAX(dt) >= DATE_SUB(CURRENT_DATE, 1)
                THEN 1.0 ELSE 0.0
            END AS timeliness
            FROM dws_user_active_da
        """,
        threshold=1.0,
        severity="BLOCK"
    ),
]

# 执行规则并生成报告
for rule in rules:
    result = execute_sql(rule.rule_sql)
    actual_value = float(result[0][0])
    passed = actual_value <= rule.threshold

    status = "✅ PASSED" if passed else "❌ FAILED"
    print(f"{status} | {rule.rule_id} | {rule.table_name}.{rule.column_name} "
          f"| {rule.rule_type.value} | 实际值={actual_value:.4f} "
          f"| 阈值={rule.threshold}")

    if not passed and rule.severity == "BLOCK":
        # 发送告警 + 阻断下游任务
        send_alert(rule)
        block_downstream_tasks(rule.table_name)
```

### 4.4 数据安全与权限

| 维度 | 措施 |
|------|------|
| **数据分级** | 公开 / 内部 / 机密 / 绝密 |
| **脱敏** | 手机号 → `138****5678`；身份证 → `3201**********1234` |
| **行列权限** | 基于角色的列级访问控制（RBAC + ABAC） |
| **审计日志** | 所有查询记录到审计表，保留1年+ |
| **加密** | 传输层 TLS；存储层 KMS 加密 |

## 五、数据服务层

### 5.1 数据 API 设计

```python
# 统一数据服务框架（FastAPI 示例）
from fastapi import FastAPI, HTTPException, Query
from pydantic import BaseModel
from typing import Optional, List
import asyncpg

app = FastAPI(title="数据服务API", version="1.0.0")

class UserProfile(BaseModel):
    one_uid: str
    nickname: str
    register_date: str
    total_orders: int
    total_amount: float
    member_level: str

@app.get("/api/v1/user/profile", response_model=UserProfile)
async def get_user_profile(one_uid: str = Query(..., description="统一用户ID")):
    """查询用户画像（缓存优先）"""
    # 1. 查询缓存
    cached = await redis.get(f"user_profile:{one_uid}")
    if cached:
        return UserProfile(**json.loads(cached))

    # 2. 查询数据仓库
    async with db_pool.acquire() as conn:
        row = await conn.fetchrow("""
            SELECT
                u.one_uid,
                u.nickname,
                u.register_date,
                COALESCE(o.total_orders, 0) AS total_orders,
                COALESCE(o.total_amount, 0) AS total_amount,
                u.member_level
            FROM ads_user_profile u
            LEFT JOIN ads_user_order_summary o ON u.one_uid = o.one_uid
            WHERE u.one_uid = $1
        """, one_uid)

        if not row:
            raise HTTPException(status_code=404, detail="User not found")

        profile = UserProfile(**dict(row))
        await redis.setex(f"user_profile:{one_uid}", 300, json.dumps(profile.dict()))
        return profile

@app.get("/api/v1/sales/dashboard")
async def get_sales_dashboard(
    start_date: str = Query(...),
    end_date: str = Query(...),
    region: Optional[str] = None,
    category: Optional[str] = None
):
    """销售看板聚合查询"""
    query = """
        SELECT
            dt,
            region,
            category,
            order_count,
            total_amount,
            avg_order_amount
        FROM ads_sales_dashboard_1d
        WHERE dt BETWEEN $1 AND $2
    """
    params = [start_date, end_date]
    if region:
        query += " AND region = $3"
        params.append(region)
    if category:
        param_idx = len(params) + 1
        query += f" AND category = ${param_idx}"
        params.append(category)
    query += " ORDER BY dt"

    async with db_pool.acquire() as conn:
        rows = await conn.fetch(query, *params)
        return {"data": [dict(r) for r in rows]}
```

## 六、调度编排

### 6.1 Airflow DAG 示例

```python
from airflow import DAG
from airflow.providers.apache.spark.operators.spark_submit import SparkSubmitOperator
from airflow.providers.ssh.operators.ssh import SSHOperator
from airflow.operators.dummy import DummyOperator
from airflow.operators.python import PythonOperator
from datetime import datetime, timedelta

default_args = {
    "owner": "data-platform",
    "depends_on_past": False,
    "email": ["data-alerts@company.com"],
    "email_on_failure": True,
    "email_on_retry": False,
    "retries": 3,
    "retry_delay": timedelta(minutes=5),
}

dag = DAG(
    "daily_dws_pipeline",
    default_args=default_args,
    description="日维度汇总层ETL",
    schedule_interval="0 2 * * *",    # 每天凌晨2点
    start_date=datetime(2026, 1, 1),
    catchup=False,
    max_active_runs=1,
    tags=["dws", "daily", "ecommerce"],
)

start = DummyOperator(task_id="start", dag=dag)

# ODS → DWD（用户域）
dwd_user = SparkSubmitOperator(
    task_id="dwd_user_register_di",
    dag=dag,
    application="/jobs/spark/dwd_user_register_di.py",
    conn_id="spark_default",
    conf={
        "spark.driver.memory": "4g",
        "spark.executor.memory": "8g",
        "spark.executor.instances": "10",
        "spark.sql.adaptive.enabled": "true",
    },
    application_args=["{{ ds }}"],   # 传入执行日期
)

# ODS → DWD（交易域）
dwd_order = SparkSubmitOperator(
    task_id="dwd_order_detail_di",
    dag=dag,
    application="/jobs/spark/dwd_order_detail_di.py",
    conn_id="spark_default",
    conf={"spark.driver.memory": "8g", "spark.executor.memory": "16g"},
    application_args=["{{ ds }}"],
)

# 数据质量检查
quality_check = PythonOperator(
    task_id="quality_check_dwd",
    dag=dag,
    python_callable=run_quality_checks,
    op_kwargs={
        "tables": ["dwd_user_register_di", "dwd_order_detail_di"],
        "dt": "{{ ds }}",
    },
)

# DWD → DWS
dws_user_active = SparkSubmitOperator(
    task_id="dws_user_active_da",
    dag=dag,
    application="/jobs/spark/dws_user_active_da.py",
    conn_id="spark_default",
    application_args=["{{ ds }}"],
)

dws_sales_summary = SparkSubmitOperator(
    task_id="dws_sales_summary_da",
    dag=dag,
    application="/jobs/spark/dws_sales_summary_da.py",
    conn_id="spark_default",
    application_args=["{{ ds }}"],
)

# ADS → 推送结果到应用系统
sync_to_redis = SSHOperator(
    task_id="sync_to_redis",
    dag=dag,
    ssh_conn_id="etl_server",
    command="python3 /scripts/sync_ads_to_redis.py {{ ds }}",
)

end = DummyOperator(task_id="end", dag=dag)

# 依赖关系
start >> [dwd_user, dwd_order] >> quality_check
quality_check >> [dws_user_active, dws_sales_summary] >> sync_to_redis >> end
```

## 八、最佳实践总结

### 8.1 建设路径建议

```
Phase 1: 统一采集 + ODS → DWD 基础分层（3-6个月）
Phase 2: 完善维度建模 + DWS + 数据质量（6-12个月）
Phase 3: 数据服务API + 数据超市（6-12个月）
Phase 4: 数据资产目录 + 自动化运营（持续）
```

### 8.2 避坑指南

| 误区 | 正确做法 |
|------|----------|
| 一上来就建大而全的中台 | 先解决1-2个核心业务场景 |
| 只重开发不重治理 | 数据质量是中台的基石 |
| 所有数据都入湖 | 按价值和频率分层存储 |
| API 返回大量数据 | 做好分页、缓存和聚合 |
| 忽略数据安全 | 分级分类 + 行列权限 + 审计 |

### 8.3 技术栈推荐

| 层级 | 推荐技术栈 |
|------|-----------|
| 采集 | Flink CDC + Filebeat + Kafka |
| 存储 | 对象存储（S3/OSS）+ 数据湖（Iceberg） |
| 计算 | Spark（离线）+ Flink（实时） |
| OLAP | Apache Doris / ClickHouse / StarRocks |
| 调度 | Apache Airflow / DolphinScheduler |
| 元数据 | Apache Atlas / DataHub |
| 数据质量 | Great Expectations / 自研规则引擎 |
| BI | Apache Superset / Metabase |
