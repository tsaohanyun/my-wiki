# 性能测试指南

## 概述

性能测试是验证系统在预期和极端负载下的响应能力、稳定性、可扩展性的关键质量保障手段。本指南涵盖负载测试、压力测试的方法论、工具配置和调优实践。

---

## 性能测试类型

### 测试类型对比

| 测试类型 | 目的 | 负载模式 | 关注指标 | 何时执行 |
|---------|------|---------|---------|---------|
| 负载测试 | 验证正常负载下性能 | 预期用户量 | 响应时间、吞吐量 | 每次发版 |
| 压力测试 | 找到系统极限 | 逐步增加到崩溃 | 最大并发数、崩溃点 | 里程碑 |
| 浸泡测试 | 检测内存泄漏等长期问题 | 持续稳定负载 | 内存、CPU趋势 | 发版前 |
| 尖峰测试 | 验证突发流量处理 | 瞬间大流量 | 响应时间、错误率 | 促销前 |
| 容量测试 | 确定系统容量上限 | 逐步增加数据量 | 数据库响应、存储 | 架构变更后 |

### 性能指标定义

```yaml
# performance-sla.yaml
service_level_agreement:
  api:
    p50_response_time: 100ms     # 50%请求响应时间
    p90_response_time: 300ms     # 90%请求响应时间
    p99_response_time: 1000ms    # 99%请求响应时间
    max_response_time: 3000ms    # 最大响应时间
    error_rate: < 0.1%           # 错误率
    throughput: >= 1000 QPS      # 吞吐量

  web_page:
    first_contentful_paint: < 1.5s
    largest_contentful_paint: < 2.5s
    time_to_interactive: < 3.5s
    cumulative_layout_shift: < 0.1

  database:
    query_p99: < 200ms
    connection_pool_usage: < 80%
    slow_query_threshold: 500ms
```

```bash

  -e -o ./report \
```

## Locust

### Locust负载测试脚本

```python
# locustfile.py
from locust import HttpUser, task, between, events
from locust.runners import MasterRunner
import time
import json
import logging

logger = logging.getLogger(__name__)

class WebsiteUser(HttpUser):
    """模拟网站用户行为"""
    wait_time = between(1, 3)  # 请求间隔1-3秒
    host = "http://localhost:8000"

    def on_start(self):
        """用户启动时登录"""
        response = self.client.post("/api/v1/auth/login", json={
            "username": "testuser",
            "password": "testpass"
        })
        if response.status_code == 200:
            self.token = response.json()["access_token"]
            self.client.headers.update({
                "Authorization": f"Bearer {self.token}"
            })

    @task(5)  # 权重5，执行频率最高
    def view_homepage(self):
        """浏览首页"""
        with self.client.get("/", catch_response=True) as response:
            if response.status_code != 200:
                response.failure(f"首页返回 {response.status_code}")

    @task(3)
    def search_products(self):
        """搜索商品"""
        self.client.get("/api/v1/products/search",
            params={"q": "手机", "page": 1, "size": 20}
        )

    @task(2)
    def view_product_detail(self):
        """查看商品详情"""
        product_id = self._get_random_product_id()
        self.client.get(f"/api/v1/products/{product_id}")

    @task(1)
    def create_order(self):
        """创建订单"""
        self.client.post("/api/v1/orders", json={
            "product_id": self._get_random_product_id(),
            "quantity": 1
        })

    def _get_random_product_id(self):
        import random
        return random.randint(1, 1000)


class AdminUser(HttpUser):
    """管理员用户行为"""
    wait_time = between(2, 5)
    host = "http://localhost:8000"
    weight = 1  # 权重较低，模拟少量管理员

    def on_start(self):
        response = self.client.post("/api/v1/auth/login", json={
            "username": "admin",
            "password": "adminpass"
        })
        if response.status_code == 200:
            self.token = response.json()["access_token"]
            self.client.headers.update({
                "Authorization": f"Bearer {self.token}"
            })

    @task(3)
    def view_dashboard(self):
        """查看管理面板"""
        self.client.get("/api/v1/admin/dashboard")

    @task(2)
    def view_orders_list(self):
        """查看订单列表"""
        self.client.get("/api/v1/admin/orders", params={"page": 1, "size": 50})

    @task(1)
    def export_report(self):
        """导出报表"""
        self.client.get("/api/v1/admin/reports/export",
            params={"type": "sales", "date_range": "last_30_days"}
        )


# 自定义事件处理
@events.test_start.add_listener
def on_test_start(environment, **kwargs):
    logger.info("性能测试开始...")

@events.test_stop.add_listener
def on_test_stop(environment, **kwargs):
    logger.info("性能测试结束")
    if isinstance(environment.runner, MasterRunner):
        # 主节点汇总结果
        stats = environment.runner.stats
        logger.info(f"总请求数: {stats.total.num_requests}")
        logger.info(f"平均响应时间: {stats.total.avg_response_time:.2f}ms")
        logger.info(f"RPS: {stats.total.current_rps:.2f}")
```

### 常见性能优化

#### 1. 数据库优化

```sql
-- 分析慢查询
EXPLAIN ANALYZE
SELECT u.name, COUNT(o.id) as order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.created_at > NOW() - INTERVAL '30 days'
GROUP BY u.id
ORDER BY order_count DESC
LIMIT 100;

-- 添加合适的索引
CREATE INDEX CONCURRENTLY idx_orders_user_created
ON orders(user_id, created_at);

-- 分区表（大数据量场景）
CREATE TABLE orders (
    id BIGSERIAL,
    user_id BIGINT,
    created_at TIMESTAMP,
    amount DECIMAL(10,2)
) PARTITION BY RANGE (created_at);

CREATE TABLE orders_2026_01 PARTITION OF orders
    FOR VALUES FROM ('2026-01-01') TO ('2026-02-01');
```

#### 2. 缓存优化

```python
# app/cache.py
import redis
from functools import wraps
import json
import hashlib

redis_client = redis.Redis(host="localhost", port=6379, db=0)

def cached(expire=300, prefix="cache"):
    """缓存装饰器"""
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            # 生成缓存key
            key_data = f"{func.__name__}:{args}:{kwargs}"
            cache_key = f"{prefix}:{hashlib.md5(key_data.encode()).hexdigest()}"

            # 尝试从缓存获取
            cached_value = redis_client.get(cache_key)
            if cached_value:
                return json.loads(cached_value)

            # 执行函数
            result = await func(*args, **kwargs)

            # 写入缓存
            redis_client.setex(cache_key, expire, json.dumps(result))
            return result
        return wrapper
    return decorator

@cached(expire=600, prefix="product")
async def get_product(product_id: int):
    """获取商品信息（带缓存）"""
    return await db.products.find_one({"id": product_id})
```

## 最佳实践

### 1. 测试环境管理
- 性能测试环境应与生产环境配置一致（等比缩小）
- 使用独立的性能测试环境，避免影响其他测试
- 测试数据量应接近生产环境

### 2. 测试执行规范
- 每次测试前清理缓存和临时数据
- 预热期（warm-up）不计入结果
- 重复测试3次以上，取平均值
- 记录完整的测试环境信息

### 3. 基线管理
- 建立性能基线（baseline）
- 每次测试与基线对比
- 性能退化超过10%需报警
- 定期更新基线

### 4. 持续性能测试
- CI中集成轻量级性能测试
- 关键接口设置性能门禁
- 性能报告自动归档和对比
