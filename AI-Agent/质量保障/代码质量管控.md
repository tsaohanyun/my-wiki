# 代码质量管控

## 概述

代码质量管控是通过规范、工具和流程三位一体的方式，确保代码在可读性、可维护性、安全性和性能等方面达到标准的质量保障体系。

## 代码规范

### Python代码规范

```toml
# pyproject.toml - 项目配置
[project]
name = "ai-agent"
version = "1.0.0"
requires-python = ">=3.11"

# Ruff - 极速Python Linter和Formatter
[tool.ruff]
target-version = "py311"
line-length = 120
fix = true

[tool.ruff.lint]
select = [
    "E",    # pycodestyle errors
    "W",    # pycodestyle warnings
    "F",    # pyflakes
    "I",    # isort
    "N",    # pep8-naming
    "D",    # pydocstyle
    "S",    # bandit (security)
    "B",    # flake8-bugbear
    "A",    # flake8-builtins
    "C4",   # flake8-comprehensions
    "UP",   # pyupgrade
    "SIM",  # flake8-simplify
    "TCH",  # flake8-type-checking
    "RUF",  # Ruff-specific rules
]
ignore = ["D100", "D104"]

[tool.ruff.lint.per-file-ignores]
"tests/**/*.py" = ["S101", "D"]

[tool.ruff.format]
quote-style = "double"
indent-style = "space"

# MyPy类型检查
[tool.mypy]
python_version = "3.11"
strict = true
warn_return_any = true
warn_unused_configs = true
disallow_untyped_defs = true
disallow_incomplete_defs = true
check_untyped_defs = true
no_implicit_optional = true

[[tool.mypy.overrides]]
module = "tests.*"
disallow_untyped_defs = false
```

### Commit规范

```yaml
# commitlint.config.js
module.exports = {
  extends: ['@commitlint/config-conventional'],

      'chore',    # 其他杂项

    'body-max-line-length': [2, 'always', 200],
  }
```

| 工具 | 类型 | 语言支持 | 特点 | 集成方式 |

### MyPy类型检查配置

```python
# app/models/user.py - 类型注解示例
from typing import Optional
from pydantic import BaseModel, EmailStr
from datetime import datetime

class UserBase(BaseModel):
    name: str
    email: EmailStr

class UserCreate(UserBase):
    password: str

class UserResponse(UserBase):
    id: int
    is_active: bool
    created_at: datetime

    class Config:
        from_attributes = True

class UserRepository:
    def __init__(self, db: "AsyncSession") -> None:
        self.db = db

    async def get_by_id(self, user_id: int) -> Optional[UserResponse]:
        """根据ID获取用户"""
        result = await self.db.execute(
            select(User).where(User.id == user_id)
        )
        user = result.scalar_one_or_none()
        if user is None:
            return None
        return UserResponse.model_validate(user)

    async def create(self, data: UserCreate) -> UserResponse:
        """创建用户"""
        user = User(**data.model_dump())
        self.db.add(user)
        await self.db.commit()
        await self.db.refresh(user)
        return UserResponse.model_validate(user)
```

### Semgrep安全规则

```yaml
# .semgrep/rules.yml
rules:
  - id: no-hardcoded-secrets
    patterns:
      - pattern: |
          $VAR = "..."
      - metavariable-regex:
          metavariable: $VAR
          regex: (?i)(password|secret|api_key|token|private_key)
    message: "检测到硬编码的敏感信息，请使用环境变量"
    severity: ERROR
    languages: [python]
    metadata:
      category: security
      cwe: "CWE-798"

  - id: no-sql-format-string
    pattern: |
      $CURSOR.execute(f"..." % ...)
    message: "SQL查询不应使用字符串格式化，应使用参数化查询"
    severity: ERROR
    languages: [python]
    metadata:
      category: security
      cwe: "CWE-89"

  - id: use-https-only
    pattern: |
      requests.get("http://...", ...)
    message: "应使用HTTPS而非HTTP"
    severity: WARNING
    languages: [python]
```

## SonarQube

### SonarQube配置

```properties
# sonar-project.properties
sonar.projectKey=ai-agent
sonar.projectName=AI Agent
sonar.projectVersion=1.0.0

sonar.sources=app
sonar.tests=tests
sonar.python.coverage.reportPaths=coverage.xml
sonar.python.xunit.reportPath=test-results.xml

# 排除文件
sonar.exclusions=**/migrations/**,**/vendor/**,**/__pycache__/**
sonar.test.exclusions=**/conftest.py

# 质量门禁
sonar.qualitygate.wait=true
```

### SonarQube质量门禁规则

```yaml
# quality-gate.yml
quality_gate:
  name: "AI-Agent-QG"
  conditions:
    # 新代码指标
    - metric: new_coverage
      operator: LESS_THAN
      threshold: "80"      # 新代码覆盖率 < 80% 则失败

    - metric: new_duplicated_lines_density
      operator: GREATER_THAN
      threshold: "3"       # 新代码重复率 > 3% 则失败

    - metric: new_bugs
      operator: GREATER_THAN
      threshold: "0"       # 新代码不允许有bug

    - metric: new_vulnerabilities
      operator: GREATER_THAN
      threshold: "0"       # 新代码不允许有漏洞

    - metric: new_code_smells
      operator: GREATER_THAN
      threshold: "10"      # 新代码代码异味 > 10 则失败

    - metric: new_security_hotspots_reviewed
      operator: LESS_THAN
      threshold: "100"     # 安全热点审查率 < 100% 则失败
```

### SonarQube CI集成

```yaml
# .github/workflows/sonar.yml
name: SonarQube Analysis

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  sonar:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # 需要完整历史用于新代码分析

      - name: 设置Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: 安装依赖
        run: pip install -r requirements-test.txt

      - name: 运行测试并生成覆盖率
        run: |
          pytest --cov=app --cov-report=xml --junitxml=test-results.xml

      - name: SonarQube扫描
        uses: sonarqube-quality-gate-action@master
        with:
          scanMetadataReportFile: target/sonar-report.json
        env:
          SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
          SONAR_HOST_URL: ${{ secrets.SONAR_HOST_URL }}
```

## 代码审查流程

### 审查清单

```markdown
## 代码审查清单

### 功能正确性
- [ ] 代码实现了需求描述的功能
- [ ] 边界条件已处理
- [ ] 错误处理完善
- [ ] 无明显的逻辑错误

### 代码质量
- [ ] 命名清晰有意义
- [ ] 函数/方法职责单一
- [ ] 无重复代码（DRY原则）
- [ ] 适当的注释（解释Why而非What）
- [ ] 遵循项目代码规范

### 安全性
- [ ] 输入已验证和清理
- [ ] SQL查询已参数化
- [ ] 敏感数据已加密
- [ ] 权限检查已实现
- [ ] 日志中无敏感信息

### 性能
- [ ] 无不必要的数据库查询（N+1问题）
- [ ] 大数据量操作已分页/分批
- [ ] 适当使用缓存
- [ ] 无不必要的循环嵌套

### 测试
- [ ] 新增代码有测试覆盖
- [ ] 测试用例覆盖正常和异常场景
- [ ] 测试命名清晰描述意图
```

### 审查流程自动化

```yaml
# .github/workflows/pr-review.yml
name: PR Quality Check

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  quality-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # 代码规范检查
      - name: Ruff Lint
        run: |
          pip install ruff
          ruff check .

      # 类型检查
      - name: MyPy Check
        run: |
          pip install mypy
          mypy app/ --ignore-missing-imports

      # 安全扫描
      - name: Security Scan
        run: |
          pip install bandit
          bandit -r app/ -f json -o bandit-report.json || true

      # PR大小检查
      - name: PR Size Check
        uses: actions/github-script@v7
        with:
          script: |
            const pr = context.payload.pull_request;
            const additions = pr.additions;
            const deletions = pr.deletions;
            const total = additions + deletions;

            if (total > 500) {
              core.warning(`PR修改了 ${total} 行代码，建议拆分为更小的PR`);
            }

            if (total > 1000) {
              core.setFailed(`PR修改了 ${total} 行代码，必须拆分`);
            }
```

## 最佳实践

### 1. 渐进式质量提升
- 新代码严格遵循标准（SonarQube New Code模式）
- 存量代码逐步修复，不阻塞发布
- 设置合理的质量门禁阈值

### 2. 工具链整合
- IDE实时检查 → pre-commit本地检查 → CI远程检查
- 所有工具配置版本化管理
- 统一工具版本避免团队不一致

### 3. 审查文化
- 审查对事不对人，关注代码而非作者
- 及时完成审查（< 24小时）
- 审查也是学习机会，鼓励提问和讨论
- 对好的代码实践给予肯定

### 4. 指标跟踪
- 定期查看SonarQube质量趋势
- 跟踪技术债务变化
- 代码审查参与度和质量
- 新增代码质量达标率
