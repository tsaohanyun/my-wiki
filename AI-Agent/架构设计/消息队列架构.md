# 消息队列架构

## 概述

消息队列（Message Queue, MQ）是分布式系统中实现**异步通信**、**流量削峰**、**系统解耦**的核心中间件。

## 消息队列解决的核心问题

```
┌──────────────────────────────────────────────────────────────┐
│                  消息队列三大核心价值                           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 异步处理                                                  │
│  ┌──────┐  同步  ┌──────┐  同步  ┌──────┐                    │
│  │ 订单 │──────→│ 支付 │──────→│ 通知 │  延迟: 200ms          │
│  └──────┘       └──────┘       └──────┘                     │
│                                                              │
│  ┌──────┐  异步  ┌──────┐       ┌──────┐                    │
│  │ 订单 │──MQ──→│ 支付 │       │ 通知 │  延迟: 20ms          │
│  └──────┘       └──────┘       └──────┘                     │
│                                                              │
│  2. 流量削峰                                                  │
│  秒杀: 10万QPS → MQ(缓冲) → 后端处理 1000QPS                  │
│                                                              │
│  3. 系统解耦                                                  │
│  订单系统 → MQ ← 库存/支付/通知/物流（独立演进）                │
└──────────────────────────────────────────────────────────────┘
```

### Kafka 核心概念

| 概念 | 说明 |
|------|------|
| **Broker** | Kafka 服务节点 |
| **Topic** | 消息的逻辑分类 |
| **Partition** | Topic 的物理分片，保证分区内有序 |
| **Replica** | 分区副本，Leader 处理读写，Follower 同步 |
| **Consumer Group** | 消费组，组内竞争消费 |
| **Offset** | 消费者在分区中的消费位置 |

```python
from kafka import KafkaProducer, KafkaConsumer
from kafka.errors import KafkaError
import json

class KafkaClient:
    """Kafka 客户端封装"""

    def __init__(self, bootstrap_servers: str = "localhost:9092"):
        self.bootstrap_servers = bootstrap_servers

    def create_producer(self) -> KafkaProducer:
        return KafkaProducer(
            bootstrap_servers=self.bootstrap_servers,
            # 序列化
            key_serializer=lambda k: k.encode("utf-8") if k else None,
            value_serializer=lambda v: json.dumps(v, ensure_ascii=False).encode("utf-8"),
            # 可靠性：等待所有副本确认
            acks="all",
            # 重试
            retries=5,
            retry_backoff_ms=100,
            # 批量发送
            batch_size=16384,
            linger_ms=10,
            # 压缩
            compression_type="lz4",
            # 幂等性（防止重复消息）
            enable_idempotence=True,
            max_in_flight_requests_per_connection=5,
        )

    def create_consumer(self, group_id: str,
                         topics: list) -> KafkaConsumer:
        return KafkaConsumer(
            *topics,
            bootstrap_servers=self.bootstrap_servers,
            group_id=group_id,
            # 反序列化
            key_deserializer=lambda k: k.decode("utf-8") if k else None,
            value_deserializer=lambda v: json.loads(v.decode("utf-8")),
            # 消费位点
            auto_offset_reset="earliest",  # 从最早消息开始
            enable_auto_commit=False,       # 手动提交
            # 批量拉取
            max_poll_records=500,
            max_poll_interval_ms=300000,
        )


# 生产者示例
```

### RabbitMQ 实现

```python
import pika
import json
from typing import Callable

class RabbitMQClient:
    """RabbitMQ 客户端"""

    def __init__(self, host: str = "localhost", port: int = 5672,
                 vhost: str = "/", username: str = "guest",
                 password: str = "guest"):
        credentials = pika.PlainCredentials(username, password)
        self.params = pika.ConnectionParameters(
            host=host, port=port, virtual_host=vhost,
            credentials=credentials,
            heartbeat=600,
            blocked_connection_timeout=300,
        )
        self.connection = None
        self.channel = None

    def connect(self):
        self.connection = pika.BlockingConnection(self.params)
        self.channel = self.connection.channel()

    def declare_exchange(self, name: str, exchange_type: str = "topic",
                          durable: bool = True):
        self.channel.exchange_declare(
            exchange=name,
            exchange_type=exchange_type,
            durable=durable,
        )

    def declare_queue(self, name: str, durable: bool = True,
                       dead_letter_exchange: str = "dlx"):
        """声明队列（带死信交换机）"""
        self.channel.queue_declare(
            queue=name,
            durable=durable,
            arguments={
                "x-dead-letter-exchange": dead_letter_exchange,
                "x-message-ttl": 86400000,  # 24小时
                "x-max-length": 1000000,
            },
        )

    def publish(self, exchange: str, routing_key: str, message: dict,
                priority: int = 0):
        """发布消息"""
        self.channel.basic_publish(
            exchange=exchange,
            routing_key=routing_key,
            body=json.dumps(message, ensure_ascii=False).encode("utf-8"),
            properties=pika.BasicProperties(
                delivery_mode=2,  # 持久化
                priority=priority,
                content_type="application/json",
            ),
        )

    def consume(self, queue: str, callback: Callable, prefetch_count: int = 1):
        """消费消息"""
        self.channel.basic_qos(prefetch_count=prefetch_count)
        self.channel.basic_consume(
            queue=queue,
            on_message_callback=callback,
            auto_ack=False,  # 手动确认
        )
        self.channel.start_consuming()


# 使用示例
def setup_mq():
    client = RabbitMQClient()
    client.connect()

    # 声明交换机和队列
    client.declare_exchange("orders", exchange_type="topic")
    client.declare_queue("order.process")
    client.declare_queue("order.notify")

    # 绑定队列到交换机
    client.channel.queue_bind("order.process", "orders", routing_key="order.*")
    client.channel.queue_bind("order.notify", "orders", routing_key="order.created")

    return client

def message_handler(ch, method, properties, body):
    """消息处理回调"""
    try:
        message = json.loads(body)
        print(f"Received: {message}")
        # 处理消息...
        ch.basic_ack(delivery_tag=method.delivery_tag)
    except Exception as e:
        print(f"Failed: {e}")
        # 拒绝消息，进入死信队列
        ch.basic_nack(delivery_tag=method.delivery_tag, requeue=False)
```

```python
# RocketMQ 事务消息流程（以订单创建为例）

class OrderTransactionProducer:
    """RocketMQ 事务消息生产者"""

    def create_order_with_transaction(self, order_data: dict):
        """
        事务消息流程：
        1. 发送 half message（预消息，消费者不可见）
        2. 执行本地事务（创建订单）
        3. 根据事务结果 commit 或 rollback
        4. 如果没有收到确认，broker 回查本地事务状态
        """

        # Step 1: 发送 half message
        msg = Message(
            topic="ORDER_TOPIC",
            tags="create",
            body=json.dumps(order_data).encode(),
        )

        # Step 2-3: 事务执行器
        transaction = Transaction(msg, self._local_transaction, self._check_local)

        # 发送事务消息
        result = self.producer.send_message_in_transaction(msg, transaction)
        return result

    def _local_transaction(self, msg: Message) -> LocalTransactionState:
        """执行本地事务"""
        try:
            order_data = json.loads(msg.body)
            # 创建订单到数据库
            order_id = self.order_db.create(order_data)
            print(f"Order {order_id} created")
            return LocalTransactionState.COMMIT_MESSAGE
        except Exception as e:
            print(f"Local transaction failed: {e}")
            return LocalTransactionState.ROLLBACK_MESSAGE

    def _check_local_transaction(self, msg: Message) -> LocalTransactionState:
        """事务回查（broker 未收到确认时调用）"""
        order_data = json.loads(msg.body)
        order = self.order_db.find_by_data(order_data)
        if order and order.status == "created":
            return LocalTransactionState.COMMIT_MESSAGE
        return LocalTransactionState.ROLLBACK_MESSAGE
```

### 幂等消费实现

```python
import hashlib
from datetime import datetime

class IdempotentConsumer:
    """幂等消息消费者"""

    def __init__(self, cache: "RedisCache"):
        self.cache = cache
        self.processed_prefix = "msg:processed:"

    async def process(self, message_id: str, handler: Callable,
                       ttl: int = 86400 * 7) -> bool:
        """处理消息（幂等）"""
        # 1. 检查是否已处理
        key = f"{self.processed_prefix}{message_id}"
        if await self.cache.get(key):
            print(f"Message {message_id} already processed, skip")
            return True

        # 2. 尝试获取处理锁（防止并发重复消费）
        lock_key = f"msg:lock:{message_id}"
        acquired = await self.cache.set(lock_key, "1", ttl=300, nx=True)
        if not acquired:
            print(f"Message {message_id} being processed by another worker")
            return True

        try:
            # 3. 执行业务处理
            result = await handler()

            # 4. 标记为已处理
            await self.cache.set(key, "1", ttl=ttl)
            return True
        except Exception as e:
            print(f"Processing failed for {message_id}: {e}")
            raise
        finally:
            await self.cache.delete(lock_key)


class RetryManager:
    """重试管理器"""

    MAX_RETRIES = 3
    RETRY_DELAYS = [1, 5, 30]  # 秒

    def __init__(self, cache: "RedisCache", dlq_topic: str):
        self.cache = cache
        self.dlq_topic = dlq_topic

    async def process_with_retry(self, message: dict,
                                   handler: Callable) -> bool:
        message_id = message.get("message_id")
        retry_key = f"msg:retry:{message_id}"

        for attempt in range(self.MAX_RETRIES):
            try:
                await handler(message)
                await self.cache.delete(retry_key)
                return True
            except Exception as e:
                retry_count = await self.cache.incr_with_ttl(retry_key, ttl=3600)
                if retry_count >= self.MAX_RETRIES:
                    # 超过最大重试次数，进入死信队列
                    await self._send_to_dlq(message, str(e))
                    return False
                else:
                    delay = self.RETRY_DELAYS[min(attempt, len(self.RETRY_DELAYS) - 1)]
                    print(f"Retry {retry_count}/{self.MAX_RETRIES} "
                          f"after {delay}s for {message_id}")
                    await asyncio.sleep(delay)

        return False

    async def _send_to_dlq(self, message: dict, error: str):
        """发送到死信队列"""
        dlq_message = {
            **message,
            "dlq_reason": error,
            "dlq_timestamp": datetime.utcnow().isoformat(),
        }
        print(f"Sent to DLQ: {message.get('message_id')}")
```

## 5. 消息队列选型对比

| 特性 | Kafka | RabbitMQ | RocketMQ |
|------|-------|----------|----------|
| **开发语言** | Java/Scala | Erlang | Java |
| **消息模型** | Topic + Partition | Exchange + Queue | Topic + Queue |
| **吞吐量** | 百万级/秒 | 万级/秒 | 十万级/秒 |
| **延迟** | 毫秒级 | 微秒级 | 毫秒级 |
| **消息可靠性** | 高（副本+ISR） | 高（镜像队列） | 高（同步双写） |
| **顺序消息** | 分区有序 | 队列有序 | 全局/分区有序 |
| **事务消息** | 支持（0.11+） | 不支持 | 支持（特色功能） |
| **延迟消息** | 不原生支持 | 插件支持 | 原生支持（18级） |
| **消息回溯** | 支持（按offset） | 不支持 | 支持（按时间） |
| **管理界面** | Kafka UI | 自带管理台 | RocketMQ Dashboard |
| **适用场景** | 大数据、日志、流处理 | 企业应用、复杂路由 | 电商、金融、事务 |

## 最佳实践

### ✅ 推荐

1. **消息持久化** - 关键消息必须持久化到磁盘
2. **手动确认** - 消费成功后手动 ack，避免自动确认导致消息丢失
3. **幂等消费** - 所有消费者必须实现幂等
4. **死信队列** - 处理失败的消息进入 DLQ，不要无限重试
5. **消息压缩** - 大消息使用 LZ4/Snappy 压缩
6. **合理分区数** - Kafka 分区数 = max(生产者并发, 消费者并发)
7. **监控告警** - 监控消费延迟（Consumer Lag）、堆积量

### ❌ 避免

1. **避免超大消息** - 单条消息不超过 1MB
2. **避免长时间处理** - 消费者处理时间不超过 max.poll.interval
3. **避免忽略顺序性** - 需要顺序的场景使用同一 partition/key
4. **避免循环依赖** - 服务间不要通过 MQ 形成消息环
