# 缓存架构设计

## 概述

缓存是通过将热点数据存储在高速存储介质中，减少对低速数据源的访问，从而提升系统性能和吞吐量的关键技术。

## 缓存架构总览

```
┌──────────────────────────────────────────────────────────────────┐
│                        缓存架构分层                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  客户端                                                          │
│  ┌──────────────┐                                                │
│  │ 本地缓存       │  L1: 进程内缓存（Caffeine/Guava）               │
│  │ TTL: 秒级     │  命中率: ~30%                                  │
│  └──────┬───────┘                                                │
│         │ Miss                                                   │
│         ▼                                                        │
│  ┌──────────────┐                                                │
│  │ 分布式缓存     │  L2: Redis Cluster / Memcached                 │
│  │ TTL: 分钟~小时 │  命中率: ~60%                                  │
│  └──────┬───────┘                                                │
│         │ Miss                                                   │
│         ▼                                                        │
│  ┌──────────────┐                                                │
│  │ 数据库        │  L3: MySQL / PostgreSQL                         │
│  │ 持久化存储     │  命中率: 100%（最终兜底）                        │
│  └──────────────┘                                                │
│                                                                  │
│  总命中率 = 1 - (1-0.3) × (1-0.6) = 1 - 0.28 = 72%              │
│  平均延迟 = 0.3×0.1ms + 0.42×1ms + 0.28×10ms ≈ 3.24ms           │
└──────────────────────────────────────────────────────────────────┘
```

## 1. 缓存策略

### Cache-Aside（旁路缓存）

最常用的策略，应用层负责缓存的读写。

```
读流程：
  App → 查缓存 → Hit → 返回数据
                → Miss → 查DB → 写缓存 → 返回数据

写流程：
  App → 更新DB → 删除缓存（而非更新缓存）
```

```python
class CacheAsidePattern:
    """Cache-Aside 模式实现"""

    def __init__(self, cache: "RedisCache", db: "Database"):
        self.cache = cache
        self.db = db
        self.default_ttl = 3600  # 1小时

    async def get(self, key: str, db_query: Callable) -> Any:
        """读取数据"""
        # 1. 先查缓存
        cached = await self.cache.get(key)
        if cached is not None:
            return cached

        # 2. 缓存未命中，查数据库
        data = await db_query()

        # 3. 写入缓存（设置 TTL）
        if data is not None:
            await self.cache.set(key, data, ttl=self.default_ttl)

        return data

    async def put(self, key: str, data: Any, db_update: Callable) -> None:
        """更新数据"""
        # 1. 先更新数据库
        await db_update(data)

        # 2. 再删除缓存（不是更新！）
        await self.cache.delete(key)

    async def put_with_delay(self, key: str, data: Any,
                              db_update: Callable, delay: float = 0.5) -> None:
        """延迟双删 - 解决主从延迟问题"""
        await db_update(data)
        await self.cache.delete(key)
        await asyncio.sleep(delay)  # 等待主从同步
        await self.cache.delete(key)  # 再次删除
```

### Read-Through / Write-Through

```
┌────────────────────────────────────────────────────┐
│           Read-Through / Write-Through              │
├────────────────────────────────────────────────────┤
│                                                    │
│  App → Cache → DB     （App 只和缓存交互）          │
│                                                    │
│  Read-Through:                                     │
│    App → 缓存 → 未命中 → 缓存加载DB → 缓存返回       │
│                                                    │
│  Write-Through:                                    │
│    App → 缓存 → 缓存同步写DB → 返回                  │
│                                                    │
│  Write-Behind (异步写回):                            │
│    App → 缓存 → 返回（缓存异步批量写DB）              │
└────────────────────────────────────────────────────┘
```

```python
class ReadThroughCache:
    """Read-Through 缓存"""

    def __init__(self, cache, loader: Callable):
        self.cache = cache
        self.loader = loader

    async def get(self, key: str) -> Any:
        cached = await self.cache.get(key)
        if cached is not None:
            return cached

        # 缓存自己加载数据（对调用者透明）
        data = await self.loader(key)
        if data is not None:
            await self.cache.set(key, data)
        return data


class WriteBehindCache:
    """Write-Behind 异步写回缓存"""

    def __init__(self, cache, db, batch_size: int = 100,
                 flush_interval: float = 5.0):
        self.cache = cache
        self.db = db
        self.batch_size = batch_size
        self.flush_interval = flush_interval
        self._write_buffer: List[tuple] = []
        self._lock = asyncio.Lock()

    async def set(self, key: str, value: Any):
        await self.cache.set(key, value)
        async with self._lock:
            self._write_buffer.append((key, value))
            if len(self._write_buffer) >= self.batch_size:
                await self._flush()

    async def _flush(self):
        """批量写入数据库"""
        if not self._write_buffer:
            return
        batch = self._write_buffer[:]
        self._write_buffer.clear()
        await self.db.batch_upsert(batch)
        print(f"[WriteBehind] Flushed {len(batch)} items to DB")
```

### 策略对比

| 策略 | 一致性 | 复杂度 | 适用场景 |
|------|--------|--------|----------|
| Cache-Aside | 最终一致 | 低 | 通用场景，读多写少 |
| Read/Write-Through | 强一致 | 中 | 对一致性要求高 |
| Write-Behind | 最终一致 | 高 | 写密集型，可容忍延迟 |

## 2. 缓存穿透

**问题**：查询不存在的数据，每次都穿透缓存打到数据库。

```
恶意请求: GET /user/99999999  (用户不存在)

缓存 ──Miss──→ 数据库 ──null──→ 缓存不存储 ──→ 下次还穿透
```

### 解决方案

```python
class PenetrationProtection:
    """缓存穿透防护"""

    def __init__(self, cache, db):
        self.cache = cache
        self.db = db

    # 方案1: 缓存空值
    async def get_with_null_cache(self, key: str, db_query: Callable,
                                   null_ttl: int = 300) -> Any:
        cached = await self.cache.get(key)
        if cached == "NULL":
            return None
        if cached is not None:
            return cached

        data = await db_query()
        if data is None:
            # 缓存空值，短 TTL
            await self.cache.set(key, "NULL", ttl=null_ttl)
        else:
            await self.cache.set(key, data)
        return data

    # 方案2: 布隆过滤器
    async def get_with_bloom_filter(self, key: str, db_query: Callable,
                                     bloom: "BloomFilter") -> Any:
        # 先检查布隆过滤器
        if not bloom.might_contain(key):
            return None  # 一定不存在

        # 可能存在，查缓存和DB
        return await self.get_with_null_cache(key, db_query)


class BloomFilter:
    """布隆过滤器"""

    def __init__(self, size: int = 1000000, hash_count: int = 7):
        self.size = size
        self.hash_count = hash_count
        self.bit_array = [False] * size

    def _hashes(self, key: str) -> List[int]:
        results = []
        for i in range(self.hash_count):
            h = int(hashlib.md5(f"{key}:{i}".encode()).hexdigest(), 16)
            results.append(h % self.size)
        return results

    def add(self, key: str):
        for h in self._hashes(key):
            self.bit_array[h] = True

    def might_contain(self, key: str) -> bool:
        return all(self.bit_array[h] for h in self._hashes(key))
```

## 3. 缓存击穿

**问题**：某个热点 key 过期的瞬间，大量并发请求同时打到数据库。

```
时间线：
  ─────────────────────────────────────────→
         hot_key 过期
              │
    1000个并发请求 ──→ 数据库（被打垮）
```

### 解决方案

```python
import asyncio
from typing import Dict

class BreakdownProtection:
    """缓存击穿防护"""

    def __init__(self, cache, db):
        self.cache = cache
        self.db = db
        self._locks: Dict[str, asyncio.Lock] = {}

    # 方案1: 互斥锁（Mutex Lock）
    async def get_with_mutex(self, key: str, db_query: Callable,
                              ttl: int = 3600) -> Any:
        cached = await self.cache.get(key)
        if cached is not None:
            return cached

        # 获取分布式锁
        lock_key = f"lock:{key}"
        if key not in self._locks:
            self._locks[key] = asyncio.Lock()

        async with self._locks[key]:
            # 双重检查
            cached = await self.cache.get(key)
            if cached is not None:
                return cached

            # 只有一个线程查询数据库
            data = await db_query()
            if data is not None:
                await self.cache.set(key, data, ttl=ttl)
            return data

    # 方案2: 逻辑过期（永不过期 + 逻辑过期时间）
    async def get_with_logical_expiry(self, key: str,
                                       db_query: Callable) -> Any:
        cached = await self.cache.get(key)
        if cached is None:
            # 首次加载
            data = await db_query()
            wrapper = {
                "data": data,
                "expire_at": time.time() + 3600,
            }
            await self.cache.set(key, wrapper, ttl=86400)  # 物理 TTL 很长
            return data

        if time.time() > cached["expire_at"]:
            # 逻辑过期，异步刷新
            asyncio.create_task(self._async_refresh(key, db_query))
            # 返回旧数据
            return cached["data"]

        return cached["data"]

    async def _async_refresh(self, key: str, db_query: Callable):
        data = await db_query()
        wrapper = {
            "data": data,
            "expire_at": time.time() + 3600,
        }
        await self.cache.set(key, wrapper, ttl=86400)
```

## 4. 缓存雪崩

**问题**：大量缓存 key 同时过期，或缓存服务宕机，请求全部打到数据库。

```
场景1: 大量key同时过期
  ────────────────────────────────→ 时间
  设置时 TTL = 3600
  100万个key同时过期 → 数据库QPS暴增 → 数据库崩溃

场景2: 缓存服务宕机
  Redis Cluster 整体不可用 → 所有请求打到数据库
```

### 解决方案

```python
import random

class AvalancheProtection:
    """缓存雪崩防护"""

    def __init__(self, cache, db):
        self.cache = cache
        self.db = db

    # 方案1: TTL 随机化
    async def set_with_jitter(self, key: str, value: Any,
                               base_ttl: int = 3600,
                               jitter_range: int = 300):
        """在基础 TTL 上加随机偏移，避免同时过期"""
        jitter = random.randint(-jitter_range, jitter_range)
        ttl = max(60, base_ttl + jitter)
        await self.cache.set(key, value, ttl=ttl)

    # 方案2: 多级缓存
    async def get_multi_level(self, key: str, db_query: Callable) -> Any:
        """L1 本地缓存 → L2 Redis → L3 DB"""
        # L1: 本地缓存
        local = self._local_cache.get(key)
        if local is not None:
            return local

        # L2: Redis
        cached = await self.cache.get(key)
        if cached is not None:
            self._local_cache.set(key, cached, ttl=60)
            return cached

        # L3: 数据库
        data = await db_query()
        if data is not None:
            await self.cache.set(key, data, ttl=3600)
            self._local_cache.set(key, data, ttl=60)
        return data

    # 方案3: 熔断降级
    async def get_with_circuit_breaker(self, key: str,
                                        db_query: Callable,
                                        breaker: "CircuitBreaker") -> Any:
        if not breaker.allow_request():
            # 熔断状态，返回降级数据
            return self._get_degraded_data(key)

        try:
            cached = await self.cache.get(key)
            if cached is not None:
                breaker.record_success()
                return cached

            data = await db_query()
            breaker.record_success()
            return data
        except Exception as e:
            breaker.record_failure()
            return self._get_degraded_data(key)
```

### Redis 缓存封装

```python
import redis.asyncio as redis
import json
from typing import Any, Optional

class RedisCache:
    """Redis 缓存封装"""

    def __init__(self, redis_url: str = "redis://localhost:6379"):
        self.redis = redis.from_url(redis_url, decode_responses=True)

    async def get(self, key: str) -> Optional[Any]:
        value = await self.redis.get(key)
        if value is None:
            return None
        try:
            return json.loads(value)
        except (json.JSONDecodeError, TypeError):
            return value

    async def set(self, key: str, value: Any, ttl: int = 3600) -> None:
        serialized = json.dumps(value, ensure_ascii=False, default=str)
        await self.redis.set(key, serialized, ex=ttl)

    async def delete(self, key: str) -> None:
        await self.redis.delete(key)

    async def delete_pattern(self, pattern: str) -> int:
        """批量删除匹配模式的 key"""
        count = 0
        async for key in self.redis.scan_iter(match=pattern, count=100):
            await self.redis.delete(key)
            count += 1
        return count

    async def get_or_set(self, key: str, factory: Callable,
                          ttl: int = 3600) -> Any:
        """获取或设置（原子操作）"""
        value = await self.get(key)
        if value is not None:
            return value
        value = await factory()
        await self.set(key, value, ttl=ttl)
        return value

    async def incr_with_ttl(self, key: str, ttl: int = 60) -> int:
        """计数器（用于限流等）"""
        pipe = self.redis.pipeline()
        pipe.incr(key)
        pipe.expire(key, ttl)
        results = await pipe.execute()
        return results[0]
```

### 延迟双删

```python
async def delayed_double_delete(cache, db, key: str, data: Any,
                                 delay: float = 0.5):
    """延迟双删策略"""
    # 1. 先删缓存
    await cache.delete(key)
    # 2. 更新数据库
    await db.update(key, data)
    # 3. 延迟后再删一次（等待主从同步）
    await asyncio.sleep(delay)
    await cache.delete(key)
```

### 基于 Binlog 的异步更新

```python
class BinlogCacheSync:
    """基于 MySQL Binlog 的缓存同步"""

    def __init__(self, cache: RedisCache):
        self.cache = cache

    async def process_binlog_event(self, event: dict):
        """处理 binlog 事件"""
        table = event.get("table")
        action = event.get("action")  # INSERT, UPDATE, DELETE
        row_data = event.get("data")

        if table == "users":
            user_id = row_data.get("id")
            cache_key = f"user:{user_id}"

            if action in ("INSERT", "UPDATE"):
                await self.cache.set(cache_key, row_data)
            elif action == "DELETE":
                await self.cache.delete(cache_key)
```

## 最佳实践

### ✅ 推荐

1. **设置合理的 TTL** - 避免缓存永不过期，建议分钟到小时级别
2. **缓存预热** - 系统启动时加载热点数据到缓存
3. **热点探测** - 实时发现并处理热点 key
4. **多级缓存** - L1 本地 + L2 分布式，逐级兜底
5. **监控缓存命中率** - 低于 80% 需要排查
6. **序列化优化** - 使用 Protobuf/MessagePack 替代 JSON
7. **大 key 拆分** - 单个 value 不超过 10KB

### ❌ 避免

1. **避免缓存穿透** - 布隆过滤器 + 空值缓存
2. **避免缓存击穿** - 互斥锁 + 逻辑过期
3. **避免缓存雪崩** - TTL 随机化 + 多级缓存 + 熔断降级
4. **避免更新缓存而非删除** - 更新缓存在并发下容易出现不一致
