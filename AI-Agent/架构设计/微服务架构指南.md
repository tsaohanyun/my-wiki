# 微服务架构指南

## 概述

本指南提供微服务架构的设计方法和最佳实践。

## 1. 微服务原则

### 单一职责

```
每个服务负责一个独立的业务功能
- 用户服务：用户管理
- 订单服务：订单处理
- 支付服务：支付处理
- 库存服务：库存管理
```

### 服务自治

```
每个服务独立部署、独立数据库
- 独立开发
- 独立测试
- 独立部署
- 独立扩展
```

### 轻量通信

```
服务间使用轻量级协议
- REST API
- gRPC
- 消息队列
```

## 2. 服务拆分

### 拆分策略

```
按业务领域拆分
- 用户域：用户、认证、授权
- 订单域：订单、支付、物流
- 商品域：商品、分类、库存
```

### 拆分原则

```
1. 高内聚、低耦合
2. 服务边界清晰
3. 数据独立
4. 团队自治
```

### 拆分示例

```
单体应用
├── 用户模块
├── 订单模块
├── 商品模块
└── 支付模块

微服务
├── 用户服务
├── 订单服务
├── 商品服务
└── 支付服务
```

### 异步通信

```python
# 消息队列
import pika

# 生产者
def publish_message(queue, message):
    connection = pika.BlockingConnection(pika.ConnectionParameters('rabbitmq'))
    channel = connection.channel()
    channel.queue_declare(queue=queue)
    channel.basic_publish(exchange='', routing_key=queue, body=message)
    connection.close()

# 消费者
def consume_messages(queue, callback):
    connection = pika.BlockingConnection(pika.ConnectionParameters('rabbitmq'))
    channel = connection.channel()
    channel.queue_declare(queue=queue)
    channel.basic_consume(queue=queue, on_message_callback=callback)
    channel.start_consuming()
```

    c = consul.Consul(host='consul', port=8500)

## 7. 服务熔断

### 熔断器模式

```python
# Hystrix
from hystrix import HystrixCommand

class GetUserCommand(HystrixCommand):
    def __init__(self, user_id):
        super().__init__(group_key='UserService')
        self.user_id = user_id
    
    def run(self):
        response = requests.get(f'http://user-service/api/users/{self.user_id}')
        return response.json()
    
    def fallback(self):
        return {'id': self.user_id, 'name': 'Unknown'}

# 使用
command = GetUserCommand(123)
user = command.execute()
```

### 限流

```python
# 令牌桶算法
import time

class TokenBucket:
    def __init__(self, rate, capacity):
        self.rate = rate
        self.capacity = capacity
        self.tokens = capacity
        self.last_time = time.time()
    
    def consume(self):
        now = time.time()
        self.tokens = min(self.capacity, self.tokens + (now - self.last_time) * self.rate)
        self.last_time = now
        
        if self.tokens >= 1:
            self.tokens -= 1
            return True
        return False
```

## 8. 分布式事务

### Saga模式

```python
# Saga协调器
class OrderSaga:
    def __init__(self):
        self.steps = [
            CreateOrderStep(),
            ReserveInventoryStep(),
            ProcessPaymentStep(),
            ConfirmOrderStep()
        ]
    
    def execute(self, order_data):
        completed_steps = []
        
        for step in self.steps:
            try:
                step.execute(order_data)
                completed_steps.append(step)
            except Exception as e:
                # 补偿已完成的步骤
                for completed_step in reversed(completed_steps):
                    completed_step.compensate(order_data)
                raise e
```
