# 安全开发生命周期

## 概述

安全开发生命周期（SDL）是将安全实践集成到软件开发每个阶段的框架，从需求分析到部署运维，确保安全性贯穿整个产品生命周期。

---

## SDL阶段概览

```
需求 → 设计 → 实现 → 验证 → 发布 → 响应
 │      │      │      │      │      │
安全   威胁   安全   安全   安全   事件
需求   建模   编码   测试   审查   响应
```

| 阶段 | 关键活动 | 交付物 |
|------|----------|--------|
| 需求 | 安全需求分析、合规要求 | 安全需求文档 |
| 设计 | 威胁建模、安全架构 | 威胁模型、安全设计 |
| 实现 | 安全编码、代码审查 | 安全代码、审查报告 |
| 验证 | 安全测试、渗透测试 | 测试报告、漏洞列表 |
| 发布 | 安全审查、最终检查 | 发布审批 |
| 响应 | 事件响应、漏洞修复 | 响应计划、补丁 |

## 威胁建模

### STRIDE威胁模型

| 威胁类型 | 描述 | 安全属性 | 示例 |
|----------|------|----------|------|
| **S**poofing | 身份伪造 | 认证 | 冒充用户登录 |
| **T**ampering | 数据篡改 | 完整性 | 修改传输数据 |
| **R**epudiation | 抵赖 | 不可否认性 | 否认执行操作 |
| **I**nformation Disclosure | 信息泄露 | 机密性 | 敏感数据泄露 |
| **D**enial of Service | 拒绝服务 | 可用性 | 服务瘫痪 |
| **E**levation of Privilege | 权限提升 | 授权 | 普通用户获取管理员权限 |

### 威胁建模流程

```python
# 威胁建模自动化工具示例
from dataclasses import dataclass
from enum import Enum

class ThreatType(Enum):
    SPOOFING = "Spoofing"
    TAMPERING = "Tampering"
    REPUDIATION = "Repudiation"
    INFO_DISCLOSURE = "Information Disclosure"
    DOS = "Denial of Service"
    ELEVATION = "Elevation of Privilege"

@dataclass
class Threat:
    id: str
    type: ThreatType
    description: str
    component: str
    stride_category: str
    risk_rating: str  # High/Medium/Low
    mitigation: str
    status: str  # Open/Mitigated/Accepted

class ThreatModel:
    def __init__(self, system_name: str):
        self.system_name = system_name
        self.threats: list[Threat] = []
        self.entry_points: list[str] = []
        self.assets: list[str] = []

    def add_entry_point(self, entry: str):
        self.entry_points.append(entry)

    def add_asset(self, asset: str):
        self.assets.append(asset)

    def analyze_threats(self):
        """基于STRIDE自动分析威胁"""
        for entry in self.entry_points:
            for threat_type in ThreatType:
                threat = Threat(
                    id=f"THREAT-{len(self.threats)+1:04d}",
                    type=threat_type,
                    description=self._generate_description(entry, threat_type),
                    component=entry,
                    stride_category=threat_type.value,
                    risk_rating=self._assess_risk(entry, threat_type),
                    mitigation=self._suggest_mitigation(threat_type),
                    status="Open"
                )
                self.threats.append(threat)

    def _generate_description(self, entry: str, threat_type: ThreatType) -> str:
        templates = {
            ThreatType.SPOOFING: f"攻击者可能伪造{entry}的身份",
            ThreatType.TAMPERING: f"攻击者可能篡改{entry}的数据",
            ThreatType.INFO_DISCLOSURE: f"{entry}可能泄露敏感信息",
        }
        return templates.get(threat_type, f"{entry}存在{threat_type.value}风险")

    def _assess_risk(self, entry: str, threat_type: ThreatType) -> str:
        # 简化的风险评估逻辑
        if "external" in entry.lower() or "public" in entry.lower():
            return "High"
        return "Medium"

    def _suggest_mitigation(self, threat_type: ThreatType) -> str:
        mitigations = {
            ThreatType.SPOOFING: "实施强认证机制（MFA、证书）",
            ThreatType.TAMPERING: "使用数字签名和完整性校验",
            ThreatType.INFO_DISCLOSURE: "加密敏感数据，实施访问控制",
        }
        return mitigations.get(threat_type, "实施对应安全控制")

# 使用示例
model = ThreatModel("E-Commerce Platform")
model.add_entry_point("Web前端 (外部)")
model.add_entry_point("REST API (外部)")
model.add_entry_point("数据库 (内部)")
model.add_asset("用户数据")
model.add_asset("支付信息")

model.analyze_threats()

# 生成报告
for threat in model.threats:
    print(f"[{threat.risk_rating}] {threat.type.value}: {threat.description}")
```

## 安全编码实践

### 输入验证

```python
# 使用Pydantic进行严格的输入验证
from pydantic import BaseModel, Field, validator, constr
from typing import Optional
import re

class SecureUserInput(BaseModel):
    username: constr(min_length=3, max_length=50) = Field(..., pattern=r'^[a-zA-Z0-9_]+$')
    email: EmailStr
    age: int = Field(ge=0, le=150)
    phone: Optional[str] = None

    @validator('phone')
    def validate_phone(cls, v):
        if v and not re.match(r'^\+?[1-9]\d{1,14}$', v):
            raise ValueError('Invalid phone number format')
        return v

    @validator('username')
    def no_sql_injection(cls, v):
        dangerous_chars = ["'", '"', ';', '--', '/*', '*/', 'xp_', 'DROP', 'DELETE']
        for char in dangerous_chars:
            if char.lower() in v.lower():
                raise ValueError(f'Invalid character in username: {char}')
        return v
```

### 安全的反序列化

```python
# 安全的JSON反序列化
import json
from typing import Any

def safe_json_loads(data: str, max_size: int = 1_000_000) -> Any:
    """安全的JSON反序列化，防止DoS"""
    if len(data) > max_size:
        raise ValueError(f"JSON data exceeds maximum size of {max_size} bytes")

    try:
        return json.loads(data)
    except json.JSONDecodeError as e:
        raise ValueError(f"Invalid JSON: {e}")

# 安全的YAML加载（防止代码执行）
import yaml

def safe_yaml_load(data: str) -> Any:
    """安全的YAML加载，禁止任意Python对象"""
    return yaml.safe_load(data)  # 不使用yaml.load()
```

## 漏洞管理

### 漏洞严重性评估（CVSS）

```python
# CVSS v3.1评分计算
from enum import Enum

class AttackVector(Enum):
    NETWORK = 0.85
    ADJACENT = 0.62
    LOCAL = 0.55
    PHYSICAL = 0.2

class AttackComplexity(Enum):
    LOW = 0.77
    HIGH = 0.44

class CVSSCalculator:
    def __init__(self, av: AttackVector, ac: AttackComplexity,
                 pr: str, ui: str, s: str,
                 c: str, i: str, a: str):
        self.av = av
        self.ac = ac
        self.pr = pr  # None/Low/High
        self.ui = ui  # None/Required
        self.s = s    # Unchanged/Changed
        self.c = c    # None/Low/High
        self.i = i    # None/Low/High
        self.a = a    # None/Low/High

    def calculate(self) -> dict:
        # 简化的CVSS计算
        impact_scores = {'None': 0, 'Low': 0.22, 'High': 0.56}
        exploitability = self.av.value * self.ac.value

        c_score = impact_scores[self.c]
        i_score = impact_scores[self.i]
        a_score = impact_scores[self.a]

        impact = 1 - ((1 - c_score) * (1 - i_score) * (1 - a_score))
        base_score = min(10, round(exploitability * impact * 10, 1))

        severity = "Critical" if base_score >= 9.0 else \
                   "High" if base_score >= 7.0 else \
                   "Medium" if base_score >= 4.0 else "Low"

        return {
            "base_score": base_score,
            "severity": severity,
            "vector": f"CVSS:3.1/AV:{self.av.name[0]}/AC:{self.ac.name[0]}/..."
        }

# 使用
calc = CVSSCalculator(
    av=AttackVector.NETWORK,
    ac=AttackComplexity.LOW,
    pr="None", ui="None", s="Unchanged",
    c="High", i="High", a="High"
)
result = calc.calculate()
print(f"Score: {result['base_score']} ({result['severity']})")
```

### 漏洞响应SLA

| 严重性 | CVSS分数 | 响应时间 | 修复时间 | 示例 |
|--------|----------|----------|----------|------|
| 🔴 Critical | 9.0-10.0 | 4小时 | 24小时 | 远程代码执行 |
| 🟠 High | 7.0-8.9 | 24小时 | 7天 | SQL注入、认证绕过 |

## CI/CD安全集成

### 完整安全流水线

```yaml
# .github/workflows/secure-pipeline.yml
name: Secure CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  # 1. 依赖扫描
  dependency-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check dependencies
        run: |
          pip install safety
          safety check --full-report

  # 2. SAST扫描
  sast:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run CodeQL
        uses: github/codeql-action/analyze@v2
        with:
          languages: python

  # 3. 密钥扫描
  secrets-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Run Gitleaks
        uses: gitleaks/gitleaks-action@v2

  # 4. 容器扫描
  container-scan:
    needs: [sast]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build and scan
        run: |
          docker build -t app:${{ github.sha }} .
          trivy image --exit-code 1 --severity HIGH,CRITICAL app:${{ github.sha }}

  # 5. 部署（仅通过所有检查后）
  deploy:
    needs: [dependency-check, sast, secrets-scan, container-scan]
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to staging
        run: echo "Deploying..."
```
