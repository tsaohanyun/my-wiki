## 1. 元数据治理概述

### 1.1 什么是元数据治理？

```
┌─────────────────────────────────────────────────────────┐
│                   元数据治理体系                          │
├───────────────┬─────────────────────────────────────────┤
│ 技术元数据     │ 表名、列名、数据类型、分区策略           │
│ (Technical)   │ ETL 调度信息、管道依赖、执行日志         │
├───────────────┼─────────────────────────────────────────┤
│ 业务元数据     │ 业务术语表、指标定义、数据所有者         │
│ (Business)    │ 分类标签、敏感等级、合规要求             │
├───────────────┼─────────────────────────────────────────┤
│ 操作元数据     │ 数据新鲜度、查询频率、使用热度           │
│ (Operational) │ 数据质量评分、SLA状态、告警历史          │
├───────────────┼─────────────────────────────────────────┤
│ 血缘元数据     │ 上游/下游依赖、转换逻辑、字段映射       │
│ (Lineage)     │ 影响分析、根因分析                       │
└───────────────┴─────────────────────────────────────────┘
```

### 1.2 治理目标

| 目标             | 说明                                     |
| ---------------- | ---------------------------------------- |
| **可发现性**     | 数据消费者能快速找到所需数据             |
| **可信任性**     | 了解数据来源、质量、所有者               |
| **可追溯性**     | 追踪数据从源头到报表的完整路径           |
| **合规性**       | 敏感数据标记、访问审计、数据保留策略     |
| **影响分析**     | 修改上游表前评估下游影响                 |

### 2.5 Python SDK 注册元数据

```python
import datahub.emitter.mce_builder as builder
from datahub.emitter.mcp import MetadataChangeProposalWrapper
from datahub.metadata.schema_classes import (
    DatasetPropertiesClass,
    SchemaMetadataClass,
    UpstreamLineageClass,
    UpstreamClass,
    DatasetLineageTypeClass,
)
from datahub.emitter.rest_emitter import DatahubRestEmitter

emitter = DatahubRestEmitter(gms_server="http://localhost:8080")

# ── 注册数据集属性 ──
dataset_urn = builder.make_dataset_urn("snowflake", "ANALYTICS.FCT_ORDERS")

dataset_properties = MetadataChangeProposalWrapper(
    entityUrn=dataset_urn,
    aspect=DatasetPropertiesClass(
        description="订单事实表，每日刷新",
        tags=["sensitive", "pii"],
        customProperties={
            "owner": "data-platform-team",
            "sla": "daily_02:00",
            "retention": "7_years",
        },
    ),
)
emitter.emit_mcp(dataset_properties)

# ── 注册血缘关系 ──
# fct_orders ← stg_orders ← raw.orders
upstream_urn = builder.make_dataset_urn("snowflake", "RAW.ORDERS")
downstream_urn = builder.make_dataset_urn("snowflake", "ANALYTICS.FCT_ORDERS")

lineage = MetadataChangeProposalWrapper(
    entityUrn=downstream_urn,
    aspect=UpstreamLineageClass(
        upstreams=[
            UpstreamClass(
                auditStamp=builder.make_audit_stamp("datahub"),
                dataset=upstream_urn,
                type=DatasetLineageTypeClass.TRANSFORMED,
            )
        ]
    ),
)
emitter.emit_mcp(lineage)

print("✅ 元数据已注册到 DataHub")
```

### 3.2 Databuilder 抽取示例

```python
# amundsen_databuilder/snowflake_extractor.py
from pyhocon import ConfigFactory
from databuilder.extractor.sql_alchemy_extractor import SQLAlchemyExtractor
from databuilder.job.job import DefaultJob
from databuilder.loader.file_system_neo4j_csv_loader import FsNeo4jCSVLoader
from databuilder.publisher import neo4j_csv_publisher
from databuilder.task.task import DefaultTask

def create_snowflake_extraction_job():
    """创建 Snowflake 元数据抽取 Job"""
    sql_alchemy_extractor = SQLAlchemyExtractor()
    csv_loader = FsNeo4jCSVLoader()

    task = DefaultTask(
        extractor=sql_alchemy_extractor,
        loader=csv_loader,
    )

    job_config = ConfigFactory.from_dict({
        # Extractor 配置
        f"{sql_alchemy_extractor.extractor_src_type}.conn_string":
            f"snowflake://{SNOWFLAKE_USER}:{SNOWFLAKE_PASSWORD}@"
            f"{SNOWFLAKE_ACCOUNT}/ANALYTICS/ANALYTICS?warehouse=ETL_WH",

        f"{sql_alchemy_extractor.extractor_src_type}.extract_sql":
            """
            SELECT
                TABLE_CATALOG as database_name,
                TABLE_SCHEMA as schema_name,
                TABLE_NAME as table_name,
                COLUMN_NAME as column_name,
                DATA_TYPE as column_type,
                ORDINAL_POSITION as column_position
            FROM INFORMATION_SCHEMA.COLUMNS
            WHERE TABLE_SCHEMA IN ('ANALYTICS', 'RAW', 'STAGING')
            """,

        # Loader 配置
        f"{csv_loader.get_scope()}.NEO4J_DESTINATION_FOLDER":
            "/tmp/amundsen/output",
        f"{csv_loader.get_scope()}.FILE_PATH":
            "/tmp/amundsen/nodes",

        # Publisher 配置
        f"neo4j_endpoint": "bolt://neo4j:7687",
        f"neo4j_user": "neo4j",
        f"neo4j_password": "amundsen",
        f"neo4j_csv_publisher.neo4j_csv_dir":
            "/tmp/amundsen/output",
    })

    job = DefaultJob(
        conf=job_config,
        task=task,
        publisher=neo4j_csv_publisher.Neo4jCsvPublisher(),
    )
    return job

# 执行
job = create_snowflake_extraction_job()
job.launch()
```

### 3.3 Amundsen 适用场景

## 5. 数据血缘追踪

### 5.1 血缘类型

```
┌───────────────────────────────────────────────────┐
│                  血缘层级                           │
│                                                    │
│  系统级    Source DB → ETL → Warehouse → BI Tool  │
│  表级      raw.orders → stg_orders → fct_orders    │
│  列级      raw.orders.id → fct_orders.order_id     │
│  作业级    Airflow DAG → dbt Model → Table         │
└───────────────────────────────────────────────────┘
```

### 5.2 基于 SQL 解析的自动血缘

```python
import sqlglot
from sqlglot import exp

def extract_lineage_from_sql(sql: str) -> dict:
    """从 SQL 中提取表级血缘"""
    parsed = sqlglot.parse_one(sql)

    # 目标表（INSERT INTO / CREATE TABLE AS）
    target_tables = []
    if isinstance(parsed, exp.Insert):
        target_tables = [parsed.this.this.name]  # INSERT target
    elif isinstance(parsed, exp.Create):
        target_tables = [parsed.this.this.name]

    # 源表（FROM / JOIN 中的表）
    source_tables = set()
    for table in parsed.find_all(exp.Table):
        source_tables.add(table.name)

    # 列级映射（简化版）
    column_mapping = {}
    for select in parsed.find_all(exp.Select):
        for col in select.expressions:
            if isinstance(col, exp.Alias):
                target_col = col.alias
                source_cols = [c.name for c in col.find_all(exp.Column)]
                column_mapping[target_col] = source_cols

    return {
        "sources": list(source_tables),
        "targets": target_tables,
        "column_mapping": column_mapping,
    }

# 使用示例
sql = """
INSERT INTO analytics.fct_orders
SELECT
    o.order_id,
    o.customer_id,
    o.amount + o.tax AS total_amount
FROM raw.orders o
JOIN raw.customers c ON o.customer_id = c.customer_id
"""

lineage = extract_lineage_from_sql(sql)
print(lineage)
# {'sources': ['orders', 'customers'],
#  'targets': ['fct_orders'],
#  'column_mapping': {'total_amount': ['amount', 'tax']}}
```

### 5.3 血缘可视化查询

```python
# OpenMetadata 血缘查询 API
def get_downstream_lineage(table_id: str, depth: int = 3) -> dict:
    """查询下游影响"""
    response = requests.get(
        f"http://localhost:8585/api/v1/lineage/table/{table_id}",
        params={"upstream": "false", "depth": depth}
    )
    return response.json()

def impact_analysis(table_name: str) -> list:
    """影响分析：修改此表会影响哪些下游"""
    lineage = get_downstream_lineage(table_name, depth=5)
    affected = []
    for node in lineage.get("nodes", []):
        affected.append({
            "entity": node["name"],
            "type": node["type"],
            "depth": node["depth"],
        })
    return affected
```

### 5.4 根因分析

```python
def root_cause_analysis(
    broken_table: str,
    metadata_client
) -> list:
    """上游根因分析：数据异常的源头"""
    upstream = metadata_client.get_upstream_lineage(broken_table, depth=10)

    root_causes = []
    for node in upstream.get("nodes", []):
        quality = metadata_client.get_quality_score(node["id"])
        freshness = metadata_client.get_freshness(node["id"])

        if quality < 0.8 or freshness > 24:
            root_causes.append({
                "table": node["name"],
                "quality_score": quality,
                "freshness_hours": freshness,
                "depth": node["depth"],
            })

    return sorted(root_causes, key=lambda x: x["depth"])
```

## 6. 平台对比与选型

| 特性            | DataHub               | Amundsen            | OpenMetadata          |
| --------------- | --------------------- | ------------------- | --------------------- |
| **维护状态**    | 活跃（LinkedIn/Acado）| 低活跃               | 活跃（Collate）       |
| **血缘**        | 表级 + 列级（强）     | 表级（弱）           | 表级 + 列级（强）     |
| **数据质量**    | 集成 GE/Soda           | 无内置               | 内置测试框架          |
| **UI/UX**       | 优秀                  | 基础                 | 优秀                  |
| **API**         | REST + GraphQL        | REST                 | REST + GraphQL + SDK  |
| **部署复杂度**  | 中（多组件）           | 低                   | 中（Docker Compose）  |
| **社区**        | 大                    | 中                   | 增长快                |
| **推荐场景**    | 企业级、大规模         | 轻量目录             | 中型团队、一站式       |

### 选型建议

```
团队规模 < 20人     → OpenMetadata（一站式，快速上手）
团队规模 20~100人   → DataHub（企业级，扩展性强）
仅需搜索 + 文档      → Amundsen（轻量）
需要血缘 + 质量 + 告警 → DataHub 或 OpenMetadata
```

## 7. 治理实践

### 7.1 数据所有者制度

```python
# 自动分配数据所有者
DATA_OWNERSHIP = {
    "analytics.fct_orders": {"owner": "data-platform", "team": "analytics"},
    "analytics.dim_customers": {"owner": "crm-team", "team": "growth"},
    "raw.events": {"owner": "data-platform", "team": "ingestion"},
}

def assign_ownership(metadata_client):
    for table, ownership in DATA_OWNERSHIP.items():
        metadata_client.update_table_owner(
            table_name=table,
            owner=ownership["owner"],
            team=ownership["team"]
        )
```

### 7.2 敏感数据分类

```yaml
# 数据分类策略
classification:
  PII:
    patterns:
      - ".*email.*"
      - ".*phone.*"
      - ".*ssn.*"
```

### 7.3 数据保留策略

```python
RETENTION_POLICY = {
    "raw.events": {"retention_days": 90, "action": "archive"},
    "analytics.fct_orders": {"retention_days": 2555, "action": "keep"},  # 7年
    "temp.*": {"retention_days": 7, "action": "delete"},
}

def enforce_retention():
    for table, policy in RETENTION_POLICY.items():
        cutoff_date = datetime.now() - timedelta(days=policy["retention_days"])
        if policy["action"] == "delete":
            execute(f"DELETE FROM {table} WHERE created_at < '{cutoff_date}'")
        elif policy["action"] == "archive":
            archive_table(table, cutoff_date)
```

### 7.4 关键实践清单

- ✅ **自动化摄取**：定期（每日/每小时）自动同步元数据
- ✅ **CI/CD 集成**：dbt 模型变更时自动更新血缘
- ✅ **搜索优化**：建立业务术语到技术表的映射
- ✅ **访问审计**：记录谁在何时查询了什么数据
- ✅ **定期清理**：标记并归档不再使用的表
- ✅ **文档驱动**：新表上线前必须注册元数据
- ✅ **KPI 监控**：元数据覆盖率、文档完整率作为团队 KPI
