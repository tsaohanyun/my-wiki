## 1. 建模方法论概述

### 1.1 Kimball vs Inmon

```
┌──────────────────────────────────────────────────────────────┐
│                     Inmon (范式驱动)                          │
│                                                              │
│  源系统 ──▶ 3NF 企业数据仓库 ──▶ 数据集市 (星型)             │
│           (自上而下，统一模型)     (面向部门)                 │
├──────────────────────────────────────────────────────────────┤
│                     Kimball (维度驱动)                        │
│                                                              │
│  源系统 ──▶ 数据集市 (总线架构)                              │
│           (自下而上，一致性维度)                              │
│           (面向业务流程，星型模型)                            │
└──────────────────────────────────────────────────────────────┘
```

| 特性        | Kimball                    | Inmon                     |
| ----------- | -------------------------- | ------------------------- |
| 核心思想    | 维度建模、总线架构          | 范式化企业仓库 + 部门集市 |
| 模型类型    | 星型 / 雪花型               | 第三范式（3NF）            |
| 起步方式    | 自下而上（单业务流程）       | 自上而下（全局模型）       |
| 实施周期    | 短（快速交付）               | 长（全局规划）             |
| 查询性能    | 优秀（反范式化）             | 一般（需要 JOIN）          |
| 推荐场景    | 中小团队、敏捷交付           | 大型企业、强一致性要求     |

> **实践建议**：大多数团队采用 **Kimball 维度建模**，以业务流程为核心快速交付。

### 1.2 数据仓库分层架构

```
┌─────────────────────────────────────────────────┐
│            Serving Layer (BI / ML / API)        │
├─────────────────────────────────────────────────┤
│  Mart Layer (面向主题域的维度模型)               │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │  Sales   │ │ Finance  │ │ Marketing│        │
│  │  Mart    │ │  Mart    │ │  Mart    │        │
│  └──────────┘ └──────────┘ └──────────┘        │
├─────────────────────────────────────────────────┤
│  Aggregation Layer (预计算汇总层)               │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │  Daily   │ │ Monthly  │ │ Cohort   │        │
│  │  Stats   │ │  Stats   │ │  Stats   │        │
│  └──────────┘ └──────────┘ └──────────┘        │
├─────────────────────────────────────────────────┤
│  Staging Layer (清洗、标准化、1:1 映射)          │
├─────────────────────────────────────────────────┤
│  Raw Layer (原始数据落地，不修改)                │
├─────────────────────────────────────────────────┤
│  Source Systems (DB / API / Files / Streams)    │
└─────────────────────────────────────────────────┘
```

## 2. 维度建模（Dimensional Modeling）

### 2.1 星型模型 vs 雪花型模型

```
─── 星型模型（Star Schema）───

              dim_customers
                   │
dim_products ── fct_sales ── dim_stores
                   │
              dim_date

─── 雪花型模型（Snowflake Schema）───

         dim_customers ── dim_customer_segments
              │
dim_products ── fct_sales ── dim_stores ── dim_regions
              │
         dim_date
```

| 特性       | 星型模型                | 雪花型模型              |
| ---------- | ----------------------- | ----------------------- |
| JOIN 数量  | 少（1 层）              | 多（多层）              |
| 查询性能   | 快                      | 较慢                    |
| 存储冗余   | 高（反范式化）           | 低（范式化）             |
| 维护复杂度 | 低                      | 高                      |
| 推荐场景   | **OLAP 分析（首选）**   | 复杂层级关系             |

### 2.2 维度表设计

```sql
-- ── 日期维度 ──
CREATE TABLE dim_date (
    date_key          INT PRIMARY KEY,        -- 20260628 (业务键)
    full_date         DATE NOT NULL,
    day_of_week       VARCHAR(10),            -- Monday
    day_number        SMALLINT,               -- 1-7
    day_of_month      SMALLINT,               -- 1-31
    day_of_year       SMALLINT,               -- 1-366
    week_of_year      SMALLINT,               -- 1-53
    month_number      SMALLINT,               -- 1-12
    month_name        VARCHAR(10),            -- January
    quarter           SMALLINT,               -- 1-4
    year              SMALLINT,
    is_weekend        BOOLEAN,
    is_holiday        BOOLEAN,
    holiday_name      VARCHAR(50)
);

-- 生成日期维度数据
INSERT INTO dim_date
SELECT
    TO_NUMBER(TO_CHAR(d, 'YYYYMMDD')) AS date_key,
    d AS full_date,
    TO_CHAR(d, 'Day') AS day_of_week,
    EXTRACT(DOW FROM d)::SMALLINT AS day_number,
    EXTRACT(DAY FROM d)::SMALLINT AS day_of_month,
    EXTRACT(DOY FROM d)::SMALLINT AS day_of_year,
    EXTRACT(WEEK FROM d)::SMALLINT AS week_of_year,
    EXTRACT(MONTH FROM d)::SMALLINT AS month_number,
    TO_CHAR(d, 'Month') AS month_name,
    EXTRACT(QUARTER FROM d)::SMALLINT AS quarter,
    EXTRACT(YEAR FROM d)::SMALLINT AS year,
    EXTRACT(DOW FROM d) IN (0, 6) AS is_weekend,
    FALSE AS is_holiday,
    NULL AS holiday_name
FROM generate_series('2020-01-01'::DATE, '2030-12-31'::DATE, '1 day') AS d;
```

```sql
-- ── 客户维度（带 SCD2 支持）──
CREATE TABLE dim_customers (
    -- 代理键
    customer_key      BIGINT PRIMARY KEY,     -- 代理键（自增）
    customer_nk       VARCHAR(50) NOT NULL,   -- 自然键（业务ID）

    -- 业务属性
    customer_name     VARCHAR(200),
    email             VARCHAR(200),
    phone             VARCHAR(50),
    address           VARCHAR(500),
    city              VARCHAR(100),
    country           VARCHAR(100),
    customer_segment  VARCHAR(50),            -- VIP / Regular / New

    -- SCD2 审计字段
    effective_from    TIMESTAMP NOT NULL,
    effective_to      TIMESTAMP DEFAULT '9999-12-31',
    is_current        BOOLEAN DEFAULT TRUE,
    version_number    INT DEFAULT 1,

    -- 数据审计
    created_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

CREATE INDEX idx_dim_customers_nk ON dim_customers(customer_nk);
CREATE INDEX idx_dim_customers_current ON dim_customers(customer_nk) WHERE is_current = TRUE;
```

### 2.3 一致性维度

```sql
-- 一致性维度：跨数据集市共享
-- 确保所有事实表使用相同的客户维度定义

-- Sales Mart
CREATE TABLE marts_sales.fct_orders (
    order_key       BIGINT PRIMARY KEY,
    date_key        INT REFERENCES dim_date(date_key),
    customer_key    BIGINT REFERENCES dim_customers(customer_key),
    product_key     BIGINT REFERENCES dim_products(product_key),
    order_amount    DECIMAL(18,2),
    quantity        INT
);

-- Marketing Mart
CREATE TABLE marts_marketing.fct_campaigns (
    campaign_key    BIGINT PRIMARY KEY,
    date_key        INT REFERENCES dim_date(date_key),        -- ← 同一日期维度
    customer_key    BIGINT REFERENCES dim_customers(customer_key), -- ← 同一客户维度
    impressions     INT,
    clicks          INT,
    cost            DECIMAL(18,2)
);

-- 跨数据集市分析（通过一致性维度 JOIN）
SELECT
    c.customer_segment,
    SUM(o.order_amount) AS total_revenue,
    SUM(cmp.clicks) AS total_clicks,
    SUM(o.order_amount) / NULLIF(SUM(cmp.cost), 0) AS roas
FROM dim_customers c
JOIN marts_sales.fct_orders o ON c.customer_key = o.customer_key
JOIN marts_marketing.fct_campaigns cmp ON c.customer_key = cmp.customer_key
WHERE c.is_current = TRUE
GROUP BY c.customer_segment;
```

## 3. 缓慢变化维度（SCD）

### 3.1 SCD 类型总览

| 类型   | 说明                           | 适用场景                   |
| ------ | ------------------------------ | -------------------------- |
| SCD 0  | 不变（保留初始值）              | 不可变属性（出生日期）      |
| SCD 1  | 覆盖更新（只保留最新值）        | 修正错误、低价值字段        |
| SCD 2  | 历史版本（新增行，保留全历史）  | **最常用**，高价值属性      |
| SCD 3  | 有限历史（新增列）              | 只需"前值"的场景            |
| SCD 4  | 历史移至独立表                  | 历史记录非常多时            |
| SCD 6  | SCD1 + SCD2 + SCD3 混合        | 复杂需求                    |

### 3.2 SCD Type 2 实现

```sql
-- ── SCD Type 2: MERGE 方式 ──
-- 当源数据中的客户属性发生变化时：
--   1. 关闭旧记录（effective_to = now, is_current = false）
--   2. 插入新记录（新版本）

MERGE INTO dim_customers AS target
USING (
    -- 从源系统获取最新数据
    SELECT
        s.customer_id AS customer_nk,
        s.customer_name,
        s.email,
        s.phone,
        s.address,
        s.city,
        s.country,
        s.customer_segment,
        s.updated_at
    FROM staging.stg_customers s
) AS source
ON target.customer_nk = source.customer_nk AND target.is_current = TRUE

-- 属性发生变化时
WHEN MATCHED AND (
    target.customer_name != source.customer_name OR
    target.customer_segment != source.customer_segment OR
    target.address != source.address
    -- ... 其他需要追踪的属性
) THEN
    -- 关闭旧记录
    UPDATE SET
        effective_to = source.updated_at,
        is_current = FALSE,
        updated_at = CURRENT_TIMESTAMP

-- 新客户
WHEN NOT MATCHED THEN
    INSERT (
        customer_key,
        customer_nk,
        customer_name,
        email,
        phone,
        address,
        city,
        country,
        customer_segment,
        effective_from,
        effective_to,
        is_current,
        version_number,
        source_system
    )
    VALUES (
        NEXTVAL('seq_customer_key'),
        source.customer_nk,
        source.customer_name,
        source.email,
        source.phone,
        source.address,
        source.city,
        source.country,
        source.customer_segment,
        source.updated_at,
        '9999-12-31'::TIMESTAMP,
        TRUE,
        1,
        'CRM'
    );
```

```sql
-- ── SCD Type 2: 插入新版本 ──
-- 在 MERGE 关闭旧记录后，需要插入变更后的新版本

INSERT INTO dim_customers (
    customer_key,
    customer_nk,
    customer_name,
    email,
    phone,
    address,
    city,
    country,
    customer_segment,
    effective_from,
    effective_to,
    is_current,
    version_number,
    source_system
)
SELECT
    NEXTVAL('seq_customer_key'),
    s.customer_id,
    s.customer_name,
    s.email,
    s.phone,
    s.address,
    s.city,
    s.country,
    s.customer_segment,
    s.updated_at,
    '9999-12-31'::TIMESTAMP,
    TRUE,
    d.version_number + 1,
    'CRM'
FROM staging.stg_customers s
JOIN dim_customers d
    ON d.customer_nk = s.customer_id
    AND d.is_current = TRUE
    AND (
        d.customer_name != s.customer_name OR
        d.customer_segment != s.customer_segment OR
        d.address != s.address
    );
```

## 4. 事实表设计

### 4.1 事实表类型

```
┌─────────────────────────────────────────────────────────┐
│                    事实表类型                             │
├───────────────┬─────────────────────────────────────────┤
│  事务事实表    │ 记录业务事件（每笔订单、每次点击）       │
│ (Transaction) │ 粒度：一行 = 一个事件                    │
├───────────────┼─────────────────────────────────────────┤
│  周期快照事实表│ 固定周期记录状态（每日账户余额）          │
│ (Periodic)    │ 粒度：一行 = 一个实体 × 一个周期         │
├───────────────┼─────────────────────────────────────────┤
│  累积快照事实表│ 追踪流程的多个里程碑                      │
│ (Accumulating)│ 粒度：一行 = 一个流程实例（如订单生命周期）│
└───────────────┴─────────────────────────────────────────┘
```

### 4.2 事务事实表

```sql
-- ── 订单事务事实表 ──
CREATE TABLE fct_order_items (
    -- 代理键
    order_item_key    BIGINT PRIMARY KEY,

    -- 维度外键
    date_key          INT NOT NULL REFERENCES dim_date(date_key),
    customer_key      BIGINT NOT NULL REFERENCES dim_customers(customer_key),
    product_key       BIGINT NOT NULL REFERENCES dim_products(product_key),
    store_key         BIGINT REFERENCES dim_stores(store_key),
    promotion_key     BIGINT REFERENCES dim_promotions(promotion_key),

    -- 退化维度（事务标识符）
    order_id          VARCHAR(50) NOT NULL,
    order_item_id     VARCHAR(50) NOT NULL,

    -- 度量（可加性）
    quantity          INT NOT NULL,
    unit_price        DECIMAL(18,2) NOT NULL,
    discount_amount   DECIMAL(18,2) DEFAULT 0,
    net_amount        DECIMAL(18,2) NOT NULL,    -- = quantity * unit_price - discount
    tax_amount        DECIMAL(18,2) DEFAULT 0,
    total_amount      DECIMAL(18,2) NOT NULL,   -- = net_amount + tax_amount

    -- 度量（半可加性 / 不可加）
    gross_margin_pct  DECIMAL(5,2),

    -- 审计
    created_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    source_system     VARCHAR(50)
)
PARTITION BY (date_key);

-- 创建索引
CREATE INDEX idx_fct_order_items_date ON fct_order_items(date_key);
CREATE INDEX idx_fct_order_items_customer ON fct_order_items(customer_key);
CREATE INDEX idx_fct_order_items_product ON fct_order_items(product_key);
```

### 4.3 周期快照事实表

```sql
-- ── 每日账户余额快照 ──
CREATE TABLE fct_daily_account_balance (
    snapshot_key      BIGINT PRIMARY KEY,
    date_key          INT NOT NULL REFERENCES dim_date(date_key),
    account_key       BIGINT NOT NULL REFERENCES dim_accounts(account_key),
    customer_key      BIGINT NOT NULL REFERENCES dim_customers(customer_key),

    -- 半可加性度量（不能跨时间简单相加）
    balance_amount    DECIMAL(18,2) NOT NULL,
    available_balance DECIMAL(18,2),
    held_amount       DECIMAL(18,2),

    -- 可加性度量
    deposit_count     INT,
    deposit_amount    DECIMAL(18,2),
    withdrawal_count  INT,
    withdrawal_amount DECIMAL(18,2),

    -- 审计
    snapshot_time     TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
PARTITION BY (date_key);

-- 生成每日快照
INSERT INTO fct_daily_account_balance
SELECT
    NEXTVAL('seq_snapshot_key'),
    TO_NUMBER(TO_CHAR(CURRENT_DATE, 'YYYYMMDD')) AS date_key,
    a.account_key,
    a.customer_key,
    a.current_balance AS balance_amount,
    a.available_balance,
    a.held_amount,
    COALESCE(t.deposit_count, 0) AS deposit_count,
    COALESCE(t.deposit_amount, 0) AS deposit_amount,
    COALESCE(t.withdrawal_count, 0) AS withdrawal_count,
    COALESCE(t.withdrawal_amount, 0) AS withdrawal_amount,
    CURRENT_TIMESTAMP
FROM dim_accounts a
LEFT JOIN (
    SELECT
        account_key,
        SUM(CASE WHEN txn_type = 'deposit' THEN 1 ELSE 0 END) AS deposit_count,
        SUM(CASE WHEN txn_type = 'deposit' THEN amount ELSE 0 END) AS deposit_amount,
        SUM(CASE WHEN txn_type = 'withdrawal' THEN 1 ELSE 0 END) AS withdrawal_count,
        SUM(CASE WHEN txn_type = 'withdrawal' THEN amount ELSE 0 END) AS withdrawal_amount
    FROM fct_transactions
    WHERE date_key = TO_NUMBER(TO_CHAR(CURRENT_DATE, 'YYYYMMDD'))
    GROUP BY account_key
) t ON a.account_key = t.account_key
WHERE a.is_active = TRUE;
```

### 4.4 累积快照事实表

```sql
-- ── 订单生命周期累积快照 ──
CREATE TABLE fct_order_lifecycle (
    order_key         BIGINT PRIMARY KEY,
    order_id          VARCHAR(50) NOT NULL,

    -- 维度外键
    customer_key      BIGINT REFERENCES dim_customers(customer_key),
    date_key_ordered  INT REFERENCES dim_date(date_key),

    -- 里程碑时间戳
    ordered_at        TIMESTAMP,
    paid_at           TIMESTAMP,
    shipped_at        TIMESTAMP,
    delivered_at      TIMESTAMP,
    returned_at       TIMESTAMP,

    -- 里程碑日期键（关联日期维度）
    date_key_paid     INT,
    date_key_shipped  INT,
    date_key_delivered INT,

    -- 周期度量
    hours_to_pay      DECIMAL(10,2),
    hours_to_ship     DECIMAL(10,2),
    hours_to_deliver  DECIMAL(10,2),
    days_total_cycle  DECIMAL(10,2),

    -- 订单状态
    current_status    VARCHAR(20),
    order_amount      DECIMAL(18,2)
);

-- 持续更新（每次有新里程碑时 MERGE）
MERGE INTO fct_order_lifecycle AS target
USING staging.stg_order_events AS source
ON target.order_id = source.order_id
WHEN MATCHED THEN
    UPDATE SET
        paid_at     = CASE WHEN source.event_type = 'payment' THEN source.event_time ELSE target.paid_at END,
        shipped_at  = CASE WHEN source.event_type = 'shipment' THEN source.event_time ELSE target.shipped_at END,
        delivered_at = CASE WHEN source.event_type = 'delivery' THEN source.event_time ELSE target.delivered_at END,
        hours_to_pay = CASE WHEN source.event_type = 'payment'
            THEN EXTRACT(EPOCH FROM (source.event_time - target.ordered_at)) / 3600
            ELSE target.hours_to_pay END,
        current_status = source.event_type
WHEN NOT MATCHED THEN
    INSERT (order_key, order_id, customer_key, ordered_at, date_key_ordered, current_status, order_amount)
    VALUES (NEXTVAL('seq_order_key'), source.order_id, source.customer_key, source.event_time,
            TO_NUMBER(TO_CHAR(source.event_time, 'YYYYMMDD')), source.event_type, source.order_amount);
```

### 4.5 无事实事实表（Factless Fact Table）

```sql
-- 记录"事件发生"而非度量值
-- 例如：某学生某天上了某门课

CREATE TABLE fct_course_attendance (
    attendance_key    BIGINT PRIMARY KEY,
    date_key          INT NOT NULL REFERENCES dim_date(date_key),
    student_key       BIGINT NOT NULL REFERENCES dim_students(student_key),
    course_key        BIGINT NOT NULL REFERENCES dim_courses(course_key),
    -- 没有度量值！仅记录关联关系
    attended          BOOLEAN DEFAULT TRUE
);

-- 另一种类型：覆盖范围（学生-课程注册）
CREATE TABLE fct_course_registration (
    registration_key  BIGINT PRIMARY KEY,
    student_key       BIGINT NOT NULL,
    course_key        BIGINT NOT NULL,
    date_key_start    INT NOT NULL,
    date_key_end      INT NOT NULL
);

-- 查询：哪些课程没有任何学生注册？
SELECT c.course_name
FROM dim_courses c
LEFT JOIN fct_course_registration r ON c.course_key = r.course_key
```

## 5. 汇总层架构

### 5.1 汇总层设计原则

```
原始事实表 (每日数亿行)
    │
    ▼

    │
    ▼
周/月汇总 (进一步聚合)
    │
    ▼
预计算看板表 (直接服务 BI)
```

### 5.2 日汇总表

```sql
-- ── 销售日汇总表 ──
CREATE TABLE agg_daily_sales (
    date_key          INT NOT NULL REFERENCES dim_date(date_key),
    customer_key      BIGINT NOT NULL REFERENCES dim_customers(customer_key),
    product_key       BIGINT NOT NULL REFERENCES dim_products(product_key),
    store_key         BIGINT REFERENCES dim_stores(store_key),

    -- 汇总度量
    total_quantity    INT,
    total_revenue     DECIMAL(18,2),
    total_discount    DECIMAL(18,2),
    total_tax         DECIMAL(18,2),
    total_net_revenue DECIMAL(18,2),
    order_count       INT,
    avg_order_value   DECIMAL(18,2),

    -- 审计
    updated_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    PRIMARY KEY (date_key, customer_key, product_key, store_key)
)
PARTITION BY (date_key)
CLUSTER BY (customer_key, product_key);

-- 物化视图或定时刷新
INSERT INTO agg_daily_sales
SELECT
    oi.date_key,
    oi.customer_key,
    oi.product_key,
    oi.store_key,
    SUM(oi.quantity) AS total_quantity,
    SUM(oi.net_amount) AS total_revenue,
    SUM(oi.discount_amount) AS total_discount,
    SUM(oi.tax_amount) AS total_tax,
    SUM(oi.total_amount) AS total_net_revenue,
    COUNT(DISTINCT oi.order_id) AS order_count,
    AVG(oi.total_amount) AS avg_order_value,
    CURRENT_TIMESTAMP
FROM fct_order_items oi
WHERE oi.date_key = TO_NUMBER(TO_CHAR(CURRENT_DATE - 1, 'YYYYMMDD'))
GROUP BY oi.date_key, oi.customer_key, oi.product_key, oi.store_key;
```

### 5.3 月度汇总 + 同比环比

```sql
-- ── 月度收入汇总 ──
CREATE OR REPLACE TABLE agg_monthly_revenue AS
WITH monthly AS (
    SELECT
        d.year,
        d.month_number,
        d.month_name,
        c.customer_segment,
        SUM(a.total_revenue) AS revenue,
        SUM(a.order_count) AS order_count,
        SUM(a.total_quantity) AS quantity
    FROM agg_daily_sales a
    JOIN dim_date d ON a.date_key = d.date_key
    JOIN dim_customers c ON a.customer_key = c.customer_key AND c.is_current = TRUE
    GROUP BY d.year, d.month_number, d.month_name, c.customer_segment
),
monthly_with_lag AS (
    SELECT
        *,
        LAG(revenue, 1) OVER (PARTITION BY customer_segment ORDER BY year, month_number) AS prev_month_revenue,
        LAG(revenue, 12) OVER (PARTITION BY customer_segment ORDER BY year, month_number) AS prev_year_revenue
    FROM monthly
)
SELECT
    year,
    month_number,
    month_name,
    customer_segment,
    revenue,
    order_count,
    quantity,
    -- 环比增长率
    ROUND(
        (revenue - prev_month_revenue) / NULLIF(prev_month_revenue, 0) * 100,
        2
    ) AS mom_growth_pct,
    -- 同比增长率
    ROUND(
        (revenue - prev_year_revenue) / NULLIF(prev_year_revenue, 0) * 100,
        2
    ) AS yoy_growth_pct
FROM monthly_with_lag
ORDER BY year DESC, month_number DESC, customer_segment;
```

## 6. 建模最佳实践

### 6.1 粒度设计

```python
# 粒度决策树
"""
1. 业务问题是什么？
   → "每笔订单多少收入？" → 事务事实表（订单行级别）
   → "每天总销售多少？"   → 日汇总表（日级别）
   → "订单多久交付？"     → 累积快照表（订单生命周期）

2. 选择最细粒度，让汇总层处理聚合
   → 事实表粒度 = 业务事件的最小原子单位

3. 避免混合粒度
   → 同一事实表中不要混入"日汇总"和"月汇总"
"""
```

### 6.2 度量分类

```sql
-- 度量类型与聚合规则

-- 可加性度量：可沿所有维度聚合
SUM(quantity)          -- ✅ 正确：可以按任意维度聚合
SUM(total_amount)      -- ✅ 正确

-- 半可加性度量：不可沿时间维度聚合
-- 余额快照表：不能简单 SUM(日余额)
SELECT balance_amount  -- ❌ 错误
FROM fct_daily_account_balance;

-- 正确做法：取最新值或平均值
SELECT AVG(balance_amount)  -- ✅ 时间平均
FROM fct_daily_account_balance
WHERE date_key BETWEEN 20260101 AND 20260131;

-- 不可加性度量：不能直接聚合
-- 比率、百分比需要先聚合分子分母
-- ❌ AVG(gross_margin_pct)
-- ✅ SUM(profit) / SUM(revenue) AS margin_pct
```

### 6.5 关键实践清单

- ✅ **粒度一致**：每张事实表必须明确声明粒度
- ✅ **使用代理键**：维度表使用代理键，解耦源系统
- ✅ **一致性维度**：跨数据集市共享维度定义
- ✅ **避免雪花型**：优先星型模型，减少 JOIN
- ✅ **半可加性注意**：余额类指标不要直接 SUM 跨时间
- ✅ **分区策略**：按日期分区，提升查询性能
- ✅ **预聚合**：高频查询建立汇总表
- ✅ **文档化**：在元数据平台记录表/列的业务定义
