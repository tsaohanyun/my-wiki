## 1. 数据质量维度

```
┌─────────────────────────────────────────────────┐
│              数据质量六大维度                     │
├──────────┬──────────────────────────────────────┤
│ 完整性   │ 数据是否缺失？行数/列数是否符合预期？  │
│ 唯一性   │ 主键是否重复？是否存在重复记录？      │
│ 时效性   │ 数据是否按时到达？延迟是否在阈值内？   │
│ 有效性   │ 数据格式、范围、枚举值是否正确？       │
│ 一致性   │ 跨表/跨系统的数据是否一致？            │
│ 准确性   │ 数据值是否与真实业务匹配？             │
└──────────┴──────────────────────────────────────┘
```

### 数据质量 SLA 示例

| 指标               | SLA 阈值                    | 告警级别   |
| ------------------ | --------------------------- | ---------- |
| 表新鲜度           | 最近更新 ≤ 24 小时          | P1 Critical|
| 行数偏差           | 与昨日对比 ≤ ±20%           | P2 Warning |
| 空值率（关键字段） | ≤ 1%                        | P1 Critical|
| 主键重复数         | = 0                         | P1 Critical|
| 外键完整性         | 100%                        | P2 Warning |

### 2.2 定义期望套件（Expectation Suite）

```python
# create_expectation_suite.py
import great_expectations as gx
from great_expectations.core.expectation_configuration import ExpectationConfiguration

context = gx.get_context()

# 创建期望套件
suite = context.add_expectation_suite("orders_quality_suite")

# 添加期望
expectations = [
    # ── 完整性 ──
    ExpectationConfiguration(
        expectation_type="expect_table_row_count_to_be_between",
        kwargs={"min_value": 1000, "max_value": 10000000}
    ),
    ExpectationConfiguration(
        expectation_type="expect_column_values_to_not_be_null",
        kwargs={"column": "order_id"}
    ),

    # ── 唯一性 ──
    ExpectationConfiguration(
        expectation_type="expect_column_values_to_be_unique",
        kwargs={"column": "order_id"}
    ),

    # ── 有效性 ──
    ExpectationConfiguration(
        expectation_type="expect_column_values_to_be_between",
        kwargs={"column": "amount", "min_value": 0, "max_value": 1000000}
    ),
    ExpectationConfiguration(
        expectation_type="expect_column_values_to_be_in_set",
        kwargs={"column": "order_status", "value_set": ["pending", "completed", "cancelled", "refunded"]}
    ),
    ExpectationConfiguration(
        expectation_type="expect_column_values_to_match_regex",
        kwargs={"column": "email", "regex": r"^[^@]+@[^@]+\.[^@]+$"}
    ),

    # ── 时效性 ──
    ExpectationConfiguration(
        expectation_type="expect_column_max_to_be_between",
        kwargs={
            "column": "created_at",
            "min_value": "2026-06-27T00:00:00Z",
            "max_value": "2026-06-28T23:59:59Z"
        }
    ),

    # ── 分布检查 ──
    ExpectationConfiguration(
        expectation_type="expect_column_mean_to_be_between",
        kwargs={"column": "amount", "min_value": 50, "max_value": 500}
    ),
    ExpectationConfiguration(
        expectation_type="expect_column_kl_divergence_to_be_less_than",
        kwargs={
            "column": "amount",
            "partition_object": {
                "bins": [0, 50, 100, 500, 1000, 5000, 10000],
                "weights": [0.3, 0.25, 0.2, 0.15, 0.07, 0.025, 0.005]
            },
            "threshold": 0.1
        }
    ),
]

for exp in expectations:
    suite.add_expectation(exp)

context.save_expectation_suite(suite)
```

### 2.3 运行检查点（Checkpoint）

```python
# run_checkpoint.py
import great_expectations as gx

context = gx.get_context()

# 配置 Checkpoint
checkpoint = context.add_or_update_checkpoint(
    name="orders_checkpoint",
    config_version=1,
    class_name="SimpleCheckpoint",
    validations=[
        {
            "batch_request": {
                "datasource_name": "snowflake_datasource",
                "data_connector_name": "default_runtime_data_connector_name",
                "data_asset_name": "orders",
                "runtime_parameters": {
                    "query": "SELECT * FROM analytics.fct_orders WHERE order_date = CURRENT_DATE"
                },
                "batch_identifiers": {"default_identifier_name": "today"},
            },
            "expectation_suite_name": "orders_quality_suite",
        }
    ],
)

# 执行检查
result = checkpoint.run()

# 检查结果
if not result["success"]:
    raise ValueError("❌ 数据质量检查未通过！")

print("✅ 数据质量检查通过")
```

## 3. Soda 数据测试

### 3.1 Soda Core（YAML 定义）

```yaml
# checks.yml - Soda 检查定义
checks for analytics.fct_orders:
  # 行数检查
  - row_count >= 1000
  - row_count <= 10000000

  # 缺失值检查
  - missing_count(order_id) = 0
  - missing_count(customer_id) = 0
  - missing_percentage(email) < 5

  # 重复值检查
  - duplicate_count(order_id) = 0

  # 有效值检查
  - valid_count(amount) >= 0
  - min(amount) >= 0
  - max(amount) <= 1000000

  # 枚举值
  - values in (order_status) must be in [pending, completed, cancelled, refunded]

  # 模式匹配
  - invalid_count(email) = 0:
      valid regex: ^[^@]+@[^@]+\.[^@]+$

  # 引用完整性
  - values in (customer_id) must exist in dim_customers.customer_id

  # 分布异常检测
  - anomaly score for row_count < default

  # Schema 检查
  - schema:
      warn: when forbidden column present [raw_data, temp_field]
      fail: when required column missing [order_id, amount, order_date]
```

## 4. 数据新鲜度监控

### 4.1 时间戳水位线

```python
import snowflake.connector
from datetime import datetime, timedelta

def check_data_freshness(
    table_name: str,
    timestamp_column: str,
    max_delay_hours: int = 24,
    snowflake_config: dict = None
) -> dict:
    """检查数据新鲜度"""
    conn = snowflake.connector.connect(**snowflake_config)
    cursor = conn.cursor()

    query = f"""
        SELECT
            MAX({timestamp_column}) as latest_record,
            COUNT(*) as total_rows,
            CURRENT_TIMESTAMP() as check_time
        FROM {table_name}
    """
    cursor.execute(query)
    result = cursor.fetchone()

    latest_record, total_rows, check_time = result
    delay = check_time - latest_record if latest_record else timedelta(days=999)

    status = "PASS" if delay <= timedelta(hours=max_delay_hours) else "FAIL"

    report = {
        "table": table_name,
        "latest_record": latest_record,
        "check_time": check_time,
        "delay_hours": round(delay.total_seconds() / 3600, 2),
        "total_rows": total_rows,
        "max_delay_hours": max_delay_hours,
        "status": status,
    }

    cursor.close()
    conn.close()
    return report
```

## 5. 数据完整性检查

### 5.1 引用完整性（Referential Integrity）

```sql
-- 检查外键完整性
WITH orphaned_records AS (
    SELECT f.order_id, f.customer_id
    FROM analytics.fct_orders f
    LEFT JOIN analytics.dim_customers d
        ON f.customer_id = d.customer_id
    WHERE d.customer_id IS NULL
)
SELECT
    COUNT(*) as orphaned_count,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM analytics.fct_orders), 2) as orphaned_pct
FROM orphaned_records;
```

### 5.2 统计分布异常检测

```python
import numpy as np
from scipy import stats

def detect_anomalies_zscore(
    df,
    column: str,
    threshold: float = 3.0
) -> pd.DataFrame:
    """使用 Z-score 检测异常值"""
    values = df[column].dropna()
    z_scores = np.abs(stats.zscore(values))

    anomalies = df.loc[values.index[z_scores > threshold]].copy()
    anomalies["z_score"] = z_scores[z_scores > threshold]

    return anomalies

def detect_volume_anomaly(
    current_count: int,
    history_counts: list,
    sensitivity: float = 3.0
) -> bool:
    """基于历史数据检测行数异常"""
    mean = np.mean(history_counts)
    std = np.std(history_counts)

    if std == 0:
        return False

    z_score = abs(current_count - mean) / std
    is_anomaly = z_score > sensitivity

    if is_anomaly:
        print(f"⚠️ 行数异常: 当前={current_count}, 历史 mean={mean:.0f}, std={std:.0f}, z={z_score:.2f}")

    return is_anomaly
```

### 5.3 Schema 演进检查

```python
def check_schema_drift(
    expected_schema: dict,
    actual_schema: dict
) -> list:
    """检测 schema 漂移"""
    issues = []

    # 检查缺失列
    for col, dtype in expected_schema.items():
        if col not in actual_schema:
            issues.append(f"❌ 缺失列: {col} (期望类型: {dtype})")
        elif actual_schema[col] != dtype:
            issues.append(
                f"⚠️ 类型变更: {col} "
                f"(期望: {dtype}, 实际: {actual_schema[col]})"
            )

    # 检查新增列
    for col in actual_schema:
        if col not in expected_schema:
            issues.append(f"ℹ️ 新增列: {col} (实际类型: {actual_schema[col]})")

    return issues
```

## 6. 与 Airflow 集成

### 6.1 Great Expectations Operator

```python
from airflow import DAG
from airflow.operators.python import PythonOperator
from great_expectations_provider.operators.great_expectations import (
    GreatExpectationsOperator
)
from datetime import datetime

with DAG(
    dag_id="data_quality_pipeline",
    schedule_interval="0 3 * * *",
    start_date=datetime(2026, 1, 1),
    catchup=False,
) as dag:

    # 数据加载
    load_data = PythonOperator(
        task_id="load_data",
        python_callable=load_to_warehouse,
    )

    # GE 质量检查
    ge_check = GreatExpectationsOperator(
        task_id="ge_quality_check",
        conn_id="snowflake_default",
        expectation_suite_name="orders_quality_suite",
        data_asset_name="analytics.fct_orders",
        data_context_root_dir="/opt/great_expectations",
        fail_task_on_validation_failure=True,
    )

    # 数据新鲜度检查
    freshness_check = PythonOperator(
        task_id="freshness_check",
        python_callable=check_data_freshness,
        op_kwargs={
            "table_name": "analytics.fct_orders",
            "timestamp_column": "created_at",
            "max_delay_hours": 24,
        },
    )

    load_data >> ge_check >> freshness_check
```

### 6.2 Soda Airflow Operator

```python
from airflow import DAG
from airflow.decorators import task
from datetime import datetime

@dag(schedule_interval="@daily", start_date=datetime(2026, 1, 1), catchup=False)
def soda_quality_pipeline():

    @task
    def run_soda_scan():
        from soda.scan import Scan

        scan = Scan()
        scan.set_data_source_name("snowflake")
        scan.add_configuration_yaml_file("/opt/soda/configuration.yml")
        scan.add_sodacl_yaml_file("/opt/soda/checks.yml")

        scan.execute()
        scan.assert_no_checks_warn_or_fail()

        return {"status": "passed"}

    @task
    def publish_report(scan_result):
        """发布质量报告到 DataHub"""
        print(f"Quality check result: {scan_result}")

    publish_report(run_soda_scan())

soda_quality_pipeline()
```

## 8. 最佳实践

### 8.1 质量门禁策略

```python
# 分级告警策略
QUALITY_POLICY = {
    "critical": {
        "tests": ["primary_key_unique", "not_null_check", "referential_integrity"],
        "action": "block_pipeline",  # 阻断管道
        "notify": ["P1"],
    },
    "warning": {
        "tests": ["volume_anomaly", "distribution_drift"],
        "action": "alert_only",      # 仅告警
        "notify": ["P2"],
    },
    "info": {
        "tests": ["schema_change", "new_column"],
        "action": "log_only",        # 仅记录
        "notify": [],
    },
}
```

### 8.2 关键实践清单

- ✅ **左移测试**：在 Staging 层就进行质量检查，而非仅 Mart 层
- ✅ **期望即代码**：将期望套件纳入 Git 版本控制
- ✅ **历史基线**：建立数据分布基线，对比检测漂移
- ✅ **分级告警**：区分阻断型和非阻断型检查
- ✅ **自动修复**：对已知异常模式设定自动修正规则
- ✅ **质量评分**：为每个表/管道计算综合质量分
- ✅ **定期回顾**：每周 Review 质量报告并更新期望
