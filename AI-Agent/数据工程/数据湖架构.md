# 数据湖架构

> 数据湖（Data Lake）是一种集中式存储库，可存储任意规模的结构化、半结构化和非结构化数据。现代数据湖基于开放表格式（Delta Lake / Iceberg / Hudi），在对象存储上提供 ACID 事务、时间旅行、Schema 演进等能力。

## 架构概览

### 传统数据湖 vs 现代数据湖

```
=== 传统数据湖（Lambda 架构）===

  批处理层  ────────▶  服务层
  (HDFS/Spark)        (Hive/Presto)
                         ▲
  流处理层  ──────────────┘
  (Kafka/Storm)

  问题：批流不一致、小文件、无事务、数据质量差


=== 现代数据湖（湖仓一体 Lakehouse）===

         ┌──────────────────────────────┐
         │     计算引擎层（解耦）         │
         │  Spark / Flink / Trino / dbt  │
         └──────────┬───────────────────┘
                    │ 开放 API
         ┌──────────▼───────────────────┐
         │     表格式层（元数据管理）      │
         │  Delta Lake / Iceberg / Hudi   │
         └──────────┬───────────────────┘
                    │
         ┌──────────▼───────────────────┐
         │     存储层（对象存储）          │
         │  S3 / HDFS / Azure Blob / GCS  │
         └──────────────────────────────┘

  优势：ACID 事务、时间旅行、Schema 演进、批流一体
```

### 湖仓一体核心特性

| 特性 | 说明 |
|------|------|
| **ACID 事务** | 原子提交，数据一致性保证 |
| **时间旅行** | 查询历史版本数据 |
| **Schema 演进** | 添加/删除/重命名列，不破坏历史数据 |
| **UPSERT/DELETE** | 支持行级更新和删除 |
| **并发控制** | 多个写入者乐观并发控制（OCC） |
| **分区演进** | 动态修改分区策略 |
| **小文件合并** | 自动合并小文件，优化查询性能 |

## 数据格式：Parquet 与 ORC

### Parquet

```python
"""
Parquet 是 Apache 开源的列式存储格式，是数据湖最常用的底层格式。

特点：
- 列式存储：只读取需要的列，减少 I/O
- 高压缩比：同类数据聚集，压缩效果好
- 谓词下推：过滤条件下推到存储层
- 嵌套数据：原生支持复杂嵌套类型
"""

# ====== PySpark 读写 Parquet ======
from pyspark.sql import SparkSession
from pyspark.sql import functions as F

spark = SparkSession.builder.appName("ParquetExample").getOrCreate()

# 写入 Parquet（支持分区、压缩）
df.write \
    .mode("overwrite") \
    .option("compression", "snappy") \
    .partitionBy("year", "month") \
    .parquet("s3a://bucket/warehouse/events/")

# 读取 Parquet
df = spark.read.parquet("s3a://bucket/warehouse/events/")

# 指定分区裁剪（谓词下推）
df_2024_jan = spark.read \
    .parquet("s3a://bucket/warehouse/events/") \
    .filter("year = 2024 AND month = 1")

# 使用 Parquet 选项优化
spark.conf.set("spark.sql.parquet.filterPushdown", "true")        # 谓词下推
spark.conf.set("spark.sql.parquet.mergeSchema", "false")           # 关闭 schema 合并（提升性能）
spark.conf.set("spark.sql.parquet.compression.codec", "snappy")   # 压缩编码
spark.conf.set("spark.sql.files.maxPartitionBytes", "134217728")  # 128MB
spark.conf.set("spark.sql.files.openCostInBytes", "4194304")      # 4MB
```

### ORC

```python
"""
ORC (Optimized Row Columnar) 是 Hive 生态优化的列式格式。

与 Parquet 对比：
- ORC 的轻量级索引更强大（Bloom Filter 内建）
- ORC 压缩率通常更高（特别是字符串列）
- Parquet 在 Spark 生态中更成熟
- ORC 在 Hive/Presto/Trino 中性能优秀
"""

# ORC 配置
spark.conf.set("spark.sql.orc.filterPushdown", "true")
spark.conf.set("spark.sql.orc.compression.codec", "zstd")
spark.conf.set("spark.sql.orc.splits.include.file.footer", "true")

# 读写 ORC
df.write.mode("overwrite").orc("s3a://bucket/warehouse/events/")
df = spark.read.orc("s3a://bucket/warehouse/events/")
```

### 格式选择指南

| 维度 | Parquet | ORC |
|------|---------|-----|
| Spark 生态 | ⭐ 推荐 | ✅ 支持 |
| Hive/Presto | ✅ 支持 | ⭐ 推荐 |
| 压缩率 | 好 | 更好 |
| 嵌套数据 | ⭐ 更好 | 一般 |
| 生态广度 | ⭐ 更广 | Hive 生态 |
| **推荐场景** | Spark 数据湖 | Hive/Presto 数据仓库 |

## Delta Lake

Delta Lake 是 Databricks 开源的数据湖存储层，在 Parquet 之上添加事务日志。

### 核心概念

```
Delta 表结构：
    /table_path/
    ├── _delta_log/           ← 事务日志（JSON 格式）
    │   ├── 00000000000000000000.json   ← 第0次提交
    │   ├── 00000000000000000001.json   ← 第1次提交
    │   ├── ...
    │   └── 00000000000000000010.checkpoint.parquet  ← 检查点
    ├── part-00001-xxx.snappy.parquet  ← 数据文件
    ├── part-00002-xxx.snappy.parquet
    └── ...
```

### PySpark + Delta Lake

```python
from pyspark.sql import SparkSession
from delta.tables import DeltaTable
from pyspark.sql.functions import *

# 初始化带 Delta 的 SparkSession
spark = SparkSession.builder \
    .appName("DeltaLakeExample") \
    .config("spark.sql.extensions", "io.delta.sql.DeltaSparkSessionExtension") \
    .config("spark.sql.catalog.spark_catalog", "org.apache.spark.sql.delta.catalog.DeltaCatalog") \
    .config("spark.delta.logStore.class", "org.apache.spark.sql.delta.storage.S3SingleDriverLogStore") \
    .getOrCreate()

# ====== 创建 Delta 表 ======
df = spark.read.json("data/events.json")

df.write \
    .format("delta") \
    .mode("overwrite") \
    .partitionBy("event_date") \
    .save("s3a://bucket/delta/events/")

# ====== 读取 Delta 表 ======
delta_df = spark.read.format("delta").load("s3a://bucket/delta/events/")

# ====== 流式写入 Delta ======
streaming_df = spark.readStream.format("kafka") \
    .option("kafka.bootstrap.servers", "kafka:9092") \
    .option("subscribe", "events") \
    .load()

query = streaming_df \
    .writeStream \
    .format("delta") \
    .outputMode("append") \
    .option("checkpointLocation", "/checkpoint/events") \
    .start("s3a://bucket/delta/events/")

# ====== UPSERT（Merge）======
delta_table = DeltaTable.forPath(spark, "s3a://bucket/delta/events/")

new_data = spark.read.parquet("s3a://bucket/staging/events_today/")

delta_table.alias("target").merge(
    new_data.alias("source"),
    "target.event_id = source.event_id"
).whenMatchedUpdateAll() \
 .whenNotMatchedInsertAll() \
 .execute()

# 条件 UPSERT
delta_table.alias("target").merge(
    new_data.alias("source"),
    "target.event_id = source.event_id AND target.event_date = source.event_date"
).whenMatchedUpdate(
    condition="source.status = 'UPDATED'",
    set={"target.status": "source.status", "target.updated_at": "source.updated_at"}
).whenNotMatchedInsert(
    values={
        "event_id": "source.event_id",
        "event_type": "source.event_type",
        "amount": "source.amount",
        "event_date": "source.event_date",
        "status": "source.status",
        "created_at": "source.created_at"
    }
).execute()

# ====== DELETE ======
delta_table.delete("event_date < '2024-01-01'")

delta_table.delete(col("status") == "DELETED")

# ====== UPDATE ======
delta_table.update(
    condition="status = 'PENDING'",
    set={"status": lit("PROCESSING"), "updated_at": current_timestamp()}
)

# ====== 表优化（小文件合并）======
delta_table.optimize.zOrderBy("event_id")  # Z-Order 优化查询
delta_table.vacuum(retentionHours=168)      # 清理旧版本文件（7天保留期）
```

### Delta Lake SQL

```sql
-- 创建 Delta 表
CREATE TABLE IF NOT EXISTS events (
    event_id STRING,
    event_type STRING,
    amount DOUBLE,
    event_date DATE,
    status STRING,
    created_at TIMESTAMP
) USING DELTA
PARTITIONED BY (event_date)
LOCATION 's3a://bucket/delta/events/';

-- 插入数据
INSERT INTO events
SELECT * FROM staging_events;

-- Merge (UPSERT)
MERGE INTO events AS t
USING staging_events AS s
ON t.event_id = s.event_id
WHEN MATCHED THEN UPDATE SET *
WHEN NOT MATCHED THEN INSERT *;

-- DELETE
DELETE FROM events WHERE event_date < '2024-01-01';

-- 表属性
ALTER TABLE events SET TBLPROPERTIES (
    'delta.logRetentionDuration' = 'interval 30 days',
    'delta.deletedFileRetentionDuration' = 'interval 7 days',
    'delta.enableChangeDataFeed' = 'true',
    'delta.autoOptimize.optimizeWrite' = 'true',
    'delta.autoOptimize.autoCompact' = 'true'
);
```

## Apache Iceberg

Iceberg 是 Netflix 开源的表格式，支持多引擎访问。

### 核心架构

```
Iceberg 表结构：
    /table_path/
    ├── metadata/
    │   ├── v1.metadata.json          ← 元数据快照（每次写入生成新版本）
    │   ├── v2.metadata.json
    │   ├── snap-xxx-1.avro           ← 快照清单文件
    │   ├── snap-xxx-2.avro
    │   └── xxx-m0.avro               ← Manifest 文件（数据文件列表）
    ├── data/
    │   ├── 00001-xxx.parquet         ← 实际数据文件
    │   └── 00002-xxx.parquet
    └── partition_data/
        └── event_date=2024-01-01/
            └── 00003-xxx.parquet
```

### Spark + Iceberg

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("IcebergExample") \
    .config("spark.sql.extensions", "org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions") \
    .config("spark.sql.catalog.iceberg", "org.apache.iceberg.spark.SparkCatalog") \
    .config("spark.sql.catalog.iceberg.type", "hive") \
    .config("spark.sql.catalog.iceberg.warehouse", "s3://bucket/warehouse") \
    .getOrCreate()

# ====== 创建表 ======
spark.sql("""
    CREATE TABLE iceberg.db.events (
        event_id STRING,
        event_type STRING,
        amount DOUBLE,
        event_date DATE,
        user_id STRING,
        metadata MAP<STRING, STRING>
    )
    USING iceberg
    PARTITIONED BY (days(event_date))
    TBLPROPERTIES (
        'write.format.default' = 'parquet',
        'write.parquet.compression-codec' = 'zstd',
        'commit.retry.num-retries' = '5',
        'history.expire.max-snapshot-age-ms' = '259200000'
    )
""")

# ====== 写入数据 ======
spark.sql("""
    INSERT INTO iceberg.db.events
    SELECT * FROM staging_events
""")

# ====== UPSERT (MERGE INTO) ======
spark.sql("""
    MERGE INTO iceberg.db.events AS t
    USING staging_events AS s
    ON t.event_id = s.event_id
    WHEN MATCHED THEN UPDATE SET *
    WHEN NOT MATCHED THEN INSERT *
""")

# ====== 分区演进（Iceberg 独有优势）======
# 添加新分区方式（不影响已有数据）
spark.sql("""
    ALTER TABLE iceberg.db.events
    ADD PARTITION FIELD bucket(16, user_id)
""")

# Iceberg 支持隐藏分区（Hidden Partitioning）
# 查询时自动应用分区过滤，不需要显式过滤分区列
spark.sql("""
    SELECT * FROM iceberg.db.events
    WHERE event_date >= DATE '2024-01-01'
""")  # 自动应用到 days(event_date) 分区

# ====== 时间旅行 ======
# 查询历史快照
spark.sql("""
    SELECT * FROM iceberg.db.events VERSION AS OF 1234567890
""")

spark.sql("""
    SELECT * FROM iceberg.db.events TIMESTAMP AS OF '2024-06-01 12:00:00'
""")

# ====== 快照管理 ======
# 查看表快照历史
spark.sql("SELECT * FROM iceberg.db.events.history").show()

# 查看快照详情
spark.sql("SELECT * FROM iceberg.db.events.snapshots").show()

# 过期旧快照
spark.sql("""
    CALL iceberg.system.expire_snapshots(
        table => 'db.events',
        older_than => TIMESTAMP '2024-06-01 00:00:00',
        retain_last => 5
    )
""")

# 合并小文件
spark.sql("""
    CALL iceberg.system.rewrite_data_files(
        table => 'db.events',
        strategy => 'binpack',
        options => map('min-input-files', '5', 'target-file-size-bytes', '536870912')
    )
""")
```

## Apache Hudi

Hudi（Hadoop Upserts Deletes and Incrementals）是 Uber 开源的表格式，专注于近实时 Upsert。

### 表类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **CoW (Copy on Write)** | 写时复制，每次更新重写整个 Parquet 文件 | 批处理、低频更新 |
| **MoR (Merge on Read)** | 读时合并，增量存储在 Avro 日志中，读取时合并 | 近实时、高频更新 |

### Spark + Hudi

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("HudiExample") \
    .config("spark.jars.packages", "org.apache.hudi:hudi-spark3-bundle_2.12:0.14.0") \
    .config("spark.sql.extensions", "org.apache.spark.sql.hudi.HoodieSparkSessionExtension") \
    .config("spark.sql.catalog.hudi", "org.apache.hudi.spark3.HoodieCatalog") \
    .getOrCreate()

# ====== Hudi 通用配置 ======
hudi_options = {
    # 基础配置
    "hoodie.table.name": "events",
    "hoodie.table.type": "MERGE_ON_READ",  # 或 COPY_ON_WRITE

    # 主键和分区
    "hoodie.datasource.write.recordkey.field": "event_id",
    "hoodie.datasource.write.partitionpath.field": "event_date",
    "hoodie.datasource.write.precombine.field": "updated_at",

    # 写入操作类型
    "hoodie.datasource.write.operation": "upsert",  # insert / upsert / bulk_insert / delete

    # Hive 同步
    "hoodie.datasource.hive_sync.enable": "true",
    "hoodie.datasource.hive_sync.database": "default",
    "hoodie.datasource.hive_sync.table": "events",
    "hoodie.datasource.hive_sync.partition_fields": "event_date",
    "hoodie.datasource.hive_sync.partition_extractor_class": "org.apache.hudi.hive.SlashDayPartitionValueExtractor",

    # 清理策略
    "hoodie.cleaner.policy": "KEEP_LATEST_COMMITS",
    "hoodie.cleaner.commits.retained": "10",
    "hoodie.cleaner.parallelism": "200",

    # 压缩（MoR 模式，合并 Avro 日志到 Parquet）
    "hoodie.compact.inline": "true",
    "hoodie.compact.inline.max.delta.commits": "5",

    # 索引类型
    "hoodie.index.type": "BLOOM",  # BLOOM / GLOBAL_BLOOM / SIMPLE / BUCKET
}

# ====== 写入数据 ======
df.write \
    .format("hudi") \
    .options(**hudi_options) \
    .mode("overwrite") \
    .save("s3a://bucket/hudi/events/")

# ====== UPSERT ======
new_df = spark.read.parquet("s3a://bucket/staging/events_today/")
new_df.write \
    .format("hudi") \
    .options(**hudi_options) \
    .option("hoodie.datasource.write.operation", "upsert") \
    .mode("append") \
    .save("s3a://bucket/hudi/events/")

# ====== 增量查询（Hudi 独有优势）======
incremental_df = spark.read \
    .format("hudi") \
    .option("hoodie.datasource.query.type", "incremental") \
    .option("hoodie.datasource.read.begin.instanttime", "20240601120000") \
    .option("hoodie.datasource.read.end.instanttime", "20240601130000") \
    .load("s3a://bucket/hudi/events/")

# ====== 时间旅行 ======
snapshot_df = spark.read \
    .format("hudi") \
    .option("hoodie.datasource.query.type", "snapshot") \
    .option("as.of.instant", "20240601120000") \
    .load("s3a://bucket/hudi/events/")

# ====== 读优化查询（MoR 模式，只读 Parquet）======
ro_df = spark.read \
    .format("hudi") \
    .option("hoodie.datasource.query.type", "read_optimized") \
    .load("s3a://bucket/hudi/events/")
```

## 三大格式对比

| 特性 | Delta Lake | Apache Iceberg | Apache Hudi |
|------|-----------|----------------|-------------|
| **起源** | Databricks | Netflix | Uber |
| **语言支持** | Scala/Java/Python/SQL | Java/Python/SQL | Scala/Java/Python/SQL |
| **引擎支持** | Spark 为主 | Spark/Flink/Trino/StarRocks | Spark/Flink/Presto/Trino |
| **ACID** | ✅ | ✅ | ✅ |
| **时间旅行** | ✅ | ✅ | ✅ |
| **UPSERT/DELETE** | ✅ | ✅ | ✅（最强） |
| **Schema 演进** | ✅ | ✅（最强） | ✅ |
| **分区演进** | ❌ | ✅ | ❌ |
| **隐藏分区** | ❌ | ✅ | ❌ |
| **增量查询** | CDF (Change Data Feed) | ✅ | ✅（最强） |
| **MoR/CoW** | 仅 CoW | 仅 CoW | ✅ MoR + CoW |
| **流写流读** | ✅（最强） | ✅ | ✅ |
| **社区活跃度** | 高 | 最高 | 高 |
| **推荐场景** | Spark 生态，Databricks | 多引擎，大规模数据 | 近实时 Upsert，CDC |

### 选型建议

```
如果团队主要使用 Spark + Databricks     → Delta Lake
如果需要多引擎支持（Spark + Flink + Trino）→ Apache Iceberg
如果核心需求是近实时 Upsert / CDC        → Apache Hudi
如果不确定，Apache Iceberg 是最通用的选择
```

## 时间旅行

### Delta Lake 时间旅行

```python
from delta.tables import DeltaTable

delta_table = DeltaTable.forPath(spark, "s3a://bucket/delta/events/")

# ====== 查询历史版本 ======

# 方式1：按版本号
df_v1 = spark.read.format("delta") \
    .option("versionAsOf", 1) \
    .load("s3a://bucket/delta/events/")

# 方式2：按时间戳
df_old = spark.read.format("delta") \
    .option("timestampAsOf", "2024-06-01 12:00:00") \
    .load("s3a://bucket/delta/events/")

# SQL 方式
spark.sql("""
    SELECT * FROM delta.`s3a://bucket/delta/events/`
    VERSION AS OF 5
""")

spark.sql("""
    SELECT * FROM delta.`s3a://bucket/delta/events/`
    TIMESTAMP AS OF '2024-06-01 12:00:00'
""")

# ====== 查看历史操作 ======
history_df = delta_table.history()
history_df.select("version", "timestamp", "operation", "operationParameters").show()

# ====== 回滚到指定版本 ======
delta_table.restoreToVersion(5)

# 按时间回滚
delta_table.restoreToTimestamp("2024-06-01 12:00:00")

# ====== Change Data Feed（CDC）======
# 需要先启用：ALTER TABLE events SET TBLPROPERTIES ('delta.enableChangeDataFeed' = 'true')
cdc_df = spark.read.format("delta") \
    .option("readChangeFeed", "true") \
    .option("startingVersion", 10) \
    .load("s3a://bucket/delta/events/")

cdc_df.select("_change_type", "event_id", "event_type", "_commit_version").show()
# _change_type: insert / update_postimage / delete
```

## 最佳实践

### ✅ 推荐做法

1. **选择一个格式并坚持**，混用会增加复杂度
2. **合理设置分区**，按日期/分类分区，避免过度分区
3. **定期执行 Vacuum/Compaction**，清理旧文件和小文件
4. **使用 ZSTD 压缩**，平衡压缩率和性能
5. **监控表元数据大小**，过多的快照会降低查询性能
6. **合理设置保留期**，`logRetentionDuration` / `deletedFileRetentionDuration`
7. **使用 Catalog 管理表**，而非直接操作文件路径
8. **启用 Change Data Feed**，支持下游增量消费

### 分区策略

| 策略 | 适用场景 | 示例 |
|------|----------|------|
| **日期分区** | 时间序列数据 | `PARTITIONED BY (event_date)` |
| **月份分区** | 数据量较小 | `PARTITIONED BY (month(event_date))` |
| **分类分区** | 明确的类别 | `PARTITIONED BY (category)` |
| **哈希分区** | 均匀分布 | `PARTITIONED BY (bucket(16, user_id))` |
| **Iceberg 隐藏分区** | 自动优化 | `PARTITIONED BY (days(timestamp))` |

> ⚠️ **分区数建议**：单个表 10,000 - 100,000 个分区。过多分区会导致元数据膨胀和查询变慢。
