# 数据库客户端工具

## 1. DBeaver

### 简介
DBeaver 是免费开源的通用数据库工具，支持几乎所有主流数据库（MySQL、PostgreSQL、SQLite、Oracle、SQL Server、MongoDB、Redis等）。

### 安装
```bash
# Ubuntu/Debian
sudo snap install dbeaver-ce

# macOS
brew install --cask dbeaver-community

# 或下载 https://dbeaver.io/download/
```

### 连接配置

**MySQL 连接：**
```
主机: localhost
端口: 3306
数据库: mydb
用户名: root
密码: ****
驱动: MySQL 8+ (com.mysql.cj.jdbc.Driver)
```

**PostgreSQL 连接：**
```
主机: localhost
端口: 5432
数据库: postgres
用户名: postgres
模式: public
```

**SSH隧道连接远程数据库：**
```
连接设置 → SSH → 启用SSH隧道
SSH主机: jump-server.example.com
SSH端口: 22
SSH用户: admin
认证方式: 密钥文件 → ~/.ssh/id_rsa

数据库主机: db-internal（SSH隧道内部地址）
数据库端口: 5432
```

### 常用功能
- **ER图生成** — 右键数据库 → 查看图表
- **数据导出** — 右键表 → 导出数据 → CSV/JSON/SQL
- **SQL格式化** — `Ctrl+Shift+F` 格式化SQL
- **执行计划** — 选中查询 → 解释执行计划
- **数据比较** — 工具 → 比较数据
- **Mock数据生成** — 右键表 → 生成测试数据

### 实用SQL模板
```sql
-- 查看表大小
SELECT
  table_name AS "表名",
  ROUND(data_length / 1024 / 1024, 2) AS "数据大小(MB)",
  ROUND(index_length / 1024 / 1024, 2) AS "索引大小(MB)",
  table_rows AS "行数"
FROM information_schema.tables
WHERE table_schema = 'mydb'
ORDER BY data_length DESC;

-- 查看慢查询
SELECT * FROM mysql.slow_log
WHERE start_time > NOW() - INTERVAL 1 DAY
ORDER BY query_time DESC LIMIT 20;
```

## 2. DataGrip

### 简介
JetBrains 出品的专业数据库IDE，深度集成SQL智能提示、重构、版本控制。适合重度数据库开发。

### 安装
```bash
# macOS
brew install --cask datagrip

# 或通过 JetBrains Toolbox 安装
```

### 核心特性配置

**智能提示增强：**
```
Settings → Editor → General → Code Completion
✅ Autopopup in 200ms
✅ Sort suggestions by relevance

Settings → Database → General
✅ Qualify object names on completion
✅ Add table alias on completion
```

**代码风格：**
```
Settings → Editor → Code Style → SQL
- 关键字大小写: 大写（SELECT, FROM, WHERE）
- 缩进: 2空格
- 对齐: 列名对齐
```

### 常用快捷键
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Enter` | 执行语句 |
| `Ctrl+Shift+Enter` | 执行语句（新tab） |
| `Ctrl+D` | 复制行 |
| `Ctrl+/` | 注释切换 |
| `Ctrl+Shift+L` | 格式化SQL |
| `Ctrl+N` | 按名称查找表 |
| `Ctrl+Shift+N` | 全局搜索对象 |
| `F4` | 跳转到表定义 |
| `Alt+Enter` | 快速修复/意图操作 |
| `Ctrl+Q` | 查看文档/DDL |

### 数据导入导出
```
# 导入CSV
右键表 → Import Data from File → 选择CSV
→ 映射列 → 设置分隔符 → 执行

# 导出
右键查询结果 → Export Data
→ 选择格式（CSV/JSON/Excel/SQL INSERT）
→ 配置选项 → 导出

# 数据库迁移（Generate Migrations）
右键表 → SQL Scripts → Generate Migration Script
```

### 版本控制集成
```
# SQL脚本版本管理
Tools → New SQL Console → 保存为.sql文件
# 文件自动纳入git管理

# 变更跟踪
Database Changes 面板显示所有DDL变更
→ 生成Flyway/Liquibase迁移脚本
```

## 3. pgAdmin

### 简介
PostgreSQL官方管理工具，Web版和桌面版都有。最专业的PostgreSQL管理工具。

### 安装
```bash
# Ubuntu/Debian
sudo apt install pgadmin4

# macOS
brew install --cask pgadmin4

# Docker运行
docker run -p 5050:80 \
  -e PGADMIN_DEFAULT_EMAIL=admin@example.com \
  -e PGADMIN_DEFAULT_PASSWORD=admin \
  -d dpage/pgadmin4
```

### 连接配置
```
# 注册服务器
右键 Servers → Register → Server

General:
  Name: Production DB

Connection:
  Host: db.example.com
  Port: 5432
  Database: mydb
  Username: app_user
  Password: ****

SSH Tunnel:
  ✅ Use SSH tunneling
  Tunnel host: jump.example.com
  Username: admin
  Authentication: Identity file
  Identity file: ~/.ssh/id_rsa
```

### 常用操作

**查询工具：**
```sql
-- 查看当前连接
SELECT pid, usename, application_name, client_addr, state, query_start
FROM pg_stat_activity
WHERE state = 'active';

-- 查看表膨胀
SELECT
  schemaname,
  tablename,
  pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS total_size,
  n_dead_tup,
  n_live_tup,
  ROUND(n_dead_tup::numeric / GREATEST(n_live_tup, 1) * 100, 2) AS dead_pct
FROM pg_stat_user_tables
WHERE n_dead_tup > 1000
ORDER BY n_dead_tup DESC;

-- 查看索引使用率
SELECT
  indexrelname,
  idx_scan,
  idx_tup_read,
  pg_size_pretty(pg_relation_size(indexrelid)) AS size
FROM pg_stat_user_indexes
ORDER BY idx_scan ASC;
```

**备份恢复：**
```bash
# 备份（命令行，pgAdmin图形界面也可操作）
pg_dump -h host -U user -d dbname -F c -f backup.dump

# 恢复
pg_restore -h host -U user -d dbname backup.dump

# SQL格式备份
pg_dump -h host -U user -d dbname -f backup.sql

# 只备份schema
pg_dump -h host -U user -d dbname --schema-only -f schema.sql
```

## 4. RedisInsight

### 简介
Redis官方GUI工具，免费，支持Redis和Redis Cluster。可视化浏览数据、分析内存、监控性能。

### 安装
```bash
# 下载安装
# https://redis.com/redis-enterprise/redis-insight/

# 或Docker
docker run -d -p 8001:8001 redis/redis-insight:latest
```

### 连接配置
```
# 添加Redis实例
Add Redis Database:
  Host: localhost
  Port: 6379
  Name: Local Redis
  # 如果有密码
  Password: ****
  # 如果使用SSL
  ✅ Use TLS
  # 如果是集群
  ✅ Clusters (自动发现节点)
```

### 核心功能

**Browser（数据浏览）：**
- 按Key模式搜索 `user:*`
- 查看不同类型（String/Hash/List/Set/ZSet/Stream）
- 直接编辑值
- TTL管理

**CLI（命令行）：**
```bash
# 直接执行Redis命令
KEYS user:*
GET user:123
HGETALL user:123
ZREVRANGE leaderboard 0 9 WITHSCORES
INFO memory
DBSIZE
```

**Memory Analysis（内存分析）：**
- 按Key分析内存占用
- 按数据类型统计
- 识别大Key和过期策略
- 生成内存快照对比

**Slowlog（慢查询）：**
```bash
# Redis中配置
CONFIG SET slowlog-log-slower-than 10000  # 10ms
CONFIG SET slowlog-max-len 128

# RedisInsight自动展示慢查询日志
```

**Pub/Sub监控：**
- 订阅频道实时查看消息
- 发布测试消息
- 监控消息吞吐量

### 常用Redis命令速查
```bash
# String
SET key value EX 3600     # 设置值+过期时间
GET key
MGET key1 key2
INCR counter

# Hash
HSET user:1 name "John" age 30
HGETALL user:1
HINCRBY user:1 age 1

# List
LPUSH queue task1 task2
RPOP queue
LRANGE queue 0 -1

# Set
SADD tags python redis
SMEMBERS tags
SINTER tags1 tags2      # 交集

# Sorted Set
ZADD leaderboard 100 "player1"
ZREVRANGE leaderboard 0 9 WITHSCORES
ZINCRBY leaderboard 10 "player1"

# Key管理
TYPE key
TTL key
EXPIRE key 3600
DEL key
SCAN 0 MATCH user:* COUNT 100
```

## 最佳实践

1. **连接管理** — 使用连接池配置，避免频繁创建连接
2. **SSH隧道** — 生产数据库始终通过跳板机+SSH隧道访问
3. **只读权限** — 开发人员在生产环境使用只读账号
4. **查询规范** — 大表查询添加LIMIT，避免 `SELECT *`
5. **定期分析** — 使用工具内置的性能分析功能定期检查
6. **备份策略** — 配置自动备份，定期验证恢复
7. **敏感信息** — 密码使用密钥管理器，不要明文存储在连接配置中
