# 游戏网络同步

多人游戏的核心技术挑战：如何在不可靠的网络上让所有玩家看到一致的游戏世界。

---

## 一、两大同步架构

### 架构对比

| 特性 | 帧同步（Lockstep） | 状态同步（State Sync） |
|------|-------------------|----------------------|
| **数据传输** | 仅传输输入 | 传输状态变化 |
| **服务端角色** | 轻量转发 | 权威计算 |
| **确定性要求** | 必须完全确定性 | 不要求 |
| **带宽消耗** | 极低 | 中等~高 |
| **回放实现** | 天然支持回放 | 需要额外录制 |
| **适用场景** | RTS、格斗、MOBA | FPS、MMO、大逃杀 |
| **典型游戏** | 星际争霸、王者荣耀 | CS:GO、Fortnite |

---

## 二、帧同步（Lockstep）

### 2.1 核心原理

```
客户端A ──输入帧N──> 服务器 ──输入帧N──> 客户端B
客户端B ──输入帧N──> 服务器 ──输入帧N──> 客户端A

所有客户端在同一逻辑帧执行相同的输入 → 相同的结果
```

### 2.2 帧同步框架

```csharp
using System.Collections.Generic;
using UnityEngine;

public class LockstepManager : MonoBehaviour
{
    public const float LOGIC_FRAME_INTERVAL = 0.033f; // 30fps 逻辑帧

    private int currentFrame = 0;
    private Dictionary<int, Dictionary<int, PlayerInput>> frameInputs = new();
    private float frameTimer = 0f;

    // 每个玩家的输入缓冲
    private Dictionary<int, PlayerInput> pendingInputs = new();

    public int LocalPlayerId { get; set; }
    public int PlayerCount { get; set; }

    void Update()
    {
        frameTimer += Time.deltaTime;

        while (frameTimer >= LOGIC_FRAME_INTERVAL)
        {
            frameTimer -= LOGIC_FRAME_INTERVAL;
            TryAdvanceFrame();
        }
    }

    // 服务器广播某帧所有玩家的输入
    public void OnReceiveFrameInput(int frame, Dictionary<int, PlayerInput> inputs)
    {
        frameInputs[frame] = inputs;

        // 如果在等待这帧，立即执行
        if (frame == currentFrame)
        {
            TryAdvanceFrame();
        }
    }

    private void TryAdvanceFrame()
    {
        // 检查当前帧所有玩家输入是否就绪
        if (!frameInputs.ContainsKey(currentFrame))
        {
            // 等待网络数据，不推进
            return;
        }

        if (frameInputs[currentFrame].Count < PlayerCount)
        {
            return; // 还有玩家输入未到达
        }

        // 执行当前帧所有输入
        ExecuteFrame(frameInputs[currentFrame]);
        currentFrame++;
    }

    private void ExecuteFrame(Dictionary<int, PlayerInput> inputs)
    {
        foreach (var kvp in inputs)
        {
            int playerId = kvp.Key;
            PlayerInput input = kvp.Value;
            GameManager.Instance.ProcessInput(playerId, input);
        }

        // 确定性物理模拟
        Physics.Simulate(LOGIC_FRAME_INTERVAL);
    }

    // 提交本地玩家输入
    public void SubmitInput(PlayerInput input)
    {
        input.Frame = currentFrame;
        input.PlayerId = LocalPlayerId;
        NetworkManager.SendInput(input);
    }
}

[System.Serializable]
public struct PlayerInput
{
    public int PlayerId;
    public int Frame;
    public float MoveX;
    public float MoveY;
    public bool Fire;
    public bool Jump;
}
```

### 2.3 确定性保证要点

```csharp
// ❌ 不确定性来源 & ✅ 解决方案

// 1. 浮点数精度
// ❌ float x = 0.1f + 0.2f; // 不同平台结果不同
// ✅ 使用定点数
public struct FixedPoint
{
    private long rawValue;
    private const int FRACTIONAL_BITS = 16;

    public static FixedPoint FromFloat(float f)
    {
        return new FixedPoint { rawValue = (long)(f * (1 << FRACTIONAL_BITS)) };
    }

    public static FixedPoint operator +(FixedPoint a, FixedPoint b)
    {
        return new FixedPoint { rawValue = a.rawValue + b.rawValue };
    }

    public float ToFloat() => rawValue / (float)(1 << FRACTIONAL_BITS);
}

// 2. 随机数
// ❌ UnityEngine.Random.Range(...)
// ✅ 确定性伪随机
public class DeterministicRandom
{
    private uint seed;

    public DeterministicRandom(uint seed)
    {
        this.seed = seed;
    }

    public uint Next()
    {
        // XorShift32
        seed ^= seed << 13;
        seed ^= seed >> 17;
        seed ^= seed << 5;
        return seed;
    }

    public int Range(int min, int max)
    {
        return min + (int)(Next() % (uint)(max - min));
    }
}

// 3. 字典遍历顺序
// ❌ foreach (var kvp in dictionary) // 顺序不确定
// ✅ 使用 SortedDictionary 或排序后的 KeyList
```

## 三、状态同步（State Sync）

### 3.1 服务端权威架构

```
客户端 ──输入──> 服务器（权威） ──状态快照──> 所有客户端
                        │
                   物理模拟 + 碰撞检测
                   伤害计算 + 状态变更
```

### 3.2 属性同步（Unity Netcode for GameObjects）

```csharp
using Unity.Netcode;

public class NetworkPlayer : NetworkBehaviour
{
    // 自动同步的变量（客户端 → 服务端 → 所有客户端）
    private NetworkVariable<int> health = new NetworkVariable<int>(
        100, NetworkVariableReadPermission.Everyone,
        NetworkVariableWritePermission.Server);

    private NetworkVariable<Vector3> networkPosition = new NetworkVariable<Vector3>(
        default, NetworkVariableReadPermission.Everyone,
        NetworkVariableWritePermission.Server);

    // 状态变化回调
    public override void OnNetworkSpawn()
    {
        health.OnValueChanged += OnHealthChanged;
        networkPosition.OnValueChanged += OnPositionChanged;
    }

    public override void OnNetworkDespawn()
    {
        health.OnValueChanged -= OnHealthChanged;
        networkPosition.OnValueChanged -= OnPositionChanged;
    }

    private void OnHealthChanged(int oldVal, int newVal)
    {
        Debug.Log($"血量变化: {oldVal} → {newVal}");
        UpdateHealthUI(newVal);
    }

    private void OnPositionChanged(Vector3 oldVal, Vector3 newVal)
    {
        // 客户端平滑插值到新位置
        targetPosition = newVal;
    }

    // 只有服务端可调用
    [ServerRpc(RequireOwnership = false)]
    public void TakeDamageServerRpc(int damage)
    {
        health.Value -= damage;
        if (health.Value <= 0)
        {
            DieClientRpc();
        }
    }

    // 服务端广播到所有客户端
    [ClientRpc]
    private void DieClientRpc()
    {
        PlayDeathAnimation();
    }

    // 客户端预测移动 + 服务端校验
    [ServerRpc]
    public void MoveServerRpc(Vector3 input, float deltaTime)
    {
        if (!IsServer) return;

        // 服务端校验：防止加速外挂
        float speed = input.magnitude * moveSpeed;
        if (speed > moveSpeed * 1.1f) // 10% 容差
        {
            // 回滚到上一帧位置
            ResetPositionClientRpc(lastValidPosition);
            return;
        }

        Vector3 newPos = transform.position + input * moveSpeed * deltaTime;
        networkPosition.Value = newPos;
    }

    private Vector3 targetPosition;
    private Vector3 lastValidPosition;
    [SerializeField] private float moveSpeed = 5f;

    void Update()
    {
        if (IsOwner)
        {
            // 本地玩家：立即移动（客户端预测）
            Vector3 input = new Vector3(
                Input.GetAxisRaw("Horizontal"), 0,
                Input.GetAxisRaw("Vertical")).normalized;

            transform.position += input * moveSpeed * Time.deltaTime;

            // 发送到服务器
            if (input.sqrMagnitude > 0.01f)
            {
                MoveServerRpc(input, Time.deltaTime);
            }
        }
        else
        {
            // 远程玩家：插值到同步位置
            transform.position = Vector3.Lerp(
                transform.position, targetPosition, Time.deltaTime * 15f);
        }
    }
}
```

```csharp
public static class Quantize
{
    // 位置精度 0.01m（厘米级）
    public static ushort QuantizeFloat(float value, float min, float max)
    {
        float normalized = (value - min) / (max - min);
        return (ushort)(normalized * 65535);
    }

    public static float Dequantize(ushort quantized, float min, float max)
    {
        float normalized = quantized / 65535f;
        return min + normalized * (max - min);
    }

    // 旋转：四元数 → 最小表示（3个float）
    public static uint PackQuaternion(Quaternion q)
    {
        // 找最大分量，只传其余3个
        int maxIndex = 0;
        float maxVal = Mathf.Abs(q.x);
        if (Mathf.Abs(q.y) > maxVal) { maxIndex = 1; maxVal = Mathf.Abs(q.y); }
        if (Mathf.Abs(q.z) > maxVal) { maxIndex = 2; maxVal = Mathf.Abs(q.z); }
        if (Mathf.Abs(q.w) > maxVal) { maxIndex = 3; }

        // 每个分量 10 bit，2 bit 索引 → 32 bit
        // ... 具体打包逻辑
        return 0;
    }
}
```

## 四、客户端预测与服务器校正

### 4.1 客户端预测（Client-Side Prediction）

```csharp
// 客户端预测：本地立即响应输入，同时发送服务器
public class ClientPrediction
{
    private struct InputState
    {
        public uint Tick;
        public Vector3 Input;
        public Vector3 PredictedPosition;
    }

    private Queue<InputState> inputHistory = new Queue<InputState>();
    private const int MAX_HISTORY = 64;

    public Vector3 PredictMove(Vector3 currentPosition, Vector3 input, float speed, float dt)
    {
        Vector3 predicted = currentPosition + input * speed * dt;

        // 记录输入历史，用于服务器校正
        inputHistory.Enqueue(new InputState
        {
            Tick = NetworkManager.CurrentTick,
            Input = input,
            PredictedPosition = predicted
        });

        if (inputHistory.Count > MAX_HISTORY)
            inputHistory.Dequeue();

        return predicted;
    }

    // 服务器确认位置后，重新模拟后续输入
    public void OnServerReconcile(uint serverTick, Vector3 serverPosition)
    {
        // 找到服务器确认的输入
        // 清除该输入之前的历史
        // 以服务器位置为起点，重新执行之后所有输入
        Vector3 correctedPosition = serverPosition;

        var remaining = new Queue<InputState>();
        foreach (var state in inputHistory)
        {
            if (state.Tick > serverTick)
            {
                correctedPosition += state.Input * 5f * 0.033f;
                remaining.Enqueue(new InputState
                {
                    Tick = state.Tick,
                    Input = state.Input,
                    PredictedPosition = correctedPosition
                });
            }
        }
        inputHistory = remaining;

        // 应用修正
        float error = Vector3.Distance(correctedPosition, serverPosition);
        if (error > 0.01f) // 误差阈值
        {
            // 平滑修正，避免瞬移
            localPlayer.position = Vector3.Lerp(
                localPlayer.position, correctedPosition, 0.3f);
        }
    }
}
```

### 4.2 实体插值（Entity Interpolation）

```csharp
// 远程实体：在两个服务器快照之间插值
public class EntityInterpolation : MonoBehaviour
{
    private struct StateBuffer
    {
        public float Timestamp;
        public Vector3 Position;
        public Quaternion Rotation;
    }

    private List<StateBuffer> stateBuffer = new List<StateBuffer>();
    private float interpolationDelay = 0.1f; // 100ms 延迟插值

    public void AddState(float timestamp, Vector3 pos, Quaternion rot)
    {
        stateBuffer.Add(new StateBuffer
        {
            Timestamp = timestamp,
            Position = pos,
            Rotation = rot
        });

        // 保留最近1秒的数据
        while (stateBuffer.Count > 0 &&
               stateBuffer[0].Timestamp < timestamp - 1f)
        {
            stateBuffer.RemoveAt(0);
        }
    }

    void Update()
    {
        if (stateBuffer.Count < 2) return;

        // 渲染延迟：当前时间 - 插值延迟
        float renderTime = Time.time - interpolationDelay;

        // 找到 renderTime 前后的两个状态
        for (int i = 0; i < stateBuffer.Count - 1; i++)
        {
            var a = stateBuffer[i];
            var b = stateBuffer[i + 1];

            if (a.Timestamp <= renderTime && b.Timestamp >= renderTime)
            {
                float t = (renderTime - a.Timestamp) / (b.Timestamp - a.Timestamp);
                transform.position = Vector3.Lerp(a.Position, b.Position, t);
                transform.rotation = Quaternion.Slerp(a.Rotation, b.Rotation, t);
                return;
            }
        }
    }
}
```

## 五、延迟补偿（Lag Compensation）

### 5.1 服务端回滚射击

```csharp
// CS:GO 风格延迟补偿
public class LagCompensation
{
    private struct HistoricalState
    {
        public float Timestamp;
        public Dictionary<int, Bounds> PlayerHitboxes;
    }

    private Queue<HistoricalState> history = new Queue<HistoricalState>();
    private const float MAX_COMPENSATION = 0.2f; // 最大补偿 200ms

    // 每帧记录所有玩家位置
    public void RecordState(float timestamp, Dictionary<int, Bounds> hitboxes)
    {
        history.Enqueue(new HistoricalState
        {
            Timestamp = timestamp,
            PlayerHitboxes = new Dictionary<int, Bounds>(hitboxes)
        });

        // 清理过旧数据
        while (history.Peek().Timestamp < timestamp - 1f)
            history.Dequeue();
    }

    // 服务端回滚判定射击
    public bool RaycastWithCompensation(
        int shooterId,
        Vector3 origin,
        Vector3 direction,
        float shooterLatency,
        out int hitPlayerId)
    {
        hitPlayerId = -1;

        // 回滚到射击者看到的时间点
        float compensatedTime = Time.time - shooterLatency;
        compensatedTime = Mathf.Max(compensatedTime, Time.time - MAX_COMPENSATION);

        // 找到最近的历史状态
        HistoricalState? closest = null;
        float closestDiff = float.MaxValue;

        foreach (var state in history)
        {
            float diff = Mathf.Abs(state.Timestamp - compensatedTime);
            if (diff < closestDiff)
            {
                closestDiff = diff;
                closest = state;
            }
        }

        if (!closest.HasValue) return false;

        // 用历史位置做射线检测
        Ray ray = new Ray(origin, direction);
        float closestDist = float.MaxValue;

        foreach (var kvp in closest.Value.PlayerHitboxes)
        {
            if (kvp.Key == shooterId) continue;

            if (kvp.Value.IntersectRay(ray, out float dist))
            {
                if (dist < closestDist)
                {
                    closestDist = dist;
                    hitPlayerId = kvp.Key;
                }
            }
        }

        return hitPlayerId >= 0;
    }
}
```

## 六、最佳实践

| 实践 | 说明 |
|------|------|
| **选对同步模型** | RTS/格斗用帧同步，FPS/MMO用状态同步 |
| **客户端预测 + 服务端权威** | 响应快且防作弊 |
| **实体插值隐藏抖动** | 远程玩家渲染延迟 100ms，换取平滑 |
| **延迟补偿射击** | 射击判定基于射击者看到的画面 |
| **带宽优化** | 差量同步 + 量化 + 压缩 |
| **丢包冗余** | UDP 包携带前 N 帧输入 |
| **时钟同步** | 使用 NTP 或自定义协议同步客户端时钟 |
| **防作弊** | 服务端校验关键逻辑，客户端只发输入 |
