## 1. DrawCall 优化

### 1.1 什么是 DrawCall

DrawCall 是 CPU 向 GPU 发出的一次绘制命令。每次 DrawCall 都有 CPU 开销（状态切换、参数设置），是渲染瓶颈的常见原因。

```
理想情况: 1 个 DrawCall 绘制 10000 个同材质物体
糟糕情况: 10000 个 DrawCall 各绘制 1 个物体
```

### 1.2 性能基准

| 平台 | DrawCall 建议上限 | 备注 |
|------|-------------------|------|
| 移动端（低端） | 100-200 | OpenGL ES 3.0 |
| 移动端（高端） | 300-500 | Vulkan/Metal |
| PC | 2000-5000 | DirectX 12 / Vulkan |
| 主机 | 5000-10000 | 专用硬件优化 |

### 1.3 批处理技术

#### Static Batching（静态批处理）

```
适用于：不移动的静态物体（建筑、地形装饰）

Unity 操作：
  1. 选中所有静态物体
  2. 勾选 Static → Batching Static
  3. Unity 在构建时合并网格

优点：运行时零开销
缺点：增加内存（合并后的网格数据），不能移动
```

#### Dynamic Batching（动态批处理）

```
适用于：共享材质的小型动态物体

条件限制（Unity）：
  - 顶点数 ≤ 300
  - 顶点属性总数 ≤ 900
  - 共享相同材质
  - 光照不冲突

自动生效，无需设置。
缺点：CPU 每帧重新计算变换，大量物体反而更慢。
```

#### GPU Instancing（GPU 实例化）

```csharp
using UnityEngine;

// 材质上需要勾选 "Enable GPU Instancing"
// 适用于大量相同网格 + 相同材质的物体（如树木、草地、子弹）

[ExecuteInEditMode]
public class GPUInstancingSetup : MonoBehaviour
{
    [SerializeField] private Mesh instancedMesh;
    [SerializeField] private Material instancedMaterial;
    [SerializeField] private int instanceCount = 10000;
    [SerializeField] private float areaSize = 100f;

    private void Start()
    {
        // 方式 1: Graphics.DrawMeshInstanced（适合静态大量物体）
        RenderInstances();

        // 方式 2: 使用 GraphicsBuffer（更灵活，支持自定义数据）
        // 见下方高级示例
    }

    private void RenderInstances()
    {
        Matrix4x4[] matrices = new Matrix4x4[1023]; // 单次最大 1023

        for (int i = 0; i < instanceCount; i += 1023)
        {
            int batch = Mathf.Min(1023, instanceCount - i);
            for (int j = 0; j < batch; j++)
            {
                int index = i + j;
                Vector3 pos = new Vector3(
                    Random.Range(-areaSize, areaSize),
                    0,
                    Random.Range(-areaSize, areaSize)
                );
                Quaternion rot = Quaternion.Euler(0, Random.Range(0, 360), 0);
                Vector3 scale = Vector3.one * Random.Range(0.8f, 1.2f);
                matrices[j] = Matrix4x4.TRS(pos, rot, scale);
            }
            Graphics.DrawMeshInstanced(instancedMesh, 0, instancedMaterial, matrices, batch);
        }
    }
}
```

#### 高级 GPU Instancing（带逐实例数据）

```csharp
using UnityEngine;

public class AdvancedGPUInstancing : MonoBehaviour
{
    [SerializeField] private Mesh mesh;
    [SerializeField] private Material material;
    [SerializeField] private int count = 10000;

    private GraphicsBuffer positionBuffer;
    private GraphicsBuffer colorBuffer;

    private void Start()
    {
        // 创建位置数据缓冲区
        Vector4[] positions = new Vector4[count];
        Color[] colors = new Color[count];

        for (int i = 0; i < count; i++)
        {
            float angle = (float)i / count * Mathf.PI * 2 * 5;
            float radius = 5f + (float)i / count * 50f;
            positions[i] = new Vector4(
                Mathf.Cos(angle) * radius,
                0,
                Mathf.Sin(angle) * radius,
                1f
            );
            colors[i] = Color.HSVToRGB((float)i / count, 1f, 1f);
        }

        positionBuffer = new GraphicsBuffer(GraphicsBuffer.Target.Structured, count, sizeof(float) * 4);
        colorBuffer = new GraphicsBuffer(GraphicsBuffer.Target.Structured, count, sizeof(float) * 4);
        positionBuffer.SetData(positions);
        colorBuffer.SetData(colors);

        material.SetBuffer("_Positions", positionBuffer);
        material.SetBuffer("_Colors", colorBuffer);
    }

    private void Update()
    {
        // 单次 DrawCall 绘制 count 个实例
        Graphics.DrawMeshInstancedProcedural(mesh, 0, material,
            new Bounds(Vector3.zero, Vector3.one * 200f), count);
    }

    private void OnDestroy()
    {
        positionBuffer?.Release();
        colorBuffer?.Release();
    }
}
```

### 1.4 SRP Batcher（Unity URP/HDRP）

```
SRP Batcher 是 Unity 可编程渲染管线（SRP）的批处理方案：

原理：不合并 DrawCall，而是减少每个 DrawCall 的 CPU 开销。
      通过缓存 CBUFFER（常量缓冲区），避免每次重新设置 Shader 参数。

启用方法：
  1. 使用 URP 或 HDRP
  2. Shader 必须使用 CBUFFER 兼容的 Uniform 声明
  3. 在 URP Asset 中勾选 "SRP Batcher"

验证：Frame Debugger 中查看是否显示 "SRP Batch"
```

### 1.5 减少材质 SetPass 数量

```
每个不同的材质/Shader Pass = 1 个额外的 SetPass Call

优化策略：
  1. 使用图集合并纹理，减少材质数量
  2. 使用 Shader 变体（Shader Keywords）替代多个 Shader
  3. 避免在运行时动态修改材质属性（会创建新材质实例）
  4. 使用 MaterialPropertyBlock 替代直接修改材质

// ❌ 错误：创建材质副本
renderer.material.color = Color.red; // 每次调用创建新材质

// ✅ 正确：使用 MaterialPropertyBlock
MaterialPropertyBlock propBlock = new MaterialPropertyBlock();
renderer.GetPropertyBlock(propBlock);
propBlock.SetColor("_BaseColor", Color.red);
renderer.SetPropertyBlock(propBlock);
```

## 2. LOD 层级细节

LOD（Level of Detail）根据物体与摄像机的距离切换不同精度的网格。

### 2.1 LOD 原理

```
距离摄像机近 → 高精度网格（面数多，细节丰富）
距离摄像机远 → 低精度网格（面数少，节省渲染和顶点处理）
非常远        → Billboard（广告牌）或不渲染
```

### 2.2 Unity LOD 配置

```csharp
using UnityEngine;

[RequireComponent(typeof(LODGroup))]
public class LODSetupHelper : MonoBehaviour
{
    [System.Serializable]
    public struct LODLevel
    {
        public Mesh mesh;
        public float screenRelativeTransitionHeight; // 屏幕占比阈值
        public bool fadeTransition;
        public float fadeTransitionWidth;
    }

    [SerializeField] private LODLevel[] lodLevels;

    private void Reset()
    {
        var lodGroup = GetComponent<LODGroup>();
        var lods = new LOD[lodLevels.Length];

        for (int i = 0; i < lodLevels.Length; i++)
        {
            var renderers = GetComponentsInChildren<MeshRenderer>();
            lods[i] = new LOD(
                lodLevels[i].screenRelativeTransitionHeight,
                renderers
            );
            lods[i].fadeMode = lodLevels[i].fadeTransition
                ? LODFadeMode.CrossFade
                : LODFadeMode.None;
        }

        // 最后一个 LOD 是 Culled（不渲染）
        var group = GetComponent<LODGroup>();
        group.SetLODs(lods);
        group.RecalculateBounds();
    }
}
```

### 2.3 LOD 层级建议

| LOD 级别 | 屏幕占比 | 三角面数建议 | 适用距离 |
|-----------|----------|-------------|----------|
| LOD0 | 60%+ | 100% (原始) | 0-15m |
| LOD1 | 30% | 50% | 15-40m |
| LOD2 | 15% | 25% | 40-100m |
| LOD3 | 5% | 10% | 100-200m |
| LOD4 | 2% | Billboard | 200m+ |
| Culled | < 1% | 0 (不渲染) | 非常远 |

### 2.4 HLOD（Hierarchical LOD）

```
Unreal Engine 专有功能：

普通 LOD: 单个物体的精度切换
HLOD:     将多个相邻物体合并为一个低精度代理网格

优势：
  - 大幅减少远处场景的 DrawCall
  - 长距离视野下的性能提升显著

配置: World Settings → HLOD System → Build HLOD
```

## 3. 对象池

对象池通过复用已创建的对象，避免频繁实例化/销毁的性能开销。

### 3.1 泛型对象池实现

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;

public class ObjectPool<T> where T : Component
{
    private readonly T prefab;
    private readonly Queue<T> pool = new();
    private readonly List<T> activeObjects = new();
    private readonly Transform parent;
    private readonly int maxSize;
    private readonly Action<T> onGet;
    private readonly Action<T> onReturn;

    public int ActiveCount => activeObjects.Count;
    public int PooledCount => pool.Count;
    public int TotalCreated { get; private set; }

    /// <summary>
    /// 创建对象池
    /// </summary>
    /// <param name="prefab">预制体</param>
    /// <param name="initialSize">初始预创建数量</param>
    /// <param name="maxSize">最大缓存数量（超出直接销毁）</param>
    /// <param name="parent">父节点</param>
    /// <param name="onGet">取出时的回调</param>
    /// <param name="onReturn">归还时的回调</param>
    public ObjectPool(
        T prefab,
        int initialSize = 10,
        int maxSize = 100,
        Transform parent = null,
        Action<T> onGet = null,
        Action<T> onReturn = null)
    {
        this.prefab = prefab;
        this.maxSize = maxSize;
        this.parent = parent;
        this.onGet = onGet;
        this.onReturn = onReturn;

        // 预热：预先创建对象
        for (int i = 0; i < initialSize; i++)
        {
            T obj = CreateInstance();
            obj.gameObject.SetActive(false);
            pool.Enqueue(obj);
        }
    }

    /// <summary>从池中获取对象</summary>
    public T Get(Vector3 position, Quaternion rotation)
    {
        T obj;

        if (pool.Count > 0)
        {
            obj = pool.Dequeue();
        }
        else
        {
            obj = CreateInstance();
        }

        obj.transform.SetPositionAndRotation(position, rotation);
        obj.gameObject.SetActive(true);
        activeObjects.Add(obj);

        onGet?.Invoke(obj);
        return obj;
    }

    /// <summary>归还对象到池中</summary>
    public void Return(T obj)
    {
        if (!activeObjects.Contains(obj))
        {
            Debug.LogWarning($"ObjectPool: Attempting to return object not from this pool: {obj.name}");
            return;
        }

        activeObjects.Remove(obj);

        if (pool.Count >= maxSize)
        {
            // 超过最大容量，直接销毁
            UnityEngine.Object.Destroy(obj.gameObject);
            TotalCreated--;
        }
        else
        {
            obj.gameObject.SetActive(false);
            obj.transform.SetParent(parent);
            onReturn?.Invoke(obj);
            pool.Enqueue(obj);
        }
    }

    /// <summary>归还所有活跃对象</summary>
    public void ReturnAll()
    {
        while (activeObjects.Count > 0)
        {
            Return(activeObjects[0]);
        }
    }

    /// <summary>预加载指定数量的对象</summary>
    public void Prewarm(int count)
    {
        for (int i = 0; i < count; i++)
        {
            T obj = CreateInstance();
            obj.gameObject.SetActive(false);
            pool.Enqueue(obj);
        }
    }

    private T CreateInstance()
    {
        T obj = UnityEngine.Object.Instantiate(prefab, parent);
        obj.name = $"{prefab.name}_Pooled_{TotalCreated}";
        TotalCreated++;
        return obj;
    }
}
```

### 3.2 对象池管理器

```csharp
using System.Collections.Generic;
using UnityEngine;

public class PoolManager : MonoBehaviour
{
    public static PoolManager Instance { get; private set; }

    [System.Serializable]
    public struct PoolConfig
    {
        public string id;
        public Component prefab;
        public int initialSize;
        public int maxSize;
    }

    [SerializeField] private PoolConfig[] poolConfigs;

    private readonly Dictionary<string, object> pools = new();
    private Transform poolRoot;

    private void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }
        Instance = this;
        DontDestroyOnLoad(gameObject);

        poolRoot = new GameObject("PoolRoot").transform;
        poolRoot.SetParent(transform);

        InitializePools();
    }

    private void InitializePools()
    {
        foreach (var config in poolConfigs)
        {
            var poolParent = new GameObject($"Pool_{config.id}").transform;
            poolParent.SetParent(poolRoot);

            var poolType = typeof(ObjectPool<>).MakeGenericType(config.prefab.GetType());
            var pool = Activator.CreateInstance(poolType,
                config.prefab, config.initialSize, config.maxSize, poolParent);

            pools[config.id] = pool;
            Debug.Log($"Pool '{config.id}' initialized with {config.initialSize} objects");
        }
    }

    public ObjectPool<T> GetPool<T>(string id) where T : Component
    {
        if (pools.TryGetValue(id, out var pool))
        {
            return pool as ObjectPool<T>;
        }
        Debug.LogError($"Pool '{id}' not found!");
        return null;
    }
}
```

### 3.3 子弹对象池示例

```csharp
using UnityEngine;

public class BulletPool : MonoBehaviour
{
    [SerializeField] private Bullet bulletPrefab;
    [SerializeField] private int initialPoolSize = 50;
    [SerializeField] private int maxPoolSize = 200;

    private ObjectPool<Bullet> pool;

    private void Awake()
    {
        pool = new ObjectPool<Bullet>(
            prefab: bulletPrefab,
            initialSize: initialPoolSize,
            maxSize: maxPoolSize,
            parent: transform,
            onGet: bullet => bullet.ResetState(),
            onReturn: bullet => bullet.StopAllCoroutines()
        );
    }

    public void Fire(Vector3 position, Vector3 direction, float speed, float damage)
    {
        Bullet bullet = pool.Get(position, Quaternion.LookRotation(direction));
        bullet.Initialize(direction, speed, damage, lifetime: 3f,
            onExpire: () => pool.Return(bullet));
    }

    public void ReturnAll()
    {
        pool.ReturnAll();
    }
}

// Bullet.cs
public class Bullet : MonoBehaviour
{
    private Vector3 direction;
    private float speed;
    private float damage;
    private float lifetime;
    private System.Action onExpire;

    public void Initialize(Vector3 dir, float spd, float dmg, float lifetime, System.Action onExpire)
    {
        this.direction = dir;
        this.speed = spd;
        this.damage = dmg;
        this.lifetime = lifetime;
        this.onExpire = onExpire;

        StartCoroutine(LifetimeRoutine());
    }

    private System.Collections.IEnumerator LifetimeRoutine()
    {
        yield return new WaitForSeconds(lifetime);
        onExpire?.Invoke();
    }

    private void Update()
    {
        transform.Translate(direction * speed * Time.deltaTime, Space.World);
    }

    private void OnTriggerEnter(Collider other)
    {
        if (other.TryGetComponent<IDamageable>(out var target))
        {
            target.TakeDamage(damage);
        }
        onExpire?.Invoke();
    }

    public void ResetState()
    {
        StopAllCoroutines();
        transform.rotation = Quaternion.identity;
    }
}

public interface IDamageable
{
    void TakeDamage(float damage);
}
```

### 4.2 避免 GC 垃圾

```csharp
using System.Collections.Generic;
using UnityEngine;

public class GCOptimization : MonoBehaviour
{
    // ===== 1. 避免在 Update 中分配 =====

    // ❌ 每帧分配新数组
    void Update_Bad()
    {
        Collider[] hits = Physics.OverlapSphere(transform.position, 5f);
        // hits 数组每帧分配，产生 GC 垃圾
    }

    // ✅ 使用预分配的数组 + NonAlloc 版本
    private readonly Collider[] hitBuffer = new Collider[32];
    private int hitCount;

    void Update_Good()
    {
        hitCount = Physics.OverlapSphereNonAlloc(transform.position, 5f, hitBuffer);
        for (int i = 0; i < hitCount; i++)
        {
            // 处理 hitBuffer[i]
        }
    }


    // ===== 2. 字符串优化 =====

    // ❌ 每帧拼接字符串（产生 GC）
    private int score = 0;
    void Update_Bad2()
    {
        GetComponent<TMPro.TMP_Text>().text = "Score: " + score;
        // 每次拼接创建新字符串
    }

    // ✅ 仅在分数变化时更新
    private int lastScore = -1;
    void Update_Good2()
    {
        if (score != lastScore)
        {
            lastScore = score;
            GetComponent<TMPro.TMP_Text>().text = $"Score: {score}";
        }
    }


    // ===== 3. 闭包陷阱 =====

    // ❌ Lambda 捕获变量，产生闭包对象
    void Start_Bad()
    {
        for (int i = 0; i < 10; i++)
        {
            var button = new UnityEngine.UI.Button();
            button.onClick.AddListener(() => Debug.Log(i));
            // 每个 Lambda 都会创建一个闭包对象（GC 垃圾）
        }
    }

    // ✅ 使用对象方法替代闭包
    void Start_Good()
    {
        for (int i = 0; i < 10; i++)
        {
            var button = new UnityEngine.UI.Button();
            int captured = i;
            button.onClick.AddListener(() => Debug.Log(captured));
            // 仍有闭包，但只捕获一个 int
        }
    }


    // ===== 4. 使用 ObjectPool 替代 List 频繁分配 =====

    // ❌ 频繁 new List
    void Process_Bad()
    {
        var nearby = new List<Enemy>(); // 每次调用分配
        // ...
    }

    // ✅ 复用 List
    private readonly List<Enemy> nearbyEnemies = new();

    void Process_Good()
    {
        nearbyEnemies.Clear();
        // 填充 nearbyEnemies...
        // Clear 不会释放内部数组，复用容量
    }


    // ===== 5. 使用 StackAlloc 替代堆分配（高级） =====

    // 对于小量临时数据，使用 stackalloc
    // 注意：仅在 unsafe 上下文或使用 Span<T> 时可用
    // Unity 2022+ 支持

    // ===== 6. foreach vs for =====

    // ❌ 旧版 Unity 中 foreach 会分配 Enumerator
    // （Unity 5.5+ 已修复，但某些自定义集合仍有问题）

    // ✅ 对于 List<T>，使用 for 循环
    private List<Transform> targets = new();
    void ProcessTargets()
    {
        for (int i = 0; i < targets.Count; i++)
        {
            // targets[i]...
        }
    }
}
```

### 4.3 资源管理

```csharp
using UnityEngine;
using System.Collections.Generic;

public class ResourceManager : MonoBehaviour
{
    // 资源引用缓存
    private readonly Dictionary<string, Object> cache = new();
    private readonly Dictionary<string, int> refCount = new();

    /// <summary>加载资源（带缓存）</summary>
    public T Load<T>(string path) where T : Object
    {
        if (cache.TryGetValue(path, out var asset))
        {
            refCount[path]++;
            return asset as T;
        }

        T loadedAsset = Resources.Load<T>(path);
        if (loadedAsset != null)
        {
            cache[path] = loadedAsset;
            refCount[path] = 1;
        }
        return loadedAsset;
    }

    /// <summary>释放资源引用</summary>
    public void Release(string path)
    {
        if (!refCount.ContainsKey(path)) return;

        refCount[path]--;
        if (refCount[path] <= 0)
        {
            cache.Remove(path);
            refCount.Remove(path);
            Resources.UnloadUnusedAssets();
        }
    }

    /// <summary>卸载所有未引用的资源</summary>
    public void UnloadUnused()
    {
        var keysToRemove = new List<string>();
        foreach (var kvp in refCount)
        {
            if (kvp.Value <= 0)
                keysToRemove.Add(kvp.Key);
        }

        foreach (var key in keysToRemove)
        {
            cache.Remove(key);
            refCount.Remove(key);
        }

        Resources.UnloadUnusedAssets();
        System.GC.Collect();
    }
}
```

## 6. CPU 优化

### 6.1 缓存友好的数据布局

```csharp
// ❌ 面向对象布局（Cache 不友好）
public class EnemyData
{
    public int health;
    public float speed;
    public Vector3 position;
}

// 遍历时每个对象的内存地址不连续，Cache Miss 频繁
public class BadManager : MonoBehaviour
{
    private List<EnemyData> enemies = new();
    void Update()
    {
        foreach (var enemy in enemies)
        {
            enemy.health -= 1; // 跳跃访问内存
        }
    }
}

// ✅ DOD 数据导向设计（Cache 友好，即 ECS 的核心思想）
public struct EnemyDataArray
{
    public int[] healths;
    public float[] speeds;
    public Vector3[] positions;
    public int count;
}

public class GoodManager : MonoBehaviour
{
    private EnemyDataArray enemies;

    void Update()
    {
        // 连续内存访问，CPU Cache 命中率高
        for (int i = 0; i < enemies.count; i++)
        {
            enemies.healths[i] -= 1;
        }
    }
}
```

### 6.2 多线程与 Job System

```csharp
using Unity.Collections;
using Unity.Jobs;
using Unity.Mathematics;
using UnityEngine;

// Unity Job System 示例：并行计算大量敌人移动
public struct EnemyMoveJob : IJobParallelFor
{
    public NativeArray<float3> positions;
    public NativeArray<float3> velocities;
    public float deltaTime;

    public void Execute(int index)
    {
        positions[index] += velocities[index] * deltaTime;
    }
}

public class JobSystemDemo : MonoBehaviour
{
    private NativeArray<float3> positions;
    private NativeArray<float3> velocities;
    private const int enemyCount = 50000;

    private void Start()
    {
        positions = new NativeArray<float3>(enemyCount, Allocator.Persistent);
        velocities = new NativeArray<float3>(enemyCount, Allocator.Persistent);

        // 初始化数据
        for (int i = 0; i < enemyCount; i++)
        {
            positions[i] = new float3(
                UnityEngine.Random.Range(-100f, 100f), 0, UnityEngine.Random.Range(-100f, 100f));
            velocities[i] = new float3(
                UnityEngine.Random.Range(-1f, 1f), 0, UnityEngine.Random.Range(-1f, 1f));
        }
    }

    private void Update()
    {
        var moveJob = new EnemyMoveJob
        {
            positions = positions,
            velocities = velocities,
            deltaTime = Time.deltaTime
        };

        // 调度并行 Job（batchSize 控制每个 Worker 的工作量）
        JobHandle handle = moveJob.Schedule(enemyCount, 64);
        handle.Complete();
    }

    private void OnDestroy()
    {
        positions.Dispose();
        velocities.Dispose();
    }
}
```

### 7.1 纹理优化

### 7.2 Shader 优化

```hlsl
// ❌ 低效 Shader：分支 + 复杂计算
float4 Frag(Varyings input) : SV_Target
{
    float4 color = 0;
    if (input.uv.x > 0.5)  // 分支在 GPU 上低效
    {
        color = tex2D(_MainTex, input.uv);
        color.rgb = pow(color.rgb, 2.2); // 逐像素 pow
    }
    else
    {
        color = tex2D(_SecondaryTex, input.uv);
        color.rgb = sin(color.rgb * 10.0);
    }
    return color;
}

// ✅ 优化 Shader：避免分支，使用 step/lerp
float4 Frag(Varyings input) : SV_Target
{
    float t = step(0.5, input.uv.x); // 无分支选择
    float4 color1 = tex2D(_MainTex, input.uv);
    float4 color2 = tex2D(_SecondaryTex, input.uv);
    float4 color = lerp(color2, color1, t);
    return color;
}
```

### 7.3 Overdraw 优化

```
Overdraw = 同一像素被多次绘制

诊断:
  Unity: Scene View → Overdraw 模式
  Unreal: r.Visibility 2

优化策略:
  1. 前向渲染 vs 延迟渲染
     - 前向: 适合透明物体多的场景
     - 延迟: 适合复杂光照、少量透明物体
  2. 排序（从近到远绘制不透明物体）
  3. 遮挡剔除
  4. 简化粒子效果（减少叠加层）
```

## 8. 最佳实践总结

### 8.1 性能优化优先级

```
1. 先 Profile，再优化（不要猜测瓶颈）
2. 渲染优化（最大收益）
   ├── DrawCall 批处理
   ├── LOD 设置
   ├── 纹理压缩
   └── 遮挡剔除
3. 逻辑优化
   ├── 对象池
   ├── 避免 GC
   ├── 缓存引用
   └── 多线程
4. 内存优化
   ├── 资源卸载
   ├── 场景分块加载
   └── 减少常驻内存
5. 微优化（收益递减）
   ├── 内联函数
   ├── 位运算
   └── 结构体 vs 类
```
