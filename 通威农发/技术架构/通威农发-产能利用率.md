# 产能利用率

> 项目：**通威农发**（通威股份农业发展有限公司）
>
> 相关Wiki：[数据库总览](通威农发-数据库总览.md) | [报表预处理中间表](通威农发-报表预处理中间表.md) | [E2D与数据同步](通威农发-E2D与数据同步.md) | [EDO设备管理](通威农发-EDO设备管理.md) | [LM精益管理](通威农发-LM精益管理.md)

## 概述

产能利用率是衡量生产资源实际使用程度的指标，涉及产线产能基准、设备产能参数、实际产出统计和利用率计算等多个维度。通威农发项目中，产能利用率相关数据分散在多个模块中，核心表包括：

| 序号 | 表名 | 所属模块 | 说明 |
|------|------|---------|------|
| 1 | lm_st_capacity_utilize_rate | 报表预处理中间表 | 产能利用率主表 |
| 2 | e2d_api_gme_prod_line | E2D与数据同步 | DIM产线产能基准数据 |
| 3 | lm_st_product_inbound_query | 报表预处理中间表 | 成品入库表（含利用率字段） |
| 4 | edo_equip_account | EDO设备管理 | 设备台账（含产能参数） |

## 1. 产能利用率主表（lm_st_capacity_utilize_rate）

> 所属模块：[报表预处理中间表](通威农发-报表预处理中间表.md)

此表是产能利用率分析的核心汇总表，按年月维度记录各公司的产能利用率数据。

### 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | varchar(64) | 主键 |
| company_code | varchar(64) | 公司编码 |
| company_name | varchar(64) | 公司名称 |
| hourly_output | varchar(64) | 工时折合产量 |
| qty | varchar(64) | 标准月产量 |
| date | varchar(64) | 年月（如：202605） |
| capacity_utilization | varchar(64) | **产能利用率** |
| team_reach_standard | varchar(64) | 单班产能达成率 |
| team_hour_output | varchar(64) | 班组时产量 |
| tenant_id | varchar(64) | 租户ID |
| create_user | varchar(64) | 创建人 |
| create_date | timestamp | 创建时间 |
| update_user | varchar(64) | 更新人 |
| update_date | timestamp | 更新时间 |
| company_id | varchar(64) | 公司ID |

### 类型说明

> ⚠️ 所有数值字段（hourly_output、qty、capacity_utilization、team_reach_standard、team_hour_output）均存储为 varchar(64)，查询时需使用 CAST 转换为数值类型再进行聚合运算。

## 2. 产线产能基准（e2d_api_gme_prod_line）

> 所属模块：[E2D与数据同步](通威农发-E2D与数据同步.md)

此表存储来自EBS系统的产线产能基准数据（DIM维度表），包含设计产能、核定产能、极限产能等多个产能层级。

### 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | varchar(64) | 主键 |
| pri_key | varchar(64) | 唯一主键 |
| organization_code | varchar(64) | 库存组织编码 |
| department_code | varchar(64) | 车间代码 |
| prodline_code | varchar(64) | 产线代码 |
| prodline_module | varchar(64) | 产线类型1 |
| start_date | timestamp | 投产日期 |
| end_date | timestamp | 停产日期 |
| uom | varchar(64) | 单位 |
| hour_qty | varchar(64) | **设计时产能** |
| standard_day_qty | varchar(64) | **标准日产能** |
| standard_month_qty | varchar(64) | **标准月产能** |
| standard_year_qty | varchar(64) | **标准年产能** |
| max_day_qty | varchar(64) | **极限日产能** |
| max_month_qty | varchar(64) | **极限月产能** |
| max_year_qty | varchar(64) | **极限年产能** |
| description | varchar(256) | 备注 |
| attribute1 | varchar(64) | 产线类型编码 |
| line_type | varchar(64) | 产线类型2 |
| check_hour_qty | varchar(64) | **核定时产能** |
| check_day_qty | varchar(64) | **核定日产能** |
| check_month_qty | varchar(64) | **核定月产能** |
| check_year_qty | varchar(64) | **核定年产能** |
| sink_hour_qty | varchar(64) | **核定沉料时产能** |
| sink_day_qty | varchar(64) | **核定沉料日产能** |
| sink_month_qty | varchar(64) | **核定沉料月产能** |
| sink_year_qty | varchar(64) | **核定沉料年产能** |
| effective_date | timestamp | 有效时间 |
| failure_date | timestamp | 失效时间 |
| last_update_date | varchar(30) | 最后修改日期 |
| tenant_id | varchar(64) | 租户ID |
| create_user | varchar(64) | 创建人 |
| create_date | timestamp | 创建时间 |
| update_user | varchar(64) | 更新人 |
| update_date | timestamp | 更新时间 |

### 产能层级说明

此表包含四个产能维度，分别对应不同的业务场景：

- **设计产能**（hour_qty → standard_year_qty）：设备设计标准，作为基准参考
- **核定产能**（check_hour_qty → check_year_qty）：实际核定值，用于日常排产
- **极限产能**（max_day_qty → max_year_qty）：短期峰值产能，用于紧急插单评估
- **沉料产能**（sink_hour_qty → sink_year_qty）：沉料工艺专用产能参数

### 索引

- PRIMARY KEY (`id`)
- INDEX `idx_E2DApiGmeProdLine_pri_key_date` (`pri_key`, `create_date`)
- INDEX `idx_gme_orgcode` (`organization_code`)

## 5. 产能利用率查询SQL模板

### 5.1 按公司/月份统计产能利用率

```sql
SELECT
    company_name,
    `date`,
    CAST(capacity_utilization AS DECIMAL(10,2)) AS capacity_utilization,
    CAST(hourly_output AS DECIMAL(10,2)) AS hourly_output,
    CAST(qty AS DECIMAL(10,2)) AS qty
FROM lm_st_capacity_utilize_rate
WHERE `date` = '202605'
ORDER BY capacity_utilization DESC;
```

### 5.2 产线产能与实际产出对比

```sql
SELECT
    p.prodline_code,
    p.prodline_module,
    CAST(p.check_day_qty AS DECIMAL(10,2)) AS check_day_qty,
    CAST(p.standard_day_qty AS DECIMAL(10,2)) AS standard_day_qty,
    q.total_output,
    q.capacity_utilization
FROM e2d_api_gme_prod_line p
LEFT JOIN lm_st_product_inbound_query q
    ON q.product_line = p.prodline_code
WHERE p.check_day_qty > 0
ORDER BY q.total_output DESC;
```

### 5.3 设备产能参数与产线基准关联查询

```sql
SELECT
    e.code AS equip_code,
    e.name AS equip_name,
    e.capacity AS equip_capacity,
    e.design_power,
    p.prodline_code,
    CAST(p.check_hour_qty AS DECIMAL(10,2)) AS line_hour_qty
FROM edo_equip_account e
LEFT JOIN e2d_api_gme_prod_line p
    ON e.line_code = p.prodline_code
WHERE e.run_state = '运行'
ORDER BY e.capacity DESC;
```
