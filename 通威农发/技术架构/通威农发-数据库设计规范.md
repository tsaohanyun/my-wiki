# 通威农发-数据库设计规范

## 概述

本文档定义了通威农发项目的数据库设计规范，确保数据库设计的一致性和可维护性。

## 1. 命名规范

### 表命名

```
[模块前缀]_[业务域]_[实体]

模块前缀：
- mes_  : MES生产执行
- wms_  : WMS仓储管理
- qms_  : QMS质量管理
- edo_  : EDO设备管理
- lm_   : LM精益管理
- mi_   : MI制造智能
- aps_  : APS计划排程
- mbm_  : MBM物料BOM
- common_ : 公共模块

示例：
- mes_work_order      : 生产工单
- wms_inventory        : 库存
- qms_inspection       : 检验记录
- edo_equipment        : 设备
```

### 字段命名

```
[表名缩写]_[属性名]

规则：
1. 全小写，下划线分隔
2. 避免使用数据库保留字
3. 布尔字段用is_前缀
4. 时间字段用_time后缀
5. 状态字段用_status后缀

示例：
- order_id        : 工单号
- order_status    : 工单状态
- create_time     : 创建时间
- is_deleted      : 是否删除
```

### 索引命名

```
[索引类型]_[表名缩写]_[字段名]

索引类型：
- pk_   : 主键
- uk_   : 唯一索引
- idx_  : 普通索引
- fk_   : 外键索引

示例：
- pk_order_id        : 主键
- uk_product_code    : 唯一索引
- idx_create_time    : 普通索引
- fk_product_id      : 外键索引
```

## 2. 字段规范

### 必备字段

```sql
-- 每个表必须包含以下字段
validflag CHAR(1) NOT NULL DEFAULT '1' COMMENT '有效标志: 1有效/0删除',
create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
create_by VARCHAR(32) NOT NULL COMMENT '创建人',
update_time DATETIME COMMENT '修改时间',
update_by VARCHAR(32) COMMENT '修改人'
```

### 数据类型选择

| 场景 | 推荐类型 | 说明 |
|------|----------|------|
| 主键 | VARCHAR(32) | 使用UUID或业务编码 |
| 外键 | VARCHAR(32) | 与主表保持一致 |
| 状态 | VARCHAR(20) | 使用枚举值 |
| 数量 | DECIMAL(10,2) | 精确到2位小数 |
| 金额 | DECIMAL(12,2) | 精确到2位小数 |
| 时间 | DATETIME | 包含时分秒 |
| 日期 | DATE | 仅日期 |
| 文本 | VARCHAR(500) | 短文本 |
| 大文本 | TEXT | 长文本 |
| 布尔 | CHAR(1) | '1'/'0' |

### 逻辑删除

```sql
-- 使用validflag实现逻辑删除
-- 查询时必须添加条件
WHERE validflag = '1'

-- 删除时更新状态
UPDATE table_name 
SET validflag = '0', 
    update_time = NOW(),
    update_by = 'user_id'
WHERE id = 'xxx';
```

## 3. 索引设计规范

### 索引原则

1. **选择性原则**：选择性高的字段优先建索引
2. **最左前缀**：组合索引遵循最左前缀原则
3. **覆盖索引**：常用查询尽量使用覆盖索引
4. **避免过度**：单表索引不超过5个

### 常用索引模式

```sql
-- 1. 主键索引（自动）
PRIMARY KEY (order_id)

-- 2. 外键索引
INDEX fk_product_id (product_id)

-- 3. 状态+时间组合索引
INDEX idx_status_time (order_status, create_time)

-- 4. 业务查询索引
INDEX idx_batch_date (batch_date, line_id)

-- 5. 唯一索引
UNIQUE uk_order_no (order_no)
```

### 索引优化建议

```sql
-- 查看表索引
SHOW INDEX FROM table_name;

-- 分析查询性能
EXPLAIN SELECT * FROM table_name WHERE ...;

-- 查看索引使用情况
SELECT 
    index_name,
    column_name,
    cardinality
FROM information_schema.statistics
WHERE table_name = 'table_name'
ORDER BY index_name, seq_in_index;
```

## 4. 分区策略

### 按时间分区

```sql
-- 按月分区（适用于日志表）
CREATE TABLE mes_operation_log (
    log_id VARCHAR(32) NOT NULL,
    operation_time DATETIME NOT NULL,
    -- 其他字段
    PRIMARY KEY (log_id, operation_time)
) PARTITION BY RANGE (TO_DAYS(operation_time)) (
    PARTITION p202601 VALUES LESS THAN (TO_DAYS('2026-02-01')),
    PARTITION p202602 VALUES LESS THAN (TO_DAYS('2026-03-01')),
    -- ...
    PARTITION p_future VALUES LESS THAN MAXVALUE
);
```

### 按业务分区

```sql
-- 按工厂分区（适用于多工厂）
CREATE TABLE mes_work_order (
    order_id VARCHAR(32) NOT NULL,
    factory_id VARCHAR(32) NOT NULL,
    -- 其他字段
    PRIMARY KEY (order_id, factory_id)
) PARTITION BY LIST (factory_id) (
    PARTITION p_factory01 VALUES IN ('F001', 'F002'),
    PARTITION p_factory02 VALUES IN ('F003', 'F004'),
    PARTITION p_other VALUES IN (DEFAULT)
);
```

## 5. SQL编写规范

### 查询规范

```sql
-- 1. 避免SELECT *
SELECT order_id, order_status, create_time
FROM mes_work_order
WHERE validflag = '1';

-- 2. 使用参数化查询
SELECT * FROM table_name WHERE id = ?;

-- 3. 避免在WHERE中使用函数
-- ❌ 错误
SELECT * FROM table_name WHERE DATE(create_time) = '2026-06-01';
-- ✅ 正确
SELECT * FROM table_name 
WHERE create_time >= '2026-06-01' 
    AND create_time < '2026-06-02';

-- 4. 使用LIMIT限制结果集
SELECT * FROM table_name 
WHERE validflag = '1'
ORDER BY create_time DESC
LIMIT 100;
```

### 更新规范

```sql
-- 1. 使用事务
START TRANSACTION;
UPDATE table_name SET field1 = 'value1' WHERE id = 'xxx';
UPDATE table_name SET field2 = 'value2' WHERE id = 'xxx';
COMMIT;

-- 2. 记录修改信息
UPDATE table_name 
SET field = 'new_value',
    update_time = NOW(),
    update_by = 'user_id'
WHERE id = 'xxx';

-- 3. 批量更新使用CASE WHEN
UPDATE table_name 
SET status = CASE 
    WHEN condition1 THEN 'status1'
    WHEN condition2 THEN 'status2'
    ELSE 'default'
END
WHERE id IN ('id1', 'id2', 'id3');
```
