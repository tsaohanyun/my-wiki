# 项目知识备忘

> 本页面记录通威农发项目实施过程中的业务知识、数据含义备注和特殊约定。当从正式文档中无法获知某个表/字段的业务背景时，可以在这里查找。

## 数据表备注

### 配料相关

| 表名 | 含义 | 备注 |
|------|------|------|
| `st_mes_distributing` | 配料记录主表（当前） | MES配料模块的核心数据表，包含理论重量、实际重量、配料属性等。`distribute_from` 字段区分来源：SND=散料，ZK=正康（某供应商/产线类型） |
| `st_mes_distributing_20250526` | 2025年5月26日启用的新配料表 | **不是归档表**，是因业务变更（表结构调整）而启用的新版配料记录表。2025年5月26日之后的新数据写入此表，之前的数据仍在 `st_mes_distributing`。视图需要UNION两张表才能覆盖完整时间范围 |
| `mes_distributing_item` | 配料明细行 | 每次配料操作的物料明细记录 |
| `mes_distributing_t` | 配料任务头 | 配料任务级信息，包含 `distribute_from`、`distributing_batch`、`team_id` 等 |

### 字段含义

| 字段 | 所在表 | 含义 |
|------|--------|------|

| `distributing_attr` | st_mes_distributing | 配料属性/混合差异标识 |
| `distributing_batch` | st_mes_distributing | 配料批次号 |
| `theory_weight` | st_mes_distributing | 理论重量（配方要求的投料量） |
| `reality_weight` | st_mes_distributing | 实际重量（实际投料量） |
| `citem_id` | st_mes_distributing | 成品物料ID |
| `yitem_id` | st_mes_distributing | 原料物料ID |
| `memo` | st_mes_distributing | 仓筒号（SILOID） |
| `region_name` | st_mes_distributing | 班组/区域名称 |
| `shift_name` | st_mes_distributing | 班次名称 |
| `workorder_no` | st_mes_distributing | 工单号 |
| `inv_org_id` | st_mes_distributing | 库存组织ID（对应工厂） |

## 视图备注

| 视图名 | 用途 | 关键维度 |
|--------|------|----------|
| `view_qms_distributing_t` | 班组级配料偏差分析 | 按日期+工厂+物料+配料批+班组+班次分组，用于分析各班组的配料精度 |
| `view_qms_distributing_bias` | 仓筒级配料偏差分析 | 按日期+工厂+成品+原料+仓筒+配料批分组，用于分析各仓筒的配料精度 |

### 视图数据来源对比

两个视图主体数据来自同一张表 `st_mes_distributing`，并都通过UNION补上 `st_mes_distributing_20250526` 覆盖完整时间范围。区别仅在于**分析视角**（班组 vs 仓筒）和**聚合方式**（`_t`的SND分支有双层子查询嵌套）。

## 业务术语

| 术语 | 含义 |

| 配料偏差 | 理论重量与实际重量的差异，用于衡量配料精度 |

## 技术约定

- **定时任务调度**：通威农发项目使用 **XXL-Job** 管理定时任务。所有定时脚本的触发由 XXL-Job 控制，代码中不需要实现触发时间逻辑，只暴露接口/入口供调度器调用即可。
- **审计字段更新规则**：ETL/转换类脚本写入数据时，`create_user`、`create_date`、`update_user`、`update_date` 四个审计字段必须按实际情况更新（新增时写 create 四项，更新时写 update 两项），不需要额外新增 ETL 标记字段。

### 绩效指标计算相关

指标计算通过IIDP平台API触发，配置存储在多张表中，结果写入三张表（职责不同）。

**配置表链路**：

| 表 | 角色 | 关键字段 |
|----|------|----------|
| `lm_indicator_standard` | 标准头表 | standard_code, standard_name, count_length |
| `lm_indicator_standard_detail` | 指标行（一级PID指向standard） | target_code, target_name, indicator_record_id, indicator(周期) |
| `lm_indicator_record` | 公式定义 | indicator_code, formula（如"生产费用人工费/合格产量"） |
| `lm_indicator_record_table_property` | 公式变量→数据源映射 | model_table(源表), custom_property_id(列名), sql_function(聚合), as_name(变量别名), standard_range(过滤条件如"科目编码 LIKE (500104%)") |

**结果表（三张表职责区分）**：

| 表 | 角色 | 写入方式 | validflag行为 |
|----|------|----------|--------------|
| `lm_st_summary_table` | **最终结果表** | 先删后写（旧数据UPDATE validflag=0再INSERT） | 有效数据validflag=1 |
| `lm_indicator_result` | **指标结果表** | 追加写入，**不做清理** | 全部validflag=1，历史全保留 |
| `lm_st_summary_log_table` | **过程日志表** | 追加写入，不清理 | 全部validflag=1 |

> 核对计算结果时以 `lm_st_summary_table` (validflag=1) 为准。

**API调用方式**：通过跳板机(2223端口)上传BAT脚本执行curl，调用IIDP的JSON-RPC接口。详见skill `iidp-indicator-calc`。

## 特殊约定

- **IIDP API Token获取**：IIDP平台没有独立REST登录接口，需从浏览器F12→Network复制 `Authorization: Bearer *** 头
- **Windows跳板机执行curl**：不能直接SSH传参（引号转义问题），必须写BAT文件(CRLF+chcp 65001)+SCP上传+SSH执行，用 `-d @文件路径` 传递JSON body

### CPT报表财务指标取数逻辑（2026-06-16整理）

所有财务指标最终都从3张源表取数，通过公式层组合运算：

**源表**：
| 表 | 取数字段 | 用途 |
|----|---------|------|
| `lm_st_balances` | `period_net`(发生额), `ending_balance`(余额) | 按科目编码过滤，取财务费用数据 |
| `lm_st_product_inbound_cost` | `quantity`(数量) | 合格产量，`物料类别='FG'` |
| `lm_st_save_variable_cost` | `cost_saving`(节约额) | 变动费用节约额（独立表） |

**取数配置链路**：`lm_indicator_standard_detail` → `indicator_record_id` → `lm_indicator_record`(公式) → `lm_indicator_record_table_property`(变量→源表映射)

**三类取数模式**：
1. **直取型**：1条配置，单科目，无公式运算（如CWZBKM11燃动总额）
2. **求和型**：N条配置，多科目，公式用`+`拼接（如CWZBKM09汽费=燃油+燃煤+蒸汽+天然气+生物质）
3. **商型**：N条费用+1条产量，公式用`/`（如DSCFY吨生产费=(折旧+非折旧+人工+燃动)/合格产量）

**关键科目编码映射**（`lm_st_balances.standard_range`过滤条件）：
| 科目编码 | 含义 | 对应指标 |
|----------|------|---------|
| `500103%` | 生产成本_燃料及动力(全部) | CWZBKM11燃动费用总额 |
| `5001030101`~`06` | 燃油/煤/电/蒸汽/天然气/生物质 | CWZBKM09汽费(子项) |
| `500104%` | 生产成本_直接人工 | CWZBKM13直接人员费用 |
| `5001050101` | 制造费用_折旧 | CWZBKM16吨折旧 |
| `5001050102` | 制造费用_非折旧 | CWZBKM01非折旧 |
| `510101%`~`510125%` | 制造费用各子项(工资/福利/社保等) | ZZFY系列 |

### lm_st_save_variable_cost 源表数据质量问题（2026-06-17发现）

**症状**：BDSCFYJYE等指标计算结果偏大

**根因**：源表存在两层问题叠加：
1. **精确重复行**：同一 `(company_id, company_name, date, cost_saving)` 组合出现多条（id不同但业务字段完全相同）
2. **同company_code关联多个company_id**：如 `2043`（越南公司）同时关联 `05bq93r29qzdw`（OU级）和 `0427l9ax70sui`（租户级），GROUP BY时被当作不同组分别聚合

**诊断SQL**：
```sql
-- 检查精确重复
SELECT company_id, company_name, date, cost_saving, COUNT(*) as cnt
FROM lm_st_save_variable_cost
WHERE company_code = '2043' AND date = '20251001'
GROUP BY company_id, company_name, date, cost_saving HAVING cnt > 1;

-- 检查同company_code多company_id
SELECT company_code, COUNT(DISTINCT company_id) as id_cnt
FROM lm_st_save_variable_cost GROUP BY company_code HAVING id_cnt > 1;
```

**修复方向**：源表去重 + ETL写入时统一company_id维度
