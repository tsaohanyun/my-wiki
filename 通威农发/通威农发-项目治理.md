# 通威农发-项目治理

> 项目：**通威农发** — 智能工厂项目治理体系（章程/SOW/组织/规范）

## 概述

项目治理是通威农发智能工厂项目的管理骨架，涵盖项目章程（正式授权文件）、SOW工作说明书（范围与职责界定）、组织架构（甲乙双方7层组织）、计划管理（滚动规划+渐近明细）、沟通管理、问题/风险管理、文档管理、人员管理和验收标准。本文档从项目章程和SOW中提取核心治理规则，供项目团队查阅。

## 项目章程核心条款

### 项目目标

智能工厂项目目标是将整体生产业务从原有系统中完整剥离，重构生产业务管理系统，统一生产业务平台，兼顾功能性、经济性，满足未来5-10年发展，助力生产业务实现**标准化、自动化、透明化、智能化、精益化**升级。

### 实施范围

| 维度 | 内容 |
|------|------|
| 实施主体 | 通威农业发展有限公司 |
| 实施范围 | 通威农发及下属国内外饲料分子公司 |
| 试点公司 | 2家 |
| 推广公司 | 65家（国内57家+越南6家+孟加拉1家+印尼1家） |
| 实施策略 | 先试点后推广 |
| 试点上线 | 2024年5月31日 |
| 推广上线 | 2024年10月1日 |

### 基本约定

1. 项目章程与合同冲突时，除双方协商一致修改外，以合同为准
2. 双方协商一致的修改，以修改后双方签署的文件为准

## 组织架构

### 项目组织结构

项目组采用**管理委员会→总监→项目经理→执行经理→实施组/开发组/精益组**七层架构。

### 核心角色

| 序号 | 组名 | 甲方 | 乙方 |
|------|------|------|------|
| 1 | 项目管理委员会 | 5人（含高层领导） | 3人（含高层领导） |
| 2 | 项目总监 | 2人 | 2人 |
| 3 | 项目经理 | 1人 | 2人 |
| 4 | 项目执行经理 | 1人 | 1人 |
| 5 | 质量管理&PMO | 3人 | 2人 |
| 6 | 规划组 | 4人（统管各实施小组方案） | 6人 |
| 7 | 实施组 | 18人 | 20人 |
| 8 | 开发组 | 2人 | 19人 |

### 职责分工

**甲方核心职责：**
- **管理委员会**：总体目标决策，听取阶段汇报
- **项目总监**：重大业务流程和问题决策
- **PMO**：监督项目执行，关键节点评估
- **项目经理**：对接乙方PM，推进项目工作，对进度质量负责
- **实施组**：业务需求提出者和决策者，负责UAT签字确认
- **开发组**：系统配置/数据集成/开发/测试
- **精益组**：辅助调研，共创提升方案

**乙方核心职责：**
- **项目经理**：制定计划，控制监督进度质量
- **实施组**：用户访谈→需求分析→蓝图方案→技术设计→SIT/UAT→培训→上线支持
- **开发组**：技术架构/集成方案/开发/数据转换
- **精益组**：参与调研，编制蓝图方案，知识转移

## 计划管理

### 编制方式

采用**滚动规划（Rolling Wave Planning）**和**渐近明细**相结合的方式：
- 蓝图方案阶段：根据合同和SOW编制项目主计划
- 蓝图评审通过后：根据方案内容细化后续阶段计划
- 以此类推，逐阶段细化

### 计划层级

| 层级 | 制定人 | 细化粒度 |
|------|--------|---------|
| 总体实施计划 | 项目经理 | 资源到角色，时间到天 |
| 阶段计划 | 项目经理 | 资源到角色，时间到天 |
| 月度计划 | 项目经理 | 资源到角色，时间到天 |
| 双周滚动计划 | 实施组长 | 任务落实到具体负责人 |
| 专题工作计划 | 项目经理 | 按需 |

### 检查机制

- **日**：各任务执行者下班前汇报完成情况，未完成需说明原因和补救措施
- **周**：每周五14:00召开周例会，总结本周+安排下周
- **不定期**：项目经理和助理检查各成员工作进度

## 沟通管理

### 沟通渠道

| 渠道 | 用途 |
|------|------|
| 金山在线文档 | 业务问题/风险反馈 |
| 会议 | 正式会议和面对面沟通 |
| i通威 | 各类业务沟通 |
| 电话 | 各类业务沟通 |
| 邮件 | 关键性工作及成果的汇报确认 |

### 沟通原则

1. 提倡面对面沟通，提高效率
2. 重要事项召开正式会议，重要结论邮件确认
3. **严禁使用微信讨论与项目有关的事**
4. 在线文件通过金山云填报
5. 发现问题或潜在风险，通过风险升级机制逐级上报

### 会议制度

| 类型 | 内容 | 召集人 | 频率 |
|------|------|--------|------|
| 阶段汇报会议 | 上一阶段总结+下一阶段规划 | 项目经理 | 每阶段 |
| 项目例会 | 进度检查/问题确认/计划安排 | 双方PM | 每周五14:00-16:00 |
| 专题性会议 | 解决特定问题 | 赛意专题负责人 | 按需 |

### 文档编码规则

格式：`TWAD_IF_XXX.###_文件名_v*.*_YYYYMMDD`

- TWAD_IF 为统一编号前缀
- XXX 为交付物清单中的数字编号
- .### 为子文档编号（如 .PLN 表示计划管理子文档）
- v*.* 版本号：草稿从v0.1开始，甲方确认后为v1.0，修改从v1.1开始
- YYYYMMDD 为日期（草稿可不填，正式版本必须填写）

### 文档传递规则

- 文档上传至通威WPS文档库，**禁止通过微信及微信群方式传递**
- 日报/周报/确认性文档通过邮件传递
- 交付物由乙方项目助理统一命名归档

## 人员管理

### 入场程序

1. 乙方执行经理每周四提交下周拟入场人员信息
2. 入场前签订《项目人员进场承诺函》
3. 入场后到甲方项目助理处报到，接受入场培训
4. 需开通权限：i通威、网络准入、VPN（按需）、堡垒机（按需）、一卡通（可选）

### 考勤规范

| 季节 | 工作时间 | 午休 |
|------|---------|------|
| 5月-9月 | 8:00-17:30 | 12:00-13:30 |
| 10月-次年4月 | 8:30-17:30 | 12:00-13:00 |

- 使用i通威"签到"功能打卡
- 出差使用"外勤签到"
- 加班也需在i通威打卡

### 现场管理

- 全办公区无烟区，违者**一人一次5万元扣除实施费**
- 按5S标准保持办公场所整洁
- 严禁使用微信/QQ等外部通讯工具讨论项目
- 禁止使用向日葵等外部远程工具

### 离场程序

- 更换离场：乙方1周内推荐替代人选，新旧人员≥1周工作交接
- 阶段任务完成离场：乙方执行经理提出计划→甲方评审→组织离场
- 手续：提交双方执行经理签字的交接表→甲方注销权限

## 验收标准

### 问题分级

| 级别 | 定义 |
|------|------|
| 1级 | 系统完全不可用，工作无法继续 |
| 2级 | 系统丧失重要服务功能，丢失重要资源 |
| 3级 | 系统丧失较少服务功能，丢失较少资源 |
| 4级 | 对业务无影响，用户工作正常 |

### 验收条件

1. 提交所有约定交付件（实施方案/配置文档/开发文档/操作说明等）
2. 所有1/2/3级问题已处理
3. 4级问题已提交解决计划
4. 系统上线后正常使用**3个月**

### 验收流程

1. 乙方提出书面验收申请
2. 甲方5个工作日内回复
3. 验收合格→双方签署《系统验收报告》
4. 验收不合格→甲方书面提出整改要求→乙方限期整改
5. 4级问题作为遗留问题清单，乙方提交解决计划

### 关键签字交付物（7份）

1. 《调研报告》
2. 《SOW项目工作说明书》
3. 《项目蓝图方案确认书》
4. 《UAT测试报告》
5. 《系统功能上线确认》
6. 《系统验收报告》
7. 《项目上线应用情况报告》

## 附录

### 项目变更控制程序

1. 提出方提交项目变更申请（PCR）
2. 双方PM共同评估，决定是否进一步评估
3. 双方授权代表签署PCR授权评估
4. 评估明确对价格/进度/合同条款的影响
5. 双方签署书面变更授权后实施
6. 未书面确认前，继续按最后协定版本执行

### 交付物接受程序

- 电子版提交给甲方PM→分发复核
- 收到交付品5个工作日内接受或提出修改要求
- 8个工作日内无回应→视为接受

### 争议解决程序

- 1级：项目团队2个工作日内内部解决
- 2级：双方PM 3个工作日内共同解决
- 仍有争议：参照合同争议解决程序
