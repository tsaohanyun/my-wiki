# 通威农发-业务调研

> 项目：**通威农发** — 6大业务域需求调研报告汇总

## 概述

业务调研是智能工厂项目蓝图方案阶段的前置工作，基于重庆通威（试点A）和揭阳通威（试点B）两家工厂现状，覆盖计划管理、生产管理、设备管理、质量管理、仓库管理、EBS解耦6大业务域。调研采用"流程→痛点→需求→收益"四段式结构，共梳理出30+个核心业务流程，为蓝图方案设计提供依据。

调研时间：2023年11月-12月
调研地点：重庆通威工厂、揭阳通威工厂、通威国际中心

## 调研总览

| 业务域 | 编号前缀 | 流程数 | 调研工厂 | 核心痛点 |
|--------|---------|--------|---------|---------|
| 计划管理 | JH | 3 | 重庆+揭阳 | 线下沟通、信息不及时、经验依赖 |
| 生产管理 | SC | 10 | 重庆永川 | 纸质记录、经验依赖、追溯困难 |
| 设备管理 | SB | 13 | 重庆 | 台账不统一、纸质记录、系统未充分利用 |
| 质量管理 | QC | 10 | 重庆 | 手工录入EBS、移动端缺失、追溯链断裂、配方管理粗放 |
| 仓库管理 | INV | 14 | 重庆+揭阳 | 待补充 |
| EBS解耦 | ERP | 11 | 总部 | 待补充 |

## 计划管理调研

### 调研对象

重庆工厂：市场销售、生产内勤、生产计划、原料采购
揭阳工厂：市场销售、生产计划、原料采购

### 核心流程

| 流程编号 | 流程名称 | 频次 | 涉及角色 |
|---------|---------|------|---------|
| JH001 | 年度计划 | 1年/次 | 生产内勤、市场销售、销售部门经理、业务员 |
| JH003 | 月度生产计划 | 1个月/次 | 生产内勤、生产计划员、品管、原料采购 |
| JH004 | 月度执行计划 | 月内不定时 | 期间计划员 |

### 年度计划

**流程**：各销售部门每年11月前收集年度销售需求 → 根据生产权限、调拨需求汇总 → 根据生产成本、产能评估生成年度生产计划 → 根据原料价格配方关联 → 生成年度预算

**痛点**：
1. 预测规则全靠与客户线下直接沟通
2. 收集方式全靠线下沟通，层层收集

**需求**：预测需求业务员在系统填报，系统自动按部门统计

### 月度生产计划

**流程**：市场销售每月25号前收集需求 → 生产主计划汇总自产/调拨/市场需求 → 根据年度计划/历史销量/成品库存/产能评估调整 → 生成月度需求计划 → 品管确认原料库存 → 采购制定采购计划 → 导入EBS

**业务规则**：
- 需求80%来源市场，20%为库存计划
- 参考去年同期及环比调整优先级
- 存在市场部预留库存（线下锁货）
- 调出订单先报先留货，留货3天
- 采购周期：周边3天、船期7-10天、集装箱25天、进口原料最快40天

**痛点**：
1. 收集方式全靠线下沟通，信息传递不及时
2. 市场份额/价格/质量/季节/温度影响预测准确性
3. 生产异常/缺料/插单导致计划变动
4. 交期答复速度慢
5. 跨公司查库存困难

**需求**：缺料警示、手机端录入、信息一致性保证、跨公司库存查询

### 期间计划

**流程**：依据月度计划+客户提货计划+已评审插单需求 → 根据可开票量/历史销量/安全库存编制期间执行计划 → 原材料检查（缺料→配方调整→计划调整） → 同配方不同规格合批 → 根据配方生成批次工单 → 下发车间中控排程

**业务规则**：
- 考虑近期销量（10天左右）
- 安全库存根据销量高低和季节变动
- 最小开机量：下单达不到不能生产
- 同配方连续生产减少换模时间
- 模具有规格区分（口径）和压缩比区分（厚度）

**痛点**：
1. 班产能管理靠人的经验
2. 排程规则靠人的经验
3. 物料齐套靠线下沟通
4. 请求响应不及时

## 生产管理调研

### 调研对象

重庆永川工厂：车间经理、中控班长

### 核心流程

| 流程编号 | 流程名称 | 频次 | 涉及角色 |
|---------|---------|------|---------|
| SC001 | 计划下达 | 1-2次/日 | 生产经理 |
| SC002 | 计划排程 | 1-2次/日 | 中控员 |
| SC003 | 工艺配方 | 1-2次/单 | 中控员 |
| SC004 | 生产准备 | 多次/单 | 小料工、包装工 |
| SC005 | 生产投料 | 多次/单 | 投料工、小料工、包装工 |
| SC006 | 生产执行 | 多批/单 | 中控员、巡视员、制粒工、膨化工、包装工 |
| SC007 | 生产入库 | 多次/单 | 叉车工 |
| SC008 | 生产报工 | 1次/单 | 生产经理 |
| SC009 | 交接班 | 1次/班 | 中控班长 |
| SC010 | 返工料 | 不定时 | 品控、仓库、生产、计划 |

### 计划下达（SC001）

**流程**：EBS下发日生产计划到中控系统 → 计划员下发《预混料分装通知单》→ 计划员下发《生产计划通知单》

**痛点**：
1. 同类型不同包装产品合单需计划员人工操作
2. 计划拆分需1小时，依赖纸质下发

**需求**：APS自动排程 + MES系统集成，取消纸质记录

### 计划排程（SC002）

**流程**：中控员根据《生产计划通知单》安排成品生产 → 参考物料结余和切换规则排程 → 中控班长按工艺路线调度

**排程参考规则**：工艺路线、配方、转产转线规则、产线产能（小时产出量）、紧急程度、最优开机时间

**痛点**：
1. 排程深度依赖中控人员经验
2. 制粒/膨化未集成到中控室
3. 人员调度通过对讲机

### 生产准备（SC004）

**流程**：小料工备料到投料口暂存（填写《小料出入库记录》）→ 包装工按节拍到仓库多次领料 → 大料/油类从地堆/筒仓直接使用

**痛点**：小料工纸质记录、包装工领料无记录事后补填

### 生产投料（SC005）

**流程**：中控员下发投料任务到看板 → 小料工按任务投料 → 铲车工大料投料（对讲机通知）→ 散装料中控系统读取流量计

**痛点**：
1. 铲车工投料后纸质记录不及时
2. 部分工厂投料口没看板
3. 物料无批次管理

### 配料管理（SC006）

**流程**：中控员按排产顺序拆分配料子批 → 按配方用量设置配方 → 设置配料筒仓 → 手动控制配料执行

**痛点**：全部依赖中控员经验

### 生产核心痛点汇总

1. EBS期间计划生产数量依靠人工计算
2. 工单管理（拆单/合并/下发）依靠人工经验
3. 生产工艺路线、工序调度深度依赖中控人员经验
4. 设备工艺参数由操作员线下管控
5. 生产备料叫料由人工指令传递
6. 物料批次管理缺失，过程料无批次管理
7. 生产过程信息及交接班信息采用纸质记录
8. 返工料执行无标准化管理
9. 物料追溯、批次追溯、质量追溯、工艺追溯数据不全

## 设备管理调研

### 调研对象

重庆工厂：机电班长、生产内勤、机修人员、标准化专员

### 核心流程

| 流程编号 | 流程名称 | 频次 | 涉及角色 |
|---------|---------|------|---------|
| SB001 | 设备台账 | 不定时 | 标准化专员、机电组长、生产内勤 |
| SB002 | 设备巡检 | 每班/1次 | 机修人员 |
| SB003 | 设备点检 | 每班/1次 | 机修人员、巡视人员 |
| SB004 | 设备保养 | 月/多次 | 机电组 |
| SB005 | 设备应急维修 | 不定时 | 机电组 |
| SB006 | 设备计划性维修 | 不定时 | 生产内勤、机电组、采购、生产经理 |
| SB007 | 设备委外维修 | 不定时 | 机电组、生产经理 |
| SB008 | 设备校检 | 年度/季度 | 机电组、生产经理 |
| SB009 | 交接班管理 | 每班/1次 | 机电组 |
| SB010 | 备品备件管理 | 每月/1次 | 采购、生产、机电组 |
| SB011 | 设备报废 | 不定时 | 生产内勤、财务 |
| SB012 | 设备调拨入库 | 不定时 | 生产经理、采购、机修组长、仓库 |
| SB013 | 设备调拨出库 | 不定时 | 生产经理、采购、机修组长、仓库 |

### 设备台账（SB001）

**流程**：新设备FBC审批 → 通威商城采购 → 到货验收 → FBC转固定资产 → EBS财务做账

**痛点**：
1. 各工厂设备编码定义不一致
2. 设备台账更新不及时
3. 设备台账存在多个系统，无集成
4. 设备资料线下管理
5. 没有完整的设备履历信息

**需求**：建立"一机一档"集团标准化，设备资料线上管理

### 设备保养（SB004）

**三层保养体系**：
- **日常保养**：生产前清洁、检查润滑部位
- **二级保养**：停机时局部解体、清洁、润滑加油、调整间隙紧固、排查易损件
- **大修保养**：停机时分解检查、修理、更换易损件

**痛点**：
1. 现有系统操作便捷性差
2. 保养内容线下管理
3. 保养依赖经验丰富的机修员
4. 保养计划与实际生产计划不协调

### 设备维修（SB005/SB006/SB007）

**应急维修流程**：生产员工通过对讲机联系机修 → 排查故障原因 → 更换备件（纸质领用） → 维修完成确认

**痛点**：
1. 维修时长未统计
2. 维修记录线下纸质文档
3. 故障原因分析未留档
4. 维修任务提报通知均线下

**需求**：系统化维修标准作业、故障原因库、自动统计维修时长

### 备品备件管理（SB010）

**流程**：机修班长/生产经理线下提交备件申请 → 内勤FBC审批 → 通威商城采购 → EBS入库 → 纸质领用记录 → EBS出库

**痛点**：
1. 手动记录入库/出库/库存
2. 库存管理不精确，库存过剩或不足
3. 员工难以追踪使用过的模具
4. 备件报废/领用流程线下管理

**需求**：安全库存提醒、系统化备品备件台账、定期检查预警

## 质量管理调研

> 来源：TWAD_IF_107.QA_调研报告.质量管理_V1.0_20231208

### 调研对象

重庆工厂：品管部经理、过程检验员（2人）、成品检查员（1人），共4人

### 调研背景

根据通威农发标准化下阶段工作重点，着重在重庆通威工厂进行质量管理体系重构，构建工厂MES系统和总部业务平台，推动质量管理向标准化、自动化、透明化、智能化、精益化方向升级。

### 组织架构

- **品管部经理**：管理配方接收与分解，监督各检验环节标准执行
- **过程检验员（2人）**：原材料感官检查、化学检验、线上巡检
- **成品检查员（1人）**：成品库存抽检，确保产品质量稳定性

### 核心流程

| 流程编号 | 流程名称 | 频次 | 涉及角色 | 现有系统 |
|---------|---------|------|---------|---------|
| QC001 | 配方接收与分解 | 不定时 | 品管部经理 | 无（手工CSV导入） |
| QC002 | 配方替换 | 不定时 | 品管部经理、生产部 | 无 |
| QC003 | 来料检验 | 不定时 | 品管部、原料库 | EBS-来料检查 |
| QC004 | 原料库存检验 | 根据周期 | 品管部、原料库 | 无 |
| QC005 | 线上质量自检 | 不定时 | 生产部 | 无（手工录入EBS） |
| QC006 | 生产过程巡检 | 不定时 | 品管部、生产部 | EBS-现场质量巡查记录 |
| QC007 | 成品入库检验 | 入库前 | 品管部、生产部、库存 | 无 |
| QC008 | 成品库存定时检验 | 根据周期 | 品管部、成品库存、生产部 | EBS-超内控产品预警 |
| QC009 | 退货流程 | 不定时 | 市场部、品管部、成品库存 | EBS-顾客退货审批单 |
| QC010 | 质量追溯 | 不定时 | 品管部 | 无（纸质查找） |

### 配方接收与分解（QC001）

**流程**：集团品管制定配方 → 下发至各工厂执行。水产类配方通过邮件CSV文件发送，品管部经理接收后手工导入EBS系统分解；畜禽类配方通过接口方式直接下发至EBS系统。

**处理逻辑**：接收配方 → 配方验证（格式/完整性） → 原料识别（匹配原料库） → 配料比例计算 → 配料调整（根据库存和生产需求）

**痛点**：
1. 禽畜类配方需手工分解和导入，效率低
2. 缺少配方数据准确性确认环节
3. 无配方版本管理，变更追溯困难

**需求**：系统集成自动化下发、分解后审核步骤、配方版本管理

### 配方替换（QC002）

**流程**：生产部调用EBS分解好的配方 → 估算原料理论需求量 → 原料不足时与市场部沟通 → 品管部门+配方师提出调整方案 → 形成业务流程闭环

**替换规则**：
- 高营养原料可替代低营养原料
- 被替换原料库存能满足当天或多天生产且新原料可及时到货时，不替换
- 重要物料不足需先与市场部沟通交货日期影响

**痛点**：替换过程复杂，配方师确认和配方变更单延迟影响生产

**需求**：优化替换流程，厂部品管内部快速沟通处理，加入版本管理和审核步骤

### 来料检验（QC003）

**流程**：接收集团来料检验标准 → 新采购原料入库前检验（感官+化学） → 感官合格后卸货入库 → 不合格则降标接收或柜收 → 紧急用料时快速检测后出结果

**处理逻辑**：查看货物对应标准 → 感官检验（手机APP录入） → 合格后卸货入库过账EBS → 卸货过程中抽取代表性样品 → 化学分析检测

**抽样规则**：多车同批次原料时，每辆车分别抽样检验，确保代表性

**痛点**：班中材料异常或来料未入库时，紧急用料难以支撑生产，导致计划执行延迟

**需求**：生产部异常时立即通知品管，对新到原料及时检查出结果

### 原料库存检验（QC004）

**流程**：品管部门根据原料检查标准，定期对库存原料取样检测 → 发现异常通知原料管理员和生产计划员 → 调整生产计划或替换原料

**痛点**：
1. 单靠周期性检查存在风险（如周期还差3天，原料已变质）
2. 投料缺少检查环节，容易产出不良品

**需求**：除周期性检查外增加不定时检查，原料使用前1-2天内进行检查

### 线上质量自检（QC005）

**流程**：操作人员按标准要求在生产过程中自行检查 → 记录存入EBS系统 → 异常上报品管部门

**处理逻辑**：按预设规则执行自检 → 检查设备状态和操作参数 → 检查原料/半成品/成品质量特征 → 检查工艺控制点 → 记录结果（含时间/操作人员/位置）

**痛点**：
1. 现场自检记录后需手工录入EBS系统
2. 缺少自检频次要求，各加工工艺难发现问题

**需求**：现场加入录入设备并设定自检项，加入检验频次要求

### 生产过程巡检（QC006）

**流程**：集团下发巡检标准 → 品管人员定时巡检生产线 → 巡检结果手工录入EBS → 异常立即通知生产部门

**痛点**：现场巡检记录后需到PC端手工录入EBS

**需求**：功能移至移动端，便于现场检查，减少纸质单使用

### 成品入库检验（QC007）

**流程**：产品到达成品仓 → 品管人员准备检查工具和标准 → 检查外观（包装完整性/标签准确性） → 抽样检验 → 功能性测试（按集团标准）

**痛点**：批量产品到达成品仓时无及时提醒，导致入库前检查被延后

**需求**：系统根据加工料到达成品仓时自动发起入库检验任务

### 成品库存检验（QC008）

**流程**：品质管理部定期对成品库存抽检 → 发现不合格产品（如霉变） → 通知生产部门讨论处理方案 → 重新加工或其他方式处理

**处理逻辑**：根据入库日期和检验周期自动生成检验任务 → 按集团标准检验

**痛点**：仅在PC端提醒，未登录时无法接收检验通知

**需求**：加入手机端APP程序及时提醒

### 退货流程（QC009）

**流程**：用户向市场部提出退货请求 → 市场部转交品管部 → 品管判定完整性/质量状况 → 完整则退回成品仓二次销售 → 质量问题则退回车间二次加工 → 客户人为原因则协商处理

**痛点**：审批操作限定在PC端，无法第一时间收到审批信息

**需求**：加入手机端APP程序及时提醒

### 质量追溯（QC010）

**流程**：查看成品包装标签（工厂+生产日期） → 查询库存位置和日常检测记录 → 查找当天使用的配方和投料情况 → 追溯原料各类信息

**痛点**：通过纸质单人工查找数据，未形成追溯性关联，缺少设备运行状况、环境温湿度等因素

**需求**：生产数据全串联形成闭环，方便追溯各数据点找到质量原因

### 质量管理核心痛点汇总

1. **手工录入普遍**：来料检验、线上自检、巡检结果均需事后手工录入EBS
2. **移动端缺失**：巡检、库存检验通知、退货审批均限定在PC端

6. **预防性不足**：原料库存仅靠周期性检查，投料前缺少检查环节

## 仓库管理调研

### 调研对象

重庆工厂：生产经理、生产班长、原料库管、成品库管、叉车工、投料工、搬运工

### 核心流程

| 流程编号 | 流程名称 | 频次 | 涉及角色 |
|---------|---------|------|---------|
| CK001 | 原料入库 | 多次/日 | 原料库管、叉车工、保安、品管员、采购员 |
| CK002 | 原料退货 | 按需 | 原料库管、叉车工、采购员、品管员 |
| CK003 | 生产领料 | 多次/日 | 原料库管、叉车工、中控员 |
| CK004 | 生产退料 | 按需 | 原料库管、叉车工、生产部 |
| CK005 | 完工入库 | 多次/日 | 成品库管、叉车工、生产部 |
| CK006 | 完工退回 | 按需 | 成品库管、叉车工 |
| CK007 | 销售出库 | 多次/日 | 成品库管、保安、开票员、叉车工 |
| CK008 | 销售退货 | 按需 | 成品库管、开票员、品管员 |
| CK009 | 采购入库（成品） | 按需 | 成品库管、叉车工、保安、品管员 |
| CK010 | 不合格物料 | 按需 | 品管员、成品库管 |
| CK011 | 调拨入库 | 按需 | 保安、开票员、库管 |
| CK012 | 调拨出库 | 按需 | 兄弟公司、库管 |
| CK013 | 借还料 | 按需 | 兄弟公司、库管 |
| CK014 | 盘点 | 1次/日 | 库管 |

### 原料入库（CK001）

**流程**：采购订单 → 入场登记 → 入场过磅 → 感官检验 → 分配卸货口 → 卸货码盘 → 二次感官检验 → 叉车入库 → 二次过磅 → 账务处理

**入库规则**：
- 存放位置：同物料同批次 → 空库位 → 同物料库位 → 有空余的库位
- 先整托后尾托，尾托放最外层
- 定包接收：小料/部分大料按标准袋装重量
- 按磅重接收：散料/油按过磅净重，大料需手工分摊到每件

**痛点**：
1. 库位纸质物料卡人工记录，库存更新不及时
2. 大料入库重量人工分摊费时易出错

### 生产领料（CK003）

**流程**：中控计算投料需求 → 筒仓散料判断 → 筒仓出库/叉车取货 → 实物投料 → 库管盘点差异 → 账务处理

**取货规则**：按入库日期先进先出，尾托先取，一次一托盘

**痛点**：
1. 取货位置依赖人的经验，存在取错货风险
2. 手工记录投料数量，事后补账
3. 手工管理易受人为因素干扰

### 销售出库（CK007）

**流程**：空车过磅 → 开票 → 装车顺序标注 → 库管分配任务 → 叉车取货装车 → 账务处理 → 二次过磅放行

**二次过磅规则**：核对重量=空车+物料总重±20kg（扣包装物）

**痛点**：
1. 发货口/取货库位线下沟通，依赖人工经验
2. 装车顺序手工填写经常出错

### 借还料（CK013）

**特殊场景**：兄弟公司间线下电话/微信申请借料，库管手工记录。当月月底前必须还包，否则转销售出库。

**痛点**：实物借走但EBS库存仍存在，存在账实不符风险

### 仓库核心痛点汇总

1. 纸质物料卡/手工记录为主，库存更新不及时
2. 取货/入库位置依赖人工经验
3. 大料重量分摊计算费时易出错
4. 借还料线下管理，账实不符风险
5. 装车顺序手工传递易出错
6. 盘点方式原始（量尺/液位计/激光测距/数托盘）

## EBS解耦调研

### 调研背景

EBS解耦调研目的是了解EBS系统现有功能的使用情况，确认哪些业务功能需要剥离到新的数智制造平台，以及系统间的交互关系。

### EBS现状流程清单

| 流程编号 | 流程名称 | 频次 | 涉及业务域 |
|---------|---------|------|----------|
| ERP01 | 主数据 | 不定时 | 采购、仓储、营销 |
| ERP02 | 原料采购到入库 | 不定时 | 采购、过磅、仓储、质量 |
| ERP03 | 原料采购退货 | 不定时 | 采购、过磅、仓储 |
| ERP04 | 非原料采购到入库 | 不定时 | 采购、仓储 |
| ERP05 | 成品采购到入库 | 不定时 | 采购、过磅、仓储、质量 |
| ERP06 | 内部调料 | 不定时 | 过磅、仓储、质量 |
| ERP07 | 需求到生产完工入库 | 不定时 | 仓储、质量、生产、计划 |
| ERP08 | 销售到发运 | 不定时 | 营销、过磅、仓储 |
| ERP09 | 销售退货 | 不定时 | 营销、过磅、仓储 |
| ERP10 | 其他出入库 | 不定时 | 仓储 |
| ERP11 | 成本核算 | 每月月底 | 财务 |

### 主数据管理现状

| 主数据 | 录入入口 | 接口情况 |
|--------|---------|---------|
| 单位 | EBS创建 | 无接口 |
| 子库存 | EBS创建 | 有对接智能库房 |
| 物料编码 | EBS维护（导入模板） | EBS同步给商城/SRM/中控 |
| 单位换算 | EBS手工维护 | 无接口 |
| 品名-成品映射 | EBS手工维护 | 无接口 |
| 客户 | FBC/营销通/EBS | FBC→EBS |
| 供应商 | SRM维护 | SRM→EBS |
| 配方 | 配方系统（畜禽）/EBS导入（水产） | 配方系统→EBS（仅畜禽） |

**物料编码规则**：行业2位+大类1位+小类2位+流水号4位
- 饲料：10原材料/20预混料/30成品/40包装物/50标签/60半成品/70待分装
- 药品：10原材料/30成品/40包装物/50标签/60半成品

### 生产完工入库关键流程（ERP07）

**EBS现状**：
1. 销售计划在预算系统维护 → EBS导入月度销售计划 → 生成月度生产计划
2. 手工在EBS下达生产工单 → 同步给中控系统
3. 生产批创建后原料检查 → 配方调整（原料短缺/返工料积压）
4. 运行请求集 → 中控获取配方/物料需求 → 生成EBS搬运单（领料单）
5. 物料消耗分**理论消耗**和**中控消耗**两种
6. 库管在EBS手工完工入库 → 入库后状态待检 → 质检后更新状态

**特殊场景**：
- 联产品产出：同配方不同包装在同一生产批
- 换包生产：包装变更需重新下达生产批（流程复杂）
- 受托加工：客户提供原料/配方/检验标准
- 委外加工：内部委外通过调料实现

### EBS解耦优化需求汇总

| 业务环节 | 优化需求 |
|---------|---------|
| 质检结果 | 基于质检结果自动更新库存批次状态 |
| 采购入库 | 由制造平台同步EBS |
| 过磅 | 过磅自动计算并同步EBS |
| 采购订单 | 同步到制造平台用于收退货 |
| 生产工单 | 由制造平台传入EBS，EBS不直接对接中控 |
| 物料消耗 | 由制造平台传入EBS |
| 完工入库 | 由制造平台传入EBS |
| 其他出入库 | 由制造平台同步EBS |
| 换包生产 | 优化简化流程 |
| 销售退货 | 质检流程线上管理 |

## 共性痛点与系统化需求

### 跨域共性痛点

| 痛点类别 | 具体表现 | 影响范围 |
|---------|---------|---------|
| 线下沟通依赖 | 需求收集/计划传递/调度通知全部线下 | 计划/生产/设备 |
| 纸质记录为主 | 领料/投料/维修/交接班/巡检全部纸质 | 生产/设备 |
| 经验依赖严重 | 排程/配方/班产能/维修全部靠人 | 计划/生产/设备 |
| 系统间无集成 | EBS/中控/设备管理/线下台账各自孤立 | 全部 |
| 追溯能力不足 | 物料批次/质量/工艺追溯数据不全 | 生产/质量 |
| 信息不一致 | 车间和市场看不同库存、多个系统台账不统一 | 计划/仓库 |

### 系统化建设需求汇总

1. **APS+MES**：构建从期间计划到生产工单的桥梁
2. **MES系统**：工艺路线/配方/工序/工单管理
3. **BATCH系统**：工序级执行、自动配料
4. **SCADA集成**：设备透明化、实时监控
5. **WMS集成**：物料备料拉动、领料线上化
6. **移动端**：岗位工作台、无纸化作业
7. **设备管理系统**：一机一档、点巡检/保养/维修全线上化
8. **追溯体系**：物料/批次/质量/工艺正反向追溯
9. **看板+驾驶舱**：生产指标实时展示
