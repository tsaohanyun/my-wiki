# 通威农发-蓝图方案-仓储与EBS集成

> 项目：**通威农发** — 仓储全流程管理 + EBS业务解耦蓝图方案

## 概述

本文档汇总通威农发智能工厂项目仓储管理与EBS解耦蓝图方案的核心内容。仓储管理实现从原料入库到成品出库的全流程数字化，通过无人过磅、WMS智能库房对接消除纸质记录和信息滞后；EBS解耦将生产、库存、质量、计划业务从EBS系统中完整剥离到数智制造平台，EBS保留采购、销售、财务核心功能，实现业务清晰分界与数据双向同步。

## 1. 业务需求矩阵

| 序号 | 流程编号 | 业务流程 | 核心功能 |
|------|---------|---------|---------|
| 1 | WH001 | 原料入库 | 采购订单→过磅→检验→卸货→入库→账务 |
| 2 | WH002 | 生产领料 | 中控计算→筒仓出库/叉车取货→投料→账务 |
| 3 | WH003 | 完工入库 | 成品打包→叉车入库→质检→账务 |
| 4 | WH004 | 销售出库 | 开票→分配库位→装车→过磅→放行→账务 |
| 5 | WH005 | 调拨管理 | 内部调料/借还料→审批→出入库→账务 |
| 6 | WH006 | 盘点管理 | 日盘/月盘→差异分析→调账 |
| 7 | ERP01-11 | EBS解耦 | 生产/库存/质量/计划业务剥离→平台+EBS双系统协同 |

## 2. 原料入库（WH001）

### 2.1 业务流程

| 步骤 | 处理方式 | 说明 |
|------|---------|------|
| 采购订单匹配 | 系统 | 系统自动匹配EBS采购订单 |
| 入场登记 | 系统+人工 | 车辆信息录入 |
| 入场过磅 | 系统 | 无人过磅系统自动获取毛重 |
| 感官检验 | 系统+人工 | 移动端录入检验结果 |
| 分配卸货口 | 系统 | 根据原料类型自动分配 |
| 卸货码盘 | 人工 | 尾托放最外层 |
| 二次感官检验 | 系统+人工 | 卸货过程中抽样 |
| 叉车入库 | 系统 | 系统指引库位（同批次→空位→同物料位） |
| 二次过磅 | 系统 | 无人过磅获取皮重，计算净重 |
| 账务处理 | 系统 | 平台入库→同步EBS过账 |

### 2.2 入库规则

| 规则类型 | 说明 |
|---------|------|
| 存放优先级 | 同物料同批次 → 空库位 → 同物料库位 → 有空余库位 |
| 整托/尾托 | 先整托后尾托，尾托放最外层 |
| 定包接收 | 小料/部分大料按标准袋装重量接收 |
| 按磅重接收 | 散料/油按过磅净重，大料手工分摊到每件 |

### 2.3 接口清单

| 编号 | 接口名称 | 数据方向 | 主要内容 |
|------|---------|---------|---------|
| 1 | 采购订单同步 | EBS→平台 | 订单号、物料、数量、供应商 |
| 2 | 过磅数据同步 | 无人过磅→平台 | 车号、毛重、皮重、净重、时间 |
| 3 | 入库账务同步 | 平台→EBS | 入库单号、物料、数量、仓库 |
| 4 | 来料检验结果 | 平台→质量管理 | 推送检验任务 |

## 3. 生产领料（WH002）

### 3.1 业务流程

| 步骤 | 处理方式 | 说明 |
|------|---------|------|
| 中控计算投料需求 | 系统 | 根据日计划+配方展开物料需求 |
| 筒仓散料判断 | 系统 | 筒仓料直接出库/非筒仓料生成取货任务 |
| 叉车取货 | 系统+人工 | 系统指引取货位（先进先出+尾托优先） |
| 实物投料 | 人工 | 按投料任务执行 |
| 差异盘点 | 系统+人工 | 库管盘点投料差异 |
| 账务处理 | 系统 | 出库过账→同步EBS |

### 3.2 关键改进

| 痛点 | 蓝图方案 |
|------|---------|
| 取货位置依赖人的经验 | 系统自动推荐库位（先进先出+尾托规则） |
| 手工记录投料数量 | 系统自动记录投料数量 |
| 事后补账 | 实时出库过账 |

## 4. 销售出库（WH004）

### 4.1 业务流程

| 步骤 | 处理方式 | 说明 |
|------|---------|------|
| 空车过磅 | 系统 | 无人过磅获取空车重量 |
| 开票 | 系统 | 匹配销售订单，生成提货单 |
| 装车顺序标注 | 系统+人工 | 系统推荐装车图谱，消除手写错误 |
| 库管分配任务 | 系统 | 自动分配库位和发货口 |
| 叉车取货装车 | 系统+人工 | 系统指引取货位 |
| 账务处理 | 系统 | 出库过账 |
| 二次过磅放行 | 系统 | 核对重量=空车+物料总重±20kg |

## 5. 特殊业务场景

### 5.1 借还料管理

| 场景 | 说明 |
|------|------|
| 业务模式 | 兄弟公司间线下电话/微信申请借料 |
| 账务规则 | 当月月底前必须还包，否则转销售出库 |
| 风险 | 实物借走但EBS库存仍存在→账实不符 |
| 蓝图方案 | 系统登记借出/归还记录，到期自动转销售或报警 |

### 5.2 不合格品管理

| 场景 | 处理方式 |
|------|---------|
| 原料不合格 | 降标接收/柜收/退货 |
| 成品不合格 | 返工/报废/降级销售 |
| 退货品 | 完整→退回成品仓；质量问题→返工 |

## 6. EBS解耦方案

### 6.1 解耦范围

| 业务域 | 解耦方向 | 核心逻辑 |
|--------|---------|---------|
| 采购业务 | EBS保留 | 采购订单在EBS创建，同步到平台用于收货 |
| 销售业务 | EBS保留 | 销售订单在EBS创建，同步到平台用于发货 |
| 生产业务 | 剥离到平台 | 工单创建/执行/报工在平台完成，完工信息回传EBS |
| 质量业务 | 剥离到平台 | 检验标准/执行/判定在平台完成，结果回传EBS更新批次状态 |
| 库存业务 | 剥离到平台 | 出入库/盘点在平台完成，库存变动实时同步EBS |
| 财务业务 | EBS保留 | 成本核算/凭证在EBS完成 |
| 计划业务 | 剥离到平台 | 年度/月度/期间/日计划在平台编制完成，下达同步EBS |
| 主数据 | 双向同步 | 物料/客户/供应商在EBS创建，同步到平台 |

### 6.3 EBS接口清单

| 序号 | 接口名称 | 数据方向 | 主要内容 |
|------|---------|---------|---------|
| 1 | 采购订单同步 | EBS→平台 | 采购订单号、物料、数量、供应商 |
| 2 | 销售订单同步 | EBS→平台 | 销售订单号、物料、数量、客户 |
| 3 | 主数据同步 | EBS→平台 | 物料编码、客户、供应商、组织 |
| 4 | 生产完工回传 | 平台→EBS | 工单号、完工数量、物料消耗 |
| 5 | 库存变动同步 | 平台→EBS | 出入库单号、物料、数量、仓库 |
| 6 | 质检结果回传 | 平台→EBS | 检验批次、结果、合格判定 |
| 7 | 计划单同步 | 平台→EBS | 计划单号、产品、计划数量、日期 |
| 8 | 过磅数据同步 | 无人过磅→平台 | 毛重、皮重、净重、车号 |

### 6.4 过渡方案

| 工厂类型 | EBS解耦策略 |
|---------|------------|
| 试点工厂（重庆/揭阳） | 上线即剥离完整方案 |
| 推广工厂（已完成PLC/SCADA升级） | 同步执行完整剥离方案 |
| 推广工厂（未完成升级） | 过渡方案：平台+手动同步EBS双轨运行 |

## 7. 仓储与EBS集成实施方法论

### 7.1 实施流程

| 阶段 | 时长 | 主要工作 | 交付物 |
|------|------|----------|--------|
| 需求调研 | 4-6周 | 过磅系统评估、WMS接口调研、EBS解耦边界确认 | 调研报告 |
| 方案设计 | 6-8周 | 仓储流程设计、EBS接口方案、过渡方案制定 | 蓝图文档 |
| 系统实现 | 12-16周 | 仓储模块开发、无人过磅对接、EBS接口开发、WMS集成 | 配置文档 |
| 测试验证 | 6-8周 | 仓储业务场景测试、EBS数据同步准确性验证、并行测试 | 测试报告 |
| 上线切换 | 3-4周 | 历史库存数据迁移、双系统并行切换、用户培训 | 上线报告 |
| 运维优化 | 持续 | 库存准确性监控、接口稳定性运维 | 运维文档 |

### 7.2 实施风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 无人过磅系统不稳定 | 入库/出库流程受阻 | 预留手动过磅降级方案 |
| EBS接口性能瓶颈 | 数据同步延迟 | 采用异步+定时批量同步策略 |
| WMS系统版本差异 | 智能库房对接复杂度高 | 制定分类对接策略 |
| 库存数据迁移准确性 | 双系统并行期账实不符 | 上线前全量盘点+逐项核对 |
| 过渡期工厂操作复杂 | 手动同步增加工作量 | 提供自动化脚本辅助 |

### 7.3 ROI与效益分析

- **直接收益**：入库效率提升40%（无人过磅+系统指引）、库存准确率提升至99%以上、销售出库装车错误率降低90%
- **间接收益**：EBS解耦消除生产业务对EBS的直接依赖、库存实时可见性提升决策速度、账实一致性消除审计风险
- **典型回收期**：1.5-2.5年
