## EBS功能测试

EBS拉通测试覆盖5个核心业务流程，验证DIM平台与EBS系统的端到端数据一致性。

### 基础数据

基础数据配置是EBS与DIM平台对接的前提条件，需在上线前完成以下主数据维护：

| 数据类型 | 关键说明 |
|---|---|
| 组织（OU与库存组织） | 上线DIM时，组织和库存组织均需配置弹性域 |
| 物料 | 物料基本信息与弹性域结构配置 |
| 货位 | EBS货位与DIM货位不直接对应，需独立维护 |
| 单位换算 | 可取物料上的换算率，也可通过分类间/分类内界面维护 |
| 子库 | 子库存基础配置 |
| 成本 | 每日运行成本请求，期间关闭前仅做参考，关闭后为实际成本 |
| 物料类别（值集） | 物料分类值集定义 |
| 工艺分类/养殖品种（值集） | 业务扩展值集 |
| 非原料领用项目 | 内部项目与外部项目管理 |
| 物料弹性域结构 | 物料弹性域定义 |
| 非原料领用科目设置 | 科目对应关系配置 |
| 非原料库存额度 | 采购额度管控 |
| 批次 | 批次状态管理（待检/合格/让步接收） |
| 库存现有量 | 现有量查询验证 |
| 生产批成本检查 | 成本数据检查与验证 |

### 库存杂项

#### 账户别名出入库流程

| 步骤 | 系统 | 说明 |
|---|---|---|
| 1. 提交账户别名申请 | DIM → EBS | 平台发起申请同步至EBS |
| 2. 账户别名审批 | EBS → FSSC → EBS | 基于快码 CUX_INV_GENERIC_FSSC 判断推送FSSC或EBS本地审批 |
| 3. 生成事务处理 | EBS | 审批通过后自动更新状态并生成出入库事务 |
| 4. 平台事务处理 | DIM | 获取EBS状态为"已审批/已完成"后执行平台出库 |

#### 非原料领用出库流程

| 步骤 | 系统 | 说明 |
|---|---|---|
| 1. 创建物料搬运单 | DIM → EBS | 仅适用于机物料领用 |
| 2. 处理搬运单 | EBS | EBS自动处理搬运单事务 |

### 配方生产批

| 流程 | 方向 | 核心说明 |
|---|---|---|
| 配方创建 | DIM → EBS | 平台配方同步至EBS |
| 月度销售计划 | EBS → DIM | EBS将已上线DIM公司的计划按物料/公司/月份汇总传递 |
| 月度调料生产计划（EBS→DIM） | EBS → DIM | 需求公司未上线DIM但供应公司已上线DIM的计划同步 |
| 月度调料生产计划（DIM→EBS） | DIM → EBS | 需求公司已上线DIM但供应公司未上线DIM的计划同步回EBS |
| 生产批创建 | DIM → EBS | 日生产计划从平台同步至EBS生成生产批 |
| 生产领料（子库转移） | DIM → EBS | 原EBS定时请求改为平台直接同步生成子库转移事务，新增专用事务处理类型 |
| 生产批物料消耗 | DIM → EBS | 平台消耗数据同步至EBS |
| 生产批完工 | DIM → EBS | 完工入库与退回同步 |
| 批次状态更新 | DIM → EBS | 理化检验合格后更新批次状态 |
| 生产批关闭/打开 | DIM → EBS | 生产批状态变更同步；特定场景打开需执行专用脚本 |
| 生产批成本检查 | EBS → DIM | 定期将成本数据同步至平台用于成本检查 |

### 采购结算

采购结算覆盖7个子场景，按物料属性及过磅方式进行区分：

| 子场景 | 过磅类型 | 关键流程 |
|---|---|---|
| 原料采购（需理化检验/收货公司磅） | 收货公司磅 | 创建订单→查询订单→一次过磅→感官检验→二次过磅→入库→批次更新→采购结算 |
| 原料采购（无需理化检验/定包） | 无需过磅 | 创建订单→查询→一次过磅→二次过磅→入库→感官检验→批次更新→结算 |
| 原料采购（需理化检验/供方磅） | 供方磅 | 创建订单→查询→一次过磅→感官检验→二次过磅→入库→批次更新→维护供方磅重→结算 |
| 原料采购（直连订单） | 直连 | 创建订单→创建委托函→委托函发货→查询→一次过磅→感官检验→二次过磅→入库→批次更新→维护供方磅重→结算 |
| 编标采购（指定过磅/虚拟磅单） | 指定过磅 | 创建订单→查询→人工过磅指定→入库（关联虚拟磅单）→结算 |
| 非原料采购（无过磅） | 无过磅 | 创建订单（商城→EBS）→查询→入库→结算；注意测试有批次序列与无批次序列两种情况 |
| 采购退货（部分退货） | 过磅接收 | 创建订单→过磅→入库→批次更新→退货→结算 |

关键要点：
- 需理化检验的物料，批次状态默认为"待检"，理化合格后批次更新为"合格"
- 无需理化检验的物料，入库即感官检验合格
- 供方磅场景必须维护供方磅重才可结算
- 直连采购订单必须创建委托函且发货状态为"已发货"才可被DIM查询
- 非原料采购订单若手工在EBS创建，结算时可能报错（校验商城订单来源）

### 销售发运

| 业务类型 | 核心流程 |
|---|---|
| 定包销售 | 车辆入场过磅 → 销售订单创建 → 收款登记订单 → 装车称重 → 销售发运 |
| 散装销售 | 入场过磅 → 提货计划创建 → 装车称重 → 自动创建销售订单 → 收款登记 → 销售发运 |

关键要点：
- EBS手工创建的销售订单需在EBS手工登记后同步平台
- 一点通/智链通创建的订单同步EBS后直接为已登记状态
- 二次过磅封签结果需同步至EBS

## 无人过磅施工方案

### 方案概述

根据重庆工厂和揭阳工厂的现场情况，对地磅周边监控设备改造及部署无人过磅系统，实现地磅的双向、不下车计量。核心目标：通过标准化、自动化、透明化、智能化升级，推动生产业务实现质量最优、效率最高、成本最低。

### 设备布局

地磅设备布局包含以下组件：

- **视频监控**：2台车牌识别摄像机（衡器两端识别车牌）+ 2台枪机摄像机（车辆前后端监控）
- **室外音柱**：1套，播放计量提示信息
- **道闸**：2个，衡器两端管控车辆上下磅
- **LED屏**：2个，衡器两端显示计量提示信息
- **工控机**：1台（门卫室内），处理计量业务信息
- **触发终端**：2台（3米立杆），扫码/刷身份证触发计量
- **打印终端**：1台，过磅票据打印
- **红外监控**：2个，地磅一侧监控停车位置
- **自助登记终端**：1台，触摸站立式终端机，17英寸，具备票据打印功能
- **车辆排队引导信息展示屏**：1台（P10 LED单色屏），排队引导提示

### 实施计划

施工按现场条件分阶段推进：

1. **基础施工**：监控杆预埋件及地笼浇筑
2. **线缆敷设**：半地埋走桥架，线缆两端加标签
3. **设备安装**：立杆、摄像机、LED屏、道闸、红外、终端等
4. **系统调试**：工控机部署、网络调试、业务逻辑联调
5. **验收交付**：功能验证、培训操作人员
