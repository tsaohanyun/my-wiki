# B13 网络安全与工控安全方案集

> 项目：**售前方案集** | 分类：**B13-网络安全与工控** | 文件数：45
> 交叉引用：[[方案集/方案集知识库-总览]] | [[方案集/B09-IT架构与中台|B09-IT架构与中台]] | [[方案集/B11-5G与物联网|B11-5G与物联网]]

## 概述

本分类汇集了工业网络安全和工控安全领域的售前方案、行业标准和技术白皮书。随着工业互联网和智能制造的推进，IT与OT（运营技术）加速融合，工业控制系统面临日益严峻的网络安全威胁。方案涵盖工控安全体系建设、等级保护合规、安全运营中心建设以及工业网络防护等核心内容。

## 1. 工控安全威胁态势

### 典型攻击事件

| 事件 | 目标 | 影响范围 | 攻击方式 |
|------|------|----------|----------|
| 震网（Stuxnet） | 伊朗核设施 | 物理破坏 | USB传播+PLC篡改 |
| BlackEnergy | 乌克兰电网 | 大面积停电 | 鱼叉钓鱼+恶意固件 |
| WannaCry | 全球工业设施 | 生产中断 | SMB漏洞+勒索加密 |
| Triton | 沙特石化 | 安全系统失灵 | SIS控制器篡改 |

### 工控安全特殊性

- **可用性优先**：不同于IT安全"机密性优先"，工控安全强调"可用性>完整性>机密性"
- **实时性要求**：不能因安全措施影响生产控制的实时性
- **长生命周期**：工控设备运行10-20年，补丁更新困难
- **异构复杂**：多种协议（Modbus/OPC/Profinet）、多代设备共存
- **物理后果**：攻击可能导致人员伤亡和环境污染

## 安全架构

### 纵深防御体系

```
企业管理层  ── IT网络 ── 防火墙/网闸隔离
    ↓
生产管理层  ── DMZ区  ── 数据采集+安全监测
    ↓
过程控制层  ── OT网络 ── 工业防火墙+白名单
    ↓
现场设备层  ── PLC/DCS ── 固件完整性验证
```

### 等级保护2.0（工控扩展要求）

- **安全物理环境**：生产区域门禁、视频监控
- **安全通信网络**：网络分区、边界防护
- **安全区域边界**：工业防火墙、协议深度检测
- **安全计算环境**：工控主机白名单、USB管控
- **安全管理中心**：集中日志审计、态势感知

### 关键安全产品

| 产品类型 | 功能 | 部署位置 |
|----------|------|----------|
| 工业防火墙 | 协议级访问控制 | OT网络边界 |
| 工控安全审计 | 协议解析+行为分析 | 交换机镜像口 |
| 主机白名单 | 进程/文件白名单控制 | 操作员站/工程师站 |
| 安全运维堡垒机 | 运维操作审计 | 维护接入点 |
| 态势感知平台 | 全网安全态势监控 | 安全管理中心 |
| 漏洞扫描 | 工控设备脆弱性检测 | 定期巡检 |

## 工业互联网安全

### IT/OT融合安全挑战

- 网络边界模糊化，传统物理隔离被打破
- IoT设备海量接入，攻击面急剧扩大
- 云平台安全责任共担模型
- 5G网络切片安全隔离
- 边缘计算节点的安全防护

### 安全运营体系

1. **资产清点**：自动发现和梳理工控资产
2. **风险评估**：识别脆弱性和威胁场景
3. **防护部署**：根据风险等级部署安全措施
4. **监测预警**：实时安全事件监测和告警
5. **应急响应**：安全事件处置和恢复
6. **持续改进**：定期评估和优化

## 方案价值

1. **合规达标**：满足等保2.0和行业监管要求
2. **风险可视**：全网安全态势一屏掌握
3. **快速响应**：安全事件分钟级发现和处置
4. **生产保障**：安全措施不影响生产连续性
5. **投资保护**：分阶段建设，灵活扩展

## 标准方法论

### 一、安全评估方法论

#### 1.1 资产识别与分级

| 阶段 | 关键活动 | 输出物 |
|------|----------|--------|
| 资产摸底 | 工控设备、网络拓扑、软件清单采集 | 资产台账 |
| 业务关联 | 资产与生产业务映射分析 | 业务-资产关联图 |
| 资产分级 | 按业务关键性划分A/B/C等级 | 资产分级清单 |

#### 1.2 威胁建模

- **攻击面分析**：网络接口、物理接口、供应链、人员
- **攻击树建模**：从攻击者目标出发，分解实现路径
- **威胁场景枚举**：结合MITRE ATT&CK for ICS框架，覆盖侦察、初始访问、横向移动、影响等阶段

#### 1.3 漏洞评估

```
评估维度：
├── 技术漏洞：未修补CVE、默认凭据、协议明文传输
├── 配置漏洞：弱口令、多余开放端口、不安全服务
├── 架构漏洞：网络边界缺失、单点故障、无冗余
└── 管理漏洞：制度缺失、职责不清、培训不足
```

#### 1.4 风险量化

- 采用 **风险 = 威胁可能性 × 资产价值 × 脆弱性严重度** 模型
- 输出风险热力图，划分高/中/低风险区域
- 制定风险处置策略：规避、降低、转移、接受

### 二、等保合规方法论

#### 2.1 等保2.0工控扩展要求实施流程

```
定级备案 → 安全建设 → 等级测评 → 监督检查 → 持续改进
  (1)        (2)         (3)         (4)         (5)
```

| 阶段 | 核心工作 | 参考标准 |
|------|----------|----------|
| ① 定级备案 | 确定系统安全保护等级，提交备案材料 | GB/T 22239 |
| ② 安全建设 | 差距分析→方案设计→措施实施→自评估 | GB/T 25070 |
| ③ 等级测评 | 委托具备资质的测评机构现场测评 | GB/T 28448 |
| ④ 监督检查 | 配合行业主管部门和公安机关检查 | 网络安全法 |
| ⑤ 持续改进 | 年度复评、安全策略优化、人员培训 | — |

#### 2.2 工控安全扩展要求逐项对照

| 安全层面 | 工控扩展要求 | 实施要点 |
|----------|-------------|----------|
| 安全物理环境 | 生产区访问控制 | 门禁系统、视频监控、访客登记 |
| 安全通信网络 | 网络架构安全 | 分区分域、冗余链路、带宽保障 |
| 安全区域边界 | 访问控制与入侵防范 | 工业防火墙、协议白名单、异常流量检测 |
| 安全计算环境 | 身份鉴别与数据完整性 | 双因素认证、数据签名、日志审计 |
| 安全管理中心 | 集中管控与审计 | 统一策略下发、日志集中分析、告警联动 |

### 三、工控安全防护方法论

#### 3.1 防护体系设计原则

1. **纵深防御**：多层防护，任一层被突破不导致全局失陷
2. **最小权限**：仅开放业务必需的通信和操作权限
3. **白名单优先**：工控环境以"已知允许"替代"已知禁止"
4. **可用性保障**：安全措施不得影响生产连续性和实时性
5. **适度安全**：平衡安全投入与业务风险，避免过度防护

#### 3.2 分区分域防护策略

| 安全域 | 防护重点 | 关键措施 |
|--------|----------|----------|
| 企业管理域 | IT系统防护 | 终端防护、邮件网关、上网行为管理 |
| DMZ域 | IT/OT安全交互 | 数据单向传输、应用代理、协议转换 |
| 生产监控域 | SCADA/HMI防护 | 工控主机加固、白名单、USB管控 |
| 过程控制域 | PLC/DCS防护 | 工业防火墙、协议深度检测、固件校验 |
| 现场设备域 | 传感器/执行器 | 物理防护、电磁屏蔽、信号完整性验证 |

#### 3.3 工控安全运维体系

```
日常运维                    安全运营
├── 资产变更管理            ├── 安全事件监控
├── 补丁/固件更新           ├── 威胁情报接入
├── 配置基线核查            ├── 日志审计分析
└── 备份与恢复演练          └── 安全态势报告
```

### 四、应急响应方法论

#### 4.1 应急响应流程（基于NIST SP 800-61 / GB/T 24363）

```
准备 → 检测与分析 → 遏制 → 根除 → 恢复 → 总结改进
(P)     (D&A)       (C)    (E)    (R)     (L)
```

| 阶段 | 工控场景关键动作 |
|------|------------------|
| 准备（Preparation） | 建立应急团队、编制应急预案、储备应急工具和备件 |
| 检测与分析（Detection & Analysis） | 安全设备告警分析、异常流量识别、现场巡检确认 |
| 遏制（Containment） | 网络隔离、切换备用系统、禁止远程访问 |
| 根除（Eradication） | 恶意代码清除、漏洞修补、固件重刷 |
| 恢复（Recovery） | 系统恢复、业务验证、逐步恢复生产 |
| 总结改进（Lessons Learned） | 事件复盘、预案更新、防护措施优化 |

#### 4.2 工控应急响应特殊要求

- **生产安全优先**：处置过程中确保人员安全和生产安全
- **隔离不断电**：优先采用网络隔离而非设备断电，避免生产事故
- **备件冗余**：关键控制器、交换机等应有冷备/热备方案
- **取证保全**：在不影响生产的前提下保存日志和现场证据
- **多方协同**：涉及生产、IT、安全、设备厂商等多部门协同

#### 4.3 应急预案体系

```
总体应急预案
├── 网络攻击专项预案（病毒爆发、勒索攻击、APT攻击）
├── 设备故障专项预案（控制器故障、网络中断、上位机宕机）
├── 数据安全专项预案（数据泄露、数据篡改、备份恢复）
└── 场景化处置手册（按系统/车间/产线编写的操作指南）
```

#### 4.4 应急演练机制

| 演练类型 | 频次 | 形式 | 参与范围 |
|----------|------|------|----------|
| 桌面推演 | 季度 | 纸面推演，验证预案逻辑 | 安全团队+管理层 |
| 功能演练 | 半年度 | 单一系统/场景实战演练 | 安全团队+生产+IT |
| 综合演练 | 年度 | 全场景、多部门联合演练 | 全相关部门 |
