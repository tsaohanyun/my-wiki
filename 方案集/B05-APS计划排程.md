# B05 APS高级计划排程方案集

> 项目：**售前方案集** | 分类：**B05-APS计划排程** | 文件数：44
> 交叉引用：[[方案集/方案集知识库-总览]] | [[B02-ERP企业资源规划]] | [[B01-MES制造执行系统]]

## 概述

本分类汇集了APS（Advanced Planning and Scheduling，高级计划与排程）系统的售前方案和实施案例。APS是连接ERP计划层与MES执行层的关键桥梁，通过数学优化算法和约束满足技术，实现有限产能条件下的最优排产。方案涵盖从长期需求计划到车间级作业调度的全链条计划管理。

## APS系统定位

### 计划体系层级

```
战略层  ── S&OP（产销协同）     → 年度/季度计划
    ↓
战术层  ── MPS（主生产计划）     → 月/周计划
    ↓        MRP（物料需求计划）  → 采购/生产建议
执行层  ── APS（详细排程）       → 日/班次/小时排程
    ↓
现场层  ── MES（制造执行）       → 工序级执行与反馈
```

### APS核心能力

| 能力 | 说明 | 算法/方法 |
|------|------|-----------|
| 有限产能排程 | 考虑设备、人员、模具等实际约束 | 遗传算法、约束规划 |
| 物料约束 | 考虑原材料和半成品供应情况 | MRP+库存优化 |
| 多目标优化 | 交期、产能利用率、换线成本平衡 | 加权评分/帕累托前沿 |
| 实时重排 | 异常事件（插单/设备故障）快速响应 | 增量式重排 |
| 甘特图可视化 | 直观展示排程结果 | 交互式拖拽调整 |
| What-if模拟 | 多方案对比评估 | 场景模拟引擎 |

## 标准方法论

### APS实施特点

APS系统与通用ERP/MES项目存在显著差异，其实施具有以下典型特征：

1. **计划算法配置**：需根据行业特性与工艺特征选配求解算法（遗传算法、约束规划、启发式规则等），并针对排程目标进行参数调优
2. **约束条件建模**：需将设备、模具、人员、物料、工艺等多维约束抽象为数学模型，是APS实施的核心难点
3. **与ERP/MES集成**：APS需双向对接ERP（订单、BOM、库存）和MES（报工、设备状态），数据质量与接口稳定性直接影响排程准确性
4. **分步上线策略**：APS不宜一次性全面切换，通常按"主计划→粗排程→详细排程"分阶段上线，逐步替换手工排产

### 实施流程

| 阶段 | 周期 | 关键活动 | 交付物 |
|------|------|----------|--------|
| 需求调研 | 2-4周 | 业务现状分析、排程痛点识别、约束条件梳理 | 需求规格说明书 |
| 方案设计 | 3-5周 | 约束模型设计、算法选型、集成方案、排程规则库 | 系统设计文档、集成规范 |
| 系统开发 | 6-10周 | 算法引擎配置、约束模型实现、接口开发、甘特图开发 | 可运行系统 |
| 集成测试 | 3-4周 | ERP/MES接口联调、排程结果验证、性能压测 | 测试报告 |
| 试运行 | 4-8周 | 并行排产、结果比对、参数调优、用户培训 | 试运行报告、SOP |
| 正式上线 | 2-4周 | 分步切换（按产线/工厂）、持续优化 | 上线验收报告 |

> **典型实施周期**：4-8个月 | **核心风险阶段**：约束建模与算法调优

### 关键风险与应对

| 风险项 | 影响 | 应对策略 |
|--------|------|----------|
| 基础数据质量差 | 排程结果不可用 | 前置数据治理，建立BOM/工艺/工时数据清洗机制 |
| 约束模型不完整 | 排程结果无法落地 | 分步建模，先覆盖核心约束再逐步扩展 |
| 算法调优困难 | 排程目标无法收敛 | 引入行业模板，结合实际反馈迭代优化参数 |
| ERP/MES集成延迟 | 数据断流、排程脱节 | 提前定义接口规范，建立数据校验与补偿机制 |
| 用户排斥新系统 | 系统上线后回退手工排产 | 并行运行期充分比对，以数据证明系统价值 |

### 预期ROI

| 指标 | 实施前（典型） | 实施后（目标） | 改善幅度 |
|------|---------------|---------------|----------|
| 计划准交率 | 65%-75% | 90%-95% | +20%-30% |
| 产能利用率 | 60%-70% | 75%-85% | +10%-20% |
| 排程编制时间 | 4-8小时/次 | 15-30分钟/次 | 缩短80%-90% |
| 库存周转天数 | 45-60天 | 30-40天 | 降低25%-35% |
| 紧急插单响应 | 2-4小时 | 5-15分钟 | 缩短95%+ |

> **投资回报周期**：通常12-18个月收回投资，主要收益来自准交率提升带来的客户满意度改善和产能释放

## 方案架构

### 功能模块

1. **需求管理**：订单接收、需求预测、优先级管理
2. **主计划**：产能规划、粗能力评估（RCCP）
3. **物料计划**：BOM展开、库存分配、采购建议
4. **详细排程**：车间级工序排程、资源分配
5. **监控看板**：计划执行监控、偏差预警
6. **报表分析**：准交率、产能利用率、计划达成率

### 约束模型

- **设备约束**：加工能力、加工范围、维护计划、可用时间
- **模具约束**：模具数量、共享规则、使用寿命
- **人员约束**：技能矩阵、班次安排、休假计划
- **物料约束**：库存水平、采购提前期、安全库存
- **工艺约束**：工序顺序、前置关系、标准工时

## 行业应用场景

### 离散制造
- **汽车零部件**：多品种小批量、JIT看板拉动排产
- **机械装备**：项目式生产、长周期物料协同
- **电子制造**：SMT线体排程、快速换线优化

### 流程制造
- **化工/制药**：批次管理、配方约束、保质期管理
- **食品饮料**：季节性波动、有效期控制、产线清洗排程
- **钢铁冶金**：连续工艺、热装热送、能耗优化排产

## 与ERP/MES集成

| 集成对象 | 数据流向 | 关键数据 |
|----------|----------|----------|
| ERP → APS | 订单、BOM、库存 | 销售订单、生产订单 |
| APS → ERP | 排程结果、采购建议 | 工单日期、需求计划 |
| MES → APS | 实际完成情况 | 工序报工、设备状态 |
| APS → MES | 派工指令 | 加工顺序、开始时间 |
| WMS → APS | 物料可用性 | 库存数量、入库计划 |

## 方案价值

1. **准交率提升**：从70%提升至90%+
2. **产能利用率**：提升10%-20%
3. **计划编制时间**：从天级缩短至小时级
4. **库存周转**：降低安全库存15%-30%
5. **插单响应**：分钟级影响评估和重排
