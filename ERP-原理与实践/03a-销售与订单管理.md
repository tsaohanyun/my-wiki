# 销售与订单管理

## 概述

销售与订单管理是需求管理的重要组成部分，涵盖从客户需求接收到货款交付的完整业务流程。本页聚焦销售业务流程、销售管理基本内容、信用管理以及数据驱动的销售分析。

> 📎 需求预测方法、物料编码、安全库存等基础概念请参见 → [[03-需求管理]]

## 核心内容

### 3.3 销售管理

#### 销售业务流程

```
客户需求 → 销售订单 → 计划部门 → 生产计划/采购计划
→ 生产部门 → 产品入库 → 发货 → 客户验货 → 货款交付
```

#### 销售管理基本内容

| 业务内容 | 主要功能 |
|----------|----------|
| 销售计划管理 | 商品计划、渠道计划、成本计划、促销计划等 |
| 产品定价管理 | 根据区域、客户、批量、交货期执行不同价格 |
| 销售报价管理 | 估算产品成本，确定价格和交货期承诺 |
| 销售订单管理 | 记录产品、数量、规格、交货时间和地点 |
| 销售收发货管理 | 分批/集中出货，退货/换货处理 |
| 销售服务管理 | 售前咨询、售中跟踪、售后安装/维护/维修 |
| 基础数据维护 | 客户资料、销售员资料、交货方式等 |

#### 信用管理

**信用管理五大要素：**

| 要素 | 定义 | 关键指标 |
|------|------|----------|
| 信用标准 | 对客户资信要求的最低标准 | DSO（销售变现天数）、坏账损失率 |
| 信用额度 | 授予客户的信用限额 | 基于总体额度和客户资信 |
| 信用条件 | 支付货款的条件 | 如2/10、n/30 |
| 信用期限 | 赊销清偿期限 | 15天、30天、60天、90天等 |
| 现金折扣 | 短期付款给予的折扣优惠 | 如5/20、n/60 |

**信用管理意义：**
- **短期**：监控应收账款回收，及时处理问题
- **长期**：提升客户质量，建立稳定守信的客户群

### 3.4 数据驱动下的销售分析

**大数据在销售中的七大作用：**

1. 差别定价策略优化（价格提高1%→利润增加8.7%）
2. 更大的客户回应率和深层客户信息
3. 客户分析、操作分析、欺诈和合规
4. 完善客户关系和客户忠诚度
5. 准确理解每个商业增长点
6. 电商推荐系统（用户画像+行为分析）
7. 广告精准投放系统

**销售分析四大维度：**
- **客户分析**：购买类型、购买周期、客户贡献率（20/80规律）
- **产品分析**：区域/季节销售规律、价格变动趋势、产品生命周期
- **组织分析**：销售人员业绩评估、能力提升与培训方向
- **趋势分析**：年度同比、季节性变化规律

## 关键概念

| 术语 | 英文/缩写 | 定义 |
|------|-----------|------|
| 信用标准 | Credit Standard | 对客户资信要求的最低标准 |
| 信用额度 | Credit Limit | 授予客户的信用限额 |
| 信用条件 | Credit Terms | 支付货款的条件 |
| 信用期限 | Credit Period | 赊销清偿期限 |
| DSO | Days Sales Outstanding | 预期销售变现天数 |
| 现金折扣 | Cash Discount | 短期付款给予的折扣优惠 |
| 销售订单 | Sales Order | 记录客户购买产品信息的单据 |

## 实际应用

### 案例分析：大数据挖掘数据价值

**亚马逊：**
- 记录用户所有行为（页面停留、搜索关键词、浏览商品）
- "买过X的人也买过Y"的精准推荐
- 标品（图书、家电）预测准，软需求产品（服装）预测难度大
- 网站设计方案持续A/B测试，追求最优转化率

**中国移动：**
- 客户流失预警：结合社交网络等新型数据源，全面获取客户真实体验
- 数据增值应用：政府服务、交通、应急等领域的数据服务

**兴趣点个性化推荐：**
- 融合时空信息的对比学习模型
- 解决用户签到数据稀疏问题
- 关注冷门用户的个性化推荐需求
