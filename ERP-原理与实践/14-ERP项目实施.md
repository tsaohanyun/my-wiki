# 第14章 ERP项目实施

> 项目：**ERP知识库**

## 概述

ERP项目的实施是指将ERP功能系统合理地应用到客户的实际业务环境中，建立ERP运行系统的过程。在ERP软件实施中有一句行话："三分软件，七分实施。"ERP项目是否成功，实施起了关键性的作用。

ERP项目实施涉及软件产品成熟度、实施人员能力、企业管理水平、领导重视程度、员工素质、企业文化等多方面因素，是一个复杂而艰巨的系统工程。

## 核心内容

### 项目实施的六大条件

1. 企业有实现现代企业制度的机制，有长远的经营战略
2. 产品有生命力，有稳定的经营环境
3. 领导班子有改革开拓精神，有决心承担项目成败责任
4. 管理工作基础扎实（档案齐全、数据可靠、规章制度完善）
5. 各级管理层理解ERP项目，有一致的、明确的、可量化的目标
6. 领导的决心和投入——ERP项目实质上是管理模式的变革

### 项目实施的四大制约因素

| 因素 | 关键问题 |
|------|----------|
| **人的因素** | 高层领导重视不够；员工信息化素质低、抵制变革；对ERP内涵认识不足（认为ERP只是用计算机代替手工操作） |
| **数据的因素** | "三分技术，七分管理，十二分数据"；基础数据缺乏、不准确、不合要求 |
| **管理的因素** | 管理薄弱，缺乏战略观念和系统观念；流程优化是实施ERP的基础和前提 |
| **技术的因素** | 不仅仅是购买软件；缺乏技术支持力量 |

### 项目实施的风险类型

| 风险类型 | 具体表现 |
|----------|----------|
| **管理风险** | 组织结构改变、员工职权变化、跨部门流程管理、数据所有权与维护问题 |
| **技术风险** | 高度集成模块的连锁故障、分布式架构的安全风险、B/S技术的非法访问风险、系统审计难度加大 |
| **人才流失** | 项目实施精英可能被咨询公司高薪挖走 |

### 项目成功实施的十大关键

1. **领导重视**——"一把手"工程
2. **产品选型**——考虑功能先进性、安全性、可扩展性、价格等
3. **建立强有力的实施团队**——高层领导+管理咨询专家+IT人员+业务骨干
4. **需求和数据分析**——需求分析越详细越好，数据必须准确
5. **相关人员积极配合**——需要业务骨干大力配合
6. **重视培训教育**——自始至终重视培训，统一思想认识
7. **规避风险**——从选型到上线全过程风险防范
8. **严格的工作纪律**——规范化管理，严格遵照执行
9. **深化改革，重视业务流程优化**——从分析现有业务流程入手
10. **健全激励机制**——建立复合型人才队伍

### 项目实施前的准备工作

```
准备工作
├── 成立项目筹备小组
│   └── 管理者代表 + 管理部门领导 + 信息部门领导 + 业务骨干
├── ERP知识培训
│   └── 高层领导及项目组成员掌握ERP基本原理和管理思想
├── 可行性分析与立项
│   └── 从技术、财务、时间、资源等方面进行可行性分析
├── 需求分析
│   └── 部门业务需求、权限设置、报表需求、数据接口开放性
├── 测试数据准备
│   └── 从各主要业务数据中抽取典型数据
└── 选型或转入开发
    └── 考虑软件功能、开发工具、文档、售后服务、价格、运行环境等
```

### 项目实施的完整流程

| 阶段 | 主要工作 |
|------|----------|
| **1. 成立三级项目组织** | 项目领导小组→项目实施小组→项目应用小组 |
| **2. 制订项目实施计划** | 项目进度计划+业务改革计划，可分期实施 |
| **3. 调研与咨询** | 企业管理现状描述→新的管理方式→业务实现与改革→达到的效果 |
| **4. 系统软件安装** | 先安装服务器，再逐步扩充工作点 |
| **5. 培训与业务改革** | 理论培训、实施方法培训、操作培训、维护培训等 |
| **6. 准备数据** | 初始静态数据+业务输入数据+业务输出数据 |
| **7. 原型测试** | 实战模拟，检验系统数据处理正确性 |
| **8. 用户化与二次开发** | 尽量减少二次开发，重点放在报表与特殊业务需求 |
| **9. 建立工作点** | 定义各工作点作业规范，形成标准管理文档 |
| **10. 并行** | ERP系统与原有系统同步运行，保持业务连续性 |
| **11. 正式运行** | 系统切换，完全转入ERP系统处理 |

### 项目实施效果的四大评价标准

| 标准 | 说明 |
|------|------|
| **系统运行集成化** | 系统全部运转，成为各部门日常工作的有效工具 |
| **业务流程合理化** | 业务处理流程趋于合理化，企业竞争力提升 |
| **绩效监控动态化** | 建立动态监控管理绩效变化的报表体系 |
| **管理改善持续化** | 建立不断自我评价和管理改善的机制 |

### 项目实施与企业变革

ERP实施引起的四大变革：

**1. 作业手段的变革**：从以纸为主到基本无纸的转变
- 员工对系统的不放心、不方便感
- 手工容易做的修改在系统里改起来麻烦
- 一个部门的数据错误可能影响其他部门

**2. 管理模式的变革**：
- 业务流程再造只是管理模式变革的一部分
- 完整的管理系统包含：决策权分配子系统、绩效评定子系统、利益分配子系统

**3. 工作关系的变革**：
- 权力分配变化——有人权力加强，有人权力削弱
- 组织扁平化导致地位和责任变化

**4. 观念的变革**：
- 规则意识：遵守预先制定的规则和程序
- 定量意识：尽可能用数据说话
- 共享意识：信息共享而非部门所有
- 安全意识：数据安全的概念和密码管理

### 业务流程再造（BPR）

BPR的四个关键词：
- **流程**：以从订单到交货的一连串作业活动为着眼点
- **根本性**：突破原有思维方式，回归零点
- **彻底性**：不是修修补补，而是彻底性变革
- **显著性**：在成本、质量、服务和速度上取得显著进展

BPR的两个思想基础：
1. **面向流程**（而非面向职能）——更直接地面对客户需求
2. **系统集成**——通过信息技术把过细的分工有机地集成

BPR实施过程：
1. 对原有流程进行全面的功能和效率分析
2. 设计新的流程改进方案并进行评估
3. 制定配套的组织结构、人力资源配置和业务规范改进规划
4. 组织实施与持续改善

## 关键概念

- **ERP实施**：将ERP功能系统应用到企业实际业务环境的过程
- **三级项目组织**：项目领导小组、项目实施小组、项目应用小组
- **BPR（Business Process Reengineering）**：业务流程再造
- **并行**：ERP系统与原有系统同步运行的过渡期
- **原型测试**：将收集的数据录入ERP系统进行实战模拟测试
- **用户化**：不涉及流程程序代码改动的系统调整
- **二次开发**：对ERP系统进行程序代码层面的修改
- **what-if分析**：假设不同场景分析其影响
- **ABCD评价**：ERP项目实施效果的分级评价标准

## 实际应用

### 案例：某金卡公司ERP项目实施

**企业背景**：中外合资大型制卡企业，上千家客户，产品保密性高，原有多个异构信息系统。

**面临的问题**：
- 科目过多（近2000个），库存商品科目200多个
- 大量业务信息通过纸面单据传递，核对困难
- 财务工作效率低下，月结需10个工作日
- 多品种小批量离散制造，质量管理要求严格

**实施过程要点**：

1. **项目组织**：设置项目指导委员会（高层支持）、项目质量经理（分阶段质量检查）、项目经理（由制造工艺部经理担任，而非IT经理）

2. **业务调研**：咨询顾问亲自参与生产计划编制过程深入了解业务；关键用户对功能需求报告签字确认

3. **方案设计**：从全局出发，各模块顾问相互沟通；业务流程追求简单和稳定

4. **系统测试**：主数据收集中发现编码不统一、重复编码等问题；对重要不同流程选择二次开发，不重要流程采取变通方法

5. **系统上线**：挑选70余名有责任心的员工完成数据收集；采用上线前盘点解决库存数据不准确问题；编写操作手册并进行最终培训（通过考核才能上岗）

**关键经验**：
- ERP项目不是一蹴而就的，实施过程中各种矛盾很正常
- "一把手"的作用：坚定信心、把握方向、资源保障、关键决策
- 项目实施各阶段员工心情呈曲折上升状态
- 进度延期越长，项目失败概率越大
