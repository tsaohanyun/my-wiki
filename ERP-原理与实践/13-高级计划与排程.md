# 第13章 高级计划与排程

> 项目：**ERP知识库**

## 概述

企业生产计划管理系统经历了从MRP→MRPⅡ→ERP的演进过程。然而在当今多元化的供应链环境下，计划对象不再局限于单一企业，还包括供应商与客户的整体资源。传统基于MRP逻辑的计划方法（假设产能无限、循序式计划等）已无法满足企业需求，取而代之的是高级计划与排程（Advanced Planning and Scheduling, APS）系统。

APS系统利用基因算法（GA）、约束理论（TOC）、运筹学（OR）、系统仿真、限制条件满足技术（CST）等先进技术，在考虑企业资源限制条件与生产现场控制规则下，制订可行的物料需求计划与生产排程计划。

## 核心内容

### APS的发展历史

| 阶段 | 时期 | 关键发展 |
|------|------|----------|
| 萌芽阶段 | 1950年代以前 | 甘特图（1917）、线性规划技术 |
| 计算机结合 | 1950-1960年代 | 大型企业配置计算机，线性规划普遍使用 |
| 企业个别开发 | 1960年代中期-1970年代 | MRP系统出现，模拟技术进入计划领域 |
| OPT/MRPⅡ/ERP结合 | 1980年代 | OPT（最优化生产技术）、人工智能、GUI出现 |
| 供应链管理结合 | 1990年代以后 | SCM思想兴起，APS嵌入ERP系统，B/S架构 |

### 传统生产计划与排程系统的七大缺点

| 缺点 | 说明 |
|------|------|
| **循序式计划方式** | 由上而下（top-down）计划，上层计划无法反映下层资源状况 |
| **功能模块无法有效整合** | 各阶段计划由独立系统完成，信息沟通困难 |
| **排程逻辑问题** | MRP无缺料、无产能限制的假设与实际差异大 |
| **独立进行需求/供给计划** | 需求计划与供给计划各自独立制订，缺乏交集 |
| **批量式计划** | 定期性批量计划，无法即时反应需求变化 |
| **单点计划范围** | 只针对单一工厂或配销中心，无法协调多厂计划 |
| **无法很好支持决策** | 缺乏what-if分析、情境分析与模拟等决策工具 |

### APS在排程方面的五大优势

1. **同步计划**：同时考虑整体供给与需求状况，进行供需平衡计划
2. **考虑资源限制的最佳计划**：运用数学模型、网络模型与模拟方法，同时考虑企业限制与目标
3. **即时性计划**：利用实时数据制订实时计划，快速处理紧急插单、设备宕机等例外事件
4. **支持决策能力**：具备what-if分析、情境分析及模拟仿真工具
5. **供需平衡的计划**：正确掌握、评估与有效计划企业的供给与需求

### APS系统功能模块

```
APS系统
├── 物料计划模块 — 中长期关键性物料计划，避免后续生产缺料
├── 生产计划模块 — 中期生产计划，物料采购计划+详细产能计划
├── 需求达交模块 — 即时性订单达交计划，ATP/CTP计算
└── 仿真平台模块 — 模拟各种变动造成的影响，支持决策
```

### APS系统与ERP系统的比较

| 比较维度 | ERP系统 | APS系统 |
|----------|---------|---------|
| 计划基础 | MRP逻辑（无限产能假设） | 有限资源能力约束 |
| 计划方式 | 循序式（由上而下） | 同步计划（供需同时考虑） |
| 优化能力 | 可行性为主，不做优化 | 基于数学模型的优化 |
| 实时性 | 批量式处理 | 即时性计划 |
| 决策支持 | 简单计划功能 | what-if分析、情境分析、模拟仿真 |
| 计划范围 | 单一企业/工厂 | 可覆盖整个供应链 |

> **关键区别**：APS不能对业务进行管理（如货物接收、发货、开发票、财务等），也不能处理数据维护（如BOM维护、工艺路线维护等）。APS是SCM的核心，能代替ERP的预测计划、DRP、MPS、MRP、CRP和生产计划。

### APS与ERP的集成方式

**方式一：分散数据模式**
- 从ERP数据库实时提取数据导入APS数据库
- 进行优化计算形成多个方案
- 通过人机界面提交给计划员决策，再导回ERP执行
- 优点：灵活配置；缺点：数据在多数据库间交换

**方式二：共用数据模式（发展趋势）**
- ERP厂商将APS技术嵌入ERP系统
- 用基于约束理论的APS技术替代无限约束策略的MRP技术
- ERP与APS共用同一数据库

### APS系统的规划过程

1. APS服务器自ERP系统下载所需数据
2. 将数据转换成APS系统所需格式
3. APS服务器将数据输入内存
4. 运算产生物料计划
5. 使用者根据需要进行仿真
6. 比对结果，选取所需计划
7. 转换成ERP系统档案
8. 将结果上传至ERP系统

### APS与SCM的关系

APS是SCM的核心，能够统一协调企业之间的长期、中期、近期计划。APS在供应链中的四大作用：

1. 通过整个供应链进行成本和服务的优化（ECR模式）
2. 最大化满足客户和消费者需求（实时报警+实时基于约束的重新计划）
3. 将需求信息更准确地传递给上游企业
4. 促进新产品的开发和推出

## 关键概念

- **APS（Advanced Planning and Scheduling）**：高级计划与排程系统
- **TOC（Theory of Constraints）**：约束理论，OPT的理论基础
- **ATP（Available to Promise）**：可承诺量，基于订单预留的主计划量
- **CTP（Capable to Promise）**：可承诺能力，考虑实际生产能力的ATP
- **OPT（Optimal Production Technology）**：最优化生产技术
- **GA（Genetic Algorithm）**：基因算法
- **what-if分析**：假设分析，测试不同方案对计划的影响
- **同步计划**：同时考虑供给与需求的计划方式
- **层次计划**：按时间跨度和业务流程分层优化的计划方法

## 实际应用

### 案例1：Visual APS优化ERP应用

**A公司**：客户经常查询订单完成情况和预期交货信息。使用可视化APS技术后，实现了对整个工厂的实时掌控。

**B公司（汽配厂家）**：业务量大、批次小、频率高的制造厂商。原来因无法及时了解交货期，经常用快递运送产品，月费用高达上百万元。通过APS技术：
- 根据订单优先级自动计算CTP，快速响应客户需求
- 优化排程技术和精益思想缩短了生产提前期
- 降低运作成本，提高行业竞争力

### 案例2：某中药企业供应链计划优化

**背景**：营收规模上百亿元，近10个生产基地、20多个仓库，产品和原料2000多种。销售按时交付率仅70%，库存高位运行，供应链数据流转依赖线下表格。

**解决方案**：采用资源智能优化协同平台，建立季度-月度-周度三级计划体系：
- **季度计划**：解决中药材采购周期长、产能分配不均衡问题
- **月度计划**：解决提取物生产周期长的需求协同问题
- **周度计划**：解决成品生产和销售协同问题

**效果**：优化了库存结构，将库存备货前移至半成品，降低了整体库存水平。
