# WMS与MES测试方法论

## 概述

本文档整合 WMS（仓储管理系统）与 MES（制造执行系统）的 UAT（用户验收测试）方法论，涵盖测试大纲结构、场景化测试设计、数据集成测试要点以及文件收集清单规范，适用于仓储与生产管理相关项目的测试培训与参考。

---

## UAT测试大纲结构

### 测试大纲组成

一套完整的 UAT 测试脚本应包含以下核心模块：

- **数据集成测试**：验证系统间数据协同、同步机制及流程配置的正确性
- **各功能模块测试**：按业务模块逐项覆盖核心操作流程
- **场景化测试**：基于真实业务场景编排多步骤端到端测试用例
- **异常与边界测试**：验证系统在异常输入或边界条件下的表现

### 测试脚本版本管理

| 字段 | 说明 |
|------|------|
| 版本号 | 如 V1.0，随迭代更新 |
| 编制依据 | 业务需求文档、功能规格说明书 |
| 适用范围 | WMS / MES 对应的功能模块 |
| 评审状态 | 待评审 / 已评审 / 已定稿 |

---

## 数据集成测试

### 数据协同

数据协同是 WMS 与 MES 系统联调的关键环节，主要验证：

- 主数据（物料、供应商、客户等）在系统间的一致性
- 单据状态（订单、出入库单、生产工单等）的实时同步
- 接口字段映射的准确性，包括编码转换、单位换算
- 数据冲突时的处理策略（覆盖 / 跳过 / 报警）

### 同步队列信息

同步队列是保障数据传输可靠性的核心机制，测试要点包括：

- 队列消息的顺序性（先进先出）
- 消息重试机制（失败后自动重发次数与间隔）
- 队列积压时的处理能力（批量消费、流量控制）
- 断点续传与幂等性验证（重复消息不会导致数据异常）

### 出入库流程配置

出入库流程配置测试覆盖以下场景：

- **入库流程**：采购入库、生产入库、退货入库、调拨入库
- **出库流程**：销售出库、生产领料出库、调拨出库、报废出库
- **流程节点验证**：单据创建 → 审批 → 执行 → 回写确认
- **异常流程**：部分收货、超量收货拦截、审批驳回后重提

## WMS模块测试项

### 仓储管理核心功能

| 测试项 | 测试内容 | 预期结果 |
|--------|----------|----------|
| 库位管理 | 库位新增、修改、冻结、解冻 | 库位状态实时更新 |
| 入库操作 | 扫码入库、上架分配、库位推荐 | 入库单据状态正确流转 |
| 出库操作 | 波次分配、拣货、复核、发货 | 出库数量与单据一致 |
| 库存盘点 | 盘点计划创建、执行、差异处理 | 盘盈盘亏记录准确 |
| 库存调拨 | 调拨申请、审批、执行 | 源库位扣减、目标库位增加 |
| 批次管理 | 批次追溯、效期管理、先进先出 | 批次信息完整可追溯 |

### 系统集成验证

- WMS 与 ERP 的单据同步（采购订单、销售订单、库存变动）
- WMS 与 MES 的物料流转信息交互
- 条码/RFID 扫描数据的实时上传与校验

## MES模块测试项

### 生产管理场景

| 测试项 | 测试内容 | 预期结果 |
|--------|----------|----------|
| 生产工单 | 工单下达、排产、开工、完工 | 工单状态全程可追踪 |
| 生产领料 | 领料申请、审批、发料、退料 | 物料消耗与工单关联 |
| 工序报工 | 工序开始、暂停、完成、质检 | 工序进度实时更新 |
| 质量检验 | 来料检验、过程检验、成品检验 | 检验结果记录并可追溯 |
| 设备管理 | 设备状态监控、维保计划 | 设备异常及时预警 |

### 生产领料场景

生产领料是 MES 的核心业务场景之一，测试用例设计需覆盖：

1. **正常领料**：根据 BOM 自动生成领料单 → 仓库确认发料 → 系统扣减库存
2. **超额领料**：领料数量超出 BOM 用量 → 触发审批流程 → 审批通过后放行
3. **退料处理**：生产完成后剩余物料退回 → 退料单生成 → 库存回滚
4. **紧急领料**：跳过标准审批流程 → 事后补录审批 → 系统标记为紧急

### 开发场景测试

针对定制开发功能，需额外关注：

- 自定义表单的字段校验（必填、格式、范围）
- 工作流引擎的正确触发与流转
- 报表数据的准确性与导出功能
- 权限控制（角色级、字段级、数据级）

## 场景化测试设计方法

### 设计原则

- **真实性**：测试场景应来源于真实业务操作，覆盖日常高频流程
- **完整性**：每个场景包含完整的操作链路，避免碎片化测试
- **可复现性**：测试步骤明确、前置条件清晰、预期结果可量化
- **独立性**：场景之间尽量解耦，减少依赖关系

### 场景用例模板

```
场景编号：SCN-001
场景名称：采购入库全流程
前置条件：
  - 供应商主数据已同步
  - 采购订单已审批通过
测试步骤：
  1. 在 ERP 创建采购订单
  2. WMS 接收同步的到货通知
  3. 仓库人员扫码收货
  4. 系统自动推荐上架库位
  5. 确认上架，库存增加
  6. 回写 ERP 入库确认
预期结果：
  - 各系统单据状态一致
  - 库存数量准确
  - 操作日志完整记录
```

## 文件收集清单规范

### 清单用途

场景化文件收集用于：

- 确保 UAT 测试过程中所需的数据文件、配置文件齐全
- 标准化测试前置条件的准备流程
- 便于测试结果的归档与回溯

### 清单结构

| 序号 | 文件名称 | 文件类型 | 所属场景 | 收集状态 | 备注 |
|------|----------|----------|----------|----------|------|
| 1 | 物料主数据 | Excel | 数据集成 | 已收集 | 含物料编码、名称、规格 |
| 2 | 供应商清单 | Excel | 数据集成 | 已收集 | 需与 ERP 一致 |
| 3 | 库位布局图 | PDF | 仓储管理 | 待收集 | 含库位编码规则 |
| 4 | BOM 清单 | Excel | 生产领料 | 已收集 | 含替代料信息 |

### 收集流程

1. **需求识别**：根据测试大纲确定所需文件列表
2. **责任分配**：明确每份文件的提供方与收集人
3. **格式约定**：统一文件格式、编码规范、命名规则
4. **质量检查**：收集完成后进行数据完整性与格式校验
5. **归档管理**：按场景分类存储，版本化管理

## 测试执行建议

### 执行策略

- **分轮次执行**：第一轮覆盖核心流程，第二轮覆盖异常与边界
- **缺陷分级**：致命 > 严重 > 一般 > 建议，按优先级修复
- **回归测试**：缺陷修复后执行回归验证，避免引入新问题
- **测试报告**：每日汇总测试进度与缺陷统计

### 关键成功因素

1. 测试数据准备充分，覆盖主要业务场景
2. 测试环境与生产环境配置一致
3. 业务人员深度参与，确保测试用例贴近实际
4. 系统集成接口提前联调，减少联调阻塞
5. 文件收集清单管理规范，减少因数据缺失导致的测试延期
