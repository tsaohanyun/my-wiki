# 飞书 Aily 企业智能客服 / AI 情报 / 智能巡检案例

> 前置阅读：[[concepts/飞书Aily-自定义智能体配置|自定义智能体配置指南]]

## 案例一：企业智能客服

### 需求分析

| 角色 | 核心需求 |
|------|---------|
| 客户（C端） | 快速获得产品咨询、订单查询、售后处理等标准问题的答案 |
| 客服专员 | 减少重复性问题处理量，专注处理复杂/情绪化客户 |
| 客服主管 | 监控智能客服拦截率、满意度、未解决问题趋势 |

### System Prompt（可直接复制）

```markdown
# 角色
你是一名专业的客服助手。你需要礼貌、耐心、准确地回答客户的问题，提供产品咨询、订单查询、售后服务等支持。

## 技能

### 技能 1：产品咨询
从知识空间检索产品手册和常见问题，给出准确、简洁的回答。
- 回答格式：先给出结论，再补充细节
- 如涉及价格，需注明"以实际下单价格为准"

### 技能 2：订单查询
当客户提供订单号时，调用订单查询 MCP 工具。
- 返回：订单状态、物流信息、预计送达时间
- 如订单异常，需主动告知原因和解决方案

### 技能 3：售后处理
- 退换货：先确认是否在 7 天无理由退换期内，再引导操作
- 投诉：表达歉意 → 记录问题 → 生成工单 → 告知处理时限
- 维修：判断是否在保修期 → 提供维修方案或寄修地址

### 技能 4：智能转人工
以下情况自动转人工：客户明确要求、情绪激烈、连续2次未解答、退款>1000元

## 限制
- 不得泄露公司内部信息（成本、利润率等）
- 不得承诺超出政策范围的补偿
- 不得代替客户做最终决定
```

### 知识空间配置

| 知识文档 | 内容说明 | 更新频率 |
|---------|---------|---------|
| 产品手册 | 全线产品参数、功能说明 | 产品上新时 |
| FAQ 库 | 高频问题标准答案（100+条） | 每周补充 |
| 退换货政策 | 退换条件、流程、时限 | 政策变更时 |
| 保修政策 | 保修期限、范围、维修流程 | 季度更新 |

### MCP 接口

**订单查询**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| order_id | string | 是 | 订单编号 |

返回示例：
```json
{
  "order_id": "ORD20260615001",
  "status": "已发货",
  "product_name": "XX产品 Pro",
  "tracking_number": "SF1234567890",
  "estimated_delivery": "2026-06-18"
}
```

## 案例二：AI 情报系统

### 需求分析

| 角色 | 核心需求 |
|------|---------|
| 市场部 | 每日行业热点、竞品营销动态 |
| 产品部 | 竞品功能更新、用户反馈趋势 |
| 管理层 | 行业政策变化、市场趋势摘要 |

### System Prompt 核心

```markdown
# 角色
你是一名专业的情报分析员，负责搜集、筛选、整理行业情报。

## 技能
### 技能 1：信息搜集
通过搜索引擎搜集行业资讯，信息源优先级：官方公告 > 权威媒体 > 行业博客

### 技能 2：内容筛选与去重
去除广告、软文，合并相同事件报道，按重要性排序

### 技能 3：情报摘要生成
标题（一句话概括）→ 摘要（3-5句）→ 影响分析 → 建议行动 → 来源链接

## 输出格式
📰 [行业] 每日情报简报 — [日期]
🔥 重大事件：[标题 + 摘要 + 影响⭐ + 建议 + 来源]
📌 行业动态：3-5条一句话摘要
📊 数据速览：关键数据点
```

### 多维表格结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 日期 | 日期 | 情报日期 |
| 标题 | 文本 | 情报标题 |
| 摘要 | 多行文本 | 内容摘要 |
| 领域 | 单选 | 行业/竞品/政策/技术 |
| 重要性 | 单选 | ⭐~⭐⭐⭐⭐⭐ |
| 来源链接 | 链接 | 原文 URL |

### 定时任务

| 任务 | 触发时间 | 执行内容 |
|------|---------|---------|
| 每日情报搜集 | 每天 07:00 | 搜集 → 筛选摘要 → 写入表格 → 推送飞书群 |
| 每周趋势报告 | 每周五 17:00 | 汇总本周 → 趋势分析 → 推送管理层 |

## 案例三：制造业智能巡检

### 需求分析

| 角色 | 核心需求 |
|------|---------|
| 巡检员 | 快速上报设备状态，异常时获得即时处理建议 |
| 设备主管 | 实时掌握设备异常分布，安排维修计划 |
| 生产经理 | 巡检完成率、异常趋势、停机损失分析 |

### 异常判断标准

| 检查项 | 正常范围 | 预警值 | 告警值 |
|--------|---------|--------|--------|
| 温度 | ≤60℃ | 60-75℃ | >75℃ |
| 振动 | ≤4.5mm/s | 4.5-7.1mm/s | >7.1mm/s |
| 噪音 | ≤85dB | 85-95dB | >95dB |

### 数据表结构

**巡检记录表**：巡检日期/设备编号/检查项/标准值/实测值/判断结果/处理建议/巡检人

**告警工单表**：工单编号/设备编号/异常描述/严重等级/处理状态/创建时间/处理人

### 工作流配置

```
触发：巡检记录表新增 → 判断结果？
├─ 正常 → 仅记录
├─ 预警 → 推送飞书提醒给设备主管
└─ 告警 → 创建告警工单 → 紧急通知设备主管+生产经理
```

### 模拟对话

```
👤 巡检员：CNC-003 温度 72 度，振动 8.2

🤖 巡检智能助手：
📋 CNC-003 巡检结果：
• 温度 72℃ → ⚠️ 预警（标准≤60℃）
• 振动 8.2mm/s → 🚨 告警（标准≤4.5mm/s）

🚨 已自动创建告警工单 JB20260616-003，已通知设备主管张工

处理建议：
1. 立即检查主轴轴承状态
2. 检查联轴器对中情况
3. 如振动持续加剧，建议停机检修
```
