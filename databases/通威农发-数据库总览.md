## 数据库架构特点

- **MySQL 5.7/8.0** 兼容，引擎统一为 InnoDB
- 字符集 **utf8mb4**，排序规则 **utf8mb4_0900_ai_ci**
- 分区表策略：按 **日期+随机码** 后缀区分（如 `_0427l9atg69iu`），包含年度分区和月度分区
- 大量使用 **decimal(22,6)** 处理重量/数量精度（饲料行业特点）
- 通用字段：`id`(PK), `warehouse_id`, `tenant_id`, `create_user`, `create_date`, `update_user`, `update_date`
- 与 **EBS (Oracle ERP)** 系统有数据接口和同步机制

## 已知视图BUG

### view_wms_ext_ng_stock_order — 企业关联错误（2026-05-14发现）

**错误定义**（snest.sql 第20889行）：
```sql
CREATE VIEW view_wms_ext_ng_stock_order AS
SELECT n.warehouse_id AS company_id, ...
FROM wms_ext_ng_stock_order n
LEFT JOIN res_enterprise re ON re.id = n.warehouse_id  -- ❌ BUG
WHERE n.bus_type = '1'
```

**问题**：`warehouse_id` 是仓库ID，不是企业ID。直接关联 `res_enterprise` 会得到错误的企业信息。

**正确关联路径**：
```
wms_ext_ng_stock_order.cargo_owner_code
  → res_inv_org.inv_org_code
    → res_inv_org.company_id
      → res_enterprise.id
```

**修正视图**：
```sql
CREATE VIEW view_wms_ext_ng_stock_order AS
SELECT
  n.warehouse_id AS warehouse_id,
  rio.company_code, rio.company_name,
  n.item_code AS material_code,
  (n.qty - n.out_qty) AS quantity,
  n.primary_cause, n.secondary_cause, n.tertiary_cause,
  DATE_FORMAT(n.create_date, '%Y-%m-%d') AS date
FROM wms_ext_ng_stock_order n
  LEFT JOIN res_inv_org rio ON n.cargo_owner_code = rio.inv_org_code
  LEFT JOIN res_enterprise re ON re.id = rio.company_id
WHERE n.bus_type = '1'
```

**关键中间表**：`res_inv_org`（库存组织表），通过 `inv_org_code` ↔ `company_id` 建立仓库到企业的正确关联。

### SQL对比分析发现的其他差异

CPT报表SQL（如1返工料比例）与简化的视图SQL之间还存在以下差异：

| 差异点 | CPT报表SQL | 简化视图SQL |
|--------|-----------|------------|
| 企业关联 | cargo_owner_code → res_inv_org → res_enterprise ✅ | warehouse_id → res_enterprise ❌ |
| bus_type | `!= '0'`（包含所有非零类型） | `= '1'`（只取影响类型） |
| 物料分类 | INNER JOIN `wms_ext_part_md` + `category LIKE '11.30.%'` | 无此过滤 |
| 状态过滤 | `state IN ('COMPLETE','PROCESS')` | 无过滤 |
| 行级过滤 | `realqty != 0` | 无过滤 |
| 用户权限 | 按当前用户仓库权限过滤 | 无过滤 |

详见 [[通威农发-CPT报表分析]] 和 [[fine-report-cpt]] 技能文档。
