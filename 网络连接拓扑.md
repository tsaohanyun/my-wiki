## 连接一：服务器 ↔ 个人电脑 (反向隧道)

### 基本信息
| 项目 | 值 |
|------|-----|
| 端口 | 2222 (服务器本地) → 22 (个人电脑) |
| 方向 | 个人电脑主动建立反向隧道 |
| 认证 | 私钥认证 |
| 密钥路径 | Windows: `C:\Users\ThinkPad\.ssh\hermes_dashboard\hermesdashborad.pem` |
| 用途 | 文件传输、远程命令执行 |

### 个人电脑端启动命令
```batch
ssh -i C:\Users\ThinkPad\.ssh\hermes_dashboard\hermesdashborad.pem -p 22222 -R 2222:localhost:22 -L 9119:localhost:9119 -o ServerAliveInterval=30 -o ServerAliveCountMax=5 -N agentuser@162.14.73.9
```

### 参数说明
| 参数 | 说明 |
|------|------|
| `-i` | 指定私钥文件 |
| `-p 22222` | 连接服务器的SSH端口 |
| `-R 2222:localhost:22` | 反向隧道：服务器2222 → 本地22 |
| `-L 9119:localhost:9119` | 本地转发：Hermes WebUI端口 |
| `-o ServerAliveInterval=30` | 每30秒发送心跳 |
| `-o ServerAliveCountMax=5` | 5次无响应断开 |
| `-N` | 不打开shell，只建隧道 |

### 服务器端使用命令
```bash
# 测试连接
ssh -p 2222 -i ~/.ssh/hermes_dashboard/hermesdashborad.pem -o StrictHostKeyChecking=no ThinkPad@localhost echo "OK"

# 推送文件到桌面
scp -P 2222 -i ~/.ssh/hermes_dashboard/hermesdashborad.pem /path/to/file ThinkPad@localhost:"C:/Users/ThinkPad/Desktop/"

# 从个人电脑拉取文件
scp -P 2222 -i ~/.ssh/hermes_dashboard/hermesdashborad.pem ThinkPad@localhost:"C:/path/to/file" /local/path/

# 在个人电脑上执行命令
ssh -p 2222 -i ~/.ssh/hermes_dashboard/hermesdashborad.pem ThinkPad@localhost "dir C:\\Users\\ThinkPad\\Desktop"
```

### 保活机制
- 客户端：`ServerAliveInterval=30` + `ServerAliveCountMax=5`
- 脚本位置：`C:\Users\ThinkPad\Desktop\ssh_connect_to_server.bat`

### 故障排查
| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| Connection refused | 反向隧道未建立 | 检查个人电脑端脚本是否运行 |
| Permission denied | 密钥问题 | 检查密钥路径和权限 |
| Connection timeout | 网络问题 | 检查防火墙、端口映射 |

## 连接二：服务器 ↔ 跳板机 (正向SSH)

### 基本信息
| 项目 | 值 |
|------|-----|
| 端口 | 2223 (服务器本地) → 22 (跳板机) |
| 方向 | 服务器主动连接跳板机 |
| 认证 | 密码认证 (sshpass) |
| 用户 | win10 |
| 密码 | Y*7@jB3#8& |
| 用途 | 访问内网API、数据库端口转发 |

### 服务器端启动命令
```bash
# 数据库隧道 - UAT
sshpass -p 'Y*7@jB3#8&' ssh -p 2223 -o StrictHostKeyChecking=no -o ServerAliveInterval=30 -o ServerAliveCountMax=3 -L 13307:172.20.194.157:3306 -N win10@localhost

# 数据库隧道 - PROD
sshpass -p 'Y*7@jB3#8&' ssh -p 2223 -o StrictHostKeyChecking=no -o ServerAliveInterval=30 -o ServerAliveCountMax=3 -L 13306:172.20.193.138:3306 -N win10@localhost

# 交互式登录
sshpass -p 'Y*7@jB3#8&' ssh -p 2223 -o StrictHostKeyChecking=no win10@localhost
```

### 跳板机反向隧道（由跳板机发起）
```bash
# 在跳板机上执行，建立到服务器的反向隧道
ssh -i hermesdashborad.pem -p 22222 -R 2223:localhost:22 -N agentuser@162.14.73.9
```

### 保活机制
- `ServerAliveInterval=30` + `ServerAliveCountMax=3`
- 跳板机Windows任务计划：AtLogOn触发 + 每10分钟检查

### 故障排查
| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| Connection refused | 跳板机SSH服务未运行 | 检查跳板机OpenSSH服务 |
| Authentication failed | 密码错误 | 确认密码 Y*7@jB3#8& |
| Tunnel端口占用 | 旧连接未释放 | kill相关SSH进程 |

## 连接三：服务器 ↔ 数据库 (通过跳板机)

### 基本信息
| 数据库 | 端口(本地) | 远程地址 | 权限 | 用途 |
|--------|-----------|----------|------|------|
| UAT | 13307 | 172.20.194.157:3306 | 读写 | 开发测试 |
| PROD | 13306 | 172.20.193.138:3306 | 只读 | 生产查询 |

### 专用账号
| 项目 | 值 |
|------|-----|
| 用户名 | db-tunnel |
| 密码 | Tunnel@2026! |
| 说明 | 仅TCP转发，nologin shell，MaxSessions=4 |

### 连接方式
```bash
# 通过MCP SQLAlchemy (推荐)
# 已配置在 ~/.hermes/config.yaml，自动连接127.0.0.1:13306/13307

# MySQL客户端直连 - PROD
mysql -h 127.0.0.1 -P 13306 -u app -p snest

# MySQL客户端直连 - UAT
mysql -h 127.0.0.1 -P 13307 -u app -p snest
```

### 同事共享方案
```bash
# 同事在自己电脑上执行，通过跳板机访问数据库
ssh -L 13306:127.0.0.1:13306 -L 13307:127.0.0.1:13307 -N db-tunnel@162.14.73.9 -p 22222
```

### 保活机制
- 依赖跳板机SSH隧道的保活
- MaxSessions=4 限制并发连接数

### 故障排查
| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| Can't connect to MySQL | 数据库隧道断开 | 重建跳板机SSH连接 |
| Too many connections | MaxSessions超限 | 清理闲置连接 |
| Access denied | 密码错误或权限不足 | 确认账号密码、validflag=1 |

## 保活监控

### 服务器端监控脚本
```bash
# 检查所有SSH隧道状态
ps aux | grep ssh | grep -v grep

# 检查端口监听状态
ss -tlnp | grep ssh

# 检查特定端口
netstat -tlnp | grep -E '2222|2223|13306|13307'
```

### 自动重连注意事项
- **个人电脑**：无监控无自动重连，需手动运行脚本
- **跳板机**：有双触发保活（AtLogOn + 每10分钟）
- **数据库隧道**：依赖跳板机连接，无独立保活
- **Hermes Cron**：no_agent模式不可完全信赖（2026-06-06曾静默失效6天）

## 常用命令速查

### 查看当前连接
```bash
# 查看SSH进程
ps aux | grep ssh

# 查看端口监听
ss -tlnp | grep -E '2222|2223|13306|13307'
```

### 重建连接
```bash
# 重建跳板机隧道（UAT + PROD）
pkill -f "ssh.*2223.*13306"
pkill -f "ssh.*2223.*13307"
sshpass -p 'Y*7@jB3#8&' ssh -p 2223 -o StrictHostKeyChecking=no -o ServerAliveInterval=30 -o ServerAliveCountMax=3 -L 13307:172.20.194.157:3306 -N win10@localhost &
sshpass -p 'Y*7@jB3#8&' ssh -p 2223 -o StrictHostKeyChecking=no -o ServerAliveInterval=30 -o ServerAliveCountMax=3 -L 13306:172.20.193.138:3306 -N win10@localhost &
```

### 测试连接
```bash
# 测试个人电脑连接
ssh -p 2222 -i ~/.ssh/hermes_dashboard/hermesdashborad.pem -o ConnectTimeout=5 ThinkPad@localhost echo "OK"

# 测试跳板机连接
sshpass -p 'Y*7@jB3#8&' ssh -p 2223 -o ConnectTimeout=5 win10@localhost echo "OK"

# 测试数据库连接
mysql -h 127.0.0.1 -P 13306 -u app -p -e "SELECT 1" snest
```

## 安全注意事项

1. **密钥安全**：私钥文件权限600，不要泄露
2. **密码安全**：跳板机密码、数据库密码不要明文存储在脚本中
3. **端口暴露**：服务器只暴露22222端口，其他端口仅本地监听
4. **数据库权限**：PROD只读，UAT读写，操作需谨慎
5. **validflag规则**：查询必须加 `WHERE validflag='1'`
