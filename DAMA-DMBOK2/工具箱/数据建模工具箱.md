# 数据建模工具箱

## 概述

本工具箱提供数据建模的方法论、模板和最佳实践。

## 1. 建模方法论

### 三范式设计

**第一范式（1NF）**：字段原子性
```
❌ 错误：skills = "Java, Python, SQL"
✅ 正确：
- developer_skill表
  - developer_id
  - skill_name
```

**第二范式（2NF）**：消除部分依赖
```
❌ 错误：order表包含product_name（依赖product_id）
✅ 正确：
- order表：order_id, product_id, quantity
- product表：product_id, product_name
```

**第三范式（3NF）**：消除传递依赖
```
❌ 错误：employee表包含department_name（依赖department_id）
✅ 正确：
- employee表：employee_id, department_id
- department表：department_id, department_name
```

### 维度建模

**星型模型**：
```
        ┌─────────┐
        │ 时间维度 │
        └────┬────┘
             │
┌─────────┐  │  ┌─────────┐
│ 产品维度 ├──┼──┤ 事实表   │
└─────────┘  │  └────┬────┘
             │       │
        ┌────┴────┐  │
        │ 客户维度 │  │
        └─────────┘  │
                     │
                ┌────┴────┐
                │ 地区维度 │
                └─────────┘
```

## 3. 建模SQL模板

### 创建表模板

```sql
-- 创建工单表
CREATE TABLE mes_work_order (
    order_id VARCHAR(32) NOT NULL COMMENT '工单号',
    order_type VARCHAR(20) NOT NULL COMMENT '工单类型',
    product_id VARCHAR(32) NOT NULL COMMENT '产品ID',
    line_id VARCHAR(32) COMMENT '产线ID',
    planned_qty DECIMAL(10,2) NOT NULL COMMENT '计划数量',
    actual_qty DECIMAL(10,2) COMMENT '实际数量',
    order_status VARCHAR(20) NOT NULL DEFAULT 'DRAFT' COMMENT '工单状态',
    start_time DATETIME COMMENT '开始时间',
    end_time DATETIME COMMENT '结束时间',
    validflag CHAR(1) NOT NULL DEFAULT '1' COMMENT '有效标志',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    create_by VARCHAR(32) NOT NULL COMMENT '创建人',
    update_time DATETIME COMMENT '修改时间',
    update_by VARCHAR(32) COMMENT '修改人',
    PRIMARY KEY (order_id),
    INDEX idx_product_id (product_id),
    INDEX idx_line_id (line_id),
    INDEX idx_order_status (order_status),
    INDEX idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='生产工单表';
```

### 索引设计原则

```sql
-- 1. 主键索引（自动创建）
PRIMARY KEY (order_id)

-- 2. 外键索引
INDEX idx_product_id (product_id)

-- 3. 查询频繁的字段
INDEX idx_order_status (order_status)

-- 4. 组合索引（最左前缀）
INDEX idx_status_time (order_status, create_time)

-- 5. 覆盖索引
INDEX idx_covering (order_status, product_id, planned_qty)
```

## 4. 数据模型评审清单

### 设计评审

- [ ] 是否符合三范式（或明确的反范式理由）
- [ ] 主键设计是否合理
- [ ] 外键关系是否正确
- [ ] 字段类型是否合适
- [ ] 是否有审计字段（create_time, update_time等）

### 性能评审

- [ ] 索引设计是否合理
- [ ] 是否有冗余字段（反范式）
- [ ] 大表是否考虑分区
- [ ] 查询SQL是否高效

### 业务评审

- [ ] 是否满足业务需求
- [ ] 是否考虑扩展性
- [ ] 是否有数据字典
- [ ] 是否有变更记录
