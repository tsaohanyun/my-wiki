# 数据质量工具箱

## 概述

本工具箱提供数据质量评估、监控和改进的实用工具。

## 1. 数据质量维度评估表

### 六大维度检查

| 维度 | 检查项 | SQL示例 | 阈值 |
|------|--------|---------|------|
| 准确性 | 数据是否正确 | 与源系统比对 | >99% |
| 完整性 | 数据是否缺失 | NULL检查 | >99% |
| 一致性 | 数据是否矛盾 | 跨表比对 | >99% |
| 及时性 | 数据是否延迟 | 时间差检查 | <1小时 |
| 唯一性 | 数据是否重复 | 重复检查 | 0% |
| 有效性 | 数据是否合规 | 格式检查 | >99% |

### 综合评估SQL

```sql
-- 数据质量综合评估
WITH quality_metrics AS (
    -- 完整性
    SELECT 
        '完整性' as dimension,
        COUNT(*) as total,
        SUM(CASE WHEN field1 IS NULL OR field2 IS NULL THEN 1 ELSE 0 END) as failed
    FROM target_table
    WHERE validflag = '1'
    
    UNION ALL
    
    -- 唯一性
    SELECT 
        '唯一性' as dimension,
        COUNT(*) as total,
        SUM(CASE WHEN cnt > 1 THEN 1 ELSE 0 END) as failed
    FROM (
        SELECT unique_field, COUNT(*) as cnt
        FROM target_table
        WHERE validflag = '1'
        GROUP BY unique_field
    ) t
    
    UNION ALL
    
    -- 有效性
    SELECT 
        '有效性' as dimension,
        COUNT(*) as total,
        SUM(CASE WHEN NOT REGEXP_LIKE(field3, '^[0-9]{4}-[0-9]{2}-[0-9]{2}$') THEN 1 ELSE 0 END) as failed
    FROM target_table
    WHERE validflag = '1'
)
SELECT 
    dimension,
    total,
    failed,
    ROUND((total - failed) / total * 100, 2) as pass_rate,
    CASE 
        WHEN (total - failed) / total >= 0.99 THEN '优秀'
        WHEN (total - failed) / total >= 0.95 THEN '良好'
        WHEN (total - failed) / total >= 0.90 THEN '一般'
        ELSE '需改进'
    END as quality_level
FROM quality_metrics;
```

## 2. 数据质量监控仪表盘

### 监控指标

```sql
-- 每日质量趋势
SELECT 
    DATE(check_time) as check_date,
    dimension,
    AVG(pass_rate) as avg_pass_rate,
    MIN(pass_rate) as min_pass_rate,
    MAX(pass_rate) as max_pass_rate
FROM dq_daily_check
WHERE check_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(check_time), dimension
ORDER BY check_date DESC, dimension;
```

### 告警规则

| 指标 | 阈值 | 告警级别 | 通知方式 |
|------|------|----------|----------|
| 完整性 | <95% | 严重 | 立即通知 |
| 完整性 | <99% | 警告 | 每日汇总 |
| 唯一性 | >0 | 严重 | 立即通知 |
| 及时性 | >2小时 | 警告 | 每日汇总 |

## 3. 数据质量改进模板

### 问题记录表

| 字段 | 说明 |
|------|------|
| issue_id | 问题编号 |
| issue_type | 问题类型（完整性/唯一性/一致性等） |
| table_name | 问题表名 |
| field_name | 问题字段 |
| issue_desc | 问题描述 |
| impact_scope | 影响范围 |
| root_cause | 根本原因 |
| solution | 解决方案 |
| owner | 负责人 |
| due_date | 截止日期 |
| status | 状态（OPEN/IN_PROGRESS/CLOSED） |

### 改进流程

```
发现问题 → 记录问题 → 分析根因 → 制定方案 → 实施改进 → 验证效果 → 关闭问题
```

## 4. 数据清洗规则模板

### 常用清洗规则

```sql
-- 1. 去除前后空格
UPDATE table_name 
SET field = TRIM(field) 
WHERE field != TRIM(field);

-- 2. 统一日期格式
UPDATE table_name 
SET date_field = STR_TO_DATE(date_field, '%Y/%m/%d')
WHERE date_field LIKE '%/%';

-- 3. 去除重复记录
DELETE t1 FROM table_name t1
INNER JOIN table_name t2
WHERE t1.id > t2.id 
    AND t1.unique_field = t2.unique_field;

-- 4. 填充默认值
UPDATE table_name 
SET field = 'UNKNOWN' 
WHERE field IS NULL;
```
