## 概述

**数据质量管理**（Data Quality Management, DQM）是数据管理体系中一项核心职能（Function），包含规划、实施和控制活动，旨在确保数据宜于消费并满足数据消费者的需求。数据质量可定义为**数据质量的维度满足业务需求的程度**，即**"适得其用"（Fit for Purpose）**。

数据质量管理的核心工作包括：基于业务需求识别关键数据、通过六大质量维度（准确性、完整性、一致性、时效性、唯一性、有效性）定义质量标准、运用PDCA循环持续改进、通过根因分析从源头解决问题，并嵌入业务流程实现预防性质量控制。

**主要业务驱动因素：** 提升利益相关方体验和组织声誉、提高组织效能、降低劣质数据风险和成本、改进组织效率和生产力。

### 同义词与相关概念

| 术语 | 同义词 | 反义/对比概念 |
|------|--------|-------------|
| 数据质量（Data Quality） | 数据品质、信息质量 | 数据缺陷（Data Defect） |
| 数据剖析（Data Profiling） | 数据探查、数据审计 | 数据修复（Data Remediation） |
| 数据清洗（Data Cleansing） | 数据净化、数据纠错 | 数据污染（Data Contamination） |
| 根因分析（Root Cause Analysis） | 问题溯源、因果分析 | 症状处理（Symptom Treatment） |
| 六大维度（Six Dimensions） | 数据质量维度框架 | — |
| PDCA（戴明环） | 持续改进循环、质量环 | 一次性修复（One-time Fix） |

## 数据质量管理语境关系图要素

| 要素 | 内容 |
|------|------|
| **定义** | 数据质量的规划、实施和控制活动，应用数据收集和处理技术，确保数据满足企业和本地消费者的需求，并适合使用 |
| **目标** | 1. 基于数据消费者的需求，制定受控的方法，使数据适合其目的；2. 在数据生命周期中定义数据质量控制的标准、要求和规范；3. 定义并实施测量、监控和报告数据质量水平的流程；4. 通过流程和系统改进，识别并倡导提升数据质量的机会 |
| **业务驱动输入** | 数据制度和标准、数据质量期望、业务要求、业务规则、数据要求、业务元数据、技术元数据、数据源和数据存储、数据沿袭 |
| **供给者** | 业务管理、主题领域专家、数据分析师、数据架构师、系统专家、数据管理员、业务流程分析师、数据供应商 |
| **方法** | 数据质量指标、数据剖析、预防和纠正措施、根本原因分析 |

## 数据质量改进核心原则

| 原则 | 说明 |
|------|------|
| **聚焦关键数据** | 聚焦于对企业和客户最为关键的数据 |
| **标准驱动** | 定义可度量的标准和期望 |
| **客观度量和透明** | 持续进行客观度量，与利益相关方共享 |
| **预防** | 重点放在防止数据出错上 |
| **根因修正** | 从根源解决问题而非停留在表象 |
| **嵌入业务流程** | 业务流程所有者对数据质量负责 |
| **体系化贯彻** | 系统化落实数据质量要求 |
| **关联服务水平** | 纳入SLA管理 |
