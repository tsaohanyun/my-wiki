## 工具与技术

数据质量工具的主要功能包括：

### 数据剖析工具

生成抽象的统计数据，使分析人员能够识别数据中的模式，进行初步质量评估和持续监测。具备数据可视化功能的剖析工具有助于数据探索流程。

### 业务规则模板和引擎

**业务规则模板（Business Rule Template）**是一种专有工具，分析人员用以获取对数据标准格式的期望。模板有助于填补业务和技术团队之间的沟通鸿沟，把业务需求转化为代码（无论该代码是嵌入在规则引擎中、数据分析工具的分析组件中，还是数据集成工具中）。一个模板包含多个部分，每部分用于针对性实施某种类型的业务规则。

### 数据解析和格式化

**数据解析（Data Parsing）**是利用预设规则分析数据的过程，以确定其内容或数值。通过提取独立组件（"标记"），创建有效模板，重新排列进标准化的表达式。发现无效模板时，应用程序可以试着将无效值转换为符合规则的值。

> 示例：可能有成千上万种不同表达形式的客户名称。一个好的标准化工具能够将客户名称的不同组件解析出来（名、中间名、姓氏、缩写、头衔、世袭称谓），并将这些组件重新排列成标准化的表达式。

### 数据转换和标准化

基于各种标准化技术构建，将原始格式和模板中的数值映射成目标表达式。标准化是一种特殊的数据转换，应用规则捕捉语境、语言学或公认的习惯用语。

### 数据增强

给数据集添加数据元素以提高数据质量（包括可用性）的过程。一些增强方式通过整合组织内部的数据集获得，也可通过购买外部数据的方式增强。

| 增强类型 | 说明 |
|----------|------|
| **时间/日期戳** | 记录数据项的创建、修改或过期时间，有助于追溯历史数据事件和根因分析 |
| **审计数据** | 记录数据血缘，用于历史追溯和验证 |
| **参考词汇表** | 业务特定术语、本体和术语表，增强理解和控制 |
| **上下文信息** | 位置、环境、访问方法和标签，供审查和分析 |
| **地理信息** | 地址标准化和地理编码，包括区域编码、市政划分、街区映射、经纬度 |
| **人口统计信息** | 年龄、婚姻状况、性别或收入；企业实体可关联年营收、员工数等 |
| **心理统计信息** | 行为、习惯或偏好分群，如产品品牌偏好、休闲活动、通勤方式等 |
| **估值信息** | 资产估值、库存和销售增强 |

### 事件管理系统

记录事件管理过程中的决策，收集问题解决、工作分配、响应时间、诊断时间、解决方案规划时间和解决时间等性能数据。这些指标为当前工作流程的有效性、系统和资源利用情况提供有价值的见解，推动数据质量控制的持续运营改进。

事件管理类数据对数据消费者也有帮助。修复数据的决策应基于对数据变化、变化原因、变化方式的认知，必须记录修改方法及其理由。

## 组织与角色

### 数据质量职能团队（Data Quality Function Team）

由业务和技术方面的数据管理专家组成，负责推动质量管理技术的应用，确保数据满足各种目的的消费需求。团队参与一系列项目，创建流程和最佳实践，解决高优先级数据问题。

### 数据质量管理员（Data Quality Steward）

与各级数据管理专员密切合作，影响业务流程和系统开发方面的制度。职责包括：
- 识别关键数据和定义质量期望
- 记录数据标准和业务规则
- 管理数据质量规则和标准
- 与业务和技术团队合作解决根因问题
- 监测数据质量并报告发现

### 数据管家（Data Custodian / Data Owner）

- **业务数据所有者**：对数据质量承担责任，在数据不符合预期时采取纠正措施
- **主题领域专家**：参与根因分析和规则确认
- **数据管理专员**：日常管理数据，承担数据质量监测的主要职责

### 数据质量治理

治理组织通过以下方式推进数据质量工作：
1. 设定优先事项并提供指导
2. 提供促进员工参与和知识共享的机制
3. 识别参与数据质量决策和活动的人员，协调相关工作
4. 确保持续进行数据质量度量
5. 共享检查结果，确定改进机会并建立共识
6. 解决差异和冲突

## 实施指南

即使由数据治理职能发起并得到高级管理者的支持，改善组织内的数据质量也不是一项容易的任务。通常，混合方法效果最佳——自上而下地提供支持、一致性保障和必要资源，自下而上地发现实际存在的问题并各个击破。

改善数据质量需要改变人们对数据的思考方式和行为。文化变革具有挑战性，需要规划、培训和强化。

### 就绪评估/风险评估

通过以下特征评估组织采取数据质量实践的准备情况：

| 评估维度 | 关键问题 |
|----------|----------|
| **数据质量认知** | 组织是否了解数据质量问题带来的阻碍和痛点？ |
| **数据实际状况** | 是否客观描述了导致痛点的数据状况？ |
| **数据相关风险** | 是否明确数据可能出现的问题及劣质数据对组织的潜在危害？ |
| **文化和技术准备** | 业务和IT团队合作是否顺畅？ |

就绪评估的结果有助于确定从哪里开始及如何快速推进，还可以为改进目标的路线图提供依据。

### 实施关键考虑因素

1. **高质量数据的价值和劣质数据成本的指标**：通过投资回报指标展示数据价值
2. **IT/业务交互操作模型**：业务人员了解数据含义，IT人员了解存储位置和方式
3. **项目执行方式变化**：项目资金必须覆盖数据质量相关步骤（分析和评估、质量期望定义、问题纠正、预防和更正、建立控制和度量）
4. **业务流程变更**：数据质量团队需对影响数据质量的流程变更进行评估和建议
5. **修复和改进项目资金**：记录成本和收益，优先进行数据改进
6. **数据质量运营资金支持**：持续运营需要持续资金保障

### 组织与文化变革

数据质量改进需要通过思维方式的转变实现：

| 需要克服的态度 | 表现 | 解决方案 |
|--------------|------|----------|
| **对现状的依赖** | "数据一直很糟糕，我们已经尽力了" | 建立改进信心和动力 |
| **责任推诿** | "他们需要处理好数据质量问题" | 打破组织内部壁垒，建立协作机制 |
| **英雄主义文化** | "没有我就做不了这个工作" | 推动客观度量与透明度 |

> 首要任务是提高员工对数据对组织作用和重要性的认识。每个接触数据的人都可能对数据质量产生影响——数据质量不仅是数据质量团队或IT团队的责任。员工需要了解劣质数据给组织带来的成本，以及导致数据质量不佳的原因。

### 预防性措施

预防是提升数据质量最有效的方式：

1. **建立数据录入控制**：防止无效或不准确的数据进入系统
2. **培训数据生产者**：确保上游工作人员了解数据对下游的影响，应根据数据的准确性和完整性而不仅仅是速度给予激励或进行评估
3. **定义和执行规则**：创建"数据防火墙"检查数据质量水平
4. **要求数据供应商提供高质量数据**：检查外部数据提供商的流程
5. **实施数据治理和管理责任制度**：定义角色和责任
6. **建立正式变更控制**：确保所有数据更改在实施前得到定义和测试，通过门控流程（Gating Processes）防止在常规流程之外直接对数据进行更改

## 行业应用

数据质量管理适用于所有依赖数据的行业和组织。关键数据的识别因行业而异，但存在跨组织的共性判断规则。

**金融行业**：客户数据（身份信息、信用记录）、交易数据的准确性和完备性是监管合规的基础。

**医疗健康**：患者数据（病历、诊断、用药）的准确性和完整性直接影响患者安全。

**零售/电商**：客户主数据、产品数据、库存数据的质量影响客户体验和运营效率。

**制造业**：供应链数据、产品质量数据的质量影响生产决策和合规。

**政府/公共部门**：人口数据、财务数据的准确性影响公共服务和监管报告。

## 方案价值

### 高质量数据的收益

| 收益类别 | 具体表现 |
|----------|----------|
| **客户信任** | 客户相信数据正确，有信心与组织打交道 |
| **员工效率** | 员工可更快速、更一致地解答问题，减少数据确认时间 |
| **收入增长** | 通过识别业务机会和成交生意获取收益 |
| **运营成本降低** | 减少客服话务量，及时解决呼叫需求 |
| **决策质量提升** | 员工更多时间用于借助数据进行洞察和正确决策 |
| **合规与信用** | 组织符合监管要求并提升信用评级 |

### 劣质数据的风险

| 风险类别 | 具体表现 |
|----------|----------|
| **声誉风险** | 损害组织声誉，导致负面媒体曝光 |
| **财务风险** | 罚款，收入损失 |
| **客户流失** | 因数据错误导致客户满意度下降 |
| **运营风险** | 业务流程中断或效率低下 |
| **合规风险** | 违反监管要求 |

### 成本收益分析（ROI）

改进数据质量的ROI基于以下因素确定：

| 维度 | 说明 |
|------|------|
| **受影响数据的关键性** | 重要性排名 |
| **受影响的数据量** | 数据规模 |
| **数据的可用期限** | 数据生命周期 |
| **受影响业务流程数量和类型** | 问题波及范围 |
| **受影响的人员数量** | 消费者、客户、供应商或员工 |
| **相关风险** | 问题引发的潜在风险 |
| **修复根因的成本** | 解决成本 |
| **变通的隐性成本** | 不修复的替代方案成本 |

> 预防问题通常比纠正问题的成本低得多（有时甚至低几个数量级）。

#### 质量改进目标设定示例

| 目标 | 度量 | 预期收益 |
|------|------|----------|
| 将客户数据完备性从90%提升到95% | 字段完备率 | 减少客户投诉，减少纠正错误时间 |

## 数据质量与其他知识领域的关系

### 与数据建模与设计的关系

数据质量通过项目设计阶段的数据建模和设计实现。数据模型允许数据元素保留质量标志（如验证通过标志）、关联业务规则。数据质量团队应与开发团队合作，确保规避数据质量风险，并充分利用有效的建模和数据处理方式提高数据质量。

### 与元数据管理的关系

元数据定义了数据表示的含义，是明示数据质量期望的主要手段。数据质量管理过程中创建的元数据应维持与数据的对应关系。元数据表达了数据的"适用性"，使得消费者能根据满足特定数据消费环境"目的"的程度评估数据质量。元数据库可保存数据质量度量结果，在整个组织中共享。

### 与主数据与参考数据管理的关系

主数据和参考数据是数据质量管理体系中的关键数据。劣质数据的关键源头之一是主数据管理薄弱。参考数据是贯彻质量规则（有效性、一致性、完整性）的重要数据来源。例如，文本值必须在定义的值域内，这些值由参考数据管理。

### 与数据集成与互操作的关系

数据流转过程中的数据集成是引发错误的常见源头。关键的元数据应与数据同步维护（集成不能忽略元数据）。在不同数据处理阶段（关键集成点）监测数据质量，可快速识别和隔离不良数据，避免在数据价值链中传播错误。

### 与数据治理的关系

数据质量问题通常是建立企业级数据治理的动因。将数据质量纳入数据治理工作使其更加有效。治理组织提供优先事项指导、员工参与和知识共享机制、差异和冲突解决。数据质量功能团队可与利益相关方合作实现业务流程工程和培训、识别关键数据和定义标准等目标。

### 与数据管理成熟度评估的关系

数据质量改进方案的持续成功取决于组织文化的改变并具备质量思维。数据质量成熟度评估有助于确定改进起点和推进路线图。

### 与数据管理组织与角色期望的关系

数据质量运营需要持续资金支持和人力资源保障。角色和责任的明确界定（包括数据所有者、数据管家、数据质量分析师）是数据质量管理有效实施的保障。

### 与其他知识领域的关系

所有数据管理知识领域都有助于提升数据质量。支持组织的高质量数据应该是所有数据管理学科的目标。跨职能的承诺和协作会产生高质量的数据。

## 附录：ISO 8000 数据质量标准

ISO 8000（国际数据质量标准）的目标是实现与应用无关的复杂数据交换。ISO声明："为了支持业务流程，以及时且具有成本效益的方式创建、收集、存储、维护、传输、处理和展示数据，既需要了解决定数据质量的特性，也需要具备测量、管理和报告数据质量的能力。"

ISO 8000将优质数据定义为**"满足既定要求的可移植（Portable）数据"**。数据可与软件应用程序分离则被视为"可移植"。满足既定要求的前提是以清晰、明确的方式定义这些要求。

**关键组成部分：**
- **第110部分**（2008年发布）：主数据的语法、语义编码和数据规范
- **第100部分**：引言
- **第120部分**：出处
- **第130部分**：准确性
- **第140部分**：完备性
- **第61部分**（制定中）：信息和数据质量管理流程参考模型，包括数据质量规划、控制、保证和改进

ISO 8000通过ISO 22745（主数据定义和交换的标准）实现，后者通过使用符合ISO 22745的开放技术词典（如ECCMA开放技术词典eOTD）标记数据，创建可移植的数据。
