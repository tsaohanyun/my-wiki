# 数据存储和运营：方法论与实践

> **所属知识体系**：[[DAMA-DMBOK2/06-数据存储和运营|DAMA-DMBOK2 第6章：数据存储和运营]]

---

## 方法论

### 存储策略选择

#### DBMS选择评估因素

选择战略性DBMS软件时需要考虑的技术因素：
1. 产品架构和复杂性
2. 容量和速度限制，包括数据流量限制
3. 应用类别（事务处理、商务智能、个人办公）
4. 特定功能需求（如是否需要支持时间类数据计算）
5. 硬件平台和操作系统
6. 软件支持工具的可用性
7. 性能基准，包括实时统计数据
8. 可扩展性
9. 软件、内存和存储要求
10. 韧性（错误处理和报告）

还需要考虑的组织因素：
- 企业对技术风险的偏好
- 是否配备训练有素的专业技术人员
- 使用成本（许可费、维护费、计算资源成本）
- 供应商声誉和支持策略、版本更新计划
- 其他客户的使用情况

企业要考虑成本效益原则，产品总费用不应超过该产品带来的经济价值。在真正引进产品并全面投产之前，最好在小型试点项目或POC中对产品进行验证。

### 运营管理流程

#### 管理数据库技术

| 阶段 | 主要活动 | 关键交付物 |
|------|---------|-----------|
| 理解 | 了解业务数据需求 | 技术需求分析报告 |
| 评估 | 评估DBMS产品，进行概念验证（POC） | 技术选型报告、POC结果 |
| 部署 | 与业务用户和开发人员合作 | 部署流程文档 |
| 监控 | 定期备份恢复测试，性能监控 | 监控报告、优化建议 |

#### 管理数据库运营

| 阶段 | 主要活动 | 关键交付物 |
|------|---------|-----------|
| 需求理解 | 定义存储需求、识别使用模式 | 存储需求文档 |
| 业务连续性规划 | 制定备份策略、恢复计划 | 业务连续性计划 |
| 实例创建 | 安装DBMS、维护多环境版本 | 实例配置文档 |
| 性能管理 | SLA设置、可用性管理、运行监控 | 性能报告、SLA合规报告 |
| 测试数据管理 | 生成和管理测试数据、数据脱敏 | 测试数据集、脱敏规则 |
| 数据迁移 | 数据迁移规划、执行和验证 | 迁移方案、验证报告 |

#### 识别使用模式

数据库有以下几种基本使用模式：
1. 事务处理
2. 大型数据集的数据读写
3. 数据处理时间分布不均匀
4. 数据处理位置分布不均匀
5. 数据处理优先级不同

#### 数据访问方法

从数据库访问数据可以采用多种标准语言、方法和格式：
- **ACID类型**：SQL、ODBC、JDBC、XQJ、ADO.NET、XML、X Query、XPath和Web Services
- **BASE类型**：C、C++、REST、XML和Java

#### 物理存储管理（SCM流程）

| 步骤 | 说明 |
|------|------|
| **配置识别** | 识别配置属性并记录在基线中 |
| **配置变更控制** | 更改配置属性和重新确定基线的流程和审批步骤 |
| **配置状态报告** | 在任何时间点记录和报告每个配置属性的基线 |
| **配置审计** | 物理配置审计和功能配置审计 |

#### 数据库访问控制管理

DBA负责管理数据资产的受控环境：
- **受控环境**：与NSA合作管理网络角色和权限、全天候监控
- **物理安全**：基于SNMP的监控、数据审计日志
- **监控**：通过对关键服务器不间断的硬件和软件监控确保持续可用
- **控制**：通过访问控制、数据库审计、入侵检测维护数据安全

#### 数据库性能管理

数据库性能取决于可用性和速度两个相互依存的方面。DBA和NSA通过以下方式管理：
1. 设置和调优操作系统及应用程序参数
2. 管理数据库连接
3. 持续调优操作系统、网络和事务处理中间件
4. 选择适当的存储设备
5. 有效预测数据库容量增长
6. 提供运营工作负荷和基准

**数据库可用性管理**：可用性指系统可以用于生产工作的时间百分比。影响可用性的4个因素：
1. **可管理性**：创造和维护数据库运行环境的能力
2. **可恢复性**：在服务中断后重新提供服务的能力
3. **可靠性**：在规定时间内提供约定水平服务的能力
4. **可维护性**：发现问题、诊断原因和维修/解决问题的能力

**导致数据库无法使用的情况**：

| 类别 | 具体情况 |
|------|---------|
| 计划内停机 | 出于维护的考虑、出于升级的考虑 |
| 计划外停机 | 服务器硬件故障、磁盘硬件故障、操作系统故障等 |
| 应用程序问题 | 安全和授权错误、严重的性能问题、恢复失败 |
| 数据问题 | 数据损坏、数据库对象丢失、数据丢失 |
| 人为失误 | 各类人为操作错误 |

**数据库性能低下的常见原因**：
1. **内存分配或竞争**
2. **锁定和阻塞**
3. **数据库统计信息不准确**
4. **编码不当**：糟糕的SQL代码是最常见的性能不佳原因
5. **低效的复杂表连接**
6. **索引不足**
7. **应用程序活动**
8. **服务器过载**
9. **数据库波动性**
10. **失控查询**

#### 管理测试数据集

软件测试占系统开发成本的近一半。测试数据可以是完全编造的、使用无意义数值生成的、或样本数据。如果生产数据包含受保护或受限数据，则必须对样本数据进行**掩码处理**。

---

#### 管理数据迁移

数据迁移是将数据从一种存储类型、格式或计算机系统转移到另一种的过程，同时尽可能减少信息损耗。

利用数据迁移技术，存储管理员可以同时完成许多日常任务：
1. 将数据从过度使用的存储设备移至独立环境
2. 根据需要将数据转移到速度更快的存储设备
3. 实施数据生命周期管理策略
4. 将数据从旧存储设备迁移到离线存储或云存储

### 工具与技术

| 工具类别 | 功能 |
|---------|------|
| **数据建模工具** | 自动化建模任务，生成DDL，支持逆向工程 |
| **数据库监控工具** | 自动监控容量、可用性、缓存性能等关键指标 |
| **数据库管理工具** | 配置、补丁、升级安装、备份恢复、数据库克隆 |
| **开发人员支持工具** | 连接数据库和执行命令的可视化界面 |

### 最佳实践

1. **识别并执行自动化机会**：缩短开发周期，减少错误和返工
2. **以重用为目标构建数据对象**：避免应用程序与数据库模式紧密耦合
3. **理解并适当运用最佳实践**：在合理理由时灵活应对偏离标准的情况
4. **数据库标准与需求相适应**：SLA既要反映DBA建议，也要考虑开发人员接受度
5. **明确DBA在项目中的角色期望**：在项目定义阶段就引入DBA
6. **在低阶环境下测试**：所有变更先在开发环境测试，再逐级升级到生产环境
7. **所有变更操作脚本化**：避免直接在数据库中更改数据
8. **采用物理命名标准**：遵循ISO/IEC 11179元数据注册库第5部分

## 组织与角色

### 关键角色

#### 数据库管理员（DBA）

DBA是数据管理专业中最常见、最被广泛认可的角色，在数据存储和运营方面发挥主导作用。

**DBA细分角色**：

| 角色 | 职责 | 详细说明 |
|------|------|---------|
| **生产DBA** | 数据运营管理的主要责任 | 确保数据库性能和可靠性、建立备份恢复机制 |
| **应用程序DBA** | 负责所有环境中的一套或多套数据库 | 为开发人员提供专业服务 |
| **过程和开发DBA** | 审查和管理程序数据库对象 | 管理存储过程、触发器、UDF等 |
| **网络存储管理员（NSA）** | 关注支撑数据存储阵列的软件和硬件 | 多元化类型的网络存储阵列 |

**生产DBA的可交付成果**：
1. 生产数据库环境，具有充足资源和容量
2. 在生产环境中建立控制数据库变更的机制和流程
3. 建立确保数据可用、完整和可恢复的机制
4. 建立错误监测机制
5. 提供与SLA相匹配的数据库服务
6. 建立数据库性能监控机制和流程

#### 其他协作角色

| 角色 | 协作内容 |
|------|---------|
| 数据管理专员 | 识别配置属性、制定保留策略 |
| 数据架构师 | 制定存储标准、数据复制策略 |
| 数据建模师 | 建立物理数据模型 |
| 网络管理员 | 协调网络和存储基础设施 |
| 安全分析师 | 制定数据安全策略 |

### 组织结构

- 生产DBA通常隶属于生产基础设施或应用程序操作支持组
- 应用程序DBA、过程和开发DBA常整合到应用开发团队中
- 网络存储管理员通常属于基础设施组

## 行业应用

### 典型应用场景

| 行业 | 应用场景 | 技术选择 |
|------|---------|---------|
| **金融** | 交易处理、风控系统 | ACID关系数据库、区块链数据库 |
| **零售** | 订单管理、库存系统 | OLTP关系数据库、键值缓存 |
| **医疗** | 电子病历、健康信息交换 | 时态数据库、区块链数据库 |
| **电信** | 网络管理、计费系统 | 空间数据库、分布式数据库 |
| **互联网** | 社交媒体、内容管理 | NoSQL数据库、图数据库 |
| **制造** | CAD/CAM、供应链管理 | 对象数据库、多维数据库 |

### 成熟度路径

| 级别 | 特征 | 能力要求 |
|------|------|---------|
| **初始级** | 手动管理，无标准流程 | 基本DBA技能 |
| **可重复级** | 建立基本备份恢复流程 | 备份恢复能力 |
| **已定义级** | 标准化的运维流程和文档 | 全面的运维管理能力 |
| **已管理级** | 性能监控和容量预测 | 数据分析和优化能力 |
| **优化级** | 自动化运维、持续改进 | 高级自动化和创新能力 |

## 实施指南

### 就绪评估/风险评估

**1. 数据丢失风险**
- SLA应制定对数据保护的一般要求
- 网络威胁在不断演变，需要对数据丢失风险进行持续评估
- 建议进行SLA审计和数据审计

**2. 与技术准备相关的风险**
NoSQL、大数据、三元组存储等新技术要求IT人员具备相应的技能和经验。DBA、系统工程师和应用开发人员及业务用户必须做好准备。

### 组织与文化变革

**常见问题**：
- 应用开发人员经常将数据管理视为开发的障碍
- DBA在适应技术变化和新的开发方法方面进展缓慢
- 过度强调数据安全、可重复使用性、数据质量对组织的长期价值，在组织中很难被认可

**克服障碍的指导原则**：
1. **主动沟通**
2. **站在对方的立场上沟通**
3. **坚持以业务为中心**
4. **乐于助人**
5. **不断学习**

## 方案价值

### 风险与挑战

| 风险类别 | 具体风险 | 影响程度 | 缓解措施 |
|---------|---------|:--------:|---------|
| **数据丢失** | 技术或程序错误导致数据丢失 | 极高 | 定期备份、恢复测试、SLA审计 |
| **系统不可用** | 生产环境停机影响业务运营 | 极高 | 集群、容错机制、故障切换 |
| **性能下降** | 数据库性能无法满足业务需求 | 高 | 性能监控、查询优化、容量规划 |
| **安全威胁** | 网络攻击和数据泄露 | 极高 | 访问控制、加密、持续安全评估 |
| **技术人才短缺** | 缺乏新技术所需的技能和经验 | 高 | 培训计划、交叉培训、外部招聘 |
| **合规风险** | 未遵守数据保留和清除法规 | 高 | 制定清晰的数据留存策略 |
| **成本超支** | 存储和许可成本超出预算 | 中 | TCO分析、存储优化、云迁移 |

### ROI分析

#### 投入成本

| 成本项 | 说明 |
|--------|------|
| 软件许可费 | DBMS软件及第三方工具许可 |
| 硬件成本 | 服务器、存储设备、网络设备 |
| 人员成本 | DBA、NSA及相关技术人员 |
| 运维成本 | 日常运维、监控、备份 |
| 培训成本 | 技术培训和技能提升 |

#### 产出价值

| 价值维度 | 说明 |
|---------|------|
| **业务连续性** | 确保系统可用，避免因停机造成的利润损失 |
| **数据资产保护** | 防止数据丢失和损坏，保护核心业务资产 |
| **运营效率** | 自动化运维减少人工干预，提升处理速度 |
| **合规保障** | 满足法规要求，避免法律和财务风险 |
| **决策支持** | 高质量的数据存储支撑数据分析和商业智能 |
| **成本优化** | 通过存储分层、数据归档、云迁移降低长期成本 |

## 数据存储和运营治理

### 度量指标

**数据存储度量指标**：
1. 数据库类型的数量
2. 汇总事务统计
3. 容量指标
4. 已使用的存储容量
5. 存储容器数量
6. 已提交和未提交数据块或页面的数据对象数量
7. 队列中的数据
8. 存储服务使用情况
9. 针对存储服务提出的请求数量
10. 使用数据存储服务应用程序性能的提升

**性能度量指标**：
1. 事务频率和数量
2. 查询性能
3. API服务性能

**运营度量指标**：
1. 数据检索时间的汇总统计信息
2. 备份大小
3. 数据质量评估
4. 可用性

**服务度量指标**：
1. 按类型划分的问题提交、解决和升级数量
2. 问题解决的时间

### 信息资产跟踪

数据存储管理工作还涉及确保组织遵守所有许可协议和监管要求。组织应对软件许可、年度支持费用、服务器租赁协议等仔细跟踪和按年进行审计。

### 数据审计和数据验证

**数据审计**：根据既定标准对数据集进行评估，通常是为了调查特定关注点。

**数据验证**：根据既定验收标准对存储数据进行评估，以确定其质量和可用性。

DBA通过以下方式支持数据审计和数据验证工作：
1. 帮助制定和审查方法
2. 进行初步数据筛选和审查
3. 开发数据监控方法
4. 应用统计、地理统计和生物统计技术优化数据分析
5. 支持采样和分析
6. 审核数据
7. 为数据发现提供支持
8. 担任与数据库管理相关问题的主题领域专家（SME）
