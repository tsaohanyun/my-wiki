# 文档和内容管理：工具与实践

> **所属知识体系**：[[DAMA-DMBOK2/09-文档和内容管理|DAMA-DMBOK2 第9章：文档和内容管理]]

---

## 工具与技术

### 企业内容管理系统（ECMS）组件

| 组件 | 功能 |
|------|------|
| 文档管理系统 | 存储、版本控制、安全、元数据管理、索引、检索、工作流 |
| 内容管理系统 | 内容生命周期管理、工作流、变更管理、分发 |
| 档案管理系统 | 重要档案保管、合规管理 |
| 数字资产管理 | 富媒体编目、存储、检索 |
| 影像处理系统 | OCR、ICR、表单处理 |
| 工作流工具 | 手动工作流、基于规则的工作流、动态规则 |

报告可以通过文档夹进行分类管理，并能轻松添加、更改或删除。报告保留策略可以设置为自动清理或归档到其他介质，如磁盘、CD-ROM、光盘存档系统（COLD）等。报告也可以存储在云存储中。以不可读或过时格式保存内容会给组织带来风险。

随着业务流程和角色交织在一起，以及供应商试图扩大其产品市场，文档管理和内容管理之间的界限变得模糊起来。一些文档管理系统拥有更高级的功能，如复合文档支持和内容复制。文字处理软件可以创建复合文档，并集成非文本元素，如电子表格、视频、音频和其他多媒体类型。复合文档可以作为用户界面的有机集合，形成单个的集成视图。

文档存储库具有管理文档的功能，包括签入签出、版本控制、协作、比较、归档、状态、从一个存储介质迁移到另一个存储介质及处置等特性。它可以提供对处于自身存储库以外的文档的一些访问和版本管理（如在文档共享或云环境中）。

### 标准标记和交换格式

计算机应用程序无法直接处理非结构化数据/内容。标准化标记和交换格式有助于数据在信息系统和互联网之间共享。

| 格式 | 说明 | 用途 |
|------|------|------|
| **XML** | 可扩展标记语言，使用元数据描述任何文档或数据库的内容、结构和业务规则 | 数据交换、内容管理、门户网站、结构化与非结构化数据整合 |
| **JSON** | JavaScript对象表示法，轻量级数据交换格式，文本格式独立于语言 | Web应用数据传输、NoSQL数据库、REST技术 |
| **RDF** | 资源描述框架，以主语-谓语-宾语三元组描述资源 | 语义网、关联数据、数据集成、大数据多样化处理 |
| **Schema.org** | 共享词汇表集合，用于页面标记 | 搜索引擎优化、富文本摘要、结构化数据互操作 |

**XML的关键能力**：
- 提供将结构化数据与非结构化数据整合到关系数据库的能力
- 可以将结构化数据与非结构化数据整合到文档、报告、电子邮件、图像、图形、音频和视频文件中
- 可以构建企业或公司门户网站（如B2B、B2C），为用户提供单一访问点
- 提供非结构化数据/内容的标识和标签，使计算机应用程序能够理解和处理这些数据

**RDF相关标准**：
- **RDFS**：为RDF数据提供数据建模词汇表
- **SKOS**（简单知识组织系统）：RDF词汇表，任何类型的分类、分类法或主题词表都可以在SKOS中表示
- **OWL**（网络本体语言）：RDF的词汇表扩展，是一种语义标记语言

RDF可以帮助处理大数据的"多样化"特征。如果可以用RDF三元组模型访问数据，则可以混合来自不同源的数据，并使用SPARQL查询语言来查找连接和模式，而无须预先定义模式。

**Schema.org**使用语义标记对内容进行标记可以帮助语义搜索引擎更轻松地索引内容。代码段是每个搜索结果下显示的文本，富文本是特定搜索的详细信息（如链接下的金星评级）。要创建富文本，网页上的内容需要使用结构化数据进行格式化，如Microdata（HTML5引入的一组标签）和Schema.org的共享词汇表。

### 受控词汇表和元数据工具

帮助开发或管理受控词汇表和元数据工具的范围包括：

1. 在组织中被用作数据指南的数据模型
2. 文档管理系统和办公生产力软件
3. 元数据库、术语表或目录
4. 分类法和分类法之间的交叉参考模式
5. 集合索引（如特定的产品、市场或安装）、文档系统、民意调查、档案、位置或异地控股
6. 搜索引擎
7. 融合非结构化数据的BI工具
8. 企业和部门同义词表
9. 已发布的报告库、内容和参考书目及其目录

### 电子举证技术

电子举证通常涉及审查大量文档。电子举证技术提供了许多功能和技术，如早期案件评估、收集、辨认、保全、处理、光学字符识别（OCR）、筛选、相似性分析和电子邮件线程分析。

**技术辅助审查**（TAR）是一个工作流或过程，在此过程中，团队可以审查选定的文档并标记其相关性。这些决定成为预测编码引擎的输入，该引擎根据相关性对剩余的文档进行审查和排序。信息治理的支持可能也是电子举证的一个特征。

## 组织与角色

### 参与者

| 角色 | 职责 |
|------|------|
| **数据管理员** | 协助定义门户网站、企业分类方案、搜索引擎索引和内容管理 |
| **数据管理专业人员** | 参与受控词表和元数据工作，协调数据建模与内容管理 |
| **档案管理人员** | 档案分类、保管、处置决策；参与风险缓解和业务连续性计划 |
| **内容管理人员** | 内容策略制定、内容生命周期管理 |
| **Web开发人员** | 网站内容管理、搜索引擎优化 |
| **图书管理员** | 受控词表理论和开发支持（通常接受过专门培训） |
| **法律团队** | 电子举证、法律保留通知、合规性指导 |
| **IT团队** | 系统支持、技术实施 |
| **外部第三方** | 供应商服务、云存储 |

### 组织架构

- 强监管行业中，档案和信息管理职能需与公司法律职能及电子举证职能密切合作。
- 若以提高运营效率为目标，档案和信息管理应与营销或运营支持团队协同。
- 若将档案和信息管理视为IT一部分，应直接向首席信息官（CIO）或首席数据官（CDO）汇报。
- 档案和信息管理功能通常嵌入在内容管理系统或企业信息管理项目中。

组织应提升内容和档案管理的地位，不应将其视为无关紧要或低优先级的功能。隐私、数据保护、机密性、知识产权、加密、符合伦理的使用和身份验证，应是文档和内容管理专业人员与其他员工及管理层和监管机构合作处理的重要问题。

## 行业应用

| 行业 | 应用场景 |
|------|----------|
| **制药/核工业** | 化合物形成、测试和处理过程的详细档案记录，需要内容级的详细元数据和标签工具 |
| **金融** | 遵守《萨班斯-奥克斯利法案》《多德-弗兰克法案》等法规，金融分类法管理 |
| **法律** | 电子举证、诉讼保留、法律发现流程 |
| **政府/军队** | 基于ISO 9001的文档层次结构，档案合规性和风险管理 |
| **医疗保健** | HL7消息管理，患者档案和合规性记录 |
| **营销/媒体** | 数字资产管理，网站内容管理，社交媒体档案管理 |
| **零售/电商** | 面向客户的网站内容管理，产品信息分类 |
| **情报/军事/科研** | 对访问、使用或检索更多关注数据和信息的控制和安全能力 |

## 方案价值

### 风险分析

| 风险类型 | 风险描述 | 缓解措施 |
|----------|----------|----------|
| **合规风险** | 不遵守法律法规要求可能导致罚款和法律责任 | 建立档案管理计划，制定保管期限表，实施合规审计 |
| **诉讼风险** | 电子举证应对不足，关键信息被篡改或销毁 | 制定诉讼应对预案，创建数据地图，实施法律保留通知 |
| **安全风险** | 敏感信息泄露，机密数据未加保护 | 实施访问控制、加密、保密方案，标记敏感数据 |
| **业务连续性风险** | 灾难导致重要档案丢失，业务无法恢复 | 识别重要档案，制订备份和恢复计划，参与BCP |
| **运营风险** | 档案保管不善，查找效率低下 | 建立受控词表和信息架构，实施ECM系统 |
| **技术风险** | 过时格式导致档案无法读取 | 采用标准化标记格式（XML、JSON），定期迁移数据 |
| **数据质量风险** | 内容不准确、不完整，影响决策 | 建立内容策略，实施元数据管理，进行文档审计 |
| **存储成本风险** | 无价值信息过度占用存储空间 | 制定保管和处置制度，定期清理无价值信息 |

### 投资回报（ROI）

| 效益类型 | 具体效益 | 衡量指标 |
|----------|----------|----------|
| **有形效益** | 提高生产力 | 检索效率提升百分比 |
| | 降低成本 | 物理存储成本下降、运营成本下降百分比 |
| | 提高信息质量 | 查准率、查全率 |
| | 改善合规性 | 合规审计通过率 |
| **无形效益** | 增进协作 | 团队协作效率提升 |
| | 简化工作程序 | 文书工作减少百分比 |
| | 降低诉讼风险 | 举证成本降低、审计跟踪数量 |
| | 提高客户满意度 | 首次联系解决率、投诉量下降 |

### 度量指标（KPI）

| 领域 | 战略层面KPI | 运营层面KPI |
|------|-------------|-------------|
| **档案管理** | 法规遵从时间、治理制度遵守率 | 运营和资本成本、培训课程数量、SLA达成率、系统集成百分比 |
| **电子举证** | 举证成本降低率、胜诉率 | 响应天数、法律保留通知速度 |
| **ECM** | ECM项目数、采用率、用户满意度 | 停机时间、用户数、存储利用率、检索性能 |

电子举证的度量，对于提高胜诉率来说至关重要。EDRM模型可以根据每个阶段的需求指引KPI的制定。EDRM还发布了电子举证指标的度量模型（参考：http://bit.ly/2rURq7R）。围绕数量、时间和成本这几个中心元素，电子举证工作主要集中在7个方面（活动、监护人、系统、介质、状态、格式和质量保证）。这些都会影响中心元素的结果。

衡量档案管理系统实施成功的标准可包括：

- 每个用户有百分之几的文档和电子邮件被认定为公司的档案
- 被认定为公司档案中有百分之几得到了控制
- 在所有存储的档案中有百分之几的档案应用了适当的保管规则

**特定ECM指标**：
- **存储利用率**：使用企业内容管理前后的存储容量对比
- **检索性能**：通过查准率（检索到的文档与实际相关的比率）和查全率（检索到的涉及所有相关文档的比率）来衡量

随着时间的推移，可以指定一些与业务解决方案的价值相关的KPI：
- **财务KPI**：ECM系统的成本、物理存储相关的成本降低及运营成本下降的百分比
- **客户KPI**：首次与客户联系时就解决了问题的比例和客户投诉的数量
- **业务流程KPI**：文书工作减少的百分比及错误减少的百分比
- **培训KPI**：管理和非管理培训课程的数量
- **风险缓解KPI**：举证成本的降低及追踪电子举证请求的审计跟踪数量

## 治理

### 信息治理框架

文档和内容治理的驱动因素包括：

1. 法律和法规遵从
2. 档案的合理处置
3. 积极准备电子举证
4. 敏感信息的安全
5. 电子邮件和大数据等风险领域的管理

成功信息治理程序的原则包括ARMA的GARP及其他原则：
- 为问责制提供高层支持
- 教育员工了解信息治理职责
- 根据正确的档案代码或分类类目对信息进行分类
- 确保信息的真实性和完整性
- 确保官方档案是电子档案，除非另有规定
- 制定制度，使业务系统和第三方与信息治理标准保持一致
- 存储、管理、提供访问、监控和审计已批准的企业信息库及档案和内容系统
- 保护机密或个人身份信息
- 控制不必要的信息增长
- 在信息的生命周期结束时对其进行处置
- 遵守信息的请求（如举证、传票等）
- 持续改进

**信息治理参考模型（IGRM）** 展示了信息治理与其他组织功能的关系，由EDRM发布（edrm.net，根据Creative Commons Attribution 3.0 Unported许可证授权）。外环包括制度、标准、流程、工具和基础架构等利益相关方，中心展示生命周期图。IGRM将责任+价值与信息资产联系起来，实现高效、有效的管理。IGRM是对ARMA的GARP的补充。

对信息治理职能的建立和持续发展而言，获得来自高管的重视和支持至关重要。需要成立一个由跨职能部门的高级管理人员组成的信息委员会或指导委员会，并定期召开会议。该委员会负责制定企业信息治理战略、运营程序、技术和标准指南、沟通和培训、监控和资金。

### 信息泛滥管理

非结构化数据的增长速度远远快于结构化数据，增加了治理难度。非结构化数据不一定与特定的业务职能或部门相关联，因此很难确定其所有权。由于无法总是从系统中推断内容的业务目的，对非结构化数据的内容进行分类也比较困难。没有得到有效管理的非结构化数据，缺少必要的元数据，会给组织带来风险。

### 以内容质量为导向的信息治理

非结构化数据的管理需要数据管理专员与其他数据管理专业人员及档案管理人员有效合作。定义高质量内容需考虑以下因素：

- **生产者**：谁创建内容？为什么创建？
- **消费者**：谁使用信息？出于什么目的？
- **时间**：何时需要信息？需要多久更新或访问一次？
- **格式**：消费者是否需要特定格式？是否有不可接受的格式？
- **分发**：信息将如何分发？消费者将如何获取信息？如何实施安全措施？

### 档案管理成熟度模型

ARMA国际与GARP联合制定的信息治理成熟度模型，描述了5个等级（参考：http://bit.ly/2sPWGOe）：

| 等级 | 特征 |
|------|------|
| 1级：未达标 | 信息治理和档案管理存在严重问题，未得到解决或仅满足最低要求 |
| 2级：发展中 | 开始认识到信息治理和档案管理能为组织带来的影响 |
| 3级：基础的 | 满足最低的法规要求 |
| 4级：主动的 | 建立了以持续改进为导向的主动信息治理职能 |
| 5级：转型成功 | 信息治理已融入企业基础架构和业务流程 |

**档案管理系统技术评估标准**：
- DoD 5015.2《电子档案管理软件应用系统设计标准》
- ISO 16175《电子办公环境中档案管理原则与功能要求》
- MoReq2《电子档案管理通用需求》
- OMG《档案管理服务（RMS）规范》

应对档案管理评估准备中发现的差距和风险进行分析，评估它们可能给组织带来的负面影响。如果没有完整的档案清单，组织将面临风险。不遵守法律法规要求可能会导致昂贵的罚款。识别和保护重要档案对于企业生存至关重要。

### 实施指南

实施企业内容管理（ECM）是一项长期投入。与任何企业级项目一样，它需要来自各个利益相关方的支持及执行委员会的资金批准。为了降低风险，应确保是内容本身而非技术驱动ECM实施决策，要根据组织需求配置工作流，实现内容流和业务流的协同，展现实施ECM的价值。

**准备评估/风险评估**：ECM实施准备评估的目的是识别内容管理方面需要改进的方面，评估组织在多大程度上能够适应流程变革以满足内容管理需求。数据管理成熟度评估模型可以帮助组织进行ECM实施的准备度评估。

ECM的关键成功因素包括：
- 高管支持、用户参与、用户培训、变革管理、企业文化和沟通
- 对现有内容进行内容审计和分类
- 适当的信息架构
- 支持内容生命周期管理
- 定义适当的元数据标签
- 能够在ECM解决方案中定制功能

**组织和文化变革**：员工适应新系统将比技术本身更具挑战。改变日常工作中的管理习惯和让员工使用ECM系统可能存在困难。采取整体的企业级内容和档案管理方法可以消除用户存储内容副本的需要。理想的解决方案是采用一个集中和安全管理的存储库，并在整个企业范围内实施明确的制度和流程。培训和沟通流程、制度和工具对于档案管理或ECM项目的成功至关重要。

### 诉讼应对方法

**诉讼应对预案**：电子举证预案应明确电子举证的目标环境，评估当前环境和目标环境之间是否存在差距；应记载电子举证活动生命周期的业务流程，明确电子举证团队的角色和职责。预案的主要内容包括：

1. 建立特定部门（法务、档案管理、IT）制度和程序清单
2. 起草有关诉讼保留、文档保留、归档和备份等主题的制度
3. 评估IT工具的能力
4. 识别和分析相关的法律问题
5. 制订沟通和培训计划
6. 识别可能提前准备以适应法律案件的材料
7. 分析供应商服务
8. 制定通知处理流程并保持预案的最新状态

**诉讼应对数据地图**：电子举证通常时间紧迫（如90天）。数据地图是信息系统的目录，描述了系统及其用途、包含的信息、保留策略和其他特征。数据地图通常会标识档案系统、原始应用程序、存档、灾难恢复副本或备份。数据地图应全面涵盖所有系统。由于电子邮件在诉讼中通常会受到审查，数据地图还应描述电子邮件的存储、处理和使用方式。

电子举证数据地图应该指明哪些档案是容易获取的，哪些不是。需要识别无法访问的数据，并记录其无法访问的原因。创建数据地图的过程，将展示文档管理过程中创建元数据的价值。
