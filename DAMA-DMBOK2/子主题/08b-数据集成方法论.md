# 数据集成方法论

## 数据集成活动全流程

数据集成和互操作遵循开发生命周期模型，从规划开始，经过设计、开发、测试和实施，实施后进行管理、监控和改进。

## 实施流程

### 规划和分析

| 活动 | 说明 | 关键输出 |
|------|------|----------|
| **定义数据集成和生命周期需求** | 理解业务目标，收集相关法律法规，考虑数据留存制度 | 需求文档、元数据 |
| **执行数据探索** | 为数据集成工作确定潜在数据来源，结合技术搜索与主题专业知识 | 数据目录、数据来源清单 |
| **记录数据血缘** | 揭示数据如何在组织中流转，记录高阶视图和详细血缘信息 | 数据血缘图 |
| **剖析数据** | 理解数据的内容和结构，评估数据质量 | 数据剖析报告 |
| **收集业务规则** | 获取定义或约束业务处理方面的语句，包括匹配规则、合并规则、存活规则和信任规则 | 业务规则库 |

**数据剖析**基本内容包括：
1. 数据结构中定义的数据格式和从实际数据中推断出来的格式
2. 数据的数量，包括空值、空白值或默认数据的级别
3. 数据值及与定义的有效值集合的紧密联系
4. 数据集内部的模式和关系（相关字段和基数规则）
5. 与其他数据集的关系

> **实践提示**：如果跳过数据剖析过程，有些影响设计的信息直到测试或实际操作之前都难以被发现。几乎在所有情况下，对数据集的预期与探索发现的实际情况都会存在差异。

**业务规则**分为四类：
1. 业务术语定义
2. 相互关联的术语的事实
3. 约束或行为断言
4. 派生

### 设计数据集成解决方案

| 设计活动 | 说明 |
|----------|------|
| **选择交互模型** | 确定中心辐射型、点对点或发布订阅，尽可能重用现有系统 |
| **设计数据服务或交换模式** | 创建或重用集成流程，避免创建多个几乎完全相同的服务 |
| **数据中心/接口/消息建模** | 对持久化数据结构和临时数据结构进行建模 |
| **映射数据源到目标** | 指明技术格式、指定转换、描述填充方式和计算规则 |
| **设计数据编排** | 设计和记录从开始到结束的数据流模式 |

**映射规范**对于每个属性应包括：
1. 指明源数据和目标数据的技术格式
2. 指定源数据和目标数据之间所有中间暂存点所需的转换
3. 描述最终或中间目标数据存储区中每个属性的填充方式
4. 描述是否需要对数据值进行转换（如通过查找表）
5. 描述需要进行哪些计算

### 开发数据集成解决方案

| 开发活动 | 说明 |
|----------|------|
| **开发数据服务** | 获取、转换和交付指定数据，建议使用一致的工具或标准供应商套件 |
| **开发数据流编排** | 使用专用工具和标准调度器（如 CTRL-M）管理执行顺序、频率和依赖关系 |
| **制定数据迁移方法** | 应用程序上线、退役或合并时的数据迁移，不是一次性过程 |
| **制定发布方式** | 确定通用消息定义（规范格式），让数据使用者订阅数据更改通知 |
| **开发复杂事件处理流** | 准备历史数据、处理实时数据流、执行触发动作 |

**数据迁移注意事项**：
- 数据迁移项目经常被低估或缺乏充分的设计
- 在没有进行适当分析的情况下迁移数据时，这些数据通常看起来与通过正常业务处理而获得的数据不一样
- 核心操作型应用程序的数据剖析过程通常会突出显示从上一代系统迁移而来的数据

### 实施和监测

- 启用已开发并通过测试的数据服务
- 对实时数据处理过程需要实时监控运行状况
- 建立表示潜在问题的度量指标及直接反馈问题的机制
- 数据交互功能必须采用与最苛刻的目标应用程序相同的服务级别进行监视
- 当触发响应的复杂性和风险增加时，应建立自动化处理和人工监控流程

### 维护元数据

- 管理和维护开发过程中创建和发现的元数据
- 记录所有系统的数据结构（源、目标和缓存），包括业务定义和技术定义
- 元数据变更需经业务和技术利益相关方审核和批准
- SOA 注册中心提供受控的可用服务信息目录

## 工具与技术

| 工具类别 | 说明 |
|----------|------|
| **数据转换引擎/ETL 工具** | 数据集成工具箱中的主要工具，支持批量和实时、物理和虚拟数据的复杂 ETL 开发和执行 |
| **数据虚拟化服务器** | 对数据进行虚拟抽取、转换和集成，可合并结构化和非结构化数据 |
| **企业服务总线（ESB）** | 面向消息的中间件，用于异构数据存储、应用程序和服务器之间的近实时消息传递 |
| **业务规则引擎** | 允许非技术用户管理业务规则，支持不改变技术代码的情况下演进解决方案 |
| **数据和流程建模工具** | 设计目标数据结构和中间数据结构，对数据流进行设计 |
| **数据剖析工具** | 对数据集内容进行统计分析，了解格式、完整性、一致性、有效性和结构 |
| **元数据库** | 包含组织数据的相关信息，如数据结构、数据内容及业务规则等 |
| **消息队列/事件流平台** | 异步消息传递和实时事件流处理（如 Kafka、RabbitMQ） |
| **API 管理平台** | API 设计、发布、监控和治理的全生命周期管理 |

## 方法原则

| 原则 | 说明 |
|------|------|
| 松散耦合 | 保持应用程序松散耦合 |
| 最小化接口数量 | 限制开发和管理的接口数量 |
| 标准规范接口 | 创建标准规范的接口 |
| 中心辐射型方法 | 优先采用中心辐射型集成模式 |
| 服务编排 | 合理编排多个流程的组织和执行 |
