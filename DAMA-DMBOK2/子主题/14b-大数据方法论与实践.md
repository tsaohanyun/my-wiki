## 方法论

### 传统数据仓库/商务智能与数据科学分析方法对比

| 对比维度 | 传统数据仓库/商务智能 | 数据科学 |
|---------|---------------------|---------|
| **分析目标** | 描述历史状态（"后视镜"式） | 预测未来行为（"前挡风玻璃"式） |
| **数据类型** | 结构化数据为主 | 结构化、半结构化、非结构化混合 |
| **处理方式** | ETL | ELT |
| **分析方法** | OLAP查询、报表 | 机器学习、统计建模 |
| **产出物** | 报告、仪表板、KPI | 预测模型、推荐系统 |

### 数据科学项目生命周期（CRISP-DM映射）

| 阶段 | 步骤描述 | CRISP-DM映射 | 关键产出 |
|------|---------|-------------|---------|
| 1 | 定义大数据战略和业务需求 | 业务理解 | 项目章程 |
| 2 | 选择数据源 | 数据理解 | 数据源清单 |
| 3 | 获取和引入数据源 | 数据准备 | 数据集、元数据目录 |
| 4 | 制定数据假设和分析方法 | 数据准备 | 假设清单 |
| 5 | 整合和对齐数据 | 数据准备 | 清洗后数据集 |
| 6 | 使用模型探索数据 | 建模 | 训练模型 |
| 7 | 部署和监控 | 部署 | 生产模型 |

### 分析建模方法

| 模型类型 | 目的 | 说明 |
|---------|------|------|
| **描述性建模** | 汇总或表示数据结构 | 使用算法定义/改善变量关系 |
| **解释性建模** | 检验因果假设 | 类似于数据挖掘和预测分析 |
| **预测性建模** | 通过训练模型进行实例学习 | 评估泛化能力 |

避免**过拟合（Over-Fitting）** 是模型训练的关键挑战，常用技术包括K折验证和数据集三部分划分。

### 大数据架构选择原则

1. 明确业务问题与数据分析目的
2. 评估数据源
3. 确定数据及时性与范围
4. 分析对其他数据结构的影响
5. 评估对现有模型数据的影响

### 大数据项目关键角色

| 角色 | 职责 |
|------|------|
| **大数据平台架构师** | 负责基础设施规划 |

| **数据分析主管** | 负责终端用户分析设计 |

### 团队建设与文化变革

- 建立**卓越中心（Center of Excellence）**
- 通过沟通和培训计划推动组织文化变革
- 培养或引进具备数据科学技能的专业人才

## 工具与技术

### Hadoop生态系统

| 组件 | 功能说明 |
|------|---------|
| **HDFS** | 分布式文件系统 |
| **YARN** | 资源管理和任务调度框架 |
| **MapReduce** | 分布式计算编程模型 |
| **Hive** | 基于Hadoop的数据仓库工具 |
| **Spark** | 内存计算框架，比MapReduce快10-100倍 |
| **Kafka** | 分布式消息队列系统 |
| **Mahout** | 机器学习库 |

### 大规模并行处理（MPP）

MPP无共享数据库技术是面向数据科学的大数据集分析标准平台，具有**线性可扩展性**，支持**在库分析**功能。

### R语言

用于统计计算和图形的开源脚本语言环境，提供线性和非结构化建模、经典统计检验、时间序列分析等统计技术。

## 大数据与数据科学治理

### 治理范围

| 治理领域 | 关键问题 |
|---------|---------|
| 数据寻源 | 有哪些数据源？最佳数据源是什么？ |
| 数据共享 | 需签订哪些共享协议和合同？ |
| 元数据 | 数据在源端的具体含义？ |
| 数据增强 | 是否需要增强数据？如何增强？ |
| 数据访问发布 | 发布什么？向谁发布？如何发布？ |

### 就绪评估/风险评估

| 评估维度 | 关键问题 |
|---------|---------|
| 业务相关性 | 计划与公司业务的一致性如何？ |
| 业务准备情况 | 业务合作伙伴是否准备好长期逐步交付？ |
| 经济可行性 | 是否有形和无形收益？ |
| 原型验证 | 能否提供产品原型以证明预期价值？ |

## 方案价值

### 风险分析

| 风险类别 | 缓解措施 |
|---------|---------|
| 数据沼泽风险 | 建立元数据目录和管理体系 |
| 数据隐私风险 | 建立数据安全制度，实施加密和掩码处理 |
| 模型偏差风险 | 建立数据伦理培训，采用中立的专业展示流程 |
| 过拟合风险 | 使用K折验证、划分训练/测试/验证集 |
| 人才短缺风险 | 考虑专业服务外包、云采购或合作 |

### 投资回报

收益维度：收入增长、成本降低、效率提升、竞争差异化、客户体验改善

度量指标：已开发模型的数量与准确性、从识别机会中实现的收入、通过规避风险减少的成本
