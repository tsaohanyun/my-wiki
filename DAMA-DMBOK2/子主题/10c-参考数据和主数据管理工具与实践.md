# 参考数据和主数据管理：工具与实践

> **所属知识体系**：[[DAMA-DMBOK2/10-参考数据和主数据管理|DAMA-DMBOK2 第10章：参考数据和主数据管理]]

---

## 工具与技术

主数据管理可以通过数据集成工具、数据修复工具、操作型数据存储（ODS）、数据共享中心（DSH）或专门的主数据管理系统实现。有些供应商提供的解决方案可以覆盖一个或多个主数据主题域，还有一些供应商则通过推广其数据集成软件产品和实施服务提供定制化的主数据解决方案。

产品、客户和参与方主数据的打包解决方案，以及配套的数据质量检查服务可以快速启动大型程序。

| 工具类别 | 说明 |
|----------|------|
| **数据集成工具** | 实现主数据的抽取、转换和加载 |
| **数据修复工具** | 识别和修正数据质量问题 |
| **操作型数据存储（ODS）** | 支持操作型报表和集成 |
| **数据共享中心（DSH）** | 实现主数据的星型架构共享 |
| **数据建模工具** | 创建逻辑和物理数据模型 |
| **元数据存储库** | 管理参考数据和主数据的元数据 |
| **MDM应用平台** | 针对产品、客户和参与方等主数据域的打包解决方案 |

---

## 度量指标

| 指标 | 说明 |
|------|------|
| **数据质量和合规性** | 数据质量仪表盘描述参考数据和主数据的质量，显示主题域实体或相关属性的置信度（百分比）及其在整个组织中的适用性 |
| **数据变更活动** | 审核可信数据的血缘，显示数据值变化情况，为向共享环境提供数据的系统提供洞察力 |
| **数据接收和使用** | 跟踪哪些系统贡献数据、哪些业务领域订阅数据 |
| **服务级别协议（SLA）** | 建立并传达给贡献者和订阅者，确保数据共享环境的使用和采用 |
| **数据管理人员覆盖率** | 关注数据负责人或责任团队及覆盖率的评估频率 |
| **总体拥有成本（TCO）** | 环境基础设施费、软件许可证费、支持人员费、咨询费、培训费等 |
| **数据共享量和使用情况** | 跟踪数据采集和使用量，表示定义、接收和订阅到数据共享环境的数据量和速度 |

## 组织与角色

### 关键角色

| 角色 | 职责 |
|------|------|
| **数据分析师** | 分析主数据和参考数据需求 |
| **数据建模师** | 创建逻辑和物理数据模型 |
| **数据管理员（数据专员）** | 负责参考数据和主数据的质量控制和维护 |
| **数据集成员** | 实现数据集成和共享 |
| **数据架构师** | 定义数据共享架构和集成方法 |
| **数据质量分析师** | 评估和监控数据质量 |
| **业务领域专家** | 提供业务上下文和术语定义 |
| **应用程序开发人员** | 实现主数据集成服务和数据共享功能 |
| **数据提供商** | 提供外部参考数据和参考目录 |
| **MDM负责人** | 统筹主数据管理工作，协调业务和技术部门 |
| **解决方案架构师** | 设计MDM技术解决方案 |

### MDM负责人

MDM负责人负责协调业务和技术团队，推动主数据管理工作的规划和实施，确保主数据管理符合业务需求和数据治理制度。该角色需要深入了解业务流程、数据模型和集成技术，并具备跨部门沟通协调能力。

### 数据专员

数据专员（数据管理员）对参考数据和主数据的质量负有直接责任，包括：
- 维护参考数据的定义和域值
- 管理主数据匹配和合并规则
- 审查和批准数据变更请求
- 监控数据质量指标
- 协调解决数据不一致问题

## 行业应用

### 典型应用场景

| 行业 | 应用场景 |
|------|----------|
| **金融/银行** | 客户主数据（360度客户视图）、产品主数据、科目主数据、外汇汇率参考数据 |
| **医疗保健** | 患者主数据、医护人员主数据、国际疾病分类代码（ICD）、诊断与治疗代码 |
| **零售/电商** | 产品主数据（SKU）、客户主数据、供应商主数据、地理位置参考数据 |
| **制造/工业** | 物料主数据、供应商主数据、产品生命周期管理（PLM）数据、产品数据管理（PDM） |
| **政府/公共服务** | 居民主数据、组织机构主数据、行政区划参考数据、行业分类代码 |
| **教育** | 学生主数据、教职工主数据、课程主数据、学历学位代码参考数据 |
| **电信** | 客户主数据、网络资源主数据、服务产品主数据、号码资源参考数据 |

## 方案价值

### 风险与挑战

| 风险类别 | 具体风险 | 影响程度 | 缓解措施 |
|----------|----------|----------|----------|
| **数据不一致** | 同一数据实体在不同系统中拥有不同值，导致业务决策错误 | 高 | 建立统一标识符管理机制，实施MDM解决方案 |
| **标识符歧义** | 一物多码或一码多物，无法准确识别实体 | 高 | 实施实体解析和标识符管理 |
| **匹配错误** | 假阳性或假阴性 | 高 | 维护匹配历史记录，支持撤销操作 |
| **数据质量问题** | 缺失值、格式错误、过期数据 | 中 | 持续监控数据质量指标 |
| **治理缺失** | 缺乏数据治理导致MDM方案沦为单纯的数据集成工具 | 高 | 建立数据治理委员会和数据管理制度 |
| **组织阻力** | 业务部门不愿放弃本地数据和流程控制权 | 中 | 开展变革管理，展示集中管理价值 |
| **参考数据变更影响** | 参考数据变更未充分沟通导致下游系统出问题 | 中 | 建立参考数据变更请求流程 |
| **架构选择错误** | 选择不适合组织的MDM架构模式 | 中 | 充分评估后选择 |

### ROI分析

| 价值维度 | 收益 | 衡量方式 |
|----------|------|----------|
| **运营效率提升** | 减少跨系统数据核对时间，降低集成成本 | 集成项目工时减少百分比 |
| **数据质量提升** | 减少因数据错误导致的业务损失和返工成本 | 数据质量评分提升 |
| **风险降低** | 确保合规，降低监管罚款和声誉损失风险 | 合规审计通过率 |
| **客户体验改善** | 统一的客户视图支持更精准的营销和服务 | 客户满意度评分 |
| **决策支持** | 可信的数据支持更准确的业务分析和决策 | 决策置信度提升 |
| **IT成本节约** | 降低数据集成维护成本，减少冗余系统 | TCO降低百分比 |
| **营收增长** | 加速产品上市，提升交叉销售和追加销售机会 | 新增收入 |
