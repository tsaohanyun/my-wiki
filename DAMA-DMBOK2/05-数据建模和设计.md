# 数据建模和设计

## 概述

数据建模（Data Modeling）是发现、分析和界定数据需求的过程，然后用被称为**数据模型**的精准形式来表示和传达这些需求。数据建模是数据管理的关键组成部分，建模过程要求组织发现并记录其数据是如何相互关联的，同时通过建模过程设计数据之间彼此的关联适配方式。用于表示数据的方案有很多种，最常用的6个方案是：关系型、维度型、面向对象型、基于事实型、基于时序型和非关系型。这些方案的模型主要建立在3个细节层面：概念、逻辑和物理。

## 核心要点速览

### 关键概念
- **数据模型组件**：实体、关系、属性、域
- **建模方案**：关系型、维度型、面向对象型、基于事实型、基于时序型、非关系型
- **模型层次**：概念数据模型（CDM）→ 逻辑数据模型（LDM）→ 物理数据模型（PDM）
- **规范化**：1NF → 2NF → 3NF → BCNF → 4NF → 5NF

### 关键方法
- **建模流程**：规划（P）→ 构建（D）→ 评审（C）→ 维护（O）
- **正向工程**：CDM → LDM → PDM
- **逆向工程**：PDM → LDM → CDM
- **质量评估**：数据模型计分卡（10项指标）
- **设计原则**：PRISM（性能、可重用性、完整性、安全性、可维护性）
