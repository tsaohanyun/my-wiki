## 四、命中率提升优化

### 4.1 aliases别名字段（P0 — 影响最大）

**问题**：当前0个页面有aliases，用户搜英文缩写/同义词直接扑空。

**方案**：每个页面frontmatter加aliases字段，收录：
- 英文缩写（MDM、ETL、CDC、RBAC…）
- 中文常用别称（主数据、数据血缘、戴明环…）
- 英文全称（Master Data Management…）

**示例**：

```yaml
# 10-参考数据和主数据管理.md
aliases: [MDM, RDM, 主数据, 参考数据, Master Data, Reference Data, 
          实体解析, Entity Resolution, 标识管理, Identity Management, 代码表]

# 03-数据治理.md
aliases: [DG, Data Governance, 数据管理专员, Data Steward, CDO, 
          首席数据官, 数据治理委员会, DGC, 联邦式治理, 集中式治理]

# 13-数据质量管理.md
aliases: [DQ, Data Quality, PDCA, 戴明环, 数据剖析, Data Profiling, 
          数据清洗, Data Cleansing, DQI, 根因分析, 六大维度]

# 08-数据集成和互操作.md
aliases: [DII, ETL, ELT, CDC, ESB, 数据集成, Data Integration, 
          消息队列, 数据虚拟化, API, SOA]

# 07-数据安全.md
aliases: [Data Security, RBAC, ABAC, 访问控制, Access Control, 
          加密, Encryption, 数据脱敏, Data Masking, 4A模型]
```

**覆盖范围**：约30-40个页面需要加，每个5-10个别名。

**预期效果**：英文缩写/同义词搜索场景命中率 **+30-40%**。

### 4.2 概述首句标准化（P0 — 语义匹配关键）

**问题**：37/50页面概述质量🟡——首句缺少中英对照定义公式，导致"什么是XX"类搜索语义匹配弱。

**反面案例**（当前第10章）：
> ❌ 在所有组织中，都存在一些需要跨业务、跨流程和跨系统使用的数据。

→ 搜"什么是主数据"搜不到，因为第一段没提"主数据管理"的定义。

**正面案例**（当前第3章）：
> ✅ 数据治理（Data Governance, DG）是对数据资产管理工作履行职权和实施管控的行为。

→ 中英文、缩写、定义一应俱全，命中率高。

**统一格式**：

```
**XX**（英文全称, 缩写）是……的……（一句话定义）。
```

**覆盖范围**：约30个页面需调整概述首句。

**预期效果**：语义搜索场景命中率 **+20-25%**。

### 4.3 孤岛页面加交叉引用（P0 — 导航可达性）

**问题**：18个页面完全没有wikilinks，进入后无法导航到相关内容。

| 孤岛类型 | 数量 | 页面 |
|---------|------|------|
| 精读笔记 | 14个 | 大部分精读笔记底部无相关页面链接 |
| 工具箱 | 2个 | 数据治理实施检查清单、数据管理工具选型指南 |
| 实践映射 | 1个 | 通威农发-DAMA映射 |
| 精读索引 | 1个 | 00-精读笔记索引 |

**方案**：每个精读笔记底部加：

```markdown
## 相关页面
- 📖 知识领域：[[DAMA-DMBOK2/03-数据治理]]
- 📖 精读笔记索引：[[DAMA-DMBOK2/精读笔记/00-精读笔记索引]]
- 📖 术语表：[[DAMA-DMBOK2/术语表]]
```

**预期效果**：导航可达性 **+10-15%**。

### 4.4 frontmatter `related` 字段全覆盖（P1）

**问题**：仅4个页面使用了frontmatter中的related字段，元数据层面关联薄弱。

**方案**：所有页面统一加3-5个related链接：

```yaml
related:
  - "[[DAMA-DMBOK2/03-数据治理]]"
  - "[[DAMA-DMBOK2/12-元数据管理]]"
  - "[[DAMA-DMBOK2/13-数据质量管理]]"
```

**预期效果**：关联发现能力 **+10%**。

### 4.5 术语表加反向链接（P1）

**问题**：术语表（14,852字符，221行）收录大量术语定义，但没有链接到对应的知识领域详细页面。

**方案**：每个术语加目标页面链接：

```markdown
| Data Governance | DG | 数据治理 | → [[DAMA-DMBOK2/03-数据治理\|详见第3章]] |
| Data Lineage | — | 数据血缘 | → [[DAMA-DMBOK2/12-元数据管理\|详见第12章]] |
| ETL | — | 抽取-转换-加载 | → [[DAMA-DMBOK2/08-数据集成和互操作\|详见第8章]] |
```

**预期效果**：术语→详情的跳转命中率 **+5-10%**。

### 4.6 同义词/相关概念段落（P2）

**方案**：在知识领域页面核心概念部分增加同义词对照表：

```markdown
### 同义词与相关概念
| 本页术语 | 同义词 | 反义/对比概念 |
|---------|-------|-------------|
| 主数据(MDM) | 核心数据、共享数据 | 交易数据、参考数据 |
| 实体解析 | 标识管理、记录链接 | — |
```

**预期效果**：精准匹配 **+5%**。

### 4.7 工具箱页面增加"适用场景"索引（P2）

**问题**：工具箱是独立文档，不知道"什么场景下该看这个工具"。

**方案**：加适用场景说明：

```markdown
## 适用场景
- 当你需要评估组织数据管理成熟度时 → 使用本模板
- 相关知识领域：[[DAMA-DMBOK2/15-数据管理成熟度评估]]
```

### 4.8 精读笔记增加英文关键词标注（P2）

**问题**：精读笔记有"一句话总结"（✅做得好），但缺少英文关键词。

**方案**：在总结后加：

```markdown
> **关键词**：Data Governance, DG, 数据治理, 数据管理专员, Data Steward, CDO
```
