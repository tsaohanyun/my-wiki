## 概述

数据安全是[[DAMA-DMBOK2/01-数据管理|数据管理]]的核心知识领域之一，包括安全制度和规程的**规划、建设与执行**，为数据和信息资产提供正确的身份验证、授权、访问和审计。

数据安全实践的核心目标：**保护数据资产，并确保其符合隐私和保密法律法规要求、合同协议约定及业务需求。**

### 业务驱动因素

| 驱动因素 | 说明 |
|---------|------|
| **降低风险** | 合规性、受托责任、声誉保护、隐私保护 |
| **促进业务增长** | 运营成功、客户信任、竞争优势、电子商务 |

### 目标

1. **赋能适当访问** — 允许对企业数据资产进行适当访问，并防止不当访问
2. **确保法规合规** — 确保符合隐私、保护和保密性的法规和政策
3. **保护利益相关方** — 确保满足利益相关方对数据隐私、数据机密性的要求

### 原则

| 原则 | 说明 |
|------|------|
| 协同合作 | IT安全管理员、数据管理专员、数据治理团队、内外部审计和法律部门协同完成 |
| 企业统筹 | 数据安全标准和制度必须在整个组织内一致地应用 |
| 主动管理 | 成功源于管理的主动性和动态性 |
| 明确的问责制 | 角色和职责必须明确界定 |
| 元数据驱动 | 数据元素的安全分类分级是数据定义的基础组成部分 |
| 减少暴露面 | 最小化敏感/机密数据的扩散 |

## 核心要点摘要

### 安全定义

> **数据安全**（Data Security）：包括安全制度和程序的规划、建设与执行，以在文化和监管考虑范围内为数据和信息资产提供正确的**身份验证、授权、访问和审计**。

### 关键概念速览

| 概念 | 要点 |
|------|------|
| **漏洞与威胁** | 漏洞是系统弱点，威胁是潜在攻击行动；需评估外部威胁和内部风险 |
| **风险分类** | 关键风险数据（CRD）、高风险数据（HRD）、中等风险数据（MRD） |
| **保密等级** | 面向普通受众 → 仅内部使用 → 机密 → 受限机密 → 注册机密 |
| **4A+1E模型** | 访问(Access)、审计(Audit)、验证(Authentication)、授权(Authorization) + 权限(Entitlement) |
| **访问控制** | RBAC（基于角色）和ABAC（基于属性），查询级访问控制解决过度授权 |
| **加密** | 哈希、对称加密（AES）、非对称加密（RSA）；数据存储安全与传输安全 |
| **数据脱敏** | 静态脱敏（永久修改）和动态脱敏（显示时改变），9种脱敏方法 |
| **安全框架** | NIST 6步风险管理框架；ISO 27001/27002信息安全管理体系 |

### 组织与角色

- 大型企业设**首席信息安全官（CISO）**
- 数据治理委员会审查和批准数据安全制度
- 外包环境中"转移控制，并非转移责任"

### 度量关键指标

- 安全补丁覆盖率、反恶意软件覆盖率
- 安全意识培训通过率
- 灾难恢复测试成功率
- 审计发现问题解决率
