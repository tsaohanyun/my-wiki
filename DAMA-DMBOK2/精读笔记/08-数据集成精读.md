# 第8章 数据集成和互操作 — 精读笔记

## 一句话总结

数据集成和互操作（DII）是管理数据在不同存储库、应用程序和组织之间流动与整合的核心职能，通过ETL、消息传递、虚拟化等技术手段，以企业视角实现数据的高效共享与复用。


> **关键词**：Data Integration, 数据集成, ETL, ELT, CDC, ESB, 消息队列, API
## 核心要点（5-8个）

1. **ETL是核心过程**：抽取（Extract）、转换（Transform）、加载（Load）是数据集成的基本流程，ELT是其变体（目标系统转换能力更强时采用）。转换包括格式变化、结构变化、语义转换、去重和重新排序。
2. **时延是关键设计维度**：从批处理（高延迟）→事件驱动/准实时→实时同步→流处理（极低延迟），不同场景需要不同级别的时延方案，需在成本与性能间权衡。
3. **交互模型决定架构**：点对点模式（接口数量呈平方增长，维护成本高）→中心辐射型（中央枢纽集中共享，推荐）→发布订阅（推拉结合），优先采用中心辐射型和规范数据模型。
4. **紧耦合vs松耦合**：紧耦合系统互相依赖（一损俱损），松耦合通过ESB/API/消息队列实现系统独立性，是现代集成架构的核心设计原则。
5. **规范数据模型降低复杂度**：企业消息格式/规范模型将N×(N-1)个转换减少到2N个，在管理3个以上系统时合理，超过100个系统时至关重要。
6. **CDC技术减少传输开销**：变更数据捕获只传输变更数据，包括基于数据（时间戳、变更列表、变更对象）和基于日志两种方式，显著降低带宽需求。
7. **数据血缘贯穿全过程**：从规划阶段记录数据来源和流转路径，到实施阶段监控数据移动，数据血缘是影响分析、合规审计和问题定位的基础。
8. **业务专家必须参与**：数据转换规则的设计和修改必须由业务专家负责，不能仅视为纯技术工作。

## 重要概念速查

| 概念 | 定义 |
|------|------|
| ETL | 抽取-转换-加载，数据集成核心过程 |
| ELT | 抽取-加载-转换，目标系统转换能力更强时使用 |
| CDC | 变更数据捕获，仅传输变更数据以减少带宽 |
| ESB | 企业服务总线，系统间消息传递的中介 |
| SOA | 面向服务架构，通过定义良好的API实现模块交互 |
| CEP | 复杂事件处理，合并多源数据识别有意义事件并自动响应 |
| DaaS | 数据即服务，按需获取数据而非本地存储 |
| iPaaS | 云化集成平台，作为云服务交付的系统集成 |
| 时延 | 从源系统生成数据到目标系统可用的时间差 |
| 规范模型 | 标准化数据共享格式的通用模型 |

## 方法论精华

**实施流程五步法：**

1. **规划和分析**：定义需求→数据探索→记录数据血缘→数据剖析→收集业务规则
2. **设计解决方案**：选择交互模型→设计数据服务/交换模式→数据建模→映射源到目标→设计编排
3. **开发解决方案**：开发数据服务→开发编排流程→制定数据迁移方法→制定发布方式→开发CEP流
4. **实施和监测**：启用服务→实时监控→建立度量指标→建立反馈机制
5. **维护元数据**：记录所有系统数据结构→元数据变更审批→SOA注册中心管理

**数据剖析五要素：**数据格式、数据数量（含空值率）、数据值有效性、数据集内部模式/关系、与其他数据集的关系。跳过剖析几乎必然导致测试或上线时发现问题。

## 实施要点

- **就绪评估**：围绕企业集成工具实现或增强来评估，关注成本合理性、增量价值和组织支持
- **工具选型**：ETL工具、数据虚拟化服务器、ESB、业务规则引擎、数据建模工具、数据剖析工具、元数据库
- **组织模式推荐**：卓越中心（CoE）模式最佳，结合集中团队的技术深度和本地团队的业务理解
- **数据共享协议**：开发接口前应制定MOU，规定数据责任、使用限制和服务级别
- **架构设计原则**：松散耦合、最小化接口数量、标准规范接口、中心辐射型优先、合理编排

## 考试/面试高频考点

1. **ETL vs ELT的区别和适用场景**：ETL在中间层转换，ELT在目标系统转换（大数据场景常见）
2. **五种时延模式的对比**：批处理、事件驱动、准实时、实时同步、流处理——各自的优缺点和适用场景
3. **点对点 vs 中心辐射型**：为什么中心辐射型优于点对点？（接口数量从N²降至N）
4. **紧耦合 vs 松耦合**：松耦合如何通过ESB/SOA实现系统独立性
5. **CDC的两种实现方式**：基于数据（时间戳/变更列表/变更对象）vs 基于日志
6. **规范数据模型的价值**：减少转换数量、降低维护成本、提高一致性
7. **数据剖析的重要性**：为什么不能跳过？预期与实际几乎总有差异
8. **数据血缘的作用**：影响分析、问题定位、合规审计
9. **SOA治理要求**：服务重用、注册中心管理、API设计规范
10. **两阶段提交**：实时同步的典型实现，适用于金融交易等场景
