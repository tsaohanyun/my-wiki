# 第3章 数据治理 — 精读笔记

## 一句话总结

数据治理是对数据资产管理工作履行职权和实施管控的行为——它不直接管理数据，而是确保数据被恰当地管理，如同财务审计与财务管理的关系，代表监督和执行之间固有的职责分离，是DAMA车轮图的核心。

## 核心要点（5-8个关键点）

1. **治理≠管理**：数据治理是"监督者"（立法+司法+行政），数据管理是"执行者"。数据治理关注"如何对数据做出决策"，而非直接管理数据。类比：数据治理与数据管理的关系 ≈ 财务审计与财务管理的关系。

2. **数据治理独立于IT治理**：IT治理专注IT投资决策和COBIT框架，数据治理只关注数据资产管理和把数据作为资产来管理。

3. **数据治理六大原则**：领导力和战略、业务驱动、共担责任、多层级的、基于框架、基于原则。三大核心特性：可持续（非一次性项目）、可融合（融入日常活动）、可衡量（有积极财务影响）。

4. **最常见驱动因素是监管合规**：驱动力归结为降低风险（一般风险管理、数据安全、隐私）和优化流程（监管合规、数据质量提升、元数据管理、开发项目效率、供应商管理）两大类。

5. **数据治理八大范围**：战略、制度、标准、监督、合规、问题管理、数据管理项目、数据资产估值。

6. **数据管理专员是治理落地的关键**：从企业全局角度确保数据高质量和有效利用，核心职责包括创建和管理核心元数据、记录规则和标准、管理数据质量问题、执行日常数据治理活动。类型包括首席/高级/企业/业务/技术/协调型数据管理专员和数据所有者。

7. **渐进式实施策略**：很少有组织一开始就将数据治理部署到整个企业范围，通常纳入大型项目（如MDM）一并实施，或按区域/部门逐步推进。四阶段：定义→制定战略→实施→嵌入。

## 重要概念速查（术语+一句话定义）

| 术语 | 定义 |
|------|------|
| 数据治理(DG) | 对数据资产管理工作履行职权和实施管控（规划、实施、监督和执行）的行为 |
| 数据治理委员会(DGC) | 企业级数据治理最高决策机构，审查、批准和采用数据标准和架构 |
| 数据管理专员(Data Steward) | 为组织最高利益代表他人管理数据资产的人，从企业全局角度确保数据高质量 |
| 数据所有者(Data Owner) | 拥有其管辖范围内有关数据决策审批权的业务数据人员 |
| CDO | 首席数据官，数据治理职能的最高负责人 |
| 数据制度 | 将数据管理原则和管理意图整编为指令性的基础规则 |
| 数据标准 | 提供比较方法，有助于定义质量和简化流程的规范 |
| 业务术语表 | 包含已达成统一定义的业务术语并与数据相关联的核心工具 |
| 数据资产估值 | 了解和计算数据对组织经济价值的过程 |
| 集中式治理 | 一个数据治理机构监督所有主题域的全部活动 |
| 联邦式治理 | 一个数据治理组织与多个业务单元协调以维持一致的定义和标准 |
| 信息鸿沟 | 需要什么信息与可以得到什么信息之间的差异，代表商业负债 |

## 方法论精华（关键流程和框架）

### 数据治理运营模型（三种类型）

| 模型 | 特征 | 适用场景 |
|------|------|----------|
| 集中式 | 一个机构监督所有主题域 | 组织规模较小或治理初期 |
| 复制式 | 每个业务单元采用同样模型和标准 | 多业务单元但流程一致 |
| 联邦式 | 一个组织与多个业务单元协调 | 大型复杂企业 |

数据治理借用政治治理的三权结构：**立法**（定义制度、标准、架构）、**司法**（问题管理和修复）、**行政**（保护和服务）。

### 数据治理实施四阶段

**阶段一：定义组织的数据治理**
1. 执行就绪评估：数据管理成熟度、变革能力、协作准备度、业务一致性
2. 对齐业务执行探索：识别现行制度有效性、数据质量分析、数据管理实践评估
3. 创建组织触点：采购与合同、预算与资金、制度合规、软件开发生命周期

**阶段二：制定数据治理战略**
- 交付成果：章程（愿景/使命/原则）、运营框架和责任、实施路线图、运营目标
- 构建运营模式须考虑：数据对组织的价值、商业模式、文化因素、监管影响

**阶段三：实施数据治理**
- 早期优先活动：定义高优先级流程、建立业务术语表、与架构团队协调、数据资产评估
- 企业数据模型由数据架构师和数据管理人员在主题域团队中共同开发维护，由DGC审查批准

**阶段四：嵌入数据治理**
- 将治理活动融入日常运营流程
- 创建数据治理兴趣社区推动本地化和学习
- 确保流程和资金到位以保证持续运作

### 问题管理流程

八类问题：权力、变革管理升级、合规、冲突、一致性、合同、数据安全和身份识别、数据质量。

升级路径：数据管理团队→业务单元数据治理委员会→数据治理委员会→数据治理指导委员会。

六项控制机制：识别/记录/跟踪/升级问题、分配行动事项、记录观点和方案、决定和沟通方案、促成客观中立讨论、升级到更高层级。

### 变革管理五大重点领域

1. **意识**：对变革需求的认知
2. **愿望**：参与和支持变革的意愿
3. **知识**：如何变革的了解
4. **能力**：执行新技能和行为的能力
5. **力度**：保持变革有序的推动力

### 数据资产估值五种方法

| 方法 | 说明 |
|------|------|
| 替换成本 | 丢失数据的替换或恢复成本 |
| 市场价值 | 企业并购时作为商业资产的价值 |
| 识别商机 | 通过利用数据识别出的机会可获得的收入 |
| 销售数据 | 将数据或洞察作为商品出售 |
| 风险成本 | 缺失/不应出现/不正确数据导致的处罚、补偿、诉讼成本 |

### 可在各知识领域标准化的内容

覆盖数据架构、数据建模和设计、数据存储和运营、数据安全、数据集成、文档和内容、参考数据和主数据、数据仓库和商务智能、元数据、数据质量、大数据和数据科学共11个领域的标准化内容清单。

## 实施要点（落地时的关键注意事项）

1. **明确业务驱动因素**：在组织内部明确数据治理的特定业务驱动因素并将其与整体业务战略保持一致。过分强调"数据治理组织"会让部分领导层感到疏远，需要用恰当的描述、运营模式和角色。

2. **渐进式实施**：很少有组织一开始就将数据治理部署到整个企业范围。通常纳入大型项目（如ERP、CRM、MDM）一并实施，或按区域/部门逐步推进。

3. **文化因素至关重要**：不论数据治理战略多么精确或独特，忽视商业文化就会降低成功概率。纪律服从程度和变革适应能力是影响运营模式的关键因素。

4. **数据制度要精炼**：数量设置相对较少，陈述简明扼要。数据制度描述"做什么"和"不做什么"，数据标准和数据流程描述"如何做"。必须有效沟通、监测、强制执行和定期复评。

5. **业务术语表是核心工具**：不仅是术语和定义的列表，每个术语还与其他元数据关联（同义词、指标、血缘、安全等级、业务规则、管理员等）。是形成共同理解、降低数据滥用风险的基础。

6. **CDO需要创建组织触点**：通过采购与合同、预算与资金、制度合规、软件开发生命周期四大触点与组织各领域建立连接，这是组织如何感知数据治理的蓝图。

7. **问题管理建立信誉**：数据问题管理建立数据治理团队的信誉，对数据消费者有直接积极影响，证明数据能够被管理且质量可以提升。

8. **可持续性是核心挑战**：数据治理不是具有明确结束时间的项目，而是需要组织承诺持续投入的企业职能。依赖业务领导力、资金和所有权。
