# 第10章 参考数据和主数据管理 — 精读笔记

## 一句话总结

参考数据和主数据管理通过管理语义一致性来协调和集成跨业务、跨系统共享的关键数据实体，确保组织拥有完整、一致、最新和权威的业务数据，是提升数据质量、降低集成成本和风险的核心实践。


> **关键词**：Master Data Management, MDM, 主数据, 参考数据, 实体解析, Entity Resolution
## 核心要点（5-8个）

1. **参考数据与主数据有本质区别**：参考数据是代码表和描述表（如国家代码、订单状态），相对稳定、规模小、复杂度低；主数据是业务实体数据（如客户、产品、供应商），持续变化、规模大、需要实体解析。
2. **主数据的三层分类**（Chisholm）：参考数据（代码表）、企业结构数据（会计科目表）、业务结构数据（客户标识符、产品代码）。主数据是这三者的集合。
3. **实体解析是MDM核心环节**：确定两个引用指向同一对象还是不同对象，包括确定性算法（规则驱动，可预测）和概率算法（统计方法，可自我优化）。假阳性（一码多物）和假阴性（一物多码）是两大风险。
4. **三种MDM架构模式**：注册模式（索引，易实现但查询复杂）、事务中心模式（主数据仅存于中心，管理好但迁移成本高）、混合模式（整合后共享，平衡两者优缺点）。
5. **记录系统vs参考系统**：记录系统是权威数据创建源（如ERP），参考系统是数据消费者获取可靠数据的权威源（如MDM系统、数据仓库），两者可以不同。
6. **黄金记录vs可信来源**：黄金记录追求"真相单一版本"但难以100%准确；可信来源强调"最佳版本"，帮助不同消费者看到对其重要的单一版本组成部分。
7. **六大基本原则**：高效共享、全局统筹、质量管控、责任到人、受控变更（合并/拆分可逆）、遵从权威（只能从记录系统复制）。
8. **组织变革是最大挑战**：MDM需要业务部门放弃对本地数据和流程的控制权，这需要有效的变革管理和高层支持。

## 重要概念速查

| 概念 | 定义 |
|------|------|
| MDM | 主数据管理，对主数据值和标识符进行控制以确保一致性 |
| RDM | 参考数据管理，对参考数据定义及域值进行控制 |
| 记录系统 | 权威系统，通过定义规则创建、获取和维护数据 |
| 参考系统 | 数据消费者获取可靠数据的权威系统 |
| 黄金记录 | 真相的单一版本（Single Version of Truth） |
| 可信来源 | 事实的最佳版本（Best Version of Truth） |
| 实体解析 | 确定两个引用指向同一对象还是不同对象的过程 |
| 假阳性 | 不同实体错误链接到同一标识符（一码多物） |
| 假阴性 | 同一实体未链接到同一标识符（一物多码） |
| 全局ID | MDM分配的唯一标识符，附加到已匹配记录 |
| 交叉引用 | 源ID与全局ID之间的映射关系 |

## 方法论精华

**MDM关键处理步骤（六步流水线）：**

```
数据模型管理 → 数据采集 → 数据验证/标准化/增强 → 实体解析 → 数据共享 → 管理责任
```

**MDM实施七阶段：**
1. 定义驱动因素和需求（P）
2. 评估和评价数据源（P）
3. 定义架构方法（D）
4. 建模数据（D）
5. 定义管理职责和维护流程（C）
6. 制定治理制度（C）
7. 实现数据共享/集成服务（D/O）

**实体解析三种匹配工作流：**
- 多重识别匹配规则：不自动合并，人工决策（高价值/高风险）
- 匹配链接规则：建立交叉引用，不更新内容（灵活可撤销）
- 匹配合并规则：合并为唯一记录（权威唯一版本）

**参考数据四种结构类型：**简单列表→交叉参考列表→分类法→本体

## 实施要点

- **遵循主数据架构**：综合考虑组织架构、记录系统数量、数据治理、延迟要求
- **监控数据移动**：确保及时提取和分发，展示共享和使用情况，识别血缘
- **管理参考数据变更**：参考数据是共享资源不能随意更改，需建立变更请求流程
- **数据共享协议**：建立SLA和指标衡量可用性和质量
- **五大度量维度**：数据质量和合规性、数据变更活动、数据接收和使用、SLA、总体拥有成本
- **先评估后实施**：了解数据结构、内容、质量及根本原因，再选择架构模式

## 考试/面试高频考点

1. **参考数据vs主数据的区别**：变化频率、数据规模、复杂度、是否需要实体解析
2. **MDM三种架构模式**：注册、事务中心、混合——各自的优缺点和适用场景
3. **实体解析的核心概念**：假阳性/假阴性、确定性/概率算法、三种匹配工作流
4. **记录系统vs参考系统**：权威数据创建源vs权威数据获取源
5. **黄金记录vs可信来源**：为什么"可信来源"比"黄金记录"更务实
6. **MDM六大原则**：高效共享、全局统筹、质量管控、责任到人、受控变更、遵从权威
7. **MDM不等于技术方案**：是人、流程和技术的知识领域，组织变革是最大挑战
8. **主数据常见域**：参与方（客户/员工/供应商）、产品、财务、法律、位置
9. **数据共享架构**：中心辐射型设计，MDM作为数据枢纽减少集成次数
10. **参考数据变更管理**：四种变更类型（外部/内部×行级/结构），需审批流程
