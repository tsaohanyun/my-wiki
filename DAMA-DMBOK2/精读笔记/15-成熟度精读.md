# 第15章 数据管理成熟度评估 — 精读笔记

## 一句话总结

数据管理成熟度评估（DMMA）基于能力成熟度模型（CMM），通过6个等级（0-5级）描述数据管理实践从无组织到优化的演变过程，帮助组织评估现状、识别差距、规划改进路线图，是数据治理的关键组成部分。


> **关键词**：CMMI, DCAM, DMM, 成熟度评估, Maturity Assessment, 数据管理能力
## 核心要点（7个关键点）

1. **DMMA是评估方法而非解决方案**：它描述当前数据管理状态和改进机会，目的是揭示优势和改进机会，而不是直接解决问题。评估应在短暂而明确的时间范围内进行。

2. **6个成熟度等级不可跳过**：第0级（无能力）→第1级（初始/临时）→第2级（可重复）→第3级（已定义）→第4级（已管理）→第5级（优化）。能力提升必须逐级开展，不能跳级。

3. **评估具有文化影响**：DMMA不仅评估技术能力，还会向利益相关方普及数据管理概念、厘清角色责任、强调数据资产价值、增强全组织意识、促进协作关系。

4. **四级评分量表统一评估**：每个评估主题按1（未开始）→2（进行中）→3（能使用）→4（有效）评分，综合映射到整体成熟度等级。雷达图可视化展示当前能力与目标能力的差距。

5. **评估按四大维度展开**：活动（实施程度）、工具（自动化程度）、标准（通用标准支持）、人员和资源（技能和角色清晰度）。

6. **主流模型各有侧重**：CMMI-DMM（通用，6大领域）、DCAM（金融行业，37项能力115项子能力）、IBM模型（55个组织委员会）、斯坦福模型（学术严谨）、Gartner EIM（分析师洞察）。

7. **评估是持续循环**：初始计划和评估→建议→行动计划→定期重新评估。DMMA的流程监管职责属于数据治理团队，执行发起人理想为CDO。

## 重要概念速查

| 术语 | 定义 |
|------|------|
| DMMA | 数据管理成熟度评估，基于框架的流程改进方法 |
| CMM | 能力成熟度模型，描述流程从初始到最佳状态的演变过程 |
| CMMI-DMM | CMMI研究院开发的数据管理成熟度模型，覆盖6大评估领域 |
| DCAM | EDM Council开发的数据管理能力评估模型，37项能力115项子能力 |
| 第1级（初始级） | 成功取决于个人能力，依赖少数专家，数据质量问题普遍存在 |
| 第2级（可重复级） | 基本流程规则已制定，角色得到定义，不再完全依赖特定专家 |
| 第3级（已定义级） | 标准化制度化，数据管理被视为组织发展的推动力量 |
| 第4级（已管理级） | 流程量化控制，能预测结果，包含性能指标 |
| 第5级（优化级） | 流程高度可预测，关注持续改进，自动化技术变更管理 |

## 方法论精华

### 评估实施总体流程（5阶段）

```
1.规划评估 → 2.执行评估 → 3.结果解读 → 4.制订改进计划 → 5.重新评估
```

| 阶段 | 核心活动 |
|------|---------|
| 规划评估 | 定义目标、选择框架、定义组织范围、定义交互方法、沟通规划 |
| 执行评估 | 收集信息（访谈/文档/调查）、执行评估与评级、讨论达成共识 |
| 结果解读 | 编制评估报告（驱动因素/结论/评分差距/建议）、管理层简报 |
| 制订改进计划 | 制定多年度改进计划、有序活动序列、时间表、预期改善 |
| 重新评估 | 定期重新评估，跟踪趋势，作为持续改进循环的一部分 |

### 四大评估维度

| 维度 | 评估问题 |
|------|---------|
| 活动 | 实施程度？执行标准？最佳实践输出？ |
| 工具 | 自动化程度？培训、按需使用、优化配置？ |
| 标准 | 通用标准支持？记录完善？治理强制执行？ |
| 人员和资源 | 人员充足？技能培训？角色职责清晰？ |

### 主流成熟度模型选择标准

| 标准 | 说明 |
|------|------|
| 可访问性 | 非技术术语表述 |
| 全面性 | 覆盖业务参与而非仅IT流程 |
| 可扩展性 | 可增加行业特定领域 |
| 内建演进路线 | 描述逻辑演进方向 |
| 非指导性 | 描述需要做什么而非必须如何做 |
| 可重复性 | 一致解释，便于对比和趋势跟踪 |
| 中立性 | 供应商选择中立 |

### 局部式 vs 企业级评估

| 维度 | 局部式 | 企业级 |
|------|--------|--------|
| 范围 | 单个业务领域 | 整个组织 |
| 深度 | 更深入细节 | 关注广泛分散部分 |
| 速度 | 更快 | 较慢 |
| 典型选择 | 受严格监管的单元 | 多职能单元 |

## 实施要点

1. **获得高管支持是前提**：评估需有执行发起人（理想为CDO），将评估与业务战略对齐。缺乏高层推动，难以获得资源和跨部门协作。

2. **首次评估选择可控范围**：范围过宽导致周期过长、结果不聚焦。可从受严格监管的职能单元（如财务报告）开始。

3. **设定明确时间框架**：评估周期过长会导致利益相关方失去热情和动力。快速完成以保持动力。

4. **充分沟通评估目的**：告知利益相关方评估目的、如何进行、参与环节、时间表。评估期间确保有明确议程，持续表达感谢。

5. **证据支持评级**：评级需基于实际证据，通过讨论达成共识。记录支持证据，使用模型标准陈述作为评级说明。

6. **可视化呈现结果**：使用雷达图等可视化工具，外环表示竞争所需能力，内环表示实际能力，差距最大区域代表最大风险。

7. **评估后必须有行动计划**：评估报告完成后必须制定改进路线图，设定可衡量的目标，否则评估将流于形式。

8. **谨慎裁剪模型**：不当修改可能导致模型失去严谨性或可追溯性。保持模型完整性，仅在必要时进行有依据的调整。

## 考试/面试高频考点

1. **6个成熟度等级及各级特征**：0级无能力→1级初始/临时→2级可重复→3级已定义→4级已管理→5级优化——各级的关键区分标志
2. **DMMA的5个目标**：评估当前状态、支持运营和战略方向、制定统一愿景、实现集成计划、设定和衡量目标
3. **6个业务驱动因素**：监管、数据治理、流程改进准备、组织变革、新技术、数据管理问题
4. **四级评分量表**：1未开始→2进行中→3能使用→4有效
5. **评估实施5阶段流程**：规划→执行→结果解读→制订改进计划→重新评估
6. **四大评估维度**：活动、工具、标准、人员和资源
7. **5大主流模型对比**：CMMI-DMM vs DCAM vs IBM vs 斯坦福 vs Gartner EIM——发布组织、评估范围、适用行业、核心优势
8. **评估方法选择的12个标准**
9. **DMMA的5大文化影响**：普及知识、厘清角色、强调资产、增强意识、促进协作
10. **风险清单**：缺乏高管支持、范围过大、参与者不投入、评估延迟、结果无法落地、模型裁剪不当
