# 第5章 数据建模和设计 — 精读笔记

## 一句话总结

数据建模是发现、分析和界定数据需求，并用精确的数据模型来表示和传达这些需求的过程，涵盖概念、逻辑、物理三个层次和关系型、维度型等六种建模方案。

## 核心要点（5-8个关键点）

1. **数据模型的四大基础组件**：实体（组织收集信息的对象）、关系（实体之间的联系）、属性（定义描述实体特征）、域（属性的全部可能值）。掌握这四者是理解所有数据模型的基础。

2. **六种数据建模方案**：关系型（事务处理）、维度型（数据仓库分析）、面向对象型（软件设计）、基于事实型（概念建模）、基于时序型（数据仓库）、非关系型（大数据）。不同方案适用于不同场景。

3. **三个抽象层次**：概念数据模型（CDM，最高抽象，仅含关键实体和关系）→ 逻辑数据模型（LDM，完整属性，经过规范化）→ 物理数据模型（PDM，与具体数据库技术相关，含性能优化）。

4. **规范化是消除冗余的核心手段**：1NF（有效主键+原子性）→ 2NF（最小主键+完整依赖）→ 3NF（无隐藏主键+非键不依赖非键）。通常"规范化模型"指满足3NF。

5. **维度建模是数据仓库的核心方案**：由事实表（度量值，占空间90%）和维度表（描述信息，占10%）组成。关键概念包括粒度、星型模式、雪花模式、渐变维度（SCD）和一致性维度。

6. **物理设计的五大技术**：视图（虚拟表简化查询）、分区（垂直/水平拆分）、逆规范化（故意冗余提升性能）、索引（加速查询）、代理键（无业务含义的系统生成键）。

7. **数据模型计分卡包含10个质量指标**：涵盖模型满足需求能力、完整性、匹配程度、结构合理性、通用性、命名标准、可读性、定义情况、企业架构一致性和元数据匹配程度。

## 重要概念速查（术语+一句话定义）

- **实体**：组织收集信息的对象，是组织内存在的"名词"（如学生、课程）
- **关系**：实体之间的联系，用基数（0/1/多）表达数量约束
- **属性**：定义、描述或度量实体特征的元素，物理层表现为列/字段
- **域**：某个属性能被赋予的全部可能值，用于标准化属性特征
- **基数（Cardinality）**：实体实例间的数量关系，用"鸡爪印"等符号表示
- **主键**：被选出的唯一识别标识符，一个实体只有一个
- **候选键**：可唯一标识实体实例的最小属性组，可有多个
- **代理键**：系统自动生成的无业务含义的计数器，通常用作主键
- **外键**：在物理模型中代表关系，存在于关系中"多"的一边
- **渐变维度（SCD）**：处理维度表中历史数据变化的策略，分Type1覆盖、Type2新行、Type3新列
- **粒度**：事实表中单行数据的含义，是维度建模的关键设计决策
- **一致性维度**：跨数据集市共享的标准化维度，确保术语和值一致
- **逆规范化**：故意将规范化模型转换为含冗余的物理表以提高查询性能
- **规范化**：运用规则消除冗余和不一致性的过程
- **抽象化**：泛化（公共属性分组为超类）和专门化（区分属性分离为子类）

## 方法论精华

### 建模流程（PDCA循环）

1. **规划（P）**：评估组织需求、确定建模标准、明确存储管理
2. **构建（D）**：概念建模→逻辑建模→物理建模（正向工程）；或反向工程从PDM→LDM→CDM
3. **评审（C）**：由不同背景的专家组成评审组，使用计分卡评估
4. **维护（O）**：需求变化时更新模型，每层变更需联动检查

### 概念建模五步法

1. 选择模式（关系/维度/基于事实/非关系型）
2. 选择模型表示方式（IE/ORM等）
3. 完成初版概念数据模型
4. 整合企业术语
5. 签署确认模型

### 物理建模关键技术选择

| 技术 | 目的 | 适用场景 |
|------|------|---------|
| 视图 | 简化查询、控制访问 | 频繁使用的复杂查询 |
| 分区 | 提升性能、便于归档 | 大表按日期/区域拆分 |
| 逆规范化 | 减少连接、提升查询速度 | 读多写少的分析场景 |
| 索引 | 加速数据检索 | 频繁查询的列（尤其是键） |
| 代理键 | 无业务含义的唯一标识 | 维度表主键 |

## 实施要点

1. **正向工程与逆向工程结合**：新建系统用正向工程（CDM→LDM→PDM），改造系统用逆向工程（PDM→LDM→CDM）
2. **命名标准遵循ISO/IEC 11179**：逻辑名用完整单词加空格分隔，物理名用缩写加下划线分隔
3. **数据模型是元数据的重要形式**：为其他管理职能（治理、仓库、分析）提供必要元数据
4. **建模是迭代过程**：与业务专家反复讨论确认，直至模型满足业务规则
5. **PRISM原则指导数据库设计**：性能（Performance）、可重用性（Reusability）、完整性（Integrity）、安全性（Security）、可维护性（Maintainability）
6. **版本管理记录五要素**：为何变更、变更对象、何时审批、谁执行、连锁影响

## 考试/面试高频考点

1. **四种可建模数据类型**：类别信息（参考数据）、资源信息（主数据）、业务事件信息（交易数据）、详细交易信息（大数据）
2. **关系型与维度型对比**：关系型面向事务处理、减少冗余、精确表达；维度型面向分析查询、事实表+维度表、利于多维分析
3. **规范化各级别要求**：1NF（原子性+有效主键）→ 2NF（完全函数依赖）→ 3NF（消除传递依赖），BCNF、4NF、5NF较少使用
4. **渐变维度三种类型**：Type1覆盖旧值、Type2新增行保留历史、Type3新增列保留有限历史
5. **标识关系与非标识关系**：标识关系中父实体主键成为子实体主键一部分（子实体为非独立实体）；非标识关系中父键仅成为子实体的普通外键
6. **键的分类**：超级键→候选键→主键（唯一一个）→备用键（未选为主键的候选键）；代理键（无业务含义）vs 业务键（自然键）
7. **逆规范化的四个目的**：预先组合减少运行时连接、创建筛选副本减少全表扫描、预先计算避免运行时资源竞争、分割数据提升安全性
8. **NoSQL四种类型**：文档数据库、键值数据库、列式数据库、图数据库——各自适用场景
