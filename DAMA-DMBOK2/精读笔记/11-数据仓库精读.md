# 第11章 数据仓库和商务智能 — 精读笔记

## 一句话总结

数据仓库将多源数据集成到通用数据模型中支撑决策分析，商务智能通过查询、报表、可视化等手段将数据转化为业务洞察，两者共同构成组织从回顾性评价到预测性分析的数据驱动决策基础设施。


> **关键词**：Data Warehouse, 数据仓库, BI, 商务智能, 数据集市, Data Mart, OLAP, 数据湖
## 核心要点（5-8个）

1. **Inmon vs Kimball两大流派**：Inmon自上而下，先建企业级数据仓库（3NF规范化），再建数据集市；Kimball自下而上，先建数据集市（星型模型），通过一致性维度整合。两者都认同数据仓库应集成、易访问、服务决策。
2. **星型模型是多维建模核心**：由事实表（定量数据）和维度表（描述属性）组成。雪花型模型是其规范化变体，减少冗余但增加查询复杂度。
3. **OLAP支持多维分析**：五种基本操作——切片、切块、钻取/上卷、旋转。三种实现：ROLAP（关系型）、MOLAP（多维矩阵）、HOLAP（混合型）。
4. **数据集市是子集**：面向特定主题域或部门的数据仓库子集，通过汇总支持快速分析，设计使用维度建模。
5. **CDC技术选型关键**：数据库日志（低复杂度/高效率）、时间戳（中等）、触发器（高复杂度/低效率）、全量加载（最低效率）。准实时方案：微批处理、消息队列、流式处理。
6. **数据仓库总线矩阵**：Kimball方法论的关键规划工具，展示业务流程与维度的交汇关系，支持增量构建和一致性维度管理。
7. **三条并行开发主线**：数据（数据管理团队）、技术（企业架构）、BI工具（分析团队）需协同推进。
8. **"以终为始"是核心原则**：以业务优先级和最终交付范围引领构建，先建立最细粒度原子数据再逐层聚合。

## 重要概念速查

| 概念 | 定义 |
|------|------|
| EDW | 企业级数据仓库，集中式，遵循企业数据模型 |
| OLAP | 在线分析处理，为多维分析提供快速查询 |
| ROLAP | 关系型OLAP，通过RDBMS实现多维技术 |
| MOLAP | 多维矩阵型OLAP，使用专门多维数据库 |
| HOLAP | 混合型OLAP，ROLAP与MOLAP的组合 |
| ODS | 操作型数据存储，包含当前或近期数据 |
| CDC | 变更数据捕获，识别增量变化数据 |
| 事实表 | 包含业务流程定量数据的表 |
| 维度表 | 存储描述性属性的表 |
| 一致性维度 | 跨数据集市共享的统一维度定义 |
| CIF | 企业信息工厂，Inmon提出的架构 |
| Data Vault | 数据保管库，另一种历史数据存储方法 |

## 方法论精华

**Inmon企业信息工厂（CIF）架构：**
```
应用程序 → 数据暂存区 → 集成与转换层 → 数据仓库（原子级）
                              ↓
                    ODS → 操作型数据集市
                              ↓
                    数据仓库 → 数据集市 → 运营报表
```
CIF特性：面向主题、集成、随时间变化、相对稳定、包含明细和汇总、反映历史变化。

**Kimball多维数据仓库：**
```
业务源系统 → 数据暂存区 → 数据展示区（星型模型） → 数据访问工具
```
关键工具：数据仓库总线矩阵（一致性维度×业务流程）。

**数据生命周期流转：**
源系统 → 暂存区（清洗/集成/增强/标准化） → 数据仓库/ODS → 数据集市/数据立方体 → 报表

**BI工具用户分组：**
- IT开发人员：高级分析工具
- 信息消费者：固定报表+简单交互
- 高级用户/分析师：自助式分析+可视化
- 高管/经理：仪表盘+计分卡
- 外部客户：嵌入式分析+门户

## 实施要点

- **八大指导原则**：聚焦业务目标、以终为始、全局设计局部建设、先细粒后聚合、倡导自助服务、同步建设元数据、合作协同、因需制宜
- **加载策略**：乐观（增加维度记录加载无法关联的事实）、悲观（创建回收区持续跟踪）
- **源到目标映射**：最困难的是确定多系统间数据元素的链接有效性和等效性
- **发布管理**：如每年3次业务发布+1次技术发布，使用MoSCoW管理范围
- **关键成功因素**：高层支持、明确范围、业务深度参与、数据质量管控、端到端元数据集成
- **数据质量认知影响满意度**：但满意度还取决于用户对数据的理解和运维团队的响应速度

## 考试/面试高频考点

1. **Inmon vs Kimball核心区别**：自上而下(3NF) vs 自下而上(星型)、企业级先建 vs 数据集市先建、企业数据模型驱动 vs 一致性维度驱动
2. **星型模型vs雪花型模型**：反范式(查询快) vs 规范化(冗余少)的权衡
3. **OLAP五种操作**：切片、切块、钻取/上卷、旋转——各自的含义
4. **三种OLAP实现**：ROLAP、MOLAP、HOLAP的对比
5. **CDC四种技术对比**：数据库日志、时间戳、触发器、全量——复杂度和效率对比
6. **数据仓库总线矩阵**：是什么？为什么重要？如何支持增量构建？
7. **事实表vs维度表**：定量数据vs描述属性，度量vs上下文
8. **数据集市的作用**：面向特定需求的子集，通过汇总加速分析
9. **EDW的四大特性**（Inmon定义）：面向主题、集成、随时间变化、相对稳定
10. **BI双重含义**：分析方法（理解业务机会）+技术栈（查询/挖掘/可视化工具）
11. **数据仓库治理**：风险管理导向、数据归档管理、沙箱隔离一次性需求
