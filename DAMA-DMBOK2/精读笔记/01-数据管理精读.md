# 第1章 数据管理 — 精读笔记

## 一句话总结

数据管理是对数据相关的规划、制度、程序和实践进行开发、执行和监督的活动，目的是在数据的整个生命周期中交付、控制和保护数据，并提升数据的价值——DAMA车轮图定义了以数据治理为核心的11个知识领域，构成了全书的知识体系框架。

## 核心要点（5-8个关键点）

1. **数据是具有独特属性的资产**：数据在使用时不会被消耗，易于复制和传输，可同时被多人使用，丢失后不易恢复。这些独特性使得数据估值和管理都不同于传统资产。

2. **数据管理是跨职能的**：需要业务角色和技术角色共同承担，涉及设计、技术、分析、语言和战略思维等多种技能，单个团队无法管理组织的全部数据。

3. **数据质量管理是核心**：低质量数据成本占收入的10%~30%（IBM估计美国低质量数据成本3.1万亿美元/年）。所有数据管理职能都会影响数据质量。

4. **元数据是管理数据的基础**：元数据描述组织拥有的数据——是什么、代表什么、来自哪里、如何流动、谁可以使用、质量如何。不管理元数据的组织通常也无法做好数据管理。

5. **数据治理置于DAMA车轮图核心**：因为治理需要在各数据管理职能之间进行平衡和一致性指导。其他10个知识领域围绕治理展开。

6. **数据战略必须源于业务战略**：数据管理的首要驱动力是让组织从数据资产中获得价值，技术应服务于数据需求，而非主导数据决策。

7. **领导者支持是关键成功因素**：CDO的角色不仅是领导项目，更要引导文化转型，助力组织采取更具战略性的数据管理方法。

## 重要概念速查（术语+一句话定义）

| 术语 | 定义 |
|------|------|
| 数据管理 | 对与数据相关的规划、制度、程序和实践进行开发、执行和监督，以交付、控制、保护数据并提升其价值 |
| 数据管理专业人员 | 在数据管理任意阶段或方面推动组织战略性目标达成的人员 |
| DIKW金字塔 | 数据(Data)→信息(Information)→知识(Knowledge)→智慧(Wisdom)的分层关系 |
| 元数据 | 用于管理和指导数据使用的数据，记录数据的定义、来源、流向、质量等上下文信息 |
| 数据血缘 | 数据从起点流动到使用点的路径，也称数据链 |
| ROT数据 | 冗余的(Redundant)、过时的(Obsolete)、不重要的(Trivial)数据 |
| 元资产 | 数据是组织描述其他资产的资产，是组织认识自身的途径 |
| CDO | 首席数据官，引领数据管理规划和文化转型 |
| 数据管理专员 | 负责特定数据域的管理、质量和合规的角色 |
| 数据治理委员会(DGC) | 由CDO领导，为数据管理战略提供支持的决策机构 |

## 方法论精华（关键流程和框架）

### DAMA车轮图（核心框架）

数据治理位于中心，外围10个知识领域：数据架构、数据建模和设计、数据存储和运营、数据安全、数据集成和互操作、文档和内容管理、参考数据和主数据、数据仓库和商务智能、元数据管理、数据质量管理。加上第2章数据处理伦理、第14章大数据和数据科学、第15-17章的成熟度/组织/变革管理，构成全书完整体系。

### 战略一致性模型（SAM）

四个基本领域：业务战略、IT战略、组织基础设施与流程、IT基础设施与流程。核心是数据与信息之间的关系——信息与业务战略和数据运营使用关联，数据与IT和支撑流程关联。

### 阿姆斯特丹信息模型（AIM）

九宫格结构：3×3矩阵（业务/信息/技术 × 战略/战术/运营），识别了中间的规划和架构层，强调信息治理和数据质量支柱。

### DMBOK金字塔（Peter Aiken）

四阶段递进：①购买有数据库功能的应用程序→②面临数据质量挑战→③需要治理支持→④利用数据优势提高分析能力。强调大多数组织在未定义好数据管理战略前就开始管理数据，如同构建空中楼阁。

### 语境关系图（基于SIPOC）

每个知识领域遵循统一结构：定义→目标→活动（P/C/D/O四阶段）→输入→成果/交付物→角色与职责→工具→方法→度量指标。活动分为规划(P)、控制(C)、开发(D)、运营(O)四个阶段。

### 数据生命周期

创建或获取→移动→转换→存储→维护→共享→使用→处置。关键影响：①创建和使用是关键要点；②质量管理贯穿全生命周期；③元数据质量管理贯穿全生命周期；④安全管理贯穿全生命周期；⑤应聚焦于关键数据。

### 数据管理战略10大组成部分

令人信服的愿景、业务场景概述、指导原则和价值观、使命和长期目标、成功衡量标准、SMART短期目标、角色和组织描述、项目组成部分描述、优先工作任务、实施路线草案。

## 实施要点（落地时的关键注意事项）

1. **数据估值是基础**：开发一致的方法评估数据价值，从获取存储成本、更换成本、影响成本、风险成本、改进成本（成本侧）和质量收益、竞争价值、销售价值、创新收入（收益侧）两方面建立评估体系。

2. **从企业级视角出发**：数据是组织的"横向"要素，应跨越"纵向"部门流动。避免将数据管理局限于眼前需求，跨部门对齐是数据治理的核心价值。

3. **平衡长期目标和短期目标**：常见的组织压力及时间和成本费用压力会阻碍更好的规划，必须权衡清晰才能做出更好的决策。

4. **数据需求驱动技术决策**：防止技术的诱惑影响数据决策，与业务战略相一致的数据需求应该推动技术决策。

5. **管理ROT数据**：组织产生了大量数据，很大一部分从未被使用。试图管理每一条数据是不可能的，应将ROT数据降至最低。

6. **建立CDO角色**：CDO需要引领数据管理规划、促使组织利用数据资产获得竞争优势，并引导文化转型。
