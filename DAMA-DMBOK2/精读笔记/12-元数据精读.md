# 第12章 元数据管理 — 精读笔记

## 一句话总结

元数据管理是规划、实施和控制"关于数据的数据"的全生命周期活动，它帮助组织理解自身数据、系统和工作流，是实现数据驱动组织的前提——要成为数据驱动型组织，就必须先实现元数据驱动。


> **关键词**：Metadata, 元数据, 数据目录, Data Catalog, 数据血缘, Data Lineage, 业务术语表
## 核心要点（5-8个）

1. **元数据分三大类型**：业务元数据（定义、规则、血缘、安全级别）、技术元数据（表名、字段属性、ETL作业、映射）、操作元数据（执行日志、错误日志、SLA、使用模式）。分类依据是生成方式而非用途。
2. **四种元数据架构**：集中式（单一存储库，高可用但同步复杂）、分布式（实时检索，始终最新但不支持全局搜索）、混合式（关键数据集中+其余实时检索，最灵活）、双向（变更可同步回源，反馈闭环但冲突处理复杂）。
3. **元数据是数据管理的基石**：如同图书馆的目录卡片，没有元数据组织就不知道拥有什么数据、数据含义、来源、流转路径、谁有权访问。
4. **业务术语表是核心工具**：记录业务概念、术语体系、定义及关系，需满足业务用户、数据管理专员、技术用户三类角色需求。
5. **数据血缘是关键应用**：分为实现态血缘（基于代码）和设计态血缘（基于映射规范），支持影响分析、问题定位和合规审计。"拼接"各段血缘可呈现端到端视图。
6. **元数据应作为产品管理**：高质量元数据是认真规划的结果，需责任到人、制定标准、建立反馈机制。流程执行者对元数据质量负责。
7. **ISO/IEC 11179是核心标准**：六部分框架覆盖数据元素的生成标准化、分类、基本属性、定义规则、命名识别、注册。
8. **八大指导原则**：组织承诺、战略规划、企业视角、宣导推广、访问便利、保证质量、审计标准、持续改进。

## 重要概念速查

| 概念 | 定义 |
|------|------|
| 业务元数据 | 关注数据内容和状态：定义、规则、血缘、安全级别 |
| 技术元数据 | 提供技术细节：表名、字段属性、ETL作业、映射 |
| 操作元数据 | 描述处理和访问细节：日志、SLA、使用模式 |
| 元数据库 | 元数据的主要管理工具，含集成层和手动更新界面 |
| 数据血缘 | 数据从源到目标的完整流转路径 |
| 影响分析 | 评估潜在变更可能影响的组件 |
| 业务术语表 | 记录业务概念、术语定义及关系的系统 |
| ISO/IEC 11179 | 元数据注册国际标准（六部分） |
| 元模型 | 元数据库的数据模型 |
| 拼接(Stitching) | 将血缘各部分连接成端到端视图 |

## 方法论精华

**元数据战略制定五步法：**
1. 启动战略规划（章程、范围、目标、沟通计划）
2. 关键利益相关方访谈（获取业务和技术基础知识）
3. 评估现有元数据来源和架构（确定问题解决难度）
4. 制定未来元数据架构（组织、治理、技术、安全）
5. 制定分阶段实施计划（验证、集成、优先级，发布路线图）

**元数据生命周期：**创建和采集 → 存储 → 集成 → 交付 → 使用 → 控制和管理

**元数据需求十一个维度：**内容、更新频率、同步情况、历史信息、访问权限、存储结构、集成要求、维护要求、管理要求、质量要求、安全要求。

**数据血缘两种分析焦点：**
- 业务焦点：从目标追溯到源系统（适合业务用户）
- 技术焦点：从源系统识别所有消费者（适合技术用户）

**元数据集成两种提取方式：**
- 专用接口：单步，直接从源到存储库
- 半专用接口：两步，先输出到文件再加载

## 实施要点

- **分步实施降低风险**：首次实施通常是验证概念的试点项目
- **就绪评估三大风险**：数据误解风险、敏感数据泄露风险、知识流失风险
- **集中式架构适用场景**：追求高度一致性的组织，支持全局搜索
- **混合式架构适用场景**：快速变化的操作元数据、大幅增长的元数据源
- **组织文化阻力是主要障碍**：从未管理到受管理环境需要大量工作和纪律
- **关键成功因素**：高管支持、跨职能合作、优秀案例示范、具体业务案例论证
- **非结构化数据元数据**：入湖对象应收集基本元数据（名称、格式、来源、版本、接收日期）

## 考试/面试高频考点

1. **元数据三大类型**：业务（内容和状态）、技术（技术细节）、操作（处理和访问）——各自的典型内容
2. **四种元数据架构对比**：集中式、分布式、混合式、双向——优缺点和适用场景
3. **"要成为数据驱动型组织，就必须先实现元数据驱动"**：理解其深层含义
4. **元数据应作为产品管理**：责任到人、制定标准、反馈机制
5. **数据血缘的两种类型**：实现态（基于代码）vs 设计态（基于映射规范）
6. **影响分析vs血缘追溯**：影响分析向前看（变更影响什么），血缘向后看（数据从哪来）
7. **ISO/IEC 11179**：元数据注册标准的六部分内容
8. **业务术语表三类用户**：业务用户、数据管理专员、技术用户
9. **元数据战略五步法**：启动→访谈→评估→制定架构→分阶段实施
10. **非结构化数据的元数据**：描述元数据、结构元数据、管理元数据、书目元数据等
11. **元数据治理角色**：元数据管理员、数据架构师、数据管理专员
