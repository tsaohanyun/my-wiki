## 概述

数据管理成熟度评估（Data Management Maturity Assessment, DMMA）是一种基于框架的流程改进方法，其核心是**能力成熟度模型（Capability Maturity Model, CMM）**——该模型描述了流程特征从初始状态到最佳状态的演变过程。这一概念源于20世纪80年代中期美国国防部为评估软件承包商而制定的标准，由卡内基梅隆大学软件工程研究所（SEI）发布。最初应用于软件开发领域，现已广泛推广到包括数据管理在内的诸多领域。

数据管理成熟度评估既可用于全面评估数据管理情况，也可聚焦于单个知识领域或特定流程。无论聚焦点范围如何，DMMA 都能帮助检查业务和 IT 在数据管理实践中的健康状况与有效性，并弥合差距。它为描述数据管理知识领域的进阶提供了共同语言，并提供了基于阶段的改进路径，该路径可根据组织的战略优先事项进行定制。因此，它既可以用来设定和衡量组织目标，也可以用于将自己的组织与其他组织或行业基准进行比较。

**数据管理成熟度评估定义：** 一种用于根据成熟度水平对处理数据的做法进行排名的方法，被组织用来描述当前数据管理的状态和改进机会。

### 目标

1. 评估关键数据管理活动的当前状态，以规划改进。
2. 支持组织的运营和战略方向。
3. 制定支持整体组织战略的数据统一愿景。
4. 实现改进的集成计划。
5. 设定和衡量组织目标，并与外部组织或行业基准进行比较。

### 业务驱动因素

| 驱动因素 | 说明 |
|---------|------|
| **监管** | 监管机构对组织的数据管理成熟度提出了保底的水平要求 |
| **数据治理** | 出于规划与合规性的目的，数据治理需要进行成熟度评估 |
| **流程改进前准备** | 组织意识到需要改进数据管理实践，从评估现状着手（如评估 MDM 准备程度） |
| **组织变革** | 合并等组织变革带来数据管理挑战，评估为应对提供规划依据 |
| **新技术** | 技术进步为管理和使用数据提供新方法，需评估成功采用新技术的可能性 |
| **数据管理问题** | 解决数据质量问题或面临其他挑战时，需明确当前状态基准以决定变革路径 |

### 原则

数据管理能力评估的主要目标是评估关键数据管理活动的当前状态，以便制订改进计划。通过明确组织的具体优势和不足，评估将组织置于成熟度等级的度量体系中。在实现其主要目标的过程中，DMMA 会对组织文化产生积极影响：

1. 向利益相关方普及数据管理概念、原则和实践方面的知识。
2. 厘清利益相关方在组织内数据相关方面的角色和责任。
3. 强调将数据作为关键资产进行管理的必要性。
4. 在全组织范围内增强对数据管理活动的认识。
5. 促进有效数据治理所需的协作关系。

## 核心概念

### 成熟度模型定义

成熟度模型通过描述流程特性的等级来定义成熟度级别。当组织对流程特性有了深入理解后，就可以评估自身的成熟度水平，并制订计划来提升相应能力。在模型等级的指导下，它可以衡量自身的进步幅度，并与竞争对手或合作伙伴进行比较。随着每个新等级的出现，流程的执行将变得更加一致、更可预测和更可靠。能力提升需要遵照事先设定好的顺序逐级开展，**不能跳过任何等级**。

### 标准成熟度等级

| 级别 | 名称 | 特征描述 |
|------|------|----------|
| 第0级 | **无能力级** | 管理活动或企业正式流程处于无组织状态；实际中很少有组织处于此级别 |
| 第1级 | **初始/临时级** | 成功取决于个人能力；使用有限工具集；很少或没有治理活动；高度依赖少数专家 |
| 第2级 | **可重复级** | 已制定基本流程规则；出现一致的工具和角色定义；角色得到定义，流程不再完全依赖特定专家 |
| 第3级 | **已定义级** | 制定和使用标准；引入可扩展的数据管理过程并实现制度化；数据视为组织发展的推动力量 |
| 第4级 | **已管理级** | 流程得到量化和控制；能预测新项目和任务的结果；开始管理与数据相关的风险；包含性能指标 |
| 第5级 | **优化级** | 过程改进目标能被量化；流程因自动化和技术变更管理而高度可预测；关注持续改进 |

### 各等级详细特征

#### 第0级：无能力级
- 数据管理活动或企业正式流程处于无组织状态。
- 该级别是理论上的起点，实际中很少有组织处于此级别。

#### 第1级：初始/临时级
- 使用有限工具集进行通用数据管理，很少或没有治理活动。
- 数据处理高度依赖少数专家，角色和职责在各部门内部分开定义。
- 数据所有者自主接收、生成和发送数据；控制措施实施不一致。
- 数据管理解决方案有限；数据质量问题普遍存在且尚未解决。
- 基础设施支持处于业务单元层面。
- **评估标准示例：** 是否存在对任意流程的控制，如记录数据质量问题。

#### 第2级：可重复级
- 出现一致的工具和角色定义支持流程执行。
- 组织开始使用集中化工具并为数据管理提供更多监管手段。
- 角色得到定义，流程不再完全依赖特定专家。
- 对数据质量问题和概念有了一定认识；主数据和参考数据概念开始得到认可。
- **评估标准示例：** 是否对组件中正式角色有了定义（如职位描述、流程文档、工具集能力）。

#### 第3级：已定义级
- 数据管理能力显现化；引入可扩展的数据管理过程并实现制度化。
- 数据管理被视为组织发展的推动力量。
- 在整个组织内部复制数据（同时实施控制措施）；全面提升数据质量。
- 协调一致的制度定义和管理；更正式的流程定义减少大量人工干预。
- 过程结果更加可预测。
- **评估标准示例：** 是否制定数据管理制度、可扩展流程的使用情况、数据模型和系统控制的一致性。

#### 第4级：已管理级
- 从第1~3级的成长中积累组织知识，能够预测新项目和任务的结果。
- 开始管理与数据相关的风险；数据管理包含性能指标。
- 从桌面到基础设施的数据管理工具实现标准化；完善的集中化规划和治理功能。
- 数据质量和全组织数据管理能力显著提高（如端到端数据审计）。
- **评估标准示例：** 与项目成功相关的指标、系统的操作指标、数据质量指标。

#### 第5级：优化级
- 数据管理实践得到优化，流程因自动化和技术变更管理而高度可预测。
- 关注持续改进；工具使数据能够在整个流程中得到可视化。
- 控制数据增长以防止不必要的重复；使用明确理解的指标来管理和测量数据质量与流程。
- **评估标准示例：** 变更管理组件和流程改进中的度量指标。

### 评估标准

每个能力级别都有与正在评估的流程相关的特定评估标准。例如，评估数据建模功能成熟度时：

- 第1级：是否有数据建模实践活动？涉及多少个系统？
- 第2级：是否已定义企业级数据建模方法？
- 第3级：企业级数据建模方法的实施推广程度如何？
- 第4级：建模标准是否得到充分有效执行？
- 第5级：是否有流程来改进建模实践？

无论哪个级别，评估标准都按同一尺度进行评估，以显示该级别内的进展情况和向下一个级别发展的趋势。评估结果可组合或可视化展示，以便了解当前状态与目标状态之间的差异。

#### 评分体系

评估采用统一的四级评分量表：

| 评分 | 等级 | 含义 |
|------|------|------|
| **1** | 未开始 | 该能力尚未启动，无相关实践或仅有个别尝试 |
| **2** | 进行中 | 能力正在建设中，存在初步实践但不完善或不一致 |
| **3** | 能使用 | 能力已建立并在组织范围内得到应用，具备基本一致性 |
| **4** | 有效 | 能力成熟且高效运行，能产生预期的业务成果，持续优化 |

每个评估主题在每个成熟度等级上都按此四级评分，综合得分可映射到整体成熟度等级（第0~5级）。评分结果可通过雷达图等可视化方式呈现（见图15-3），其中外环表示组织在竞争中取得成功所需的能力水平，内环表示通过评估所确定的实际能力水平，两个环之间距离最大的区域代表组织面临的最大风险。此类报告帮助组织设定优先级，也可用来衡量一段时间内的进展情况。

根据语境关系图中的类别，标准可划分为四大维度：

| 评估维度 | 说明 |
|---------|------|
| **活动** | 该活动或流程的实施程度如何？是否定义了有效和高效执行的标准？是否产生最佳实践输出？ |
| **工具** | 该活动在多大程度上实现了自动化并受到通用工具支持？工具培训、按需使用、优化配置及长期技术规划情况？ |
| **标准** | 该活动在多大程度上受到通用标准的支持？记录完善程度？是否由治理活动和变更管理强制执行？ |
| **人员和资源** | 组织是否配备足够的人员？需要哪些特定技能、培训和知识？角色和职责是否界定清晰？ |

### 主要成熟度模型详细介绍

#### CMMI数据管理成熟度模型（CMMI-DMM）

由能力成熟度模型研究院（CMMI Institute）开发，为以下数据管理领域提供评估标准：

1. **数据管理策略**
2. **数据治理**
3. **数据质量**
4. **平台与架构**
5. **数据运营**
6. **支持服务流程**

在每个流程中，模型识别待评估的子流程。例如，"数据质量"部分考虑"数据质量策略"和"数据质量评估、分析和清洗"。模型还考虑了数据管理各流程之间的关系，如利益相关方的一致性需要，以及业务流程与数据质量管理之间的关系。

#### DCAM（数据管理能力评估模型）

由企业数据管理委员会（EDM Council）——一家总部位于美国的金融服务行业倡导组织——开发。该模型描述了与建立可持续数据管理职能相关的37项能力和115项子能力，评分重点关注利益相关方的参与程度、流程的正式性及证明能力实现的组件存在情况。

#### IBM数据治理委员会成熟度模型

综合了55个组织组成的委员会的意见制定。委员会成员协作定义了一组通用的可观察和可预测的共同标准，围绕4个关键类别组织：

| 类别 | 包含内容 |
|------|---------|
| **成果** | 数据风险管理和合规性、价值创造 |
| **推动因素** | 组织架构和意识、策略、管理 |
| **核心域** | 数据质量管理、信息生命周期管理、信息安全与隐私 |
| **支持域** | 数据架构、分类和元数据、审计信息、日志和报告 |

该模型既以成熟度框架的形式呈现，也以一组评估问题的形式呈现，其答案指示组织的成熟度级别。

#### 斯坦福数据治理成熟度模型

斯坦福大学为本校使用而开发，重点在于数据治理而非数据管理。分为基础部分（意识、正规化、元数据）和项目组件部分（数据管理、数据质量、主数据），各部分说明人员、策略和能力的驱动因素，阐明每个成熟度级别的特征，并为每个级别提供定性和定量测量。

#### Gartner企业信息管理成熟度模型（EIM）

建立了评估愿景、战略、指标体系、治理、角色和职责、生命周期和基础设施的标准。

## 方法论

### 评估实施总体流程

数据管理成熟度评估需要有计划地进行。评估应在短暂而明确定义的时间范围内进行，评估的目的是揭示当前的优势和改进机会，而不是解决问题。

| 阶段 | 活动 | 说明 |
|------|------|------|
| **1. 规划评估** | 定义目标 | 明确评估的驱动因素和目标，获得高管和业务部门的理解与支持 |
| | 选择框架 | 基于当前状态和评估目标审查可选框架，选择能提供有意义指导的模型 |
| | 定义组织范围 | 确定评估范围（局部式 vs 企业级），首次评估通常选定可控范围 |
| | 定义交互方法 | 确定信息收集方式（研讨会、访谈、调查、组件审查） |
| | 沟通规划 | 制定沟通计划，确保利益相关方了解评估目的、过程及期望 |
| **2. 执行评估** | 收集信息 | 收集访谈记录、系统文档、数据调查、策略手册、元数据等材料 |
| | 执行评估与评级 | 多阶段评估，参与者对标准评分，通过讨论达成一致意见，以证据支持 |
| **3. 结果解读** | 报告评估结果 | 编制包含驱动因素、结论、评分差距、建议的评估报告 |
| | 制定管理层简报 | 为管理层准备总结优势、差距和建议的执行简报 |
| **4. 制订改进计划** | 确定行动与路线图 | 制定多年度数据管理改进计划，包含有序活动序列、时间表、预期改善 |
| **5. 重新评估** | 周期性再评估 | 定期重新评估，跟踪趋势，作为持续改进循环的一部分 |

### 评估方法选择

在选择 DMM 框架时，应考虑以下标准：

| 标准 | 说明 |
|------|------|
| **可访问性** | 实践应以非技术术语表述，传达活动的功能本质 |
| **全面性** | 框架涵盖广泛的数据管理活动，包括业务参与而不仅是 IT 流程 |
| **可扩展性和灵活性** | 结构可增加行业特定或附加领域，可根据需求全部应用或部分裁剪 |
| **内建演进路线** | 描述每个功能的逻辑演进方向 |
| **行业通用 vs 特定** | 某些组织受益于行业特定方法，另一些则受益于通用框架 |
| **抽象或详细程度** | 实践和评估标准表达详细，确保与组织工作可关联 |
| **非指导性** | 描述需要执行的内容，而非指导必须如何执行 |
| **按主题组织** | 将数据管理活动置于适当场景，支持单独评估并可识别依赖关系 |
| **可重复性** | 框架得到一致解释，支持可重复结果，便于组织间对比和趋势跟踪 |
| **中立性** | 由中立独立组织支持，供应商选择中立，避免利益冲突 |
| **技术中立性** | 重点应是实践而非工具 |
| **培训支持** | 有全面培训支持，专业人员能掌握并优化框架使用 |

### DAMA-DMBOK框架的使用

DAMA-DMBOK 可用于为 DMMA 准备或制定标准。执行负责人将看到分段职能（知识领域）和相应任务（活动）之间的直接联系。根据测量的领域、支持活动、相关性和可用时间，可以将 DMBOK 知识领域、活动和可交付成果（工作产品）配置到特定的 DMM 框架中。这种快速检查表法可用于确定需要更深入分析的区域、表示差距或指出需要补救的热点。DMBOK 还提供一个额外优势——有一个庞大的专业知识人士群体将 DMBOK 作为跨多个行业的指南，并围绕其使用创建了实践社区。

### 评估范围选择

大多数 DMM 框架设计旨在适用于整个组织，但首次评估通常最好选定一个可控的范围。

**局部式评估 vs 企业级评估：**

| 维度 | 局部式评估 | 企业级评估 |
|------|-----------|-----------|
| **范围** | 单个业务领域或项目 | 整个组织或多个职能单元 |
| **深度** | 可更深入了解细节 | 关注广泛存在但分散的部分 |
| **完成速度** | 更快 | 较慢 |
| **典型选择** | 受严格监管的职能单元（如财务报告） | 研发、制造和融资等多单元 |
| **输入/角色/工具** | 可能超出被评估功能范围 | 通常是跨企业且多层次的 |
| **结论应用** | 可汇总和加权形成企业级评估 | 可直接用于组织层面决策 |

### 评估交互方法

实施 DMMA 时，信息收集活动可能包括：

- **研讨会**：集体讨论和评分
- **访谈**：深入获取具体领域信息
- **调查**：广泛收集各方意见
- **组件审查**：检查实际证据

应采用符合组织文化的方法，最大限度减少参与者的时间投入，使评估快速完成。在所有情况下，都需要通过让参与者对标准进行评分来正式确定评估结果。许多情况下还包括对组件和其他证据的实际检查和评估。

### 评估优化流程

评估评级一般按如下方式进行：
1. 根据评级方法审查结果，为每个工作产品或活动给定初步评级。
2. 记录支持证据。
3. 与参与者一起审查，就每个领域的最终评级达成共识（根据重要性使用不同权重）。
4. 使用模型标准陈述和评估人员评论作为评级的说明。
5. 开发可视化工具，更直观地展示评估结果。

### 沟通规划

沟通有助于成熟度评估的整体成功。沟通将面向评估参与者和其他利益相关方。在评估开始之前，应告知利益相关方：

1. 数据管理成熟度评估的目的。
2. 评估应如何进行。
3. 对方参与的是哪个环节。
4. 评估活动的时间表。

在评估活动期间，应确保有明确的议程，包括解答后续问题的计划，不断提醒参与者评估活动的目标和目的，对参与者持续表达感谢并描述下一步计划。

## 组织与角色

### 成熟度管理治理

DMMA 是一系列数据治理活动的重要组成部分，这些活动都有各自的生命周期。数据管理成熟度评估的生命周期包括：初始计划和初始评估 → 建议 → 行动计划 → 定期重新评估。生命周期本身也应受到管理。

### DMMA流程监督

DMMA 的流程监管职责属于**数据治理团队**。如果组织尚未建立正式的数据治理机制，监管工作将默认由发起 DMMA 的指导委员会或管理层负责。

| 角色/机构 | 职责 |
|-----------|------|
| **执行发起人**（理想为 CDO） | 确保数据管理活动的改进直接与业务目标对接 |
| **数据治理团队** | 流程监管，确保评估方法和结果的规范性 |
| **指导委员会/管理层** | 未建立正式治理机制时的默认监管方 |
| **各职能部门** | 在整体评估的执行、方法、结果和路线图中拥有发言权 |
| **利益相关方** | 提供输入、参与评分、审查结果、推动行动计划 |

### 度量指标

度量指标是所有改进策略的核心组成部分，也是一种关键的沟通工具。

| 指标类别 | 说明 |
|---------|------|
| **DMMA评级** | 反映组织能力水平的概况，可附有自定义权重和推荐目标状态 |
| **资源利用率** | 以人员数量形式表达数据管理成本（如"每人花费10%时间手动汇总数据"） |
| **风险敞口** | 应对风险场景的能力与 DMMA 评级的对应关系 |
| **支出管理** | 数据管理成本分摊方式及其对可持续性和价值的影响 |
| **DMMA输入** | 覆盖范围、调查水平和范围细节的完整性度量（计数、系统数量、团队等） |
| **变革速度** | 组织提升自身能力的速率，基于基线和定期重新评估追踪趋势 |

其他相关指标包括：
- 数据管理的可持续性
- 实现倡议的目标和宗旨
- 沟通的有效性
- 教育和培训的有效性
- 采纳变革的速度
- 数据管理的价值
- 对业务目标的贡献度
- 风险降低的情况
- 运营效率的提升情况

## 行业应用

### 自我评估 vs 第三方评估

| 对比维度 | 自我评估 | 第三方评估 |
|---------|---------|-----------|
| **执行方** | 组织内部团队 | 外部独立评估机构 |
| **优势** | 成本较低、内部知识充分、过程可控 | 客观中立、专业性高、跨行业基准参考 |
| **劣势** | 可能存在偏见、缺乏客观性 | 成本较高、需较长时间建立对组织的理解 |
| **适用场景** | 首次评估、内部摸底、日常跟踪 | 正式认证、行业对标、监管需要 |

### 定制化注意事项

评估可以根据组织的需求进行调整，但要谨慎修改评估模型。如果评估模型被不当裁剪或修改，可能导致其失去严谨性或可追溯性。因此，在进行自定义评估时，应保持模型的完整性不变。

### 行业特定模型选择

- **金融服务**：DCAM 提供行业特定的最佳实践框架。
- **教育行业**：可参考斯坦福数据治理成熟度模型。
- **通用行业**：CMMI-DMM、IBM 模型、Gartner EIM 更具通用性。
- **需要行业标杆对比**：选择有广泛行业应用的模型（如 CMMI-DMM）。

## 方案价值

### 典型风险与缓解措施

| 风险 | 描述 | 缓解措施 |
|------|------|---------|
| **缺乏高管支持** | 评估无高层推动，难以获得资源和跨部门协作 | 确保有执行发起人（如 CDO）；将评估与业务战略对齐 |
| **范围过大** | 首次评估范围过宽，导致周期过长、结果不聚焦 | 首次评估选择可控范围（单个业务域或项目） |
| **参与者不投入** | 业务和 IT 参与者对评估缺乏兴趣或敷衍了事 | 充分沟通评估目的和收益；最大限度减少时间投入 |
| **评估延迟** | 评估周期过长，利益相关方失去热情和动力 | 设定明确的时间框架；快速完成以保持动力 |
| **结果无法落地** | 评估报告完成后无行动计划或行动得不到执行 | 将评估直接关联到改进路线图；设定可衡量的目标 |
| **模型裁剪不当** | 过度裁剪导致评估失去严谨性和可追溯性 | 保持模型完整性；在必要时才进行有依据的调整 |
| **文化阻力** | 组织对变革存在抵触情绪 | 通过评估结果展示改善空间；引导形成共同愿景 |
| **评级分歧** | 参与者对同一主题评分差异大，无法达成共识 | 通过讨论和证据审查调和不同意见；建立评分共识机制 |

### 投资回报（ROI）

DMMA 为组织带来的价值体现在以下方面：

| 价值维度 | 具体收益 |
|---------|---------|
| **战略对齐** | 使数据管理活动与业务战略方向保持一致 |
| **问题识别** | 明确数据管理中的优势、劣势和差距，确定高价值改进机会 |
| **优先级排序** | 帮助管理层确定资源分配的优先级 |
| **共同语言** | 为跨部门沟通提供统一的数据管理成熟度语言 |
| **路线规划** | 提供基于成熟度等级的改进路径和里程碑 |
| **进度追踪** | 通过定期重新评估量化改进成果 |
| **风险管理** | 识别与数据管理能力不足相关的业务风险 |
| **文化建设** | 增强全员数据管理意识，促进数据治理文化形成 |
| **行业对标** | 与竞争对手或行业基准进行比较，明确竞争定位 |
| **合规保障** | 满足监管机构对数据管理成熟度的最低要求 |

### 组织与文化变革

建立或增强数据管理职能包括对流程、方法和工具的改进。伴随着这些变化，文化也必须随之发生变化。组织和文化变革的起点是承认事情可以变得更好。度量功能通常能推动有意义的变革。DMMA 根据成熟度等级对组织进行定位，并提供改进路线图。实施 DMMA 的成果应成为组织内更大范围讨论的一部分。在有效的数据治理支持下，DMMA 的结果可以整合不同的观点，形成共同愿景，并加速组织的进步。

| 工具 | 说明 |
|------|------|
