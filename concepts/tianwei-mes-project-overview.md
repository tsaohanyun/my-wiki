# 天味家园B栋MES项目概览

## 项目概况

### 项目背景

面对食品行业数字化智能化发展的变化与挑战，天味家园通过数据化运营平台（[[mes]]）的实施，引入先进的管理理念和模式，打造食品生产执行层中的关键环节。通过建立一体化平台，可实现对生产数据的实时采集、展示及MES的接入，实现数据的分析利用及基于数据决策运营，解决信息孤岛效应，促进天味的生产[[digital-transformation|数字化转型]]。

本项目全称为"老车间工艺和物料防呆防错信息化优化项目"，部署于天味家园B栋食品加工车间，覆盖火锅底料从原料预处理到成品包装的完整生产流程。

### 项目目标

- **工艺防呆防错**：通过系统指导操作配料、校验配料数量，确保投料顺序及工艺参数符合工艺要求，减少人为失误
- **物料追溯与管控**：实现从原料投料到成品产出的全流程物料追溯，自动记录投料、报工数据
- **生产数据实时采集**：对炒制过程中的投料顺序及工艺参数进行实时控制和记录
- **账务数据集成**：与SAP系统对接，实现生产数据的账务回传与同步

### 项目范围

项目覆盖B栋老车间的核心生产工段，包括：

- 大料预处理（姜、蒜、辣椒等原材料的切碎、蒸煮）
- 大料配料（按工艺要求分装到不同容器）
- 小料预处理（小料投料、半成品报工与线边出入库）
- 炒料炒制（按工艺单要求顺序投料炒制）
- 内外包装（辅料消耗记录及成品产出）
- 返工品管理（拆解、重新投料炒制）
- 车间备料与物料流转

系统同时与WMS（仓储管理系统）和SAP进行接口集成，实现备料需求下达、物料退料、外包装工单报工等12个接口的互联互通。

## 核心功能清单

以下为项目13个核心功能模块总览：

| 序号 | 功能项名称 | 功能描述 | 关键性 |
|------|-----------|----------|--------|
| 1 | 工单管理 | 根据生产计划生成各工段工单 | 高 |
| 2 | 大料预处理 | 姜、蒜、辣椒等原材料进行切碎、蒸煮等加工处理的记录 | 高 |
| 3 | 大料配料 | 将原料（如盐、糖）及经过预处理生产出的半成品（如姜粒、蒜粒）等按照火锅底料炒制的工艺要求，分装到不同大小的容器中 | 高 |
| 4 | 小料预处理 | 按生产执行单自动归集小料各项原材料投料、半成品报工数据并自动触发线边出入库 | 高 |
| 6 | 炒料炒制 | 根据工艺单要求的顺序，将各种物料倒入炒锅中进行炒制，系统支持对炒制过程中的投料顺序及工艺参数进行控制 | 高 |
| 7 | 工艺配方单 | 每一个炒制步骤需要投放的物料及炒锅工艺参数的要求，用于指导车间按工艺要求生产 | 中 |
| 8 | 内外包装 | 对辅料的消耗进行记录，以统计生产成本，同时记录内包装半成品及外包装成品的产出 | 中 |
| 9 | 返工品拆解 | 返工原材料拆解为各半成品，完成之后产生MOM端生产执行单、投料记录、报工记录 | 中 |
| 10 | 车间备料 | 根据工单的物料需求，并结合当前车间的库存，计算出需要准备多少物料 | 中 |
| 11 | 返工品投料配置 | 不合格的内包半成品进行拆包后重新投入炒锅进行炒制，配置每一种炒制半成品允许投放哪些半成品及各种半成品的上限 | 低 |
| 12 | 委外物料配置 | 计划人员在将车间的成品订单拆分出半成品生产工单时，将这些半成品的物料区分开来，委外其它车间生产 | 低 |
| 13 | 工序产品配置 | 每个工序配置生产产品所属的物料类别，为将"生成订单"自动拆分成"生产工单"提供基础数据配置 | 低 |

> **注**：序号5在原始文档中未出现，可能为编号跳跃或合并到其他模块。

## 项目交付内容

### 硬件交付

项目部署采用超融合架构，服务器硬件资源需求如下：

| 类别 | 规格描述 |
|------|----------|
| CPU | 物理CPU 128C，超线程（双线程），共256线程 |
| 内存 | 1024G RAM |
| 存储 | 约48TB磁盘存储（超融合双副本） |
| 网络 | 生产环境网段（预留60个IP）、测试环境网段（预留40个IP） |
| 安全设备 | VPN服务、核心交换机 |
| 终端设备 | PDA手持终端、电子秤（接口联调） |

硬件高可靠要求涵盖：硬件健康检测、虚拟机高可用（HA）、虚拟机热迁移、数据多副本保护、网络连通性探测等。

### 软件交付

| 类别 | 内容 |
|------|------|
| MES核心平台 | 13个功能模块（见上方核心功能清单） |
| WMS接口集成 | 12个双向接口（备料、退料、报工、返工等） |
| SAP集成 | 与PCE版SAP对接，实现账务数据回传 |
| 应用部署 | 基于K8s容器化部署，镜像仓库（Harbor）管理 |
| 大数据平台 | 数据分析与可视化 |
| 运维支持 | VPN安全访问、监控告警、日志管理 |

### 系统集成架构

系统采用多层架构部署：

1. **网络接入层**：内网访问，通过核心交换机路由
2. **安全层**：VPN + 实施服务器访问控制
3. **应用层**：MES应用集群，基于容器化部署
4. **数据层**：中间件与数据存储、大数据集群
5. **外部集成层**：与WMS、SAP系统的接口互联
