# 详细设计文档编写规范

> 详细设计是蓝图方案到系统开发之间的核心交付物。本文基于智能工厂生产业务平台8大模块（计划/配方/生产批组/质量/仓储/生产/设备/精益）的详细设计实践，提炼通用编写规范。关联：[[feed-industry-solution]] [[smart-factory-planning]]

## 文档定位

详细设计文档承上启下：
- **上承蓝图方案**：将蓝图中的业务流程细化为可开发的功能规格
- **下接开发实现**：开发团队依据详细设计编码，测试团队据此编写用例

与[[blueprint-writing-methodology]]的区别：蓝图侧重业务流程和功能清单，详细设计侧重每个功能的字段级规格和交互逻辑。

## 文档整体结构

```
1 概述
  1.1 项目背景
  1.2 项目目标
2 整体架构
  2.1 整体业务流程
  2.2 整体应用架构
  2.3 整体接口架构
3~N 详细设计方案-{模块名}
  N.1 业务流程总图
  N.2 业务功能清单
  N.3 详细功能设计
    N.3.x 功能名称
附件-备忘录
```

## 功能描述标准模板

每个功能模块的详细设计必须包含7个部分：

### 1. 功能描述

```
简要说明：<一句话功能用途>
前置条件：<使用前必须满足的条件，含基础数据准备清单>
后置条件：<执行后的结果/产出>
操作流程：<关键步骤箭头连接>
```

**前置条件**尤为重要——它不仅说明依赖关系，还暗示了基础数据维护的范围。如标准化点检的前置条件列出了审核类型、角色管理、点检项目、点检区域等10项基础数据。

### 2. 功能逻辑

按按钮/操作逐条描述，格式：

```
【按钮名】：
操作方式：点击按钮>打开弹窗 / 点击按钮 / ...
逻辑说明：触发行为、涉及字段和规则
```

常见操作类型及标准描述：

| 操作 | 标准描述 |
|------|----------|
| 新增 | 点击按钮>打开弹窗/跳转页面，填写相关信息 |
| 编辑 | 对已有数据进行修改 |
| 删除 | 点击删除指定数据 |
| 保存 | 暂存当前数据，不改变状态 |
| 确认/提交 | 完成操作，状态变更 |
| 下发/下达 | 将计划/任务下发至执行层 |
| 审批 | 通过/驳回/转办，状态流转 |
| 导出 | 导出选中行明细数据 |

### 3. 页面流转说明

描述页面间的跳转关系：

```
点击【按钮A】，打开页面X。
页面X中，点击【按钮B】，打开页面Y。
```

### 4. 业务规则

功能特有的业务约束和计算规则。无特殊规则时写"无"。

### 5. 页面原型

列出所有原型图编号和名称，编号格式：`图{章节号}-{序号}`

### 6. 页面字段定义（三区模式）

每个页面按**查询区→功能区→表格区**三个区域定义：

**查询区**：筛选条件，含输入方式（下拉/模糊查询/日期范围/文本框）和查询逻辑

**功能区**：操作按钮，可分主功能区和子功能区（明细子表操作）

**表格区**：数据展示，每个字段需说明4个属性：

| 属性 | 说明 |
|------|------|
| 字段 | 字段名称 |
| 字段逻辑 | 值的来源和计算逻辑 |
| 是否必填 | 是/否 |
| 是否可编辑 | 是/否 |

表格区可分主表和子表。自动生成的字段（如单号）标记为"必填是/可编辑否"。

### 7. 权限控制与接口需求

权限按角色×操作矩阵定义（查询/新增/删除/编辑/提交）。

接口需说明：编号、名称、数据方向（上行/下行）、说明、交互方式（同步/异步）。

## 编码规则

常见单号编码格式：`前缀_+YYMMDD+4位流水号`

示例：ZBJH_2411010001（标准化计划，2024年11月1日第1号）

## 审批流模式

审批操作统一为：通过、驳回、终止、转办

两种审批维度：
- **公司级**：公司内部审批链（日/周/月稽查）
- **总部级**：片区→总部审批链（片区稽核→总部督导）

复杂审批流示例（合理化建议）：
```
提议 → 提议审批 → 实施反馈 → 标准化专员确认 → 部门经理复评
→ 总经理评估 → 片区推广评估 → 总部推广评估
```

## 定时任务设计

周期性任务参数模型：

| 参数 | 说明 | 取值 |
|------|------|------|
| 执行周期 | 任务频率 | 每天/每周/每月/每年 |
| 执行频次 | 周期内次数 | 1, 2, ... |
| 下次执行时间 | 定时触发时间 | 精确到小时 |

自动更新逻辑：定时任务到达「下次执行时间」→ 生成任务数据 → 根据周期更新「下次执行时间」

## 多终端设计

| 终端 | 特点 | 典型功能 |
|------|------|----------|
| PC端 | 完整功能，复杂表单 | 计划制定、积分管理、指标配置 |
| APP端 | 轻量操作，拍照上传 | 点赞、积分查看、任务执行 |
| PAD端 | 车间现场作业 | 原料入库、垛口任务 |
| CS端 | 设备直连操作 | 投料管理、制粒膨化管理 |

## 精益管理模块参考

智能工厂精益管理模块包含9大业务模块，作为详细设计的完整参考 ^[raw/articles/smart-factory-lean-detail-design.md]：

| 模块 | 核心功能 | 功能数量 |
|------|----------|----------|
| 标准化点检 | 计划制定→任务执行→整改验证 | 8个功能 |
| 职业病危害检查 | 检测标准→任务执行→整改审批 | 3个功能 |
| 职业健康体检 | 机构维护→标准定义→计划→结果 | 4个功能 |
| 劳保用品管理 | 清单→发放标准→发放任务 | 3个功能 |
| 安环事故管理 | 公司级+总部级事故汇总 | 2个功能 |
| 优秀行为点赞 | APP端点赞 | 1个功能 |
| 积分管理 | APP端查看+PC端管理 | 2个功能 |
| 合理化建议 | 类型→推广→流程 | 4个功能 |
| 指标档案配置 | 档案→标准定义→预警 | 3个功能 |

### 标准化点检设计要点

标准化点检是最复杂的模块，核心设计模式：

1. **计划→任务生成**：按周期和生成维度自动生成任务
2. **生成维度**：按点检类别/区域/区域+类别/审核类型，决定任务数量
3. **整改闭环**：不合格项→发起整改→整改验证→审批闭环
4. **多层级审核**：公司检查（选角色执行）vs 总部检查（选人员执行）

### 指标档案设计要点

指标档案配置体现了从数据到指标的完整链路：

1. **指标定义**：编码+名称+关联字段+计量单位
2. **计算公式**：基于数据表字段的表达式
3. **数据溯源**：Model→Table→字段名→数据字段→SQL函数
4. **标准与预警**：设定标准值→超阈值自动预警

## 常见陷阱

- **前置条件不能省略**：直接影响基础数据维护范围
- **功能逻辑和查询区字段不要重复**：功能逻辑是操作行为，查询区是筛选条件
- **表格区子表必须标注从属关系**：如"子表【点检明细】"
- **是否必填/可编辑要逐字段明确**：自动生成字段标"必填是/可编辑否"
- **审批流要分清维度**：公司级和总部级审批链不同
- **定时任务要写更新逻辑**：不能只写"定时生成"，必须说明下次执行时间如何更新
- **编码规则要具体到前缀+日期格式+流水号位数**
- **多终端功能要分别设计字段和操作**
