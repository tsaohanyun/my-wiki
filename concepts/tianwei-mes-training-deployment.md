项目：**天味家园B栋MES**（老车间工艺和物料防呆防错信息化优化项目）

## 一、系统部署方案

### 1.1 部署概述

MES平台采用超融合架构部署，覆盖从网络接入到应用层的完整技术栈：

```
网络接入 → 安全发布 → 安全访问 → 服务器硬件 → 硬件虚拟化
→ 应用集群部署 → 数据库和中间件 → 大数据平台 → 运维支持 → 网络端口开放
```

系统分为**测试环境**和**生产环境**，通过网络VLAN隔离：

| 环境 | 网段 | 状态 |
|------|------|------|
| 测试环境 | 192.168.230.0/24 | 已分配 |
| 生产环境 | 192.168.240.0/24 | 待分配 |

### 1.2 网络拓扑

业务交互采用核心交换机统一访问模式：
- **车间访问**：车间工位IPC通过车间交换机 → 核心交换机 → MES服务器
- **办公区访问**：办公PC通过办公交换机 → 核心交换机 → MES服务器
- **外部系统**：SAP/WMS通过核心交换机与MES对接

### 1.3 安全架构

**安全访问**：采用 VPN + 实施服务器方式，企业内网部署VPN服务端，分配账号给研发和运维人员，通过VPN权限控制访问范围。

**安全发布**：企业将镜像仓库 Harbor 端口 NAT 映射到公网地址，绑定域名和证书，限制仅供应商固定IP可访问。镜像不包含敏感数据（数据库信息、密钥等已从镜像抽离，通过环境变量注入）。

### 1.4 服务器硬件资源

| 项目 | 规格 |
|------|------|
| 物理CPU | 128核（超线程双线程 = 256线程） |
| 内存 | 1024GB RAM |
| 存储 | 约48TB（超融合双副本） |

高可靠性要求：
- **硬件高可用**：CPU/内存/硬盘/网卡/RAID卡/电源健康检测 + 告警
- **计算层高可用**：虚拟机HA、快照、热迁移、动态资源调度
- **存储层高可用**：多副本保护、数据仲裁、热备盘、IO QOS
- **网络高可用**：控制面/转发面/三层转发可靠性、VXLAN可靠性

### 1.5 企业需准备资源

关键需要企业决策或准备的资源包括：
- 网络接入（内网带宽、VLAN划分）
- VPN服务端部署及账号分配
- Harbor镜像仓库域名及证书
- 物理服务器及超融合平台
- 内网DNS域名解析（约25个域名）

虚拟化、虚拟机创建、应用集群部署、中间件、大数据平台、运维支持等由实施方完成。

## 三、培训方案

### 3.1 培训策略

培训按岗位分批进行，每批次包含**关键用户培训**和**终端用户培训**两个阶段：
1. 先培训关键用户（车间管理层），使其掌握系统操作并协助UAT测试
2. 再由关键用户指导终端用户（一线操作工）进行实际操作

### 3.2 培训分组

| 培训批次 | 岗位 | 培训目的 |
|----------|------|----------|
| 炒料室关键用户 | 炒料室管理 | 炒料工段系统操作 + UAT测试 |
| 小料室关键用户 | 小料室管理 | 小料工段系统操作 + UAT测试 |
| 外包关键用户 | 外包管理 | 内外包业务操作 + UAT测试 |
| 内包关键用户 | 内包管理 | 内包业务操作 + UAT测试 |
| 内勤用户 | 车间二线内勤 | 数据录入与查询 + UAT测试 |
| 炒料室终端用户 | 炒料室操作工 | 实际操作培训 |

### 3.3 培训内容覆盖

每个岗位培训内容包含：
- 系统登录与基础操作
- 对应业务模块功能演示与实操
- 异常场景处理流程
- UAT测试任务分配
