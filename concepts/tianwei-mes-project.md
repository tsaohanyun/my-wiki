# 天味家园B栋MES项目

## 项目概述

**全称**：老车间工艺和物料防呆防错信息化优化项目

**部署位置**：天味家园B栋食品加工车间

**核心目标**：
- **防呆防错**：通过系统校验替代人工核对，消除配料和炒制环节的人为错误
- **工艺账务管理**：细化车间物料流转的全链路数据采集与账务同步
- **数据化运营**：实现生产数据的实时采集、分析与基于数据决策运营

**业务范围**：覆盖火锅底料从原料预处理到成品包装的完整生产流程，包括大料（盐、糖、姜蒜等）、小料（干料、溶液）、炒制、内外包装全环节。

**典型产品**：好人家牛油麻辣、第三代厚火锅、鲜番茄汤锅、黄焖鸡调料四种成品为首批上线产品。

## 系统架构

```
                    ┌─────────────────────────────┐
                    │       MES 平台（博依特）       │
                    │  数字工厂 | 生产管理 | 追溯    │
                    │  仓储管理 | 看板 | 报表       │
                    └──────┬──────────┬────────────┘
                           │          │
                    12接口 │          │ 28接口
                    HTTP   │          │ HTTP
                           │          │
              ┌────────────┘          └────────────┐
              │                                    │
     ┌────────┴────────┐                  ┌────────┴────────┐
     │   WMS（仓储）    │                  │   SAP（PCE版）   │
     │ 备料/退料/包装   │                  │ BOM/工单/物料移动 │
     └─────────────────┘                  └─────────────────┘
```

**外围系统集成**：
- **WMS**：12个核心接口，覆盖备料需求下达/反馈/签收/关闭、退料、外包装工单、返工出库
- **SAP**：28个接口（MES→SAP 21个，SAP→MES 5个，WMS转SAP 2个），覆盖BOM同步、工单报工、物料移动
- **一物一码系统**：条码追溯对接
- **PLC设备**：炒锅工艺参数下发与实时采集

## 重大决策记录

以下决策从16场项目会议中提炼，按影响范围排序：

1. **不对接旧版ES-SAP**（6.30月度会决议）
   - 老车间MES只对接新的PCE版SAP，不考虑旧版ES-SAP
   - 新版SAP按10月1日上线计划，若延期则MES重新制定上线策略

2. **煮椒吸水比通过虚拟物料号解决**（6.30月度会决议）
   - 在MES中设置虚拟煮椒半成品物料号，记录实际产出和平均吸水比
   - 回传SAP时仍按干辣椒管控模式，不含水用量

3. **上线时间三次推迟**（从会议纪要追踪）
   - 初版 V1.0：9月30日上线
   - 修订版 V2.0：调整为10月30日（因SAP版本切换+详设变更）
   - 最终版 V2.1：10月30日第一阶段 + 11月15日第二阶段

4. **小料干料预混需专项方案**（6.30月度会决议）
   - 存在小料配料环节干料与大料提前预混的业务场景
   - 需组织研发、生产做专项讨论并输出解决方案

5. **炒锅设备联动与手动双模式**（详设评审确定）
   - 联机模式：MES下发工艺单到PLC，控制投料顺序和工艺参数
   - 手动模式：关闭联动需二级密码授权，操作工按物料指引手动投料

6. **上线策略从单机转为SAP联调**（版本演变）
   - V1.0原定单机上线（不依赖SAP/WMS）
   - 最终改为SAP联调上线，实现三系统数据闭环
