# B栋MES接口集成方案

本文档梳理B栋MES系统与WMS、SAP等外围系统的接口集成方案，涵盖接口清单、SAP集成方向、单机上线风险及系统联调测试方案。

## MES-WMS接口清单

MES与WMS之间共计12个核心接口，采用HTTP协议实时触发，覆盖备料、退料、外包装工单及返工等生产执行业务场景。

### 接口总览

| 序号 | 接口名称 | 传递方向 | 说明 |
|------|----------|----------|------|
| 1 | 备料需求下达接口 | MES→WMS | MES备料需求生效时调用WMS接口下达备料信息，支持新增、删除检查、删除三种操作类型 |
| 2 | 备料信息反馈接口（罐区） | WMS→MES | WMS反馈罐区备料详情（批号、数量、仓库等），MES生成本地备料记录 |
| 3 | 备料信息反馈接口 | WMS→MES | WMS反馈非罐区备料详情，MES提供独立服务与罐区接口分开 |
| 4 | 备料信息签收接口 | MES→WMS | MES对实物进行签收或拒收，签收后WMS同步移库数据给SAP（311移动类型） |
| 5 | 备料需求关闭接口 | WMS→MES | WMS通知MES对已完成或异常的备料需求进行整单关单操作 |
| 6 | 车间物料退料信息下达接口 | MES→WMS | MES触发退料时推送退料信息给WMS，MES同步给SAP（303移动类型） |
| 7 | 车间物料退料确认接口 | WMS→MES | WMS确认退料结果（接收/拒收），MES同步给SAP（接收：305，拒收：304） |
| 8 | 外包装工单下达接口 | MES→WMS | 外包装工单生效时推送工单信息给WMS，用于成品入库调度 |
| 9 | 外包装工单报工接口（产成品入库） | WMS→MES | WMS调用MES进行包装工单报工，WMS给SAP传入库，MES给SAP进行工时报工 |
| 10 | 取消外包装工单报工接口 | WMS→MES | WMS调用MES取消已报工的包装工单，WMS给SAP传取消入库，MES给SAP取消工时报工 |
| 11 | 返工出库反馈接口 | WMS→MES | WMS将立体库返工品出库信息同步给MES，一步签收，返工品从成品仓返回车间 |
| 12 | 返工出库签收接口 | MES→WMS | MES对返工实物进行签收或拒收，签收后WMS同步移库数据给SAP |

### 关键接口字段说明

**备料需求下达接口**（MES→WMS）核心字段：

| 字段 | 说明 |
|------|------|
| `inWarehouse` | 目的仓库（SAP库存地点） |
| `externorderkey` | 备料指令单号 |
| `planDate` | 计划送货时间（yyyy-MM-dd hh:mm:ss） |
| `type` | 类型：1-删除检查，2-新增，3-删除 |
| `sku` | 物料号 |
| `istank` | 是否罐区：1-是，0-否 |
| `quantity` | 数量 |
| `factoryCode` | 工厂编码 |
| `outWarehouse` | 出库仓库（SAP库存地点） |

**外包装工单下达接口**（MES→WMS）核心字段：

| 字段 | 说明 |
|------|------|
| `productCode` | 产品编码 |
| `batch` | 生产批次（10位） |
| `outputNum` | 计划数量 |
| `factory` | 工厂代码（区分家园立库和双流立库） |
| `sapbillno` | SAP生产订单号（12位） |

**退料下达接口**特殊说明：`source` 字段用于标识退料来源，家园固定传递"F"，双流固定传递"A"。

### 通用接口规范

- 传输协议：HTTP
- 触发方式：实时触发
- 返回格式统一为 `{ code, data, message/msg }` 或 `{ code, data, message, success }`
- 认证方式：通过 `token` 字段进行接口鉴权

## MES-SAP集成（PCE集成方向）

MES与新SAP通过PCE中间件集成，共计28个接口，其中B栋项目涉及26个（备件接口不涉及B栋项目）。

### 接口分类

| 方向 | 数量 | 说明 |
|------|------|------|
| MES调用SAP接口 | 21个 | 开发计划启动时间为8月1日 |
| SAP调用MES接口 | 5个 | 开发计划启动时间为8月15日，8月20日开始交付第一个接口 |

### 主要集成业务

- **MES推送方向**：订单创建与执行、物料消耗提交、返工业务、盘点业务等在MES端发生的业务数据
- **MES拉取方向**：[[bom-management|BOM]]主数据、工艺主数据、客商主数据、工艺工时、跨车间物流、预留信息
- **SAP推送方向**：个别成本、跨工厂业务等由SAP主动调用MES

### 旧SAP对接

旧SAP对接共计17个接口，均为MES调用旧SAP接口，同样涵盖订单创建与执行、物料消耗提交以及工艺工时、跨车间物流、预留信息的拉取。

### 一物一码系统对接

MES向一物一码系统推送工单数据，共计1个接口。

## 单机上线风险说明

MES系统在B栋单机上线期间，由于部分接口尚未联调完成，存在以下风险项。详见[[mes-standalone-deployment|单机上线方案]]。

### 风险清单

| 序号 | 风险项 | 风险说明 | 影响范围 |
|------|--------|----------|----------|
| 1 | 基础数据变更 | 系统初始数据导入后，后续数据变化时MES数据更新不及时 | 生产工单、任务单建立失败 |
| 2 | 车间退料 | 无接口时需在SAP二次制单，录错导致账务不符 | MES与SAP账务差异 |
| 3 | 称重接收 | B栋新增场景，称重差异成本需发SAP成本中心 | SAP库存虚高、工作量增加 |
| 4 | 生产工单创建 | SAP创建工单后需在MES系统导入生成，操作不熟悉 | 工单无法创建 |
| 5 | 生产工单投入与产出 | 需在SAP分别制出库单、入库单、报工单 | 系统间账务差异 |
| 6 | 盘点分摊 | 盘点后账务差异按用量分摊到工单，不及时则账务不符 | SAP单据录入错误 |
| 7 | 跨车间/跨工厂调拨 | 两个系统均需出入库操作 | 账务差异、工作量增加 |

### 各风险项详细应对方案

#### 1. 基础数据变更

**风险预防：**
- 物料类别变更较少，新增物料类别随物料主数据自动新建
- 物料主数据每周进行导入确认
- BOM发生变更或新增时通知生产工艺，导出BOM清单后在MES系统中导入
- 生产版本按当前版本管理，每日编订计划前确认有无生产版本变更

**应对方案：**
- 物料主数据缺失导致工单建立失败时，根据页面报错提示查询SAP数据并在MES新增物料档案
- BOM缺失时，通知生产工艺在MES系统导入数据，无法找到对应BOM时至PLM查询
- 生产版本错误且在生产前未发现的，产品由质量、工艺按不良品决定处理方案

#### 2. 车间退料

**风险预防：** 通过现场标识、移动制单、定时盘点保证MES数据与实物流转一致；由MES汇总退料情况输出报表辅助内勤确认。

**应对方案：** 出现账务差异时核对MES与SAP退料账务，修正SAP单据异常。

#### 3. 称重接收

**风险预防：** MES与电子秤对接保证重量录入正确；MES汇总称重差异输出报表。检查已有称重接收接口的通用性。

**应对方案：** 出现账务差异时核对并修正SAP单据或新建单据。

#### 4. 生产工单创建

**风险预防：** SAP配置导出模板避免二次筛选；MES导入时对工作中心等字段进行校验。

**应对方案：** 工单无法创建时再次导出导入，异常情况联系项目团队或关键用户。

#### 5. 生产工单投入与产出

**风险预防：** 通过条码化避免MES与实物差异；MES提供报表自动汇总，内勤根据工单汇总数据在SAP录入；工时按理论工时录入。

**应对方案：** SAP单据录入错误时与MES对账，对错误单据进行撤回或冲销。

#### 6. 盘点分摊

**风险预防：** SAP制单前查看MES是否完成盘点，MES根据盘点分摊结果在报表上修正工单投入产出。

**应对方案：** SAP单据错误时与MES对账，进行撤回或冲销。

#### 7. 跨车间/跨工厂调拨

**风险预防：** 通过移动端在实际发货收货时现场完成MES账务制单，再根据MES记录完成SAP单据。

**应对方案：** SAP单据错误时与MES对账，进行撤回或冲销；统计业务频率核定工作量，针对新增管控环节与车间人员开会宣贯。

## 系统联调测试方案

MES-SAP-WMS三系统共同完成业务功能[[integration-testing|集成测试]]。

### 测试目标

验证MES-SAP-WMS集成接口及相关业务的端到端流程。

### 测试业务范围

| 业务模块 | 测试内容 |
|----------|----------|
| 备料管理 | 备料需求下达→WMS备料反馈→MES签收/拒收→关单 |
| 退料管理 | MES退料下达→WMS确认→SAP同步（303/304/305移动类型） |
| 外包装工单 | 工单下达→产成品入库报工→取消报工 |
| 返工管理 | 返工出库反馈→MES签收/拒收 |
| 称重接收 | 电子秤对接→称重数据采集→差异报表 |
| 盘点分摊 | 盘点执行→差异分摊→SAP账务同步 |

### 测试准备

| 准备项 | 说明 |
|--------|------|
| 网络环境 | 炒锅设备交换机电源改为常通电，避免设备断电导致网络中断 |
| 数据采集 | 设备数据采集完毕 |
| 数据核对 | 基础数据核对完毕 |
| 测试物料 | 用自来水模拟物料，物料桶贴二维码 |
| 测试设备 | 电子秤、PDA、拖车（贴二维码）、炒锅（贴二维码） |

### 测试步骤

1. **配方维护**：工艺人员在MES平台维护测试用配方信息
2. **现场配料**：操作工根据测试配方信息进行配料操作
3. **拉料到炒锅**：将配好物料通过贴码拖车拉到炒锅旁
4. **扫码校验**：通过PDA扫描拖车、炒锅二维码进行绑定校验
5. **工艺数据下发**：炒制工艺数据下发到设备PLC，验证PLC和触摸屏数据一致性
6. **工步执行（第1步）**：
   - 验证MES收到"执行中"信号
   - 验证油温数据变化与现场设备一致
   - 验证火力、温度不符合工艺要求时设备侧异常提醒
   - 验证物料桶扫码物料诊断
   - 验证温度达标后第一步完成信号及投料提醒
   - 验证投料完成后MES接收信号及下一步计时
7. **工步执行（第N步）**：
   - 验证工步信号下发及工艺参数正确显示
   - 验证搅拌频率、火力、温度异常提醒
   - 验证物料诊断及提升机解锁
   - 验证炒制时长达标后工步完成信号
   - 循环执行至最后一步
8. **排料完成**：验证排料阀开关与MES计时
9. **特殊场景1**：未收到物料诊断成功信号时设备告警
10. **特殊场景2**：炒锅设备跳闸后恢复通讯，领班权限手动触发工步完成
11. **特殊场景3**：一个工步多个物料桶，逐步扫码、延时告警、脉冲信号验证

### 测试总结

测试完成后需总结测试准备、执行、问题跟踪过程中的主要问题及改进建议。各方对测试结果认可后，对交付方案进行签字确认。
