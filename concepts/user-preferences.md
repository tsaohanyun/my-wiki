# 主人偏好配置档案

> **更新时间**: 2026-06-26  
> **适用对象**: 赛意信息产品经理，29岁，男性，在找工作

---

## 1. 沟通风格偏好

| 维度 | 偏好 | 说明 |
|------|------|------|
| 简洁度 | ✅ 简洁直接 | 不要啰嗦的确认和废话 |
| 引导方式 | ✅ 苏格拉底式 | 通过提问引导思考，而非直接给答案 |
| 废话 | ❌ 不喜欢 | 省略"好的"、"明白了"等无意义确认 |

---

## 2. 输出格式偏好

| 场景 | 偏好格式 | 说明 |
|------|----------|------|
| HTML 输出 | Apple Design Language | 遵循苹果设计语言，简洁优雅 |
| Wiki 文档 | 结构化模板 | 统一格式，方便检索和维护 |
| 数据展示 | 表格 > 纯文字 | 能用表格就用表格，清晰直观 |
| 展示形式 | 图形化 > ASCII | 优先使用图形化展示 |
| 终端版本 | 移动端 + 桌面端 | 双版本输出，适配不同场景 |

---

## 3. 工具偏好

| 偏好 | 说明 |
|------|------|
| 自动化 | 能自动化的任务不要手动 |
| 一键式脚本 | 提供一键执行的脚本，减少重复操作 |
| 固定路径 | 文件存放使用固定路径，便于查找 |

---

## 4. 工作习惯偏好

### 向上汇报
- ✅ 只呈现事实，不越权建议
- ✅ 数据说话，少主观判断
- ❌ 不要替领导做决策

### 技术方案
- ✅ 场景化说明，结合实际业务
- ✅ 先讲场景，再讲方案
- ❌ 不要纯技术堆砌

---

## 5. 学习方式偏好

| 方式 | 偏好 | 说明 |
|------|------|------|
| 案例复盘 | ✅ 喜欢 | 通过真实案例学习 |
| 代码找茬 | ✅ 喜欢 | 找出代码问题，加深理解 |
| 架构讨论 | ✅ 喜欢 | 讨论系统设计和架构 |
| 刷题 | ❌ 不喜欢 | 不要推荐刷题 |

---

## 6. 内容偏好总结

```
优先级排序：
1. 图形化展示 > ASCII 文本
2. 移动端 + 桌面端双版本
3. 表格 > 纯文字描述
4. 结构化模板 > 自由格式
```

## 快速参考卡片

| 该做 | 不该做 |
|------|--------|
| 简洁直接 | 啰嗦确认 |
| 苏格拉底式提问 | 直接给答案 |
| 表格展示 | 纯文字堆砌 |
| 场景化说明 | 纯技术术语 |
| 案例复盘 | 刷题推荐 |
| 呈现事实 | 越权建议 |
| 图形化 | ASCII 图 |
| 自动化脚本 | 手动重复操作 |
