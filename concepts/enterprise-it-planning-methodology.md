# 企业架构与IT规划方法论

> 企业架构（EA）是建设企业信息化的顶层设计蓝图，回答"企业去哪里、IT怎么建"的战略问题。IT规划是EA落地的具体路径。

## 企业架构（EA）四大视图

源自《企业架构咨询方法论》，将企业信息化划分为四个相互关联的视图：

### 1. 业务架构（Business Architecture）
- 企业的商业内容、业务流程、组织架构
- 连接业务战略与IT系统的桥梁

### 2. 信息架构（Information Architecture / 数据架构）
- 数据资产的定义、组织和管理
- 数据标准、数据模型、数据流向

### 3. 应用架构（Application Architecture）
- 应用系统的组合与交互关系
- 系统边界、接口定义、集成方式

### 4. 技术架构（Technical Architecture）
- 基础设施、网络、硬件、中间件
- 技术标准、平台选型、部署方案

**方法论周期：** 每个架构的建立遵循"计划 → 评价 → 执行"的循环方法，逐步完善。

## IT规划四步法

源自《如何做企业IT规划》，系统性IT规划的四个阶段：

### 第一步：确认业务目标
- 明确集团战略定位及业务驱动力
- 明确IT驱动力（IT必须如何做才能支持业务）
- 明确管理模式及其对IT管理的影响

### 第二步：评估IT现状
- 记录并评估现有信息技术架构
- 绘制应用系统清单及数据接口模型
- 审核IT投资分析和管理流程
- 从功能层面和技术层面评估系统健康状况

### 第三步：设计IT目标环境
- 确定未来的应用架构、数据架构、技术架构
- 定义标准方向、章程和原则
- 确定IT组织架构及管理模式
- 参考全球最佳实践进行设计

### 第四步：制定行动计划
- 确定现状与蓝图的差距
- 定义IT项目优先级和分阶段实施蓝图
- 进行IT投资成本分析
- 制定IT项目管理方法

## 核心原则

1. **EA是业务与IT的桥梁** — 用双方都能理解的语言描述业务与技术间的关联
2. **顶层设计先行** — 先做规划再做建设，避免"一次性方案"和系统孤岛
3. **分阶段落地** — 基础设施先行，核心业务系统优先，逐步扩展集成
4. **持续迭代** — IT规划不是一次性工作，需要定期复盘和调整
