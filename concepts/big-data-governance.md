# 大数据治理

## 定义

大数据治理是在数据的全生命周期内，对其进行管理的原则性方法，目标是确保数据的**安全、及时、准确、易用**。数据生命周期包括：创建→处理→存储→查看→报告和使用→保留和检索→销毁。

## 治理的必要性

| 维度 | 问题描述 |
|------|----------|
| 数据规模 | 数据量增加至传统技术无法处理的量级 |
| 时效性 | 业务与数据使用方对数据延迟更加敏感 |
| 多样性 | 结构化、半结构化、无结构化数据并存 |
| 使用门槛 | 缺乏有效的元数据标签解释 |
| 价值密度 | 海量数据中有价值的信息比例低 |
| 真实性 | 数据质量参差不齐，多拷贝导致不一致 |
| 有效性 | 数据合规性与隐私保护要求 |

## 核心技术架构

### 全链路大数据平台

数据处理的六个阶段：

1. **数据上报** — 传感器、设备、业务系统的原始数据产出
2. **数据采集** — 批量/流式采集接入
3. **数据同步** — 跨系统、跨环境的数据同步
4. **数据加工** — ETL/ELT 处理，数据清洗与转换
5. **数据分发** — 将加工后的数据推送至消费端
6. **数据服务** — API 化的数据访问服务

### 数据仓库分层 ^[raw/articles/big-data-governance-day1.md]

| 层级 | 缩写 | 说明 | 存储格式 |
|------|------|------|----------|
| 原始数据层 | ODS | 原始数据，csv、文本文件 | 行式存储 |
| 明细数据层 | DWD | 列式存储，ORC 等格式 | 列式存储 |
| 服务数据层 | DWS | 明细数据按标准聚合，提供可复用数据 | 列式存储 |
| 数据应用层 | ADS | 常规应用部署 | 多样 |

### 元数据管理

- **数据目录**：可用数据列表、归属权、敏感度
- **数据词典**：存储方式（File/Object/Table）、字段含义、类型、取值
- **管理工具**：Atlas（开放式元数据管理）、Griffin（大数据质量开源框架）

### 统一平台能力 ^[raw/articles/big-data-governance-day1.md]

- 数据质量检测平台
- 数据安全平台
- 元数据平台
- 数据一站式开发平台
- 数据访问能力服务化（API 服务，屏蔽底层数据源差异）
- 数据资产监控和管理

## 成本管理 ^[raw/articles/big-data-governance-day2.md]

### 成本结构

```
大数据引擎
├── 存储引擎：HDFS、HIVE 等
├── 计算引擎：Flink、Hive、MapReduce 等
└── 数据服务引擎：应用引擎

成本组成：存储成本 + 计算成本 + 数据服务成本
```

### 成本公式

**总成本 = 不可控成本 + 可控成本**

不可控成本（基础设施等）与可控成本（存储策略、计算优化等）的分离是成本优化的起点。

### 存储优化策略

- **减表**：清理无用表
- **减副本**：降低冗余副本数
- **减列**：移除无用字段
- **减行**：清理过期数据
- **减分区**：合并或删除过期分区

### 生命周期矩阵

数据分层对应不同生命周期策略：DOG（原始）→ DWD → DWS → TDMC → APP → 临时表 → 开发环境，层级越低保留期越短。

### 治理实施方法

PDCA 循环 + 元数据驱动 + 专项数据治理 + 自驱式数据管理

## 数据规范治理 ^[raw/articles/big-data-governance-day3.md]

### 面临的问题

数据孤岛、数据不一致、数据难用、成本高

### 治理思路

1. 制定模型规范和标准
2. 确定治理目标和方法
3. 元数据驱动治理

### 模型分层规范

**业务数据化 → 数据资产化（面向实事建模） → 数据业务化（面向领域建模）**

### 业界模型建设方法论

- **范式建模** — 第三范式，减少冗余
- **维度建模** — 维表+事实表，大厂常用，常结合范式建模
- **DATA Vault** — 面向企业数据仓库的建模方法
- **Anchor** — 超级规范化建模

### 指标定义规范 ^[raw/articles/big-data-governance-day3.md]

核心原则：**唯一性、可扩展性、易理解**

指标派生路径：
- 数据域 → 维度 → 维度模型 → 统计周期/粒度/业务限定 → **派生指标**
- 数据域 → 业务过程 → 事实模型 → 原子指标 → 衍生指标 → **派生指标**

### 治理体系 ^[raw/articles/big-data-governance-day3.md]

**四大目标**：问题标准化、治理可量化、过程策略化、运营有抓手

**资产健康评估**：
- 模型：问题诊断 → 评估打分
- 质量：问题诊断 → 评估打分
- 成本：问题识别 → 评估打分

结合治理评估体系的运营机制，实现治理收益的持续评估。
