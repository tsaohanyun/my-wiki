# 实施方法论

> 信息系统和软件项目的规范化交付方法论，覆盖从项目启动到持续运营的全生命周期。

## 用友IPM五阶段（智能工厂（离散版）实施方法论V2.0）

IPM（Intelligent Plant implementation Methodology）是用友针对离散制造行业智能工厂项目打造的实施方法体系。2017年发布V2.0。

### 阶段一：项目启动
- 组建项目团队（双方组织架构与职责）
- 售前交接与内部交接
- 制定项目主计划（WBS分解）
- 召开项目启动大会

### 阶段二：蓝图设计
- 业务调研与需求分析（问卷 + 现场访谈）
- 现状分析（AS-IS）与将来流程定义（TO-BE）
- 三大解决方案：业务解决方案、数据采集方案、技术部署方案
- 方案评审与客户签字确认

### 阶段三：系统构建
- 技术路线确定与开发环境部署
- 客户化开发（功能开发 + 接口开发 + 报表开发）
- 集成测试（功能测试 + 数据采集测试 + 接口测试 + 性能测试）
- 仿真模拟应用（全业务覆盖，极端情况模拟）

### 阶段四：上线运行
- 上线环境发布与配置
- 最终用户培训与考核
- 动态数据准备与转换
- 系统切换上线

### 阶段五：持续支持
- 日常运行支持与三层支持体系
- 项目总结与验收
- 实施转运维交接
- 持续优化与版本升级

**项目管理要点：** 质量管理、问题管理、变更管理、风险管理、里程碑确认
**文档体系：** 5个里程碑文档 + 20个关键文档 + 42个必选模板 + 30个可选模板

## 六步法实施方法论

源自《项目方法论分享》（优刻得），适用于通用信息化项目：

1. **项目准备** — 售前交接、组建团队、制定计划、召开启动会、标准产品安装
2. **蓝图设计** — 现状调研、需求分析、编写业务蓝图、方案汇报确认
3. **客户化部署** — 系统环境部署、客户化开发、单元测试
4. **整体测试** — 集成测试、定制开发测试、问题跟踪
5. **系统上线** — 用户培训与模拟、上线准备、系统初始化、上线切换
6. **项目收尾** — 项目总结、项目验收、项目移交

**过程管控：** 周报月报机制、变更控制、问题管理流程

## 汉得中台实施方法论

源自汉得信息《中台实施方法论》，适用于企业中台项目建设：

### 四阶段模型
1. **准备阶段** — 团队组建、计划制定、环境准备
2. **规划与设计阶段** — 业务调研、中台架构设计、技术方案确定
3. **实现与交付阶段** — 开发实现、测试验证、用户培训
4. **发布运营阶段** — 系统上线、运营支持、持续优化

### 方法论的六个核心维度
任务（做什么）、人员（谁来做）、流程（先后顺序）、时间（周期）、工具（用什么）、成果（交付什么）
