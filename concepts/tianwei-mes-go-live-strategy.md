# 天味家园B栋MES上线策略

## 一、上线策略总述

本项目（天味家园B栋老车间工艺和物料防呆防错信息化优化项目）以**防呆防错**为核心目标，细化车间工艺账务管理，涉及计划管理、配料炒制防呆防错、仓库管理三大业务域，覆盖数字工厂、生产管理、生产追溯、仓储管理、告警管理等核心功能模块。

### 1.1 上线模式：单机上线 vs SAP联调上线

项目在实际推进过程中，面临两种上线路径的选择与迭代：

- **单机上线（独立运行模式）**：MES系统先行独立部署，不依赖外围系统（SAP、WMS、一物一码），先在MES平台内部跑通核心业务流程（工单管理、配料、炒制、仓储等），验证功能闭环。该模式适合快速验证、降低上线耦合风险。
- **SAP联调上线（集成联机模式）**：MES与SAP、WMS、一物一码等外围系统同步开放接口，实现三系统数据集成联调。该模式实现完整的业务闭环（含账务同步），但对接口稳定性、数据一致性要求高。

在实际策略演进中，项目组经历了从"单机验证→联调上线"的逐步推进过程。最终方案（V1.6版）采用**分步联机**策略：上线当天选择一条牛油产线完成炒锅联机上线，其余产线使用非联机模式，待第一条产线运行稳定后，依次在其余产线推进联机。

上线策略还涉及与[[tianwei-mes-project-overview]]的总体规划对齐，确保分阶段交付与业务需求一致。

## 二、上线范围与版本演进

上线策略报告经历了4个主要版本的迭代，反映了项目范围的细化和时间表的调整：

### 版本 I（初始版本，2023年8月10日）

- 第一阶段计划上线时间：**9月30日**
- 第二阶段计划上线时间：**10月15日**
- 上线功能：生产管理（工单管理、配料、炒制、内外包装等）、基础数据、生产追溯、仓储管理、运行监测、告警管理、系统管理、报表管理
- 外围系统集成：SAP、WMS
- 特点：功能范围最广（含运行监测、告警管理），但部分功能尚在规划中

### 版本 II（修订版本，2023年10月10日）

- 第一阶段计划上线时间调整为：**10月30日**
- 第二阶段计划上线时间调整为：**11月15日**
- 新增功能：小料报工、报工记录管理、投料记录管理、生产看板（炒锅炒制看板）
- 外围系统集成：SAP、WMS
- 新增上线准备工作的详细分解（功能验证确认、基础数据收集、系统初始化与配置、硬件准备）
- 特点：时间表后移一个月，功能模块更加完善

### 版本 III（进一步完善，2023年10月15日）

- 第一阶段计划上线时间：**10月30日**
- 第二阶段计划上线时间：**11月15日**（新增告警平台）
- 明确了上线参考场景：好人家牛油麻辣、第三代厚火锅、鲜番茄汤锅、黄焖鸡调料四种成品
- 新增详细的上线准备工作项任务表（含责任人、完成时间）
- 新增终端用户培训计划（10月20日-25日）
- 新增上线工作通讯方式（项目通讯群、上线日报）
- 新增**上线风险与应对策略**章节（7大风险+系统灾备方案）
- 特点：全面的风险管控和灾备机制

### 版本 V1.6（最终版本）

- 第一阶段计划上线时间最终调整为：**11月17日**
- 第二阶段计划上线时间最终调整为：**12月5日**
- 外围系统集成扩展：SAP、WMS、**一物一码系统**
- 新增炒锅产线**分步联机上线**策略：
  - 11月17日：2#炒锅联线运行切换
  - 11月17-18日：1#炒锅联线运行切换
  - 11月17-19日：3#、4#炒锅联线运行切换
- 系统上线运行优化期：**11月17日-12月1日**
- 特点：采用渐进式联机上线，降低单次切换风险

## 三、关键风险与应对

策略报告识别了7大上线风险及其应对措施，并与[[mes-risk-management]]的通用方法论保持一致。

### 风险1：生产现场负荷高、临时工多

- **风险描述**：生产旺季现场生产任务重、效率要求高；临时工多、人员流动大
- **应对措施**：
  1. 为临时工建立车间内岗位通用账号
  2. 硬件（PDA等）登记下发和收回，避免丢失
  3. 顾问现场跟进使用情况，快速解决问题

### 风险2：煮椒下线油污严重难以操作

- **风险描述**：煮椒下线油污严重，难以在生产同时操作PDA
- **应对措施**：
  1. 无法在线称重时，使用手动输入代替
  2. 对一批物料采用数据补录方式推进
  3. 在配料时（加冰糖环节）对煮椒成品进行称重

### 风险3：基础数据收集及导入不及时

- **风险描述**：数据收集及整理工作量大，无法全部准时完成；新工艺要求发版变更
- **应对措施**：
  1. 优先完成计划生产的及生产频率高的数据，保证生产有数据可用
  2. 明确系统内发版与更新流程，固化相应文件
  3. 所有工艺变更在系统做新发版本，下发新版本号，保留旧版本避免数据升级异常

### 风险4：现场炒锅通讯异常

- **风险描述**：炒锅在晚班偶尔出现通讯延迟超5分钟；交换机使用设备电源可能断联
- **应对措施**：
  1. 炒锅切换为【解锁提升机】，MES切换为【非联机模式】，切换需车间班组长确认密码
  2. 通讯异常后生产单位需第一时间通报异常

### 风险5：上线期间出现工艺变更

- **风险描述**：上线期间可能出现配方调整、BOM变更等工艺变更
- **应对措施**：
  1. 系统内发新版本，保留旧版本，新旧版本并行过渡
  2. 变更需经工艺团队确认后统一更新

### 风险6：上线首日炒锅产线切换风险

- **风险描述**：多产线同时切换可能导致大面积生产中断
- **应对措施**：采用分步联机上线，首日仅切换一条牛油产线，稳定后再推进

### 风险7：系统灾备

- **风险描述**：系统运行中出现严重故障需要回退
- **应对措施**：
  1. 上线前完成全量数据备份
  2. 制定回退方案和灾备切换流程
  3. 保留手工操作流程作为应急备份

## 四、上线切换方案（V1.6最终版）

### 4.1 切换时间表

| 日期 | 事项 | 说明 |
|------|------|------|
| 11月17日 | 2#炒锅联线运行切换 | 首条产线联机上线 |
| 11月17-18日 | 1#炒锅联线运行切换 | 第二条产线 |
| 11月17-19日 | 3#、4#炒锅联线运行切换 | 剩余产线 |
| 11月17日-12月1日 | 系统运行优化期 | 持续监控与优化 |
| 12月5日 | 第二阶段上线 | 告警平台等剩余功能 |

### 4.2 切换前检查清单

- [ ] MES系统部署完成，功能验证通过
- [ ] SAP接口联调完成，数据一致性验证通过
- [ ] WMS接口联调完成，备料流程验证通过
- [ ] 基础数据导入完成并核对
- [ ] 终端用户培训完成
- [ ] PDA、电子秤等硬件到位并调试
- [ ] 炒锅通讯测试通过
- [ ] 系统灾备方案就绪
