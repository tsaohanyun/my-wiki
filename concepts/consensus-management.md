# 共识管理

## 定义

共识管理是对协作过程中共识达成、固化与扩展的系统性管理方法。核心在于确保团队/组织在关键决策上达成明确、可追溯、可演进的共识。

## 共识分类 ^[raw/articles/consensus-management.md]

```
共识
├── 书面共识
│   ├── 明确确认共识（最高级别）
│   │   └── 典型文件：蓝图
│   │       特征：需共识方正面书面答复同意
│   │       要求：对文件明确内容作层级拆分说明
│   └── 达成共识
│       └── 典型文件：会议纪要
│           特征：通报即可，可包含"不回应视为同意"
└── 口头共识
    ├── 明确支持共识
    └── 未反对共识
```

## 共识层级选择原则

共识层级由**关键性**和**责任性**决定：

> 越关键、越需要承担责任的决策，需要使用越高层级的确认方式。

## 评价维度 ^[raw/articles/consensus-management.md]

共识质量由两个维度评价：

| 维度 | 说明 |
|------|------|
| 固化程度 | 从口头到书面，从低到高 |
| 共识范围 | 人员范围 × 内容范围 |

## 共识演进规律 ^[raw/articles/consensus-management.md]

1. **方向**：口头 → 书面，小范围 → 大范围
2. **优先级**：共识范围扩大 > 共识固化（范围扩大优先级略高）
3. **人员不均匀性**：关键相关方拥有更高权重，需考虑人员代表性
4. **终极形态**：体系文件的建设与出台

## 关键洞察

- 达成共识的**内容范围**与**人员范围**会影响固化工作的开展
- 共识范围包含人员范围和内容范围两个维度
- 扩大人员范围时需关注关键相关方的权重与代表性
