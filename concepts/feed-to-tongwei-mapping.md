### 3.1 配方管理（Formula）

| 工艺配方（物料→投料口映射） | **按预处理清单+微配属性自动匹配**投料工段 | TH003 | mes_base_material_special（特殊物料表） |

| 调料配方版本确认（基地间调拨） | **品管选择使用方/生产方配方**，自动齐套检查 | TH005 | — |

#### 关键行业洞察 → 项目理解

> 饲料行业的配方管理**核心痛点**在于：营养配方由营养师在专业系统中制定（非SME平台），而生产执行需要将营养配方**逐层分解**为可执行的工艺配方。通威的TH001-TH006流程正是为了解决这个"从营养到工艺"的翻译问题，其中**自学习优化**机制让分解准确率随使用次数提升，逐步降低品管经理的人工介入。

#### 关键行业洞察 → 项目理解

> 饲料生产执行最大的行业特点是**工段制连续生产 + 频繁换产**。通威项目通过以下关键设计来应对：

#### 关键行业洞察 → 项目理解

> 饲料仓储管理与其他行业最大的差异：
> 1. **多模态入库**：大宗原料呈现汽运/船运/油车/集装箱等多种运输方式，通威项目在MA001下细分为**5个入库子流程**分别处理
> 2. **散料与袋装并存**：大宗原料散装（筒仓）+成品袋装（40kg标准包），不同包装方式对应不同的入库策略和出库计量方式
> 3. **投料计量精准化**：饲料投料从传统的"均重出库"升级为"流量秤计量出库"，是数据驱动精准管理的关键改进

#### 关键行业洞察 → 项目理解

> 饲料行业质量管理的**双通道检验体系**是其独特之处：
> - 感官检验：快速判断原料外观/气味/质地，是饲料行业特有的"第一时间防线"
> - 理化检验：实验室精密检测水分/蛋白/脂肪等指标
> 
> 通威项目通过 **PDA条码打印+扫码送检** 打通了检验全链路，同时 **SPC（QC014）** 将质量管控从"事后检验"升级为"事前预防"，与IIoT平台的数据采集能力深度绑定。

#### 关键行业洞察 → 项目理解

> 饲料设备管理的行业特征：
> - **四大主机为核心**：粉碎机、制粒机、膨化机、超微粉碎机是饲料生产的"心脏"，其运行状态直接影响产量和质量
> - **淡旺季明显**：受养殖周期影响，饲料生产有明显的淡旺季，设备维保需要在淡季集中安排——通威系统的**淡旺季模型（EQ002）** 就是为此设计
> - **故障知识库共享**：多工厂模式下，一个工厂遇到过的设备故障经验可以通过知识库共享，减少重复排障

### 4.1 典型饲料行业IT架构 vs 通威实现

```
    ┌──────────────┐               ┌──────────────┐
    │    ERP系统    │               │  SAP/EBS     │ ← 财务与采购核心
    ├──────────────┤               ├──────────────┤
    │ 配方系统(独立)│               │  配方系统    │ ← 专业营养配方
    ├──────────────┤               ├──────────────┤
    │ 生产管理(自研)│               │  SME数智平台  │ ← 生产业务主平台
    ├──────────────┤               ├──────────────┤
    │ 仓储(手工/自研)│    ───▶      │  SWMS智能仓储 │ ← 仓储物流管理
    ├──────────────┤               ├──────────────┤
    │ 质检(纸质)   │               │ 工厂MES      │ ← 车间执行层
    ├──────────────┤               ├──────────────┤
    │ 设备(纸质)   │               │ IOT平台       │ ← 设备数据采集
    └──────────────┘               ├──────────────┤
                                   │  FBC审批平台  │ ← 移动端审批
                                   └──────────────┘
```

### 4.2 饲料行业特有的系统集成点

| 集成环节 | 行业特殊性 | 通威方案 |
|---------|-----------|---------|
| 配方系统→MES | 配方需逐层分解为工艺配方 | 接口推送+自动分解引擎（TH001-TH003） |
| IOT→质量SPC | 设备实时数据用于质量前馈控制 | IOT数据→SPC控制图→超限预警 |
| EBS→库存 | 饲料大宗原料价格波动大，需实时库存成本 | SAP与SWMS双向数据同步 |
| 移动端→现场 | 饲料工厂现场环境（粉尘/噪音）不适合固定工位 | PDA扫码+PDA检验+叉车平板 |
