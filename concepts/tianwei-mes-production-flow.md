# 天味MES-生产流程与业务规则

本文档围绕**一颗火锅底料从原料到成品的完整旅程**，串联MES系统13个核心功能模块，重点阐述每个环节的**防呆防错业务规则**。

项目背景详见 [[concepts/tianwei-mes-project|天味家园B栋MES项目]]。

## 生产流程总览

```
生产订单导入 → 工单拆解 → 车间备料 → WMS配送
                                          ↓
原料预处理 ──→ 配料 ──→ 拉料 ──→ 炒制 ──→ 内外包装 → 成品入库
(大料/小料)   (大料/小料)         (设备联动)  (投料+报工)
                                          ↑
                                     返工品拆解 ──→ 返工品投料
```

支撑配置：工艺配方单、BOM配置、工序产品配置、委外物料配置、炒锅通讯配置、小料干料配置

## 1. 计划与工单

**入口**：【小博智造】→【生产管理】→【工单管理】

### 业务流程

1. **生产订单导入**：从SAP导入生产订单需求，系统根据BOM和各项配置自动拆解
2. **工单生成**：订单拆解为各环节的生产工单
3. **工单审核下发**：审核后系统自动生成：
   - **生产执行单**（大料/炒制/内外包直接生成）
   - **配料任务单**（大料/小料分别生成）
   - 小料需选择锅数后才生成小料执行单和溶液执行单

### 防呆规则

- 工单审核后不可修改，确保执行数据与计划一致
- 小料工单必须选择锅数，系统据此计算配料量
- 工单状态全生命周期管控：未开始→执行中→已完成

## 2. 车间备料

**入口**：【小博智造】→【生产管理】→【车间备料】

### 业务流程

1. **自动备料**：对指定时间范围内未执行的工单进行物料需求计算
2. **分单**：按收货仓库自动形成不同备料单据
3. **WMS下推**：将备料需求下达给WMS，WMS执行备料作业
4. **反馈签收**：WMS反馈实际备料信息，车间扫码签收或拒收
5. **备料关闭**：已完成或异常的备料需求由WMS触发关闭

### 防呆规则

- 按收货仓库分单，一个库存地点一张单据
- 签收/拒收按整行操作，不可部分签收
- 签收后WMS自动同步SAP（311移库），拒收则不同步
- 罐区和非罐区使用不同接口服务

## 3. 原料预处理

### 3.1 大料预处理（B栋1F）

**原料**：姜、蒜、番茄、干辣椒等

**加工流程**：
- 姜蒜→姜粒/蒜粒（切制）
- 番茄→番茄丁（切制）
- 干辣椒→糍粑辣椒（煮制，即"煮椒"）

**防呆规则**：
- 每种预处理按生产执行单归集投料和报工数据
- 煮椒环节：系统通过**虚拟煮椒半成品物料号**记录实际产出和平均吸水比，作为品控依据
- 回传SAP时仍按干辣椒管控模式，不含水用量
- 煮椒下线油污严重时，可切换为手动输入代替在线称重

### 3.2 小料预处理

**原料**：各类小料原料、溶液半成品

**加工流程**：
1. **小料称重配料**：按配方单称重各原料
2. **进搅拌罐**：投入搅拌罐混合
3. **干料配料**：部分干料需提前预混
4. **报工**：溶液生产执行单整批快速报工

**防呆规则**：
- 投料支持扫码物料标签或手工选择，多次投料自动累加
- 自动产生物料线边仓出库记录
- 溶液报工时自动触发半成品入库
- 数据来源由列表按钮自动带出，不可编辑

## 4. 配料

### 4.1 大料配料

**入口**：【小博智造】→【生产管理】→【大料配料】（APP端）

**业务场景**：将原料（盐、糖）及半成品（姜粒、蒜粒、糍粑辣椒等）按火锅底料炒制工艺要求，分装到不同大小的容器中。

**业务流程**：
1. 系统依据炒制工艺单数据，生成每个容器的配料任务单
2. 操作工按配料任务单逐容器完成配料
3. 界面分为"未完成"和"已完成"两个标签页

**防呆规则**（核心！）：
- 系统指导操作工按配料任务单操作，**校验每个物料的配料数量**
- **校验容器内物料组合**是否与工艺要求一致
- 配料数量超出允差范围时系统拦截
- 按车间筛选，默认第一个车间

### 4.2 小料配料

- 小料按配方单配料后进搅拌罐
- 干料和溶液分别处理
- 报工后自动触发线边仓出入库

## 5. 炒制（核心环节）

**入口**：【小博智造】→【生产管理】→【炒料炒制】（APP端）

这是整个生产流程中**防呆防错要求最严格**的环节。

### 5.1 炒制拉料

配料完成后，根据炒制环节的生产需要，将容器按要求**抬上板车送至正确的生产产线**。

- 系统校验板车上的物料是否对应正确的炒锅
- 校验物料批次和数量

### 5.2 炒制投料

系统提供**两种模式**：

| 模式 | 操作方式 | 授权要求 |
|------|----------|----------|
| **设备联动模式** | MES下发工艺单到PLC，控制投料顺序和工艺参数，自动采集数据 | 默认模式 |
| **手动模式** | 关闭联动后，操作工按物料指引在各工步手动投入物料并报工 | 需二级密码授权 |

**炒制流程（设备联动）**：
1. 平台维护配方信息
2. 现场配料完成
3. 拉料到炒锅
4. **PDA扫码校验**物料与工步对应
5. 工艺数据下发到设备PLC
6. 按工步执行（第1步→后续步骤）
7. 整锅完成确认

### 5.3 防呆规则（精华）

- **投料顺序控制**：必须按工艺单规定的顺序投入物料，不可跳步
- **物料校验**：每步投料前扫码校验物料编码、批次、数量
- **工艺参数监控**：实时采集炒锅温度、时间等关键参数
- **二级密码机制**：切换为非联机模式需车间班组长确认密码
- **炒锅通讯异常降级**：通讯延迟超过5分钟时，解锁提升机+MES切手动模式
- **整锅完成确认**：所有工步执行完毕后统一报工

## 6. 内外包装

### 业务流程

1. **内包装**：完成炒制半成品、内包半成品及包材的投料
2. **内包报工**：对内包半成品进行产量汇报
3. **外包装**：外包装工单下达给WMS，WMS执行B栋三楼辅料包仓库出库
4. **成品入库**：WMS完成包装后调用MES报工接口，MES向SAP报工

### 与WMS联动

```
MES 外包装工单生效 → WMS出库调度 → WMS报工 → MES确认 → SAP报工
```

- 外包装工单生效时才下推WMS（开始生产才传）
- WMS报工类型：0=正常报工，1=取消报工
- 取消报工时WMS给SAP传取消入库，MES给SAP取消工时报工

## 7. 返工品处理

### 业务场景

生产过程中出现不合格品需要返工：从成品仓库（立库）出库→返回车间→拆解→重新投料。

### 流程

1. **返工出库**：WMS将立库返工品出库信息同步给MES
2. **MES签收**：车间确认收到返工品，签收或拒收
3. **返工品拆解**：在MES中创建拆解执行单，将返工品拆解为原材料
4. **拆解对接SAP**：自动产生MOM端生产执行单、投料记录、报工记录，对接SAP生产订单投料和入库
5. **返工品投料**：拆解后的物料重新进入配料/炒制流程

### 防呆规则

- 返工出库一个返工单号组成一条报文，多个返工单号不可在同一报文
- 签收后WMS同步SAP移库（311），拒收则不同步
- 物料盆视为物料桶管理：赋容器码，拉料时作为容器

## 跨模块关键规则

### 工艺配方单版本管理

- 所有工艺变更在系统中做新发版本，下发新版本号
- **保留旧版本**，避免工艺数据升级异常
- 明确系统内发版与更新流程，并固化相应文件

### 容器编码管理

- 物料桶统一赋码，作为配料和拉料的基本载体
- 物料盆如未取消，视为物料桶进行配置
- 容器码在拉料校验和投料校验中作为关键校验项

### 批次追溯链路

- 从原料入库→预处理→配料→炒制→包装→成品入库，全链路批次关联
- 支持工艺参数追溯和生产批次追溯两个维度
- 炒制环节的工艺参数（温度、时间、投料量）与批次绑定

### 基础数据配置项

以下配置项在上线前需完成收集和录入：

| 配置项 | 数据来源 | 说明 |
|--------|----------|------|
| BOM配置 | 物控部+数字中心 | 产品物料清单 |
| 工艺配方单 | 工艺部 | 每个步骤的物料和工艺参数 |
| 工序产品配置 | 工艺部+生产部 | 产品与工序的对应关系 |
| 委外物料配置 | 生产部 | 委外加工物料标记 |
| 小料干料配置 | 小料组+物控部 | 干料配料方式 |
| 返工品投料配置 | 工艺部+生产部 | 返工品重新投料规则 |
| 炒锅通讯配置 | 设备部+实施方 | 炒锅PLC通讯参数 |
| 称重设备配置 | 生产部 | 称重设备与工位绑定 |
| 条码规则配置 | 实施方 | 物料/容器/设备条码规则 |
